        public TField getMneGrp()
        Returns the MneGrp

        Field description: While defining the formula for a ratio there could be need to hold intermediate calculations. This is typically essential when the formula is to be defined in several steps. User could provide a mnemonic group name and do an intermediate calculation and re-use such calculation while defining the final formula for the ratio. Example : It is required to calculate Total Debt to Net Income Ratio as a % figure. This is calculated as follows; (Total Debt/Net Income)*100 Total Debt = Existing Loan Repayments + Proposed Loan Repayments Net Income = Total Income - Total Deductions Total Income = Montly Income + Monthly Incentives + Monthly Other Income Total Deduction = Monthly Tax + Monthly other deductions Assume that the following data group records are available

        Data Group (as defined in SA.DATA.GROUP)Comprising Data Types (In SA.DATA.TYPES)
        EXIST.LOAN.REP (For existing loan repayments)EX.LOAN.REP (Repayment obligation towards existing loan)
        PROP.LOAN.REP (For proposed loan repayments)PR.LOAN.REP (Repayment obligation towards proposed loan)
        PR.LOAN.OTH (Other payment obligations towards proposed loan like insurance charges etc.,)
        TOT.INC (For Total Income)MONTHLY.INC (Monthly Income)
        MONTHLY.INC (Monthly Incentives)
        MONTHLY.OTH (Monthly other Income)
        TOT.DED (For Total Deduction)MONTHLY.TAX (Monthly Tax Deduction)
        MONTHLY.ODED (Monthly other deductions)
        The formula requires the following: 1) Sum of data groups EXIST.LOAN.REP and PROP.LOAN.REP. This can be stored in a mnemonic, say REP 2) Difference between data groups TOT.INC and TOT.DED representing net income. This can be stored in a mnemonic, say NI 3) Dividing REP by NI 4) Multiplying the out after step 3 by a constant 100. This formula can therefore be defined as under;
        CALC.PARAMADD
        DATA.GROUPREP (mnemonic for total repayments)
        CALC.CONSTANT
        CALC.PARAMDIV
        DATA.GROUPNI (mnemonic for net income)
        CALC.CONSTANT
        CALC.PARAMMUL
        DATA.GROUP
        CALC.CONSTANT100
        Now the formula for calculating the mnemonic can be defined.
