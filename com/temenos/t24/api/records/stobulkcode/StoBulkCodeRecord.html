      • addAcctWithBank

        public void addAcctWithBank(TField param)
        Appends the specified AcctWithBank to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Identifies the Bank where the Beneficiary maintains his Account. If an input of less than 10 characters has been made by the User, the System will check against the Agency file to see if the input corresponds to a valid Agent. If this is the case, the short name of the Agent will be displayed and any expansion of the multivalue field will be refused. If the input does not correspond to a valid Agent, the System will then allow the User to expand the field to enter further information. For BACS payments, this field will define the bank to whom payment is to be made and is only used for Standing Order statement purposes to print the name of the bank and branch to whom payment is to be made. Validation Rules: Up to 35 type S (uppercase alpha or numeric) characters. Multivalue field. A maximum of 4 lines of 35 characters each will be allowed. (Optional Input; No Default Value.)

        Parameters:
        param - the AcctWithBank to append
        Returns:

        public TField getBankSortCode()
        Returns the BankSortCode

        Field description: Defines the Bank Sorting Code of the bank and branch to whom the payment is to be sent. It is only applicable to the BAC payments. Validation Rules: 6 numeric characters. This bank sorting code is only used in United Kingdom for the BACS payments. In Standing Order records, this field will be mandatory when the Pay Method selected is equal to BC.
