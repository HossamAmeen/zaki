      • getColumn

        public TField getColumn()
        Returns the Column

        Field description: Identifies the position where the 'field' should be printed. You can print at a position on the current line by just specifying the column, "CC" or you can print at a specific column and line, "CC,LL". The former is usually used for printing the passbook entries, whereas the latter is used for printing headings, customer details etc. You can also specify an offset from the current line, "CC,+N" or "CC,-N". This is used when the entry line consists of two or more printed lines. The PRINT.ON field, associated with COLUMN, will determine WHEN the data is printed. Validation Rules: 1-9 Any characters "CC", "CC,LL", "CC,-N" or "CC,+N"

        Returns:

      • getField

        public TField getField()
        Returns the Field

        Field description: Specifies the field to be printed. The data printed can be any field from the following files: ACCOUNT CUSTOMER COMPANY TRANSACTION CURRENCY ACCOUNT.STATEMENT STMT.ENTRY CATEGORY DEPT.ACCT.OFFICER The TRANSACTION and STMT.ENTRY files contain details concerning the 'current' entry. The other files contain static information concerning the customer and account. The special fields are as follows: CREDIT +ve amount from the entry DEBIT -ve amount from the entry BALANCE the new account balance TODAY todays date TELLER.ID the current teller id. Free format text can also be entered enclosed in double quotes, "xxxxx". Validation Rules: 1-40 Any character a) FILE.NAME>FIELD.NAME b) "text" c) Special field. (i FILE.NAME>FIELD.NAME Must be one of the files listed above and the field must be a valid data field defined on the file's STANDARD.SELECTION record. Special Field Must be one of the field names listed above.

        Parameters:
        Returns:
        TField - The Field

      • getFormat

        public TField getFormat()
        Returns the Format

        Field description: Specifies the format of the output. The following format specifications are normally used:

        20L20 chars - left justified
        15R15 chars - right justified
        11R,amount formatted with commas
        11R2,with 2 decimal places
        11RX,decimal places of the currency
        11RX,Nnegative sign not displayed
        11*RX, padded with "*"
        DATEdate in the format dd.mm.yy
        ACCOUNT number using the mask from the COMPANY record
        Validation Rules: 10 Any characters. Any valid FMT UniVerse Basic mask.

      • getPrintOn

        public TField getPrintOn()
        Returns the PrintOn

        Field description: Determines when the data, specified by FIELD, COLUMN & FORMAT, should be printed. Controls the output of the data. Depending on the code entered the data will be printed when: NEW A new passbook is issued. Eg The customer's name & address. NEW1 Additional details that can be printed on page 2 when a new passbook issued. FIRST A new page is printed. Eg The balance brought forward. LINE An entry is printed. Ie The transaction amount, debit/credit column, balance etc. LAST The last line of the page is printed. Eg The balance carried forward. END The last page of the passbook is printed. Eg New book issued message. Validation Rules: 1-5 Alpha characters NEW, FIRST, LINE, LAST, END

        Parameters:
        Returns:
        TField - The PrintOn
