      • getAllowedProcess

        public TField getAllowedProcess()
        Returns the AllowedProcess

        Field description: This field specifies the Default process type for a transaction if none is associated to its own definition: ‘SAO’: direct authorization attempt for the SEPA.FOLLOW.UP whatever the status of the customer account might be. ‘SNP’: direct authorization attempt for the SEPA.FOLLOW.UP if no non overridable message forbade the booking entry on the customer account (if one is involved). Otherwise, the SEPA.FOLLOW.UP is only validated with a ‘FNAO’ status. ‘RET’: direct authorization attempt for the SEPA.FOLLOW.UP through the same transit account as the one used for the inward entry. ‘MAN’: the SEPA.FOLLOW.UP is produced in the ‘FHLD’ non validated status. ‘NO’: SEPA.FOLLOW.UP not generated.

        Validation rules Value upto 3 type A(AlphaNumeric) and user can input only 'NO' 'MAN' 'RET' 'SAO' and 'SNP' Associated Multi Value set of Field ALLOWED.VALUE to ALLOWED.MAP

        Returns:
