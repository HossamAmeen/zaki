      • getChngStatus

        public TField getChngStatus()
        Returns the ChngStatus

        Field description: Defines the status of the associated PRIN.CHANGE. Once the account officer has submitted a request for PRINCIPAL change, the CHNG.STATUS will be set to REQUESTED for the associated PRIN.CHANGE and CHANGE.DATE and ORDER.ID. If the dealer accepts the change request, they must do so by setting the CHNG.STATUS to ACCEPTED. If the dealer does not wish to accept the change, the change fields must be removed from the FD.FID.ORDER record. Once the dealer has Accepted the change and authorised the contract, it cannot be removed. Changes accepted in error can effectively be reversed by adding a change for the opposite amount to the FD.FID.ORDER record for the same CHANGE.DATE, which must also be ACCEPTED by the dealer. There will only ever be one REQUESTED change for the same CHANGE.DATE. Fields CHANGE.DATE to CHNG.STATUS form a related multi value set. Fields ORDER.ID to CHNG.STATUS form a related sub value set. Validation Rules: REQUESTED, ACCEPTED or ACTIONED. (Optional input) Multi value sets cannot be deleted. Expansion of the CHANGE fields is not allowed. A status of ACTIONED cannot be changed. REQUESTED changes can only be set to ACCEPTED, or back to REQUESTED.

        Returns:

      • getPrinChange

        public TField getPrinChange()
        Returns the PrinChange

        Field description: A change to the principal of placement pending, generated by a request to change the Order. The customer may require a change to the existing principal of NOTICE order. This is entered by the account officer in the PRIN.CHANGE field of the order, with the corresponding CHANGE.DATE when the change is to be effective. On authorisation of the order, the placement is automatically updated with the PRIN.CHANGE and CHANGE.DATE, the associated CHNG.STATUS field is set to REQUESTED and the associated ORDER.ID field is updated with the key to the FD.FID.ORDER record which generated the request. The dealer must accept the request for principal change by changing the CHNG.STATUS to ACCEPTED. Once the change is accepted, the confirmation is sent to the FID.BANK, and the status is then set to ACTIONED on the placement, and the corresponding Order. The principal change will be made during the end of day on the CHANGE.DATE requested. A negative PRIN.CHANGE denotes that a principal decrease is requested. Fields CHANGE.DATE to CHNG.STATUS form a related multi value set. Fields ORDER.ID to CHNG.STATUS form a related sub value set. Validation Rules: 1-14 numeric amount, with optional leading minus (-). This is a NOINPUT field.

        Parameters:
        Returns:
        TField - The PrinChange
