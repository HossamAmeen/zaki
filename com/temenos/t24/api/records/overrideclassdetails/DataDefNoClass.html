      • getDataDefNo

        public TField getDataDefNo()
        Returns the DataDefNo

        Field description: Indicates which data item, as defined in field 1 (DATA DEF), the decision fields 4 to 6 refer to. The number in this field identifies which multi-value from field 1, DATA DEF, the decision fields 4 to 6 refer to. For example, the number '1' indicates that it is the data item defined by field 1.1, the number '2' indicates that it is the content of field 1.2 and so on. Each individual element of the override message which is to be used as a decision criterion need only be defined once in field 1 (DATA DEF). The same element can then be referenced as many times as required within the body of the decision table by a simple number. Validation Rules: 1 or 2-digit numeric. Multivalue field in association with fields 2 to 6, subvalued in association with fields 4 to 6. (Mandatory input) The number must correspond to an existing value in DATA DEF (field 1).
