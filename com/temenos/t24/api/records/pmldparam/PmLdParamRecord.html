      • getPmMaxPeriod

        public TField getPmMaxPeriod()
        Returns the PmMaxPeriod

        Field description: Where Position Management (PM) is being actively used and the LD module is installed this record controls how the two modules interact. In cases where it is common to enter long term loans i.e 10 years or more you may wish to set this value to limit the updates sent to PM to a more reasonable period. For example if you want to enter a 25 year term loan with monthly settlements of Interest and/or Commission you can limit the number of months of info sent to PM. It is unlikely you would use PM information (broken down on a monthly basis) that far in advance, and to avoid the overhead at input time you can set this field to say 60 (representing 60 months). If you do require this level of information contact Temenos on advice on system settings which may need tuning. Validation Rules: Optional Input Must be a number between 1 and 999

      • getShowFwdInt

        public TField getShowFwdInt()
        Returns the ShowFwdInt

        Field description: Defines whethere interest and commission events are to be projected into the future. The next interest and commission movement will always be shown in the Position Management system. Subsequent events will not be included unless this field is defined. It may contain the following values: BOTH - Show fixed and variable rate contract future payments VARIABLE - Show only future payments for variable interest contracts FIXED - Show only future interest and commission payaments for fixed rate contracts. A fixed rate contract is a contract where INT.RATE.TYPE = 1, all other types are termed variable. Commission may only be fixed rate. Future interest and commission movements will be shown using the position class defined in POS.FWD.FIX.INT, POS.FWD.VAR.INT or POS.FWD.COMM if defined, otherwaise the movements will be reflected as cash flows in the same way as the first interest or commission payment. Future interest is calculateed using the current interest rate for all contracts. If a contract is of interest rate type PERIODIC AUTOMATIC, the current rate will be used, unless the field PROJECT.R.TYPE is set to YES in the LMM.INSTALL.CONDS table. In this case the future rate will be projected on R schedule dates using the current setting of the PERIODIC.INTEREST table. This can be recalcualted on the period specified in FWD.R.TYPE.FQU. Validation Rules: Optional input First 2 characters must be LD
