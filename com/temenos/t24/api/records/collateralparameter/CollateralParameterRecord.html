      • getOnlineUpdate

        public TField getOnlineUpdate()
        Returns the OnlineUpdate

        Field description: This field controls online update of cash collaterals. If cash collateral such as account, deposits (LD, MM), securities and fiduciary are attached to a limit record; any change in these collateral value are updated during the End of Day process. If such collateral, if value to be updated online then this field is to be marked as "Y". (LIMIT record where the cash collateral are attached, also to be marked "Y" in the online update field along with Online update field marked "Y" in COLLATERAL.TYPE) In such cases the collateral is recalculated when the limit reference is drawn. If the field is marked "N" or left blank then no online update process is done. Validation Rules: Optional Input field Input can be Y,N or null Input allowed only if the execution value(field 5) contains value F (FED) Online update of cash collateral is done only when online update fields in COLLATERAL.PARAMETER, COLLATERAL.TYPE & LIMIT records are marked "Y"

        Returns:
        TField - The OnlineUpdate

This parameter controls the first step in the cycle of archiving collateral records. The content of the field specifies a period of time as follows:

        nnDnn days
        nnMnn months
        nnYnn years
        In each case, this period of time must elapse before a set of associated collateral records are moved from the live file and placed on the history file. This elapsed period is taken from the date of the collateral right's EXPIRY DATE. Any collateral main file (object) records are moved to history together with their associated right. The status of all records moved is set to 'MAT'. This process is performed as part of the collateral application's end-of-day. The archiving of history records (their complete removal from disk) is handled by the Archiving module. Validation Rules: 2 to 3 alphanumeric characters; nnx where nn is a number in the range 1 to 99 and x is either 'D', 'M' or 'Y'. (Optional input)

        Set the specified value for RevaluationInfo
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Determines to what extent a history of the revaluation of a collateral record is kept (ie. how many previous values). A numeric parameter which determines how many values are stored on the collateral main file in REVALUED DATE & REVALUED AMOUNT. Each time the collateral is revalued, the system records the previous value of the EXECUTION VALUE and the date at which the revaluation took place in the fields mentioned above. This parameter determines how much of this historical information is kept in the system by limiting the number of multivalues used. When the maximum number of values defined in this field has been reached, the system automatically discards the oldest each time a new one is added. Zero is allowed. If no value is given, the value one is assumed. Validation Rules: 1 to 2-digit numeric. (Optional input)

        Field description: Identifies which descriptive field from the report lines file is used to enrich the source & target line fields on the collateral allocation table. The value of this field determines which of the values in the F.RE.STAT.REP.LINE field DESC is used to provide an enrichment to the collateral allocation fields SOURCE LINE & TARGET LINE. These source & target fields are recorded on the collateral allocation table using the CRF line numbers (ie. those numbers which form the second part of the key to records on F.RE.STAT.REP.LINE). In order to clarify the meaning of the line numbers on this table, this parameter may be used to instruct the system to display against them one of the available descriptive lines from the report lines file.
