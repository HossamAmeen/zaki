        public TField getLinkFile()

        Field description: Indicates the file to be used to extract the data from the fields specified in field LINK.FLD. If specified, the LINK.FILE along with fields LINK.NAME, LINK.FLD & OPERATION are used to determine the ID to be used for reading the file name specified in field GLOBUS.FILE.         public TField getXmlGroup()
        Returns the XmlGroup

        Field description: Indicates the parent element under which the element mentioned in field XML.LEVEL will be created as a child element. Example : For extracting the following XML document <?xml version="1.0" encoding="ISO-8859-1"?> <VALUATIONREPORT> <PORTFOLIO.INFO id="200000-1"> <PORTFOLIO.ID>200000-1</PORTFOLIO.ID> <REF.CCY>USD</REF.CCY> <VALUATION.DATA id="200000-1.LD0022200021..20000824.5.5.D"> <SECURITY.NO>LD0022200021</SECURITY.NO> <SHORT.NAME>*** Commitment Assets - </SHORT.NAME> <SECURITY.CCY>USD</SECURITY.CCY> <NO.NOMINAL>25000.00</NO.NOMINAL> </VALUATION.DATA> </PORTFOLIO.INFO> </VALUATIONREPORT> For PORTFOLIO.INFO - both XML.LEVEL & XML.GROUP are same as it contains directly data elements and there are no parent elements for PORTFOLIO.INFO. For VALUATION.DATA - XML.LEVEL is VALUATION.DATA and XML.GROUP is PORTFOLIO.INFO as VALUATION.DATA is the sub element of the element PORTFOLIO.INFO. Validation Rules: 1-35 Alphanumeric characters The value should confirm to the rules specified for naming the Elements in XML document

        Parameters:
        Returns:
        TField - The XmlGroup
