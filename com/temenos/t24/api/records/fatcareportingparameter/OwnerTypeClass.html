      • getOwnerAddRecord

        public TField getOwnerAddRecord()
        Returns the OwnerAddRecord

        Field description: If the OWNER ADDRESS is set to DE.ADDRESS (for joint owners), the field is used to identify the record from where the address of the customer will be picked up and updated in FATCA TAX BASE. For example, if the ADDRESS.RECORD is set as PRINT.1, for customer ID (100252) in Company 1 (GB0010001), the system will read the DE ADDRESS record GB0010001.C-100252.PRINT.1 and get the address. If there is no matching DE ADDRESS record, then there will be no default. Validation Rules: A valid carrier with the suffix (e.g. PRINT.1).

      • getRelIdentifier

        public TField getRelIdentifier()
        Returns the RelIdentifier

        Field description: If set to FCSI, the related customers will be identified from FATCA CUSTOMER SUPPLEMENTARY INFO (FCSI) record. If left blank, the related customers will be from Customer or Account (based on Relation level defined in FATCA AGGREGATION PARAMETER). For beneficial and substantial ownership, the relationships will be only from FCSI whereas for joint owners, the relationships can be at FCSI level or customer or account level. So, if the OWNER TYPE is Beneficial or substantial, only FCSI is allowed. Validation Rules: FCSI or Blank.

        Parameters:
        Returns:
        TField - The RelIdentifier
