      • getRenumberRoutine

        public TField getRenumberRoutine()
        Returns the RenumberRoutine

        Field description: Specifies the name of the subroutine used to generate the new account number when an ACCOUNT is to be converted. When accounts are converted the account to be converted is renumbered. The number used must either be entered in the AC.CCY.CONVERSION record for the account, or allocated automatically using a subroutine. Since each site may decide on a different numbering convention, a subroutine can be written to return the renumbered account number. For example account numbers may be 10 digits, the leading digit is zero. A convention for the renumbered accounts could be to replace the leading '0' with a '9'. In order to do this a routine must be written to return the modified account number, the name of the routine must be specified here. For each ACCOUNT record selected an AC.CCY.CONVERSION record will be created and the specified subroutine will be executed. The account number returned will be stored in the RENUMBER.ACCT.NO field of the created record. Subroutine Details ------------------ The subroutine must have no arguments, the account number to be renumbered should be passed in the common variable COMI, the new value should also be returned in this variable. Any error should be returned in ETEXT. Validation Rules: Must be defined in PGM.FILE as type S. Only allowed if APPLICATION is ACCOUNT.

        Parameters:
        Returns:
        TField - The RenumberRoutine

      • setRenumberRoutine

        public void setRenumberRoutine(TField value)
        Set the specified value for RenumberRoutine
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies the name of the subroutine used to generate the new account number when an ACCOUNT is to be converted. When accounts are converted the account to be converted is renumbered. The number used must either be entered in the AC.CCY.CONVERSION record for the account, or allocated automatically using a subroutine. Since each site may decide on a different numbering convention, a subroutine can be written to return the renumbered account number. For example account numbers may be 10 digits, the leading digit is zero. A convention for the renumbered accounts could be to replace the leading '0' with a '9'. In order to do this a routine must be written to return the modified account number, the name of the routine must be specified here. For each ACCOUNT record selected an AC.CCY.CONVERSION record will be created and the specified subroutine will be executed. The account number returned will be stored in the RENUMBER.ACCT.NO field of the created record. Subroutine Details ------------------ The subroutine must have no arguments, the account number to be renumbered should be passed in the common variable COMI, the new value should also be returned in this variable. Any error should be returned in ETEXT. Validation Rules: Must be defined in PGM.FILE as type S. Only allowed if APPLICATION is ACCOUNT.

        Parameters:
        value - The value to set
        Returns:
