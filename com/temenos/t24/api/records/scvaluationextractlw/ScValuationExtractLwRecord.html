If the position is for a Forward Rate Agreement then this field will contain the value in the INTEREST.RATE field of the original contract. If the position is for an Account then this field will be blank. If the position is for a Money Market contract or for a Loans and Deposits contract then this field will be blank.         Set the specified value for Yield
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field contains a percentage figure equating to the yield of the position. This figure contains different data depending on the module that gave rise to the portfolio position; If the position has been generated from a Fiduciary contract (FD) the this field will contain the INTEREST.RATE from the FD.FID.ORDER record that gave rise to the position. If this position has been generated from a LOANS.AND.DEPOSITS (LD) contract then this field is taken from the INTEREST.RATE field on the LOANS.AND.DEPOSIT record that gave rise to the position. If this position has been generated from a MONEY.MARKET (MM) contract then this field is taken from the INTEREST.RATE field on the MONEY.MARKET record that gave rise to the position. If this position has been generated from a Security Position then if the Security giving rise to the position is a share then this field will be updated from the PROCE.EARN.RATIO field on the SECURITY.SUPP record for the security. If the security is a bond then this field will be populated from the YIELD.TO.MAT field in the SECURITY.SUPP record for the bond. If the YIELD.TO.MAY figure is blank then it will calculate a yield using the Interest Rate from the SECURITY.MASTER record for the bond and the cost price for the position. Validation Rules:
