        public void addCollatCode(java.lang.CharSequence param)
        Appends the specified CollatCode to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The collateral code for which reallocation from the source line to the target line is to take place. The collateral codes in this field indicate which categories of collateral may be considered by the reallocation process for debit balances occurring under the corresponding source lines. The codes entered in this field must exist on the collateral code file. Codes may be defined in a set of multivalues, each one of which contains a corresponding SOURCE.LINE and TARGET.LINE. Within a single value, further sub-values may be used to form a group of codes which all share the same source and target lines. Multivalue field in association with field range SOURCE.LINE to PERCENTAGE. subvalue field (no association). Validation Rules: 1-3 digit numeric. Multivalue field. (Mandatory input) Sub-values may not be duplicated. Duplicate conditions (combinations of source line, code & type) are not permitted.

        Parameters:
        param - the CollatCode to append

        public void addSourceLine(java.lang.CharSequence param)
        Appends the specified SourceLine to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The source line from which amounts are to be reallocated. The source line is the reporting line in which debit balances occur that are eligible for reallocation to other lines on the basis of the value and type of collateral supporting them. These lines are defined by use of the CRF report line definitions on F.RE.STAT.REP.LINE. Input to this field must, when appended to the REPORT ID (field 1), form the key to a record on the report lines file. This record must be defined as either 'DETAIL' or 'LINK' (ie. total lines may not be used). Source lines may be defined in a set of multivalues, each one of which contains a corresponding TARGET LINE . Within a single value, further sub-values may be used to form a group of source lines which all share the same target line. Multivalue field in association with fields COLLAT.CODE. COLLAT.TYPE. TARGET.LINE. PERCENTAGE.. , subvalue field (no association). Validation Rules: 4-digit numeric. Multivalue field. (Mandatory input) Sub-values may not be duplicated. Duplicate conditions (combinations of source line, code & type) are not permitted.

        Parameters:
        param - the SourceLine to append
