        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field will contain the cost of the position. If the position is for a Securities (SC) contract then the system will first see if there is a MARGIN.CONTROL record connected to the SECURITY.MASTER record for the Security. If there is the margin rate will be taken from the MARGIN.CONTROL record. If there is no MARGIN.CONTROL record connected to the Security then the margin rate will be taken from the MARGIN.RATE field of the SUB.ASSET.TYPE record the position is connected to. If the position is for a Derivatives (DX) contract then this field contains the variation margin amount for value-dated positions. Validation Rules: No input field.         Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field contains a percentage figure equating to the yield of the position. This figure contains different data depending on the module that gave rise to the portfolio position; If the position has been generated from a Fiduciary contract (FD) the this field will contain the INTEREST.RATE from the FD.FID.ORDER record that gave rise to the position. If the security is a bond then this field will be populated from the YIELD.TO.MAT field in the SECURITY.SUPP record for the bond. If the YIELD.TO.MAY figure is blank then it will calculate a yield using the Interest Rate from the SECURITY.MASTER record for the bond and the cost price for the position. If this position has been generated from a Swap (SW) contract then this field is taken from the INTEREST.RATE field from the SWAP.BALANCES record, of the side of the SWAP transaction represented by this set of fields.
