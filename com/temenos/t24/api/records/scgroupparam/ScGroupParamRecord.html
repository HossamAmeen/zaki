      • addUserFields

        public void addUserFields(TField param)
        Appends the specified UserFields to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Multi-valued field to define additional parameters for grouping. This field allows ACCOUNT.MANAGER, CUST.NUMBER, LIQUIDATION.PERIOD, ORDER.DATE, PORTFOLIO.TYPE, PREMIUM.PRICE, STOCK.EXCHANGE, CUST.ACC.NO,CU and CU.EX.RATE.ACC. The grouping rules are that, if these parameters are defined in SC.GROUP.PARAM, all the above parameters except CUST.ACC.NO and CU.EX.RATE.ACC shall be the same across all the orders for grouping to take place. However if CUST.ACC.NO and CU.EX.RATE.ACC are defined then they should be the same for a specific portfolio, if the orders carrying such portfolio are to be grouped. Portfolio.type is a J descriptor linked to the SEC.ACC.MASTER and then to MANAGED.ACCOUNT. The value in GRP.PORT.DESC field in the linked MANAGED.ACCOUNT record should be the same for all the portfolios occurring across the orders that are sought to be grouped. Note: This facilitates grouping of orders by Portfolio type such as PEP portfolios, which may be defined in MANAGED.ACCOUNT file. Validation Rules: Multi-value, optional field. Allowed values are ACCOUNT.MANAGER, CUST.NUMBER, LIQUIDATION.PERIOD, ORDER.DATE, PORTFOLIO.TYPE, PREMIUM.PRICE,STOCK.EXCHANGE, CUST.ACC.NO,CU and CU.EX.RATE.ACC

        Parameters:
        param - the UserFields to append
        Returns:
        nothing
