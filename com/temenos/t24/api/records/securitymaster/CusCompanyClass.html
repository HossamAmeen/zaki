      • getLimitRef

        public TField getLimitRef()
        Returns the LimitRef

        Field description: Identifies the LIMIT REFERENCE number to be checked for limit control on Securities Issuer Limits. The ISSUER field must be populated with a relevant customer number before input will be accepted. A limit must be set up for the issuer with a limit amount allocated before input will be accepted. For a limit to be active then relevant SEC.ACC.MASTER records (normally for the Banks own books) should have theCHECK.LIMIT field set to Y. If the above conditions are met then limit checks will take place for all trades in this security for therelevant books. If a limit is exceeded then OVERRIDE messages will be generated when the trade takes place. When a valid limit reference is entered then on Authorisation Limits will be checked and updated for all tradesin this security for the relevant books. If a valid limit reference is removed or changed then limits will be backed out for the old reference for alltrades in this security for the relevant books. Validation Rules: 3 to 7 numeric characters limit reference code. (Optional input). Input in this field must be a valid limitreference number. If no limit exists with a limit amount allocated then an error message will be displayed. Multivalued associated with fields 82 (CUS.COMPANY) to 87 (MC.RESERV1)

        Parameters:
        Returns:
        TField - The LimitRef
