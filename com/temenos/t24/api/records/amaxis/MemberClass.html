      • getMbrAllocation

        public TField getMbrAllocation()
        Returns the MbrAllocation

        Field description: This field contains the allocation mode. DIRECT means that the MEMBER field contains the actual modelling position code (usually the internal security code). If this is used, the position allocation is done directly to the corresponding member. You do not have to set up a rule for that member. RULES or blank indicates that the allocation is done using rules describe with fields ENTITY, OPERAND, VALUE, TO.MEMBER, TO.PCT. AUTOMATIC When an item is allocated to a MEMBER which has its MBR.ALLOCATION set as AUTOMATIC the MEMBER will be replaced with the entity value. This allows for a dynamic member list, hence if the entity returns the currency we would dynamically generate member labels for each currency. DIRECT type members are process before RULES. Validation Rules: DIRECT, RULES or AUTOMATIC Optional input. Blank is processed as RULES.

        Parameters:
        Returns:
        TField - The MbrAllocation
