      • addCreditTxnId

        public void addCreditTxnId(TField param)
        Appends the specified CreditTxnId to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Multi-value field to determine the purchase transaction towards which the redemption would be allocated. Input allowed only for Debit TRANS.TYPE (redemptions). In case of redemptions, the same would be allocated towards the credit transaction specified. If the number of units in the Credit Transaction is more than the redemption amount, then units would be redeemed only to the extent of the redemption amount and balance units would remain. If the credit transaction units are not sufficient to cover the redemption amount, then the redemptions would be allocated towards the remaining Credit transactions on FIFO basis. If there are no other credit transactions, the redemption would only be executed to the extent of units available. If the order in which units are to be redeemed is not provided here, the redemptions would be allocated on a FIFO basis If order is being created from MF.ORDER.BY.CUST, this would be mapped from MF.ORDER.BY.CUST (CREDIT.TXN.ID or BULK.CR.TXN depending on order type) Validation Rules: Valid MF.TRADE ID that pertains to a purchase for the fund and by the customer in the order

        Parameters:
        param - the CreditTxnId to append
        Returns:
