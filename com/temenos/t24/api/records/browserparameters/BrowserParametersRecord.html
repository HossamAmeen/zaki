      • addAttributes

        public void addAttributes(java.lang.CharSequence param)
        Appends the specified Attributes to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Field can be multivalued depending on which attributes are required. Valid options are :- CHECKFILE.ENRICHMENT If this option is selected then checkfile enrichments will be run dynamically. Otherwise enrichments will only be run when a contract is committed or validated. MAXIMIZE.WINDOWS If this option is selected then windows will be always be displayed in the maximized state when opened. This does not apply to the main window. MULTILINE.VERSION.HEADERS If this option is selected then browser will display both lines of the VERSION header (fields HDR.1.00 to HDR.2...132) in the user's language. If no heading has been defined for the users language, the default language heading will be used. If no heading has been defined at all, the version description will be used. If no description or heading has been defined, the application name and version will be used. This setting is off by default as multiline headers can take up a lot of room in composite screens. NO.AUTOCOMPLETE If this option is selected then auto-completion of field values is switched off. NO.LANG.IN.VERSION If this option is selected then the language prefix (e.g. "GB") will not be displayed in field labels. NO.SELECTION If this option is selected the enquiry selection fields are not displayed on the enquiry result screen. NO.SORT.OPTIONS If this option is selected then lists of values will not be sorted in to an alphabetical list, but will be displayed in the order in which they were defined.

        Parameters:
        param - the Attributes to append
        Returns:
        nothing

      • addNonStdApps

        public void addNonStdApps(TField param)
        Appends the specified NonStdApps to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the non-standard applications that require additional processing when the Browser product is used. The Browser product uses a form of caching when the user requires a new contract screen to be displayed. If the application is marked as being non-standard then the new contract screen will not be cached, thus ensuring the the application is run every time. This is important for such applications as TELLER. This setting should only be used by System Administrators that are fully aware of the consequences of setting this field. This field is applicable only till R07.005 and lower releases Validation Rules: Up to 35 characters. Must match an application name from the PGM file.

        Parameters:
        param - the NonStdApps to append
        Returns:
        nothing

        public void addValidSkin(TField param)
        Appends the specified ValidSkin to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the names of the 'skins' allowed for displaying screens in the Browser product. A 'skin' determines the 'look and feel' of the Browser screens in terms of colours, fonts, graphics, etc. Each Browser installation will be delivered with a number of predefined skins that can be used. This field defines those that are present. The skin displayed used by Browser screen is defined on a user basis by the application BROWSER.PREFERENCES. Validation Rules: Up to 20 (type A) alphanumeric characters. Multivalue field (Optional input. Default value = blank.)
