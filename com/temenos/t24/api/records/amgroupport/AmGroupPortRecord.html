      • addSecAccNo

        public void addSecAccNo(java.lang.CharSequence param)
        Appends the specified SecAccNo to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Allows the user to specify more than one portfolio for a particular Group. The Portfolio entered must have the same ACCOUNT.OFFICER as created for the Group. If the field Account officer holds a value, then the field SEC.ACC.NO holds only those portfolios which has this Account Officer. If not, the Account officer of the Group must be higher in the hierarchy which is specified in the DEPT.ACCT.OFFICER file. Validation Rules

        A Multi-valued field.

        Optional Input. Must exist in the SEC.ACC.MASTER file.

        Any portfolio input must belong to the Account officer, that is identified in the associated ACCOUNT.OFFICER field.

        If group type holds a value then the portfolios must be linked to a AM.PORT.TYPE

        Must be at least one portfolio without a tax wrapper if it is a rebalancing group.

        If the field RBL.TO.DEF.PORT in AM.PARAMETER is set to yes, then the field SEC.ACC.NO must have at least one portfolio with port type defined in DEF.PORT.TYPE in AM.PARAMETER. A group cannot have two portfolios with default port type.

      • getInvestmentProgram

        public TField getInvestmentProgram()
        Returns the InvestmentProgram

        Field description: The axis of the support model must match with the axis of the group level matrix. i.e. AXIS.X of the support model must match with AXIS.X of the group model, same rule applies for AXIS.Y also; else error message is raised. Group matrix can have two axis and the supporting matrix can have both or any one of the group matrix axis, but it cant be the other way. Mandatory field because rebalancing must happen at group level and maximising the tax benefits must happen at sub portfolio level. If a link matrix is attached to any one of the model i.e. either supporting model or group level model, then a link matrix of the same type (axis should be in line with the attached linked matrix) has to be attached to the same axis member of the other model. Validation Rules:

        Alpha numeric

        Parameters:
        Returns:
        TField - The InvestmentProgram

      • setSecAccNo

        public void setSecAccNo(java.lang.CharSequence param, int idx)
        Set the specified SecAccNo to a specific position in collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Allows the user to specify more than one portfolio for a particular Group. The Portfolio entered must have the same ACCOUNT.OFFICER as created for the Group. If the field Account officer holds a value, then the field SEC.ACC.NO holds only those portfolios which has this Account Officer. If not, the Account officer of the Group must be higher in the hierarchy which is specified in the DEPT.ACCT.OFFICER file. Validation Rules

        A Multi-valued field.

        Optional Input. Must exist in the SEC.ACC.MASTER file.

        Any portfolio input must belong to the Account officer, that is identified in the associated ACCOUNT.OFFICER field.

        If group type holds a value then the portfolios must be linked to a AM.PORT.TYPE

        Must be at least one portfolio without a tax wrapper if it is a rebalancing group.

        If the field RBL.TO.DEF.PORT in AM.PARAMETER is set to yes, then the field SEC.ACC.NO must have at least one portfolio with port type defined in DEF.PORT.TYPE in AM.PARAMETER. A group cannot have two portfolios with default port type.

        Parameters:
        param - the SecAccNo to append
        idx - the position in collection
        Returns:
        nothing
