      • getBalanceToUse

        public TField getBalanceToUse()
        Returns the BalanceToUse

        Field description: When calculating the amount that needs to be transferred to/from an Account the Cash Pool rules can use one ofthree possible values as the Account balance. 1. ONLINE CLEARED The balance will be based on the Account cleared balance. This balance will not include any uncleared items (forexample cheques deposited with a future exposure date). 2. ONLINE ACTUAL The balance will be based on the Account uncleared balance. This balance will include uncleared items (forexample cheques deposited with a future exposure date). 3. WORKING This is the 'worst case' scenario balance as it will be impacted by unauthorised debits and would show a lowerbalance than the actual balance if there were entries due to be passed to this account. This balance type is morecritical for on-line sweeps since during the eod any unauthorised items are placed on hold and the working balanceadjusted accordingly. 4.VALUE.DATED This balance reflects the current balance after excluding any forward dated FT, Teller or DC entries. If backvalue dated adjustment is required then the balance to use can be only Value.dated. 5. USER DEFINED To arrive Account balance for sweeping purpose based on Bank's requirement instead of any of the above Balance touse option, it can be defined as USER.DEFINED and a valid "S" type of routine from PGM.FILE can be specified inBALANCE.ROUTINE. The balance as arrived by this routine will be used and checked against the Condition as definedin AC.CASH.POOL. To invoke user defined Balances, the rule should have NULL value in BALANCE.TO.USE for thecorresponding record in AC.SWEEP.TYPE. 6.WORKING LESS LOCKED AMT This option will bring the balance of Working balance less of Locked amount in the account record. When using SHARED.BALANCES,BALANCES.TO.USE must be WORKING. When BACK.VALUE.ADJ is set to YES, BALANCE.TO.USE must be VALUE.DATED 7.BOOKED LESS LOCKED This option will bring the online actual balance less of locked amount in the account record. 8. BOOKED WITH FWD LESS LOCKED This option will bring the consolidated balance by considering the online actual balance along with the forwardmovements but deducting the locked amount in the account record. The user must enter one of the ABOVE possible values, there is no default. Validation Rules: Values allowed are ONLINE CLEARED, ONLINE ACTUAL, VALUE.DATED ,WORKING LESS LOCKED AMT, WORKING, USER DEFINED,BOOKED LESS LOCKED or BOOKED WITH FWD LESS LOCKED. When using SHARED.BALANCES,BALANCES.TO.USE must be working. When BACK.VALUE.ADJ is set to YES, BALANCE.TO.USE must be VALUE.DATED

        Returns:
