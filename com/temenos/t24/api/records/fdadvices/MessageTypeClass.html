        public TField getMessageType()
        Returns the MessageType

        Field description: Specifies which message type, if any is to be produced for the Activity. Message types allowed are linked to the activity code and only certain types may be defined. If no message is required the user may enter the value 'NONE' to supress any output. If a USE.RECORD is specified the contents of the MESSAGE.TYPE field in the USE.RECORD will be used. A series of default messages is defined for each activity code, and will be produced if ALL is specified. Default Values if ALL specified: Event Type Message ===== ==== ======= New Contract ) FIXED ORDER 3320 Start Contract ) FIXED PLACEMENT 320 Change Contract ) NOTICE ORDER 3330 Amend Contract )} NOTICE PLACEMENT 330 Reimbursement ) Request ) Interest Payment NOTICE ORDER 3350 NOTICE PLACEMENT 350 Contract Maturity FIXED ORDER 3324 FIXED PLACEMENT 324 NOTICE ORDER 3330 NOTICE PLACEMENT 330 Contract Reversal FIXED ORDER 3320 FIXED PLACEMENT 320 NOTICE ORDER 3330 NOTICE PLACEMENT 330 Rate Change FIXED ORDER 3320 FIXED PLACEMENT 320 NOTICE ORDER 3335 NOTICE PLACEMENT NONE Payment FIXED NOTICE PLACEMENT 202 Advice to Receive FIXED NOTICE PLACEMENT 210 All Other Cases NONE Validation Rules:

        3 or 4 numeric DE.MESSAGE type. or NONE or ALL. (Optional field)

        Mandatory input when USE.RECORD is null.

        For Order related activities message type must be in the range 3000-3100, NONE or ALL.

        For payment activity codes (202 or 210), the only acceptable value is the same as the activity code.

        Message type 320 or 3320 is only allowed for Fixed Orders/ Placements.

        Message type 324 or 3324 is only allowed for Fixed Orders/ Placements.

        Message type 330 or 3330 is only allowed for Notice Orders/ Placements.

        Message type 350 or 3350 is only allowed for Notice Orders/ Placements interest payment activity.

        Message type 335 or 3335 is only allowed for Notice Orders/ Placements rate change activity.

        Input must exist on DE.MESSAGE if numeric.

      • getPrintFormat

        public TField getPrintFormat()
        Returns the PrintFormat

        Field description: Specifies the application format for the message required for the activity. This field allows the user to define different formats of the same basic message type for different activities. The element defined will become the second element of the resultant DE.FORMAT.PRINT record key. It is recommended that each activity code should use a different value where the same message type is used for different activities. This will allow different advices to be produced specific to the activity. Note that this field has NO effect or SWIFT output. Validation Rules: 1-4 numeric code or 'FD' followed by 4 numeric characters. (Optional input)

        Parameters:
        Returns:
        TField - The PrintFormat
