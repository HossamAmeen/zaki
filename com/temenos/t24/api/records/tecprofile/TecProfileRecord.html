      • getLogSets

        public TField getLogSets()
        Returns the LogSets

        Field description: Specifies the number of Log sets to be created when flushing the records in TEC.OUTPUT file. When the number exceeds, the first log set record will be overwritten with the new flushed details. This in turn provides self-cleaning mechanism for TEC.OUTPUT file storage space Example:- Let FLUSH.TIME = 30 seconds and LOG.SETS = 5 When flushing TEC events for the 1st time (after 30 seconds), Log set number 1 will be used. TEC.OUTPUT record will be created with the ID as (session number).(log set number).(TEC.ITEM) [as 6330.1.READ.SIZE, 6330.1.WRITE.SIZE etc]. When flushing for the 2nd time, details will be available in the TEC.OUTPUT record 6330.2.READ.SIZE, 6330.2.WRITE.SIZE etc. When Log set number is reached and details are flushed for the next time, TEC.OUTPUT record 6330.1.READ.SIZE will be overwritten. Validation Rules: : Default value is 10

        Parameters:
        Returns:
        TField - The LogSets

      • getQueueMaxSize

        public TField getQueueMaxSize()
        Returns the QueueMaxSize

        Field description: A threshold for storing TEC event messages in TEC.OUTGOING.MESSAGES file that can be consumed and dispatched to T24monitor by T24MONITOR.DISPATCHER service. The count of TEC event messages stored in TEC.OUTGOING.MESSAGES queue will be measured by each session for every 1000th message and will continue to push messages. When the threshold is reached or recognized in one session, TEC events will not be flushed further from that session. However, the other sessions may continue to flush messages by the time it measures the count and recognizes the threshold. This size does not reflect the total number of messages in TEC.OUTGOING.MESSAGES but acts as a threshold for every session in flushing the messages. Validation Rules: : Default value is 20000

        Returns:

This in turn provides self-cleaning mechanism for TEC.OUTPUT file storage space Example:- Let FLUSH.TIME = 30 seconds and LOG.SETS = 5 When flushing TEC events for the 1st time (after 30 seconds), Log set number 1 will be used. TEC.OUTPUT record will be created with the ID as (session number).(log set number).(TEC.ITEM) [as 6330.1.READ.SIZE, 6330.1.WRITE.SIZE etc]. When flushing for the 2nd time, details will be available in the TEC.OUTPUT record 6330.2.READ.SIZE, 6330.2.WRITE.SIZE etc. When Log set number is reached and details are flushed for the next time, TEC.OUTPUT record 6330.1.READ.SIZE will be overwritten.
