      • getFundId

        public TField getFundId()
        Returns the FundId

        Field description: This field is to hold SECURITY.MASTER id whose TR.ISSUER is the ISSUER in SC.TRAIL.FEES.HOLDING record. This filed is to hold the valid SECURITY.MASTER id for which the average price will be given by Issuer This filed is a part of multi value set with AVG.RECAL.PRICE Validation Rules: Input to this field should be a valid record from SECURITY.MASTER application Input to this field can not be blank when AVG.RECAL.PRICE is inputted Cannot include new SECURITY.MASTER ids to the multi value set when RECAL.WITH.AP is already set to YES Cannot include new SECURITY.MASTER ids to the multi value set when SETT.STATUS is defined as 'COMPLETE' i.e SETT.STATUS as COMPLETE and New fund ids cannot be defined at the same time Isser of SC.TRAIL.FEES.HOLDING record and TR.ISSUER of SECURITY.MASTER record should match AVG.RECAL.PRICE can not be blank when FUND.ID is inputted Input to this field can not be duplicated

        Parameters:
        Returns:
