      • getLimitLevel

        public TField getLimitLevel()
        Returns the LimitLevel

        Field description: Denotes the level of the Limit record from which the data was extracted. This field may contain the values: GLOBAL when extracted from a GLOBAL limit record PRODUCT when extracted from a PRODUCT level limit record, including products under a Global limit. SUB for SUB product limits, including those sub-products under a Global limit. Transaction data is extracted at both PRODUCT and SUB product level (although not at Global level), so a single transaction may appear twice in the CUSTOMER.POSITION file with 2 different limit references in the key. In enquiries based on transaction information, records with a value of SUB in this field should gemerally be ignored, as all information will be included in the PRODUCT level record. GLOBAL records will only contain a value for LIMIT records defined at that level.

        Returns:
        TField - The LimitLevel
