      • getTransType

        public TField getTransType()
        Returns the TransType

        Field description: This field contains transaction type identification code by which items will be matched in the automatic matching process. This allows the user to determine how certain transactions should be matched (eg. for items on a S.W.I.F.T. statement, the user may wish to match miscellaneous [NMSC] transactions by Value.Date and Amount but match cheques [NCHK] by Amount only). If this field is not required, it should be left blank. For entries related to payment and transfer instructions, including related charges messages, not sent through S.W.I.F.T. or where an alpha description is preferred. Format: Naaa The last three characters must be one of the following codes, indicating the reason for the entry: BOE Bill of Exchange BRF Brokerage Fee CHG Charges and Other Expenses CHK Cheques CLR Cash Letter/Cheques Remittance COL Collections (used when entering a principal amount) COM Commission DCR Documentary Credit (used when entering a principal amount) DIV Dividends Warrants EQA Equivalent Amount ECK Eurocheques FEX Foreign Exchange INT Interest LBX Lock Box1. LDP Loan Deposit MSC Miscellaneous RTI Returned Item SEC Securities (used when entering a principal amount) STO Standing Order TCK Travellers Cheques TRF Transfer VDA Value Date Adjustment For entries being first advised by the statement (items originated by the Account Servicing Institution): Format: Faaa The last three characters must be an appropriate code indicating the reason for the entry (i.e. the codes listed above). Validation Rules: 4 character multi-valued alphanumeric field
