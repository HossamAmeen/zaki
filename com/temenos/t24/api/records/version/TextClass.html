      • getText

        public TField getText()
        Returns the Text

        Field description: May contain text, a prompt associated with the input field. Text or Prompt Indicates the Text that User wants to define for the field specified in FIELD NO (Field 13). This is a sub-value field which can be expanded by itself to allow the Text to be entered in the various languages defined in Language code field. If no Text is entered by the User, the Version screen will be presented without any Text for this specific field. Spaces which are created by the User in this field will not be ignored. Optionally the number of the field can also be displayed by the System in front of the Text itself. If required, it should be defined before the Text as follows: 1. For Single fields enter in this field N, NN or NNN according to the size of the field i.e. N if field number is between 1 and 9, NN if field number is between 10 and 99 and NNN if field number is between 100 and 999. Example : N MNEMONIC 2. For Multivalue fields enter in this field N, NN, or NNN as appropriate followed by a space and XX. Example : N XX RELATION 3. For Sub-value fields enter in this field N, NN or NNN (as appropriate) followed by a space, XX, a second space and a second XX.Example : N XX XX FIELD NAME The Language codes define the language of the Text in the corresponding group of fields (e.g. if the second LANGUAGE.CODE specifies French, the system assumes that second group of text fields are in French and will display them for users whose profiles specify French). Validation Rules: Up to 39 type A (alphanumeric) characters Subvalue field (in association with Fields 17 to 20) associated with Fields 13 to 23. (Optional input. No default). Input in this field is not allowed when FIELDS PER LINE (Field 3) = 1 unless FIELD NO (Field 13) equals '*'. Input in this field is not allowed when FIELD NO is missing. The maximum length of Text which can be input in this field (including the field number itself if specified) cannot exceed 39 characters unless FIELD NO (Field 13) contains an '*'. Input in this field is not allowed when the corresponding Language code has not been defined. If, for example, this subvalue field has been expanded by means of the ')' sign and LANGUAGE CODE (Field 4) has not been expanded, input in sub-value 2 will then be refused because the language 2 was not previously defined.

        Returns:

      • getTxt040078

        public TField getTxt040078()
        Returns the Txt040078

        Field description: Indicates the Text that the User wants to define across the screen or page (from column 40 to 78) for a remark line specified in FIELD NO (Field 13) by the symbol '*'. This field is only used when a remark line ('*' in FIELD NO) across the screen is required. Spaces which are created by the User in this field will not be ignored. This is a sub-value field which can be expanded by itself to allow the Text to be entered in various languages. This will be allowed when the corresponding Language codes have been defined in LANGUAGE CODE (Field 4). The Language codes define the language of the Text in the corresponding group of Fields 17 to 20 e.g. if Field 4.2 specifies French, the System assumes that Texts in Fields 17.2 to 20.2 are in French and will display them for Users whose Profiles specify French. Validation Rules: Up to 39 type A (alphanumeric) characters Subvalue field (in association with Fields 17 to 20) associated with Fields 13 to 23. (Optional input. No default-Text will be generated by the System). Input in this field is only allowed when FIELD NO (Field 13) contains the value '*'. Input in this field is not allowed when FIELD NO is missing. Input in this field is not allowed when the corresponding LANGUAGE CODE (Field 4) is missing. If, for example, this sub-value field has been expanded by means of the ')' sign and the LANGUAGE CODE field has not been expanded, input in sub-value 2 will then be refused because the language 2 was not previously defined.

        Returns:
