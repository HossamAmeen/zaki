        Appends the specified BankNarr to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Provides instructions or additional information from the Sending bank for the receiver, intermediary account with institution, or beneficiary institution. For outward payments, any details input will be relayed to the Receiver bank. For Inward payments/receipts which are processed directly from Delivery, all transactions where bank to bank information has been found will be placed in hold status (IHLD) to force the user to review the bank to bank information from the sending bank. Whenever this field is used in a message transmitted between countries with different languages, it is recommended that English is used. The values of this field gets updated to handoff file in position 5.16, which can be mapped to tag 72 on SWIFT messages, and must never be used for information for which another field is intended, or for instructions not required in a message. Validation Rules: Up to 35 Type S characters (uppercase alpha or numeric). Multi-valued field. A maximum of 6 lines of 35 characters each will be allowed. If this field is used, the first multivalue must contain a code word followed by further information. All Swift code words must be present on the SWIFT.CODE.WORDS file, and must be enclosed between slashes "/". Further code words and information can be entered in subsequent multivalues, but additional information contained on a new line must be preceded by double slashes "//".

        public TField getAccountNumber()
        Returns the AccountNumber

        Field description: Identifies the Account number for the specified set of settlement instructions. It defines the account to be credited or debited on the contract value date and for the appropriate contract schedules. Should the account indicated be a Nostro account then further input may be required in the appropriate fields listed below : o ACCT.WITH.BANK o ACC.WITH.ADD o BEN.ACCOUNT The system will default to the first current account of the customer for the currency (SET.CURRENCY). The account indicated in this field will be used to default the remaining customer/ account fields within the settlement instructions if no further details are specified. Validation Rules: 1-16 numeric characters relating to a Customer account or a Nostro Account. Or, 3-10 type A (alpha-numeric) account mnemonic. (Mandatory input. Default = Current account in settlement ccy). This field will only be defaulted if the settlement currency field (SET.CURRENCY) is left blank. Must be of the same currency as defined in the settlement currency (SET.CURRENCY). This field can be changed at any time, but after the Value date of the contract, it will only affect future contract schedule movements.

      • getSetCurrency

        public TField getSetCurrency()
        Returns the SetCurrency

        Field description: Identifies the currency for contract settlements. This field is part of a set of settlement instructions to which credits and debits of a swap contract are made. There must be a set of settlement instructions for each currency defined on the contract. Therefore if the asset leg currency is the same as the liability leg currency, then one set of settlement instructions with a settlement currency of the same must be defined. The settlement currency is defaulted to this currency. If the leg currencies are different then two sets of settlement instructions are required (one for each currency). Both settlement currencies are defaulted - one to each leg of the contract. Validation Rules: 1-3 Characters Type CCY (Currency code). (Mandatory input. Defaults to leg currencies). Must be a valid currency code as defined on the Currency file.

        Returns:

      • getSetPayReceipt

        public TField getSetPayReceipt()
        Returns the SetPayReceipt

        Field description: Specifies the type of entries that are to be posted to the account number specified in the corresponding set of settlement instructions. Valid definitions are as follows : o null All accounting entries are to be posted to the corresponding settlement account number. o PAYMENT Only "payment" accounting entries are to be posted to the corresponding settlement account number. o RECEIPT Only "receipt" accounting entries are to be posted to the corresponding settlement account number. The set up of this field should be either : o null or, o a set of defined instructions for both "PAYMENT" and "RECEIPT". The main key to the settlement instructions to identify the associated account number is the Currency (SET.CURRENCY). If a set of settlement instructions for a currency is defined with a type of "PAYMENT", and a set of associated instructions for "RECEIPT" is not defined, then the same settlement account will be used for both payment and receipt type entries. Validation Rules: 1-8 Alpha-numeric characters. (Optional input. Default = null). Valid definitions are null, "PAYMENT", and "RECEIPT".

        Returns:
        TField - The SetPayReceipt

      • getSetType

        public TField getSetType()
        Returns the SetType

        Field description: Specifies the type of movements that are to be posted to the account specified in the corresponding set of settlement instructions. Valid definitions are as follows : o null All movements are to be posted to the corresponding settlement account number. o PRINCIPAL Only "prinicpal" movements are to be posted to the corresponding settlement account number. o INTEREST Only "interest" movements are to be posted to the corresponding settlement account number. Either : o one set of instructions should be defined for the currency with type set to null o or, two sets of instructions should be defined for the currency - one each type, "PRINCIPAL and "INTEREST". The main key to the settlement instructions to identify the associated currency account number is the currency (SET.CURRENCY). Validation Rules: 1-12 Alpha-numeric characters. (Optional input. Default = null). Valid definitions are null, "PRINCIPAL", and "INTEREST".
