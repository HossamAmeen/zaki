      • addLineAttribute

        public void addLineAttribute(java.lang.CharSequence param)
        Appends the specified LineAttribute to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The field indicates the characteristic of this line. Some lines may have a financial angle to it(dealing with amounts, currencies etc) like LENDING and some may not share that characteristic(like INTERNET.SERVICES). Currently to state that the products of this line deal with amounts, this field may be stated as CCY. A new option SINGLE is added along with the existing options CCY and REPLAY. The option SINGLE would indicate that there can be only one unique arrangement for each customer for products belonging to this line. For example, a customer may take as many LENDING products that he may require – but there can be only one RELATIONSHIP.PRICING arrangement for a customer. Validations would be added based on this field whilst taking NEW arrangement to throw error when multiple arrangements are taken under a single product. Option EXTERNAL.FINANCIAL is introduced to indicate that the product line is External financial Validation Rules Maximum of 35 Alphanumeric characters. Accepts CCY or NULL Non-mandatory field. Line attribute CCY is mandatory when line attribute EXTERNAL.FINANCIAL is given Products AX and XP should be installed if line attribute EXTERNAL.FINANCIAL is given

        Parameters:
        param - the LineAttribute to append
        Returns:
        nothing

      • insertLineAttribute

        public void insertLineAttribute(TField param, int pos)
        Insert the specified LineAttribute to the specified position in the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The field indicates the characteristic of this line. Some lines may have a financial angle to it(dealing with amounts, currencies etc) like LENDING and some may not share that characteristic(like INTERNET.SERVICES). Currently to state that the products of this line deal with amounts, this field may be stated as CCY. A new option SINGLE is added along with the existing options CCY and REPLAY. The option SINGLE would indicate that there can be only one unique arrangement for each customer for products belonging to this line. For example, a customer may take as many LENDING products that he may require – but there can be only one RELATIONSHIP.PRICING arrangement for a customer. Validations would be added based on this field whilst taking NEW arrangement to throw error when multiple arrangements are taken under a single product. Option EXTERNAL.FINANCIAL is introduced to indicate that the product line is External financial Validation Rules Maximum of 35 Alphanumeric characters. Accepts CCY or NULL Non-mandatory field. Line attribute CCY is mandatory when line attribute EXTERNAL.FINANCIAL is given Products AX and XP should be installed if line attribute EXTERNAL.FINANCIAL is given

        Parameters:
        param - the LineAttribute to append
        pos - The position where to insert the value.
