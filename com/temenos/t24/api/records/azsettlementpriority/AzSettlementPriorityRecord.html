      • getLinkToSettPrior

        public TField getLinkToSettPrior()
        Returns the LinkToSettPrior

        Field description: Specifies whether Settlement Priority rules should be followed for appropriation or the Settlement Priority rules of another record id should be followed. Validation: Input must be either 'NONE' or id of an existing AZ Settlement Priority application. NOINPUT for SYSTEM record. Linked settlement priority record should not have 'NONE' in its LINK.TO.SETT.PRIOR field. Account based settlement priority record ids(id starts with 'A') only be linked to another account id record. SYSTEM id cannot be linked to customer id records. Product based settlement priority ids can only be linked to another product id record. Input on other fields not allowed when this field is defined. Reversal of the record allowed only after reversing the linked record. When no values are inputted in any fields, this field will be defaulted to 'NONE' Field Properties: Standard T24 alphanumeric field. A maximum of 20 characters may be entered.
