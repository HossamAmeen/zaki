      • addCreateOveList

        public void addCreateOveList(java.lang.CharSequence param)
        Appends the specified CreateOveList to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field is used to determine which applications will create records on the EB.OVERRIDE.APPROVED file for OVERRIDE suppression. If an application is entered here & an OVERRIDE situation is encountered during that application then the OVERRIDE will be stored in the EB.OVERRIDE.APPROVED file. If an application is used that is in the OVE.SUPPRESSION field encounters an OVERRIDE it will read the EB.OVERRIDE.APPROVED record to see if the OVERRIDE has already been accepted & if so will not show the OVERRIDE again. Validation Rules: Either SEC.OPEN.ORDER or SC.EXE.SEC.ORDERS applications. Cannot be input unless the SC product is installed.

        Parameters:
        param - the CreateOveList to append
        Returns:
        nothing

        public TField getExactMatch()
        Returns the ExactMatch

        Field description: When checking if an OVERRIDE message has already been accepted, the system needs to know how to match the OVERRIDE message. YES - The OVERRIDE suppression programs will check the message exactly, including all values in the message. If any of the values do not match then the OVERRIDE will be shown. If the OVERRIDE message is exactly the same as the one stored in the EB.OVERRIDE.APPROVED file then the OVERRIDE message will be suppressed. NO - The OVERRIDE suppression programs will only check the Internal ID of the OVERRIDE, therefore if any of the text changes in the OVERRIDE the message will still be suppressed. E.g. If the OVERRIDE CASH.FLOW.OVERDRAFT creates the message "15/01/02 18599 CASH FLOW OVERDRAFT GBP -2234.99" has already been created in a previous application but the application we are in creates the same OVERRIDE with the message "15/01/02 18599 CASH FLOW OVERDRAFT GBP With EXACT.MATCH set to "NO" the system will compare the two OVERRIDE message ID's which are both CASH.FLOW.OVERDRAFT & therefore suppress the message. With EXACT.MATCH set to "YES" not only are the message ID's checked but also the text. In this example the ID's match but the text is different because the overdraft amount has changed. Therefore this message will not be suppressed & will be displayed. Validation Rules: YES or NO Cannot be input unless the SC product is installed.

        public void setExactMatch(java.lang.CharSequence value)
        Set the specified value for ExactMatch
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: When checking if an OVERRIDE message has already been accepted, the system needs to know how to match the OVERRIDE message. YES - The OVERRIDE suppression programs will check the message exactly, including all values in the message. If any of the values do not match then the OVERRIDE will be shown. If the OVERRIDE message is exactly the same as the one stored in the EB.OVERRIDE.APPROVED file then the OVERRIDE message will be suppressed. NO - The OVERRIDE suppression programs will only check the Internal ID of the OVERRIDE, therefore if any of the text changes in the OVERRIDE the message will still be suppressed. E.g. If the OVERRIDE CASH.FLOW.OVERDRAFT creates the message "15/01/02 18599 CASH FLOW OVERDRAFT GBP -2234.99" has already been created in a previous application but the application we are in creates the same OVERRIDE with the message "15/01/02 18599 CASH FLOW OVERDRAFT GBP With EXACT.MATCH set to "NO" the system will compare the two OVERRIDE message ID's which are both CASH.FLOW.OVERDRAFT & therefore suppress the message. With EXACT.MATCH set to "YES" not only are the message ID's checked but also the text. In this example the ID's match but the text is different because the overdraft amount has changed. Therefore this message will not be suppressed & will be displayed. Validation Rules: YES or NO Cannot be input unless the SC product is installed.

        value - The value to set
