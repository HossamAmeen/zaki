      • getGetNewSig

        public TField getGetNewSig()
        Returns the GetNewSig

        Field description: Specifies whether a new signature is to be captured for this signature box. If this field is changed to 'Y' for one or more signature boxes on a card then signatures will be captured when the record is validated. The signature image will be placed into each box flagged 'Y' in turn. Validation Rules: 1 alphanumeric Multivalue associated with fields 1 to 4. Cannot be changed to 'Y' unless operating on a valid capture terminal as defined in the SIGNATURE.PARM file. Entry must be 'Y' or 'N' Default is 'N' if not entered. On entering a new signature card defaults to 'Y' for box 1. No input allowed for a box number greater than the number defined in SIGNATURE.PARM. For the history restore function defaults to 'Y' for each box which contained a signature when the record was reversed. Cannot be changed to 'Y' on a non-display station whilst within a zero authorisers version of SIGNATURE.

        Parameters:
        Returns:
        TField - The GetNewSig

        public TField getName()
        Returns the Name

        Field description: Defines a name for each signature on the card. Non mandatory name for each signature. This name will be appended to the SIGNATURE.BOX label in the SIGNATURE.PARM file for display on screen if the DISPLAY.NAME flag is set for this signature box. Validation Rules: 1 - 20 alphanumeric Signifies the beginning of the multivalue set of fields 1 to 4. May not be entered for a signature box number greater than the number defined in SIGNATURE.PARM.
