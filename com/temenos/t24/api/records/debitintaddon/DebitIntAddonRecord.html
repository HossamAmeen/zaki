      • getDefaultFreeAmt

        public TField getDefaultFreeAmt()
        Returns the DefaultFreeAmt

        Field description: Specifies the Free Amount, in local currency equivalent, to be deducted from the total charge calculated subject to the Minimum and Maximum Amounts, if there is no entry in F.CURRENCY for the Currency of the Account. Amounts for adjusting Debit Interest Addon Charges in specific Currencies can be specified in the multivalue associated group of fields, K.CURRENCY to MAXIMUM.AMOUNT. The Default Amounts specified in fields DEFAULT.FREE.AMT to DEFAULT.MAX.AMT are used if there is no entry in K.CURRENCY for the Currency of the Account. The calculated amount is converted to local currency before making adjustments specified in fields DEFAULT.FREE.AMT to DEFAULT.MAX.AMT. The Default Free Amount is subtracted from the calculated local currency charge and further adjustments are made as specified in fields DEFAULT.MIN.AMT and DEFAULT.MAX.AMT. The result is converted to the Currency of the Account. Note : (i) The result may be zero, but not negative, i.e. the Customer will not receive a Credit. (ii) Even if the Free Amount is greater than or equal to the calculated charge, any Minimum specified in DEFAULT.MIN.AMT is still applicable. Validation Rules: 1-14 numeric characters plus a decimal point. (Optional input) The number of decimals may not exceed the number for local currency.

        Parameters:
        Returns:
        TField - The DefaultFreeAmt

      • getDefaultMaxAmt

        public TField getDefaultMaxAmt()
        Returns the DefaultMaxAmt

        Field description: Specifies the Maximum Amount, in local currency equivalent, to be charged if there is no entry in K.CURRENCY for the Currency of the Account. Amounts for adjusting Debit Interest Addon Charges in specific Currencies can be specified in the multivalue associated field group K.CURRENCY to MAXMIMUM.AMOUNT. The Default Amounts specified in fields DEFAULT.FREE.AMT to DEFAULT.MAX.AMT are used if there is no entry in Field 10 for the Currency of the Account. The calculated amount is converted to local currency equivalent before making adjustments specified in fields DEFAULT.MIN.AMT and DEFAULT.MAX.AMT. If after the Default Free Amount has been subtracted from the calculated local currency charge, the result is greater than the Default Maximum Amount, the Default Maximum Amount is used. The result is converted to the Currency of the Account. Validation Rules: 1-14 numeric characters plus a decimal point. (Optional input, default value is zero.) The number of decimals may not exceed the number for local currency.

The calculated amount is converted to local currency before making adjustments specified in fields DEFAULT.MIN.AMT and DEFAULT.MAX.AMT. If after the Default Free Amount has been subtracted from the calculated local currency charge, the result is less than the Default Minimum Amount, the Default Minimum Amount is used. Otherwise the Default Maximum Amount is checked. The result is converted to the Currency of the Account. Validation Rules: 1-14 numeric characters plus a decimal point. (Optional input, default value is zero.) The number of decimals may not exceed the number for local currency.
