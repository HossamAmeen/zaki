      • getBaseCcyRank

        public TField getBaseCcyRank()
        Returns the BaseCcyRank

        Field description: Indicates whether a currency should always be the base currency for currency conversions, in the event that a base currency has not been supplied to the conversion routine. The actual value is defined in the CURRENCY.PARAM record for this currency. Each currency can optionally be assigned a unique base currency rank in the same manner as currency rank. The values are stored in the application RANK.BASE.CURRENCY. If two currencies are involved in a conversion and both have a base ccy rank defined then the currency with the lowest rank will take preference. N.B. It has been stated that rates to the EURO (EUR) will always be quoted as the number of other currency units to 1 EUR, even when 1 unit of the other currency is worth more than 1 EUR. Example : assuming that 0.71 GBP were worth 1 EUR then the effect of not supplying a base currency or rate to the conversion routine would result in the base currency being set to GBP and the rate expressed as the inverse of 0.71 i.e 1.408451. Setting the EUR to have a base currency rank of 1 would result in the EUR being the base currency and the rate to be 0.71. Great care should be taken when defining a currency to have a base currency rank. There are in fact very few instances where this should be used. One example as discussed is the EURO.
