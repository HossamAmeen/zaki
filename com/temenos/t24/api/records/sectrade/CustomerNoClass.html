Where Securities differs from any other Application is that within one transaction e.g. a Purchase of Securitesyou have the following entries:- Credit securities to the clients portfolio. Debit Client financial account with total cost of Securities. CreditP L with entries for commissions, charges etc. Credit Brokers Financial account or Nostro with Total amountpayable. Debit P L with entries representing Brokers charges to you. To keep the integrity within any particular transaction therefore pairs of Transaction codes have to be used. These are stored within the File SC.TRANS.TYPE. Here you must specify your Security Debit and Credit codes forany particular transaction and state whether they are financial credits or debits. The system will NOT ALLOW mismatched transaction codes to be used. Note: Proceedure to be followed:- 1. Open TRANSACTION codes. 2. Open SC.TRANS.NAME records. 3. Open SC.TRANS.TYPE records.
