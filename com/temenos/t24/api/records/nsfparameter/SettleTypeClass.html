      • getSettleType

        public TField getSettleType()
        Returns the SettleType

        Field description: Defines the type of settlement. The is an option field with values 'PTL/POL/SET'

        Validation Rule:

        This is a mandatory multivalued field. User must define all three options.

        There will be no duplicate values allowed which means user has to define all three options of SETTLE.TYPE. PTL - Paid Through Limit - Represent underlying Debit transaction to the account was paid through the limit available at the time of transaction. POL - Paid Outside Limit - represent underlying Debit transaction to the account was paid in excess of the limit available at the time of transaction. SET - Automatic Settlement - represent underlying Debit transaction to the account was settled irrespective of account or limit balance available at the time of transaction.

        Parameters:
        Returns:
        TField - The SettleType
