      • getPlanAmt

        public TField getPlanAmt()
        Returns the PlanAmt

        Field description: For SIP, this would be amount that is proposed to be invested in regular intervals and for SWP and SSP, this could be either number of units to be redeemed/switched or amount to be redeemed/switched based on UNIT.AMOUNT field. If the PLAN.PERCENT field is set to Yes, then this amount would reflect the quantity to be redeemed as per the frequency set. For example, if this field has a value of 10 (for SSP and SWP), then 10 % of available quantity would be either redeemed or switched. If the PLAN.AMT is proposed to be increased after a specified period, the Multi value could be expanded and PLAN.AMT could be changed with new Start and End Dates. Validation Rules: Standard T24 nominal / amount ( NOMAMT ) format.

        Parameters:
        Returns:

      • getPlanType

        public TField getPlanType()
        Returns the PlanType

        Field description: Defines the type of the plan. There could be three kinds of plans: SIP – Systematic Investment Plan SWP – Systematic Withdrawal Plan SSP – Systematic Switching Plan Plans could be defined simultaneously (it is possible to have an SIP running for a period, followed by a Switch or SSP. But they would not run simultaneously. i.e. There cannot be overlap of plans within a particular period. The plans need not necessarily be entered at the time of placing the order. They could be entered separately at a later date using this application. Input allowed only if plans are allowed for the fund ( field PLAN.ALLOWED in MF.FUND.MASTER must be set to 'YES'). SSP would be allowed only if switch is allowed for the fund ( field ALLOW.FUND.SWITCH in MF.FUND.MASTER must be set to 'YES' ) Validation Rules: Multi-value field. Allowed values are SIP / SWP / SSP.

        Parameters:
        Returns:
        TField - The PlanType
