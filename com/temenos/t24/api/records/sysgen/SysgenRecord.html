        public TField getAccountMask()
        Returns the AccountMask

        Field description: Used to set the account mask on the COMPANY file (e.g. ######-##-###); must include the checkdigit. The mask for internal accounts is hardcoded (###-#####-####). NOTE: The checkdigit can be changed on the COMPANY record after the SYSGEN has been run, providing no customer accounts have been entered. Examples: 1. No = 1234; Mask = ##### - ##### Display = 1234 2. No = 1234567; Mask = ##### - ##### Display = 12 - 34567 3. No = 1234; Mask = R##### - ##### Display = 00000 - 01234 4. No = 1234567; Mask = R##### - ##### Display = 00012 - 34567 Validation Rules: 4 to 20 alphanumeric characters. Mandatory input if Ac is installed Validation raises error if AC product is not available in the PRODUCTS field list Input in this field must contain 4 '#' characters as a minimum. The number of '#' characters defines the maximum length of the account number. When ACCT.CHECKDIG.TYPE = 1, input in this field must contain 12 '#' characters. When ACCT.CHECKDIG.TYPE = 2, input in this field must contain 10 '#' characters. Characters '0...9', 'A...Z', 'a...z' are not allowed. The only exception being that the 1st character can be 'R' which means that the account number will be right adjusted and that it may contain leading zeros.
