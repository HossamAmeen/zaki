        public TField getRestDate()

        Field description:

        To calculate the date of the rest period, the system will add to the effective date of the key the value defined in REST.PERIOD (Field 3). If that date is a holiday then system will take the next date as REST.DATE. This REST.DATE can also be arrived based on base country setup done in ST.RATE.PARAMETER table

      • getRestPeriod

        public TField getRestPeriod()
        Returns the RestPeriod

        Field description: Indicates the period to which a rate applies.

        'R' for Remaining, NND for Rest period in Days where D = Number of Days, NNNM for Rest Period in Months where M= Number of Months and NNY for rest period in years where Y= Number of years. e.g. a 10 day rest period would be written as 10D, a 120 month rest period as 120M and a 20 year rest period as 20Y. R must be the last rest period entered. Multi value field. (Mandatory input. No default.) The number of days or months must be entered in logical order i.e. in ascending sequence. For days the range 01-31 will be allowed (if both days and months are defined) while for months 01-360 will be accepted. For years the range 01-30 is accepted.

        Days greater than 31 will be allowed when the interest is defined only in terms of days(D) i.e., 01D, 03D, 045D...9999D. In this case days range is from 01 to 9999.

        When 'R' is defined, at least one additional rest period must be defined. When more than one rest period is defined, 'R' must be entered on the last entered rest period.

        Parameters:
        Returns:
        TField - The RestPeriod
