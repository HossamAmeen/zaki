      • getCashflowTolerance

        public TField getCashflowTolerance()
        Returns the CashflowTolerance

        Field description: The cashflow tolerance is an exclusion value, expressed as a percentage, which when applied to the market value of the portfolio (based on the portfolio reference currency) returns the absolute value above which a flow, investment or withdrawal, will exclude the portfolio from the composite performance calculation. This test is applied during the EOD AM.COMP.BATCH batch and checks each days flows for the month under consideration for the composite calculation, if there is a flow which breaks the cashflow tolerance on any one day during the month, then the portfolio is excluded from that months composite performance calculation. The amount of time a portfolio will be excluded from the composite is determined by the excluded cashflow duration (field 13) and whether or not there have been other flows breaking this tolerance in the following month. e.g if the portfolios market value is $1,000,000 on the 15th July 2001 and the cashflow tolerance is set at 10%. Then an investment or withdrawal greater than $100,000 on the 15th of July 2001 will exclude the portfolio from inclusion in July 2001's composite performance calculation. If the portfolio's value on the 16th July 2001 is $950,000 then the investment/withdrawal limit for exclusion will be $95,000. Validation Rules: 0-8 numeric characters, format 999.9999 (optional field)

        Parameters:
        Returns:
        TField - The CashflowTolerance

      • getCloseDate

        public TField getCloseDate()
        Returns the CloseDate

        Field description: This is the end date of the composite and indicates the last day on which composite figures will be calculated.This date will be default to the first day of the month entered and can only be input when all links with the composite have been unplugged. e.g. - Today is the 13th September 2001 and you enter 15th September 2001 the system will validate the entry and change it to the 1st September 2001. - Today is the 13th September 2001 and you enter 15th October 2001 the system will validate the entry and change it to the 1st October 2001. - Today is the 13th September 2001 and you enter 2st September 2001, the system will validate the entry and change it to the 1st September 2001. Validation Rules: Standard date format YYYYMMDD (optional) The composite end date must be the 1st day of a Month. e.g. 01 May 2001. Any dates between the 1st and 31st will be modified to the 1st of the current month. The end date must be greater than the start date. The end date must be greater than or equal to the last date the monthly composite figures were generated.
