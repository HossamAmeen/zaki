        public TField getValidationRtn()
        Returns the ValidationRtn

        Field description: If the value in the associated field FIELD.NAME is not present in the corresponding field VALIDATION.FLD in VERSION record, the value in this field and the associated fields are appended to the VERSION record. This field accepts a user-defined subroutine or an EB.API record of type METHOD which implements an interface defined in the EB.API record VERSION.FIELD.VALIDATION.RTN.HOOK. The name of routine will be used to validate the input into the associated field FIELD.NAME. See the EB.API record VERSION.FIELD.VALIDATION.RTN.HOOK for the full list of supported interfaces, initially EB.TemplateHook.validateField(). This field supports the RecordLifecycle.validateField() method. The RecordLifecycle class is in the com.temenos.t24.api.hook.system package which is in EB_TemplateHook.jar shipped with T24. Validation Rules: Routine name must start with @, the text after the @ must be a valid record on the PGM.FILE. The PGM.FILE>TYPE must be 'S'. Field APPL.FOR.SUBR on the PGM record must equal the application defined in the key to the Version.Control record.
