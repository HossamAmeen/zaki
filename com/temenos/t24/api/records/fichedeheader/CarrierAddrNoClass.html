        public TField getCarrierSeqNo()
        Returns the CarrierSeqNo

        Field description: SWIFT and Telex use a sequence number to identify each message. The last sequence number used is held on the System Status record as SWIFT MF SEQUENCE and TELEX MF SEQUENCE for outgoing messages and SWIFT SF SEQUENCE and TELEX MF SEQUENCE for incoming messages. When a SWIFT or Telex message is sent, 1 is added to SWIFT MF SEQUENCE or TELEX MF SEQUENCE respectively and its value is stored in the Carrier Seq No for the appropriate copy of the message. Similarly the appropriate fields are updated when an inward message is received. Sites using ST400 - SWIFT 2 link have a session number added to the sequence number in the format 'XXXXXYYYYY' where 'XXXXX' is the Sequence Number and 'YYYYY' is the Session Number. Validation Rules: Fields CARRIER.ADDR.NO to TELEX BATCH NO form a related multi value set. 1-10 numeric characters. This is a NOINPUT field.

      • getCopyNumber

        public TField getCopyNumber()
        Returns the CopyNumber

        Field description: Specifies the ID of a message referred to by this set of data. It is usually the occurrence number within a multivalue set, but this is not the case if the copy has been subdivided (see FRAME NO). Example: A statement of more than 2000 characters is to be sent via SWIFT, but has gone into the repair queue because the customer's SWIFT address is not on file. Once the address has been added (to the DE.ADDRESS file), the message can be resubmitted. The new message will be split into two messages, each less than 2000 characters. The Msg Disposition of the first will start as 'Repair' and then change to 'Resubmitted' and the Copy Number will be 1. The Msg Disposition of the second and third sets will change from 'Formatted' to 'WACK' (awaiting acknowledgement) and after that to 'ACK'. Both will have Copy Numbers of 2 and Frame Numbers of 1 and 2. After the 1st attempt at Formatting: 26.1 COPY NUMBER 1 32.1 MSG DISPOSITION REPAIR 33.1 MSG ERROR CODE Customer 1234 address missing. After being successfully formatted and sent: 26.1 COPY NUMBER 1 32.1 MSG DISPOSITION RESUBMITTED 26.2 COPY NUMBER 2 27.2 FRAME NUMBER 1 32.2 MSG DISPOSITION ACK 26.3 COPY NUMBER 2 27.3 FRAME NUMBER 2 32.3 MSG DISPOSITION ACK For print carrier sets the field might optionally contain the customer id to whom the advice has been sent - if a MDR.CUSTOMER has been specified in the DE.PRODUCT. Validation Rules: Standard T24 numeric field with maximum of 3 characters. This is a NOINPUT field. Fields CARRIER.ADDR.NO to TELEX BATCH NO. form a related multi value set.

        public TField getFrameNumber()
        Returns the FrameNumber

        Field description: Specifies the sequence number of a segment from a SWIFT Message that has been divided up because it contains more than 2000 characters. The maximum number of characters allowed in a SWIFT message is 2000. If a Message exceeds 2000 characters but needs to be sent via SWIFT (this only occurs with statements, Message Type 950), then the SWIFT Formatting module automatically divides it up into 'Frames' of less than 2000 characters.
