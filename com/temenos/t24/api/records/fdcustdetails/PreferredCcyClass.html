      • addPrefBank

        public void addPrefBank(java.lang.CharSequence param)
        Appends the specified PrefBank to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: For the associated PREFERRED.CCY this field allows all the banks where the customer prefers the funds to be placed to be specified. If the customer makes a Fiduciary deposit and the order is the only in the pooled group used to generate the placement, the first preferred bank in the list will be used as the default FID.BANK when the placement is made, providing it is in the same currency as the PREFERRED.CCY. Validation Rules: 1-6 numeric customer number or 3-10 alphanumeric customer mnemonic. (Optional input.) No duplicates allowed within one currency. At least one bank is mandatory when the associated PREFERRED.CCY is present. Customer must exist on the Customer file and be defined as a BANK on the ACCOUNT.CLASS file. Cannot be defined as an EXCLUDED.BANK.

        Parameters:
        param - the PrefBank to append
        Returns:
        nothing
