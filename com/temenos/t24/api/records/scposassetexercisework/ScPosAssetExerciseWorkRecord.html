      • getAccruedInt

        public TField getAccruedInt()
        Returns the AccruedInt

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field contains the amount of interest (income) that has been accrued for the position but not yet been capitalised. If the position is for an account (AC) then this field will contain the sum of the ACCR.CR.AMOUNT, ACCR.CR2.AMOUNT, ACCR.DR.AMOUNT and ACCR.DR2.AMOUNT fields. If the position is for a Money Market (MM) or Loans and Deposits (LD) contract then if the contract is a deposit then this field will contain the amount from the OUTS.CUR.ACC.I.PAY from the LMM.ACCOUNT.BALANCES record. If the contract is a loan then this field will contain the OUTS.ACCRUED.INT field from the LMM.ACCOUNT.BALANCES record for the contract. If the position represents a Fiduciary contract (FD) then this field will be updated from the OTS.INTEREST field on the FD.BALANCES record for the contract. If the position represents a FOREX (FX) contract or a Forward Rate Agreement then this field will contain 0. This field will not be updated if the position represents a Miscellaneous Deal. If this position is for a Security then this field represents the amount of accrued interest on the security position. If the position is for a Swap (SW) contract then then this field will contain the amount from the field INTEREST.AMOUNT from the SWAP.BALANCES record of the side of the SWAP transaction represented by this set of fields. Validation Rules: No input field. Automatically updated by the system whenever the portfolio valuation data is rebuilt.

      • getForwardAccAmt

        public TField getForwardAccAmt()
        Returns the ForwardAccAmt

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field is only updated if the position is for an Account. The value dated account balance (contained in the V.DATE.NOMINAL field) is taken from the ONLINE.ACTUAL.BAL field on the ACCOUNT file. The trade dated account balance (held in the NO.NOMINAL field) includes any accounting movements resulting from the Securities module except dividends and coupons that have not yet reached Ex-Div Date. This is because the Securities movements these accounting movements relate to will be included in the portfolio valuation as a trade dated security position. Thus this field contains the difference between the NO.NOMINAL and V.DATE.NOMINAL fields for an Account position. Validation Rules: No input field. Automatically updated by the system.

If the position is for a Money Market (MM) contract then this field will contain the principal amount from the PRINCIPAL field on the MM.MONEY.MARKET record that gave rise to the position. If the position is for a Miscellaneous Deal (MD) then this field will always contain zero. If the position is for a Security positin then this field will contain an estimated value of the security position calculated using the market price in the associated MARKET.PRICE field. If the security position is short then the figure will be preceded by a '-' sign. This figure will only include securities that have updated the security position via transactions that have settled before the next working day. If the position is for a Swap contract (SW) then this field will contain the Net Present Value of the side of the SWAP transaction represented by this set of fields.         Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field contains a percentage figure equating to the yield of the position. This figure contains different data depending on the module that gave rise to the portfolio position; If the position has been generated from a Fiduciary contract (FD) the this field will contain the INTEREST.RATE from the FD.FID.ORDER record that gave rise to the position. If this position has been generated from a LOANS.AND.DEPOSITS (LD) contract then this field is taken from the INTEREST.RATE field on the LOANS.AND.DEPOSIT record that gave rise to the position. If this position has been generated from a MONEY.MARKET (MM) contract then this field is taken from the INTEREST.RATE field on the MONEY.MARKET record that gave rise to the position. If this position has been generated from a Security Position then if the Security giving rise to the position is a share then this field will be updated from the PROCE.EARN.RATIO field on the SECURITY.SUPP record for the security. If the security is a bond then this field will be populated from the YIELD.TO.MAT field in the SECURITY.SUPP record for the bond. If the YIELD.TO.MAY figure is blank then it will calculate a yield using the Interest Rate from the SECURITY.MASTER record for the bond and the cost price for the position. If this position has been generated from a Swap (SW) contract then this field is taken from the INTEREST.RATE field from the SWAP.BALANCES record, of the side of the SWAP transaction represented by this set of fields.
