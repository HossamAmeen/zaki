      • getChgCalcBasis

        public TField getChgCalcBasis()
        Returns the ChgCalcBasis

        Field description: This field is meant for the user to define the basis of calculation of the charge. Depending on the value in this field, the system will derive the CHG.BASE.AMT. In other words, the value in this field will determine the base amount on which the charge is to be calculated. Accepted values are FACILITY.ORIG/FACILITY.OS/LOAN.ORIG/LOAN.OS/PART.SHARE /ADHOC. FACILITY.ORIG would mean that charges would be calculated on the original facility amount. FACILITY.OS would mean that charges would be calculated on the outstanding facility amount or in other words the undrawn portion of the facility. LOAN.ORIG would mean that charges would be calculated on the original loan amount. LOAN.OS would mean that charges would be calculated on the outstanding loan amount. PART.SHARE would mean that charges would be calculated participant wise taking the paticipant's share as the base amount. ADHOC would mean that the charges would be calculated based on the amount input by the user. The system would not derive any amount for calculation when this option is chosen and the user input of a CHG.BASE.AMT is mandatory. Validation Rules: Allowed inputs are FACILITY.ORIG, FACILITY.OS, LOAN.ORIG, LOAN.OS, PART.SHARE, ADHOC. If ADHOC is given, the CHG.BASE.AMT is mandatory input.

        Parameters:
        Returns:
        TField - The ChgCalcBasis

        public void setChgCalcBasis(TField value)
        Set the specified value for ChgCalcBasis
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field is meant for the user to define the basis of calculation of the charge. Depending on the value in this field, the system will derive the CHG.BASE.AMT. In other words, the value in this field will determine the base amount on which the charge is to be calculated. Accepted values are FACILITY.ORIG/FACILITY.OS/LOAN.ORIG/LOAN.OS/PART.SHARE /ADHOC. FACILITY.ORIG would mean that charges would be calculated on the original facility amount. FACILITY.OS would mean that charges would be calculated on the outstanding facility amount or in other words the undrawn portion of the facility. LOAN.ORIG would mean that charges would be calculated on the original loan amount. LOAN.OS would mean that charges would be calculated on the outstanding loan amount. PART.SHARE would mean that charges would be calculated participant wise taking the paticipant's share as the base amount. ADHOC would mean that the charges would be calculated based on the amount input by the user. The system would not derive any amount for calculation when this option is chosen and the user input of a CHG.BASE.AMT is mandatory. Validation Rules: Allowed inputs are FACILITY.ORIG, FACILITY.OS, LOAN.ORIG, LOAN.OS, PART.SHARE, ADHOC. If ADHOC is given, the CHG.BASE.AMT is mandatory input.
