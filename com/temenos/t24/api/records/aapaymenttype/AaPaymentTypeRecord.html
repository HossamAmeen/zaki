      • getCalcType

        public TField getCalcType()
        Returns the CalcType

        Field description: This fields denotes different types of calculated payments that could be applied. T24 supports five types of calculated payments: • Constant – results in equal repayments. This is used for ‘Annuity’ arrangements and requires both a Term Amount and Interest property to be specified. • Linear – the Term Amount repayment remains fixed over the lifetime of the arrangement. Optional properties such as Interest, Charge, and Tax may be included. The actual amounts calculated by these properties will be added to the fixed Term Amount repayment. • Actual – is used for repayment of calculated property classes (i.e. Interest, Charge, and Tax) and will be determined on each payment schedule date. • Progressive – On choosing this type, a progressive percentage should be defined in the field PROG.PAY.PERC. System will calculate an amount which will be progressing by the rate specified in PROG.PAY.PERC. Exactly same as CONSTANT type when PROG.PAY.PERC is defined as 0. Optionally user routine could be attached for this type. • Other – indicates that a user routine will calculate the repayment amounts. • Accelerated - In this calculation type payment amount would be calculated using Monthly frequency (e0Y e1M e0W e0D e0F) even if user has given the payment frequency as weekly or Bi-weekly. Then the amount calculated for monthly will then be factored to the payment frequency defined in payment schedule to arrive at the final payment amount for each period i.e. divided by 2 or 4 based on whether frequency is Bi-weekly or weekly.

        This field is Mandatory.

        Options are CONSTANT, LINEAR, ACTUAL, PROGRESSIVE, ACCELERATE or OTHER.

        • Fixed Equal – is a blended payment that mandatorily includes principal and interest components and optionally can include charges. The FER is the amount the customer has adviced / agreed to repay towards the loan on a regular basis. In case of FER, the term can be defined or left blank for system to calculate the term based on FER specified.

        • TRANSACTION - is used to generate due bill whenever there is a Disbursement activity on an loan Arrangement and the due amount calculated based on the disbursement amount. Account Property class and Schedule type are mandatory.

        Returns:
        TField - The CalcType

      • getType

        public TField getType()
        Returns the Type

        Field description: This field indicates whether the payment type is ‘Calculated’ or ‘Manual’. The majority of the defined payment types will be defined as ‘Calculated’. During the setup of each arrangement it may be possible for the user to negotiate a different payment amount than the one determined by T24. For arrangements with ad-hoc payment schedules a financial institution can create a ‘Manual’ payment type which indicates that the payment amounts will be entered manually and not be calculated by T24.

        This field is Mandatory.

        Input allowed is CALCULATED or MANUAL.

      • setCalcType

        public void setCalcType(TField value)
        Set the specified value for CalcType
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This fields denotes different types of calculated payments that could be applied. T24 supports five types of calculated payments: • Constant – results in equal repayments. This is used for ‘Annuity’ arrangements and requires both a Term Amount and Interest property to be specified. • Linear – the Term Amount repayment remains fixed over the lifetime of the arrangement. Optional properties such as Interest, Charge, and Tax may be included. The actual amounts calculated by these properties will be added to the fixed Term Amount repayment. • Actual – is used for repayment of calculated property classes (i.e. Interest, Charge, and Tax) and will be determined on each payment schedule date. • Progressive – On choosing this type, a progressive percentage should be defined in the field PROG.PAY.PERC. System will calculate an amount which will be progressing by the rate specified in PROG.PAY.PERC. Exactly same as CONSTANT type when PROG.PAY.PERC is defined as 0. Optionally user routine could be attached for this type. • Other – indicates that a user routine will calculate the repayment amounts. • Accelerated - In this calculation type payment amount would be calculated using Monthly frequency (e0Y e1M e0W e0D e0F) even if user has given the payment frequency as weekly or Bi-weekly. Then the amount calculated for monthly will then be factored to the payment frequency defined in payment schedule to arrive at the final payment amount for each period i.e. divided by 2 or 4 based on whether frequency is Bi-weekly or weekly.

        This field is Mandatory.

        Options are CONSTANT, LINEAR, ACTUAL, PROGRESSIVE, ACCELERATE or OTHER.

        • Fixed Equal – is a blended payment that mandatorily includes principal and interest components and optionally can include charges. The FER is the amount the customer has adviced / agreed to repay towards the loan on a regular basis. In case of FER, the term can be defined or left blank for system to calculate the term based on FER specified.

        • TRANSACTION - is used to generate due bill whenever there is a Disbursement activity on an loan Arrangement and the due amount calculated based on the disbursement amount. Account Property class and Schedule type are mandatory.

        Parameters:
        value - The value to set
        Returns:
        nothing
