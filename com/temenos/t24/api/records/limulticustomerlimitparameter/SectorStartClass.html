      • getSectorStart

        public TField getSectorStart()
        Returns the SectorStart

        Field description: Customer segments for which rules differ from the default pre-condition. Specific sector codes (or) the Start and End values for a range of Sectors. When this field is set to yes then at the time of inputting a contract for the customer, the validation would check for all the possible matches, find whether there are any Limit records in existance. If only one is available then it will be defaulted at the LIMIT.REFERENCE of the contract. If there are more than one, the user is shown the choices thorough a drop-down list attached to the field LIMIT.REFERENCE in respective contract (eg: LD Contract). Validation Rules: SECTOR.START may be given without SECTOR.END but SECTOR.END not allowed without SECTOR.START.

        Returns:
