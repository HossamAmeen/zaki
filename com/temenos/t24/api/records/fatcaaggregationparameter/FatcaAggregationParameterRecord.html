      • addRelationCode

        public void addRelationCode(TField param)
        Appends the specified RelationCode to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: A number of relationships for a customer can be defined in T24. However, only certain relationships (Joint Owner, Same Owner, beneficial ownership, etc.) will have a bearing on the aggregated balance calculation. The relation codes, that need to be considered for aggregating the balances, need to be specified in this field. For example, if Customer 100291 has the following relationships defined: Relation Code: 9 Joint Owners Relation Customer: 100292 Relation Code: 7 Signatory Relation Customer: 100295 In T24, for 100292, the reverse relationship code of 99 (based on RELATION table set-up) will automatically be updated. Relation Code: 99 Joint Relation Customer: 100291 If both relation codes 9 and 99 are defined in Relation Code field, the aggregate balance for Customer 100291 will be arrived at by consolidating the balances of all accounts/deposits, positions, etc. held by both customers 100291 and 100292. The balance of 100292 will be the sum of all holdings/account balances of 100292 and 100291. If only relation code 9 is defined in parameter, the balance of 100292 will only consider the holdings of 100292 and not that of client 100291. The balances of account (if any) held by Customer 100295 will not be considered for aggregation Validation Rules: Should be a valid ID from RELATION table

        Parameters:
        param - the RelationCode to append
        Returns:
        nothing
