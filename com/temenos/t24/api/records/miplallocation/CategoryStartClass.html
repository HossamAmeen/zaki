      • getNetted

        public TField getNetted()
        Returns the Netted

        Field description: This field indicates whether MI.AUTO.ENTRY records should be netted when they are created. If this flag is set to VALUE then entries which have the same :

        CATEGORY
        ACCOUNT.OFFICER
        PRODUCT.CATEGORY
        BOOKING.DATE
        VALUE.DATE
        CURRENCY
        Will be netted together if the account officer from which the entries were created are the same, e.g. takes into account the value date and booking date of the incoming records. When this flag is set to MONTHLY the value.date is set to the end of the month it had value in and the booking date will be set to the system date on which the allocation was run. This option provides for a higher level of netting and will result in fewer records. Validation Rules: Only Input of VALUE, MONTHLY or NONE will be excepted. Optional Input. If left blank then the entry will not be netted.

        Parameters:
        Returns:
        TField - The Netted
