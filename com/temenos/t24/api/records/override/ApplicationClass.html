      • addConOverride

        public void addConOverride(java.lang.CharSequence param)
        Appends the specified ConOverride to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Conditional Override (CON.OVERRIDE) Dispo overrides can be conditional on whether the customer has sufficient funds on their accounts. If the customer has sufficient funds then other Dispo overrides such as &#145;Limit has expired&#146; can be prevented from going to DISPO. You can define whether certain overrides are dependent on other overrides. In this field the user can specify the other override that this particular override is dependent on. The user is able to enter more than one override per application. If an override has conditional overrides, and one of these conditional overrides is present on the contract, then the override will go to the dispo queue. However if none of the conditional overrides are on the contract then the override will not invoke dispo processing, that is, it will not go to the dispo queue. Validation Rules: Field is optional Must be a valid Dispo Override. Ie. DISPO.ALLOWED field is set to YES.

        Parameters:
        param - the ConOverride to append
        Returns:
        nothing

      • getTransactionInd

        public TField getTransactionInd()
        Returns the TransactionInd

        Field description: Transaction Code Indicator (TRANSACTION.IND) Certain types of accounting transactions are exempt from Dispo processing depending on the application used to generate the transaction/contract. For example, fees generated by a Funds Transfer transaction can cause the account to go into overdraft, this overdraft override is exempt from Dispo processing. This field indicates whether the exempt indicator on the transaction code should be used in determining if the override is exempt. If the field is set to YES, then the system must look at the exempt indicator on the transaction code. Validation Rules: Valid entries for this field are: YES, NO, or empty field. YES in this field indicates that the transaction file is checked for the exempt indicator NO, or null in this field indicates that the transaction file will not be checked, and the Dispo instructions specified on the OVERRIDE file should be followed

        Parameters:
        Returns:
        TField - The TransactionInd

        public void setTransactionInd(java.lang.CharSequence value)
        Set the specified value for TransactionInd
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Transaction Code Indicator (TRANSACTION.IND) Certain types of accounting transactions are exempt from Dispo processing depending on the application used to generate the transaction/contract. For example, fees generated by a Funds Transfer transaction can cause the account to go into overdraft, this overdraft override is exempt from Dispo processing. This field indicates whether the exempt indicator on the transaction code should be used in determining if the override is exempt. If the field is set to YES, then the system must look at the exempt indicator on the transaction code. Validation Rules: Valid entries for this field are: YES, NO, or empty field. YES in this field indicates that the transaction file is checked for the exempt indicator NO, or null in this field indicates that the transaction file will not be checked, and the Dispo instructions specified on the OVERRIDE file should be followed
