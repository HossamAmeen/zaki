      • getAgregateBal

        public TField getAgregateBal()
        Returns the AgregateBal

        Field description: This field is used in conjunction with CASHFLOW type rules. These are essentially the same as a Two Way sweep rule except there is a controlling limit on how much can be taken from higher level account. The CASHFLOW.AMT and AGREGATE.BAL fields control how much can be taken from the higher level account. There is no limit on how much can be transferred to the higher level account. Example A cashflow limit of USD 100,000.00 is set in field CASHFLOW.AMT. The link account is credited today with USD 50,000.00 and is automatically transferred up to the higher level account. On the next day the fields will show: CASHFLOW.AMT 100,000.00 AGREGATE.BAL 50,000.00 This means that the amount that can be taken to cover shortfalls is now USD 150,000.00 i.e. the net amount of the two fields. The next transaction causes an overdraft on the link account of USD 200,000.00 so ideally it would need that amount to bring the balance to zero (it's minimum balance). However, as only USD 150,000.00 is allowed that amount is transferred down. The next day the fields will then show: CASHFLOW.AMT 100,000.00 AGREGATE.BAL -100,000.00 So now the net amount is zero so no further funds can be taken until there have been more transfers to the higher level account to change the AGREGATE.BAL value. Note the CASHFLOW.AMT is not changed by the system and would remain as USD 100,000.00. For take-on purposes the AGREGATE.BAL field can be given a starting value at first input only, once authorised the field is no longer changeable by the user. Validation Rules: Only valid for CASHFLOW rule types. Initial input allowed, no change to authorised records.

      • getRule

        public TField getRule()
        Returns the Rule

        Field description: This determines what type of sweeping will occur within the pool between the source account and the link account. The value entered is the id of an AC.SWEEP.TYPE record which is user configurable, the actual type is set within that record. However, they must be based on one of four types of rules used within cash pooling: MAINTENANCE If the source account balance falls below the specified amount in the minimum amt field then the link account is debited and the source account credited with the amount required (subject to user controlled conditions) or with the funds available. SURPLUS If the balance in the source account is above the amount specified in the maximum amt field then the surplus is debited from the source account and credited to the link account. TWO-WAY This is like a combination of the two above. First the balances are checked for a maintenance sweep. If the specified balance is not below the minimum amount then it will then be checked for a surplus amount. CASHFLOW A cash flow type rule will only occur between those accounts linked to the main master account. It is similar to the two way rule but with a restriction on the amount of money that the Main Master (MM) account can pass down to the account below it. Basically when a pool is being set up a limit will be set on the maximum amount that the main master will pass down. This is held in the CASHFLOW.AMT field. There is an additional check on how much money has been passed up to the MM. This running total of monies transferred is held in the AGREGATE.BAL field. The available amount is the net of the CASHFLOW.AMT and AGREGATE.AMT fields. Example A cashflow limit of USD 100,000.00 is set in field CASHFLOW.AMT. The link account is credited today with USD 50,000.00 and is automatically transferred up to the higher level account.       • setRule

        public void setRule(java.lang.CharSequence value)
        Set the specified value for Rule
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This determines what type of sweeping will occur within the pool between the source account and the link account. The value entered is the id of an AC.SWEEP.TYPE record which is user configurable, the actual type is set within that record. However, they must be based on one of four types of rules used within cash pooling: MAINTENANCE If the source account balance falls below the specified amount in the minimum amt field then the link account is debited and the source account credited with the amount required (subject to user controlled conditions) or with the funds available. SURPLUS If the balance in the source account is above the amount specified in the maximum amt field then the surplus is debited from the source account and credited to the link account. TWO-WAY This is like a combination of the two above. First the balances are checked for a maintenance sweep. If the specified balance is not below the minimum amount then it will then be checked for a surplus amount. CASHFLOW A cash flow type rule will only occur between those accounts linked to the main master account. It is similar to the two way rule but with a restriction on the amount of money that the Main Master (MM) account can pass down to the account below it. Basically when a pool is being set up a limit will be set on the maximum amount that the main master will pass down. This is held in the CASHFLOW.AMT field. There is an additional check on how much money has been passed up to the MM. This running total of monies transferred is held in the AGREGATE.BAL field. The available amount is the net of the CASHFLOW.AMT and AGREGATE.AMT fields. Example A cashflow limit of USD 100,000.00 is set in field CASHFLOW.AMT. The link account is credited today with USD 50,000.00 and is automatically transferred up to the higher level account. On the next day the fields will show: CASHFLOW.AMT 100,000.00 AGREGATE.BAL 50,000.00 This means that the amount that can be taken to cover shortfalls is now USD 150,000.00 i.e. the net amount of the two fields. The next transaction causes an overdraft on the link account of USD 200,000.00 so ideally it would need that amount to bring the balance to zero (it's minimum balance). However, as only USD 150,000.00 is allowed that amount is transferred down. The next day the fields will then show: CASHFLOW.AMT 100,000.00 AGREGATE.BAL -100,000.00 So now the net amount is zero so no further funds can be taken until there have been more transfers to the higher level account to change the AGREGATE.BAL value. Note the CASHFLOW.AMT is not changed by the system and would remain as USD 100,000.00. Validation Rules: Must be a valid ID on the AC.SWEEP.TYPE file. If GROUP.ID has BACK.VALUE.ADJ as YES in AC.CP.GROUP.PARAM, then CASHFLOW rule not allowed.
