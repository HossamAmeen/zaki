        public TField getDate()
        Returns the Date

        Field description: This field indicates the date of the event. If a multiple period contract is involved and the date inserted in this field does not represent the maturity date this field will represent the end date of the period and the starting date for the next period. Validation Rules: Standard date format. A valid date is mandatory for each schedule type. If a date is input it must be a valid date format. An override will be required if the date is not defined on the HOLIDAY table. An override will be required if the date, as defined on the HOLIDAY table, is a non-working day. Input prohibited for dates prior to the value in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. Input prohibited for dates later than the value in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. Schedules can only be defined for dates greater than the process date except for I + C schedules when the process date may equal an interest period end date as in the case of back valued contracts. "P" schedule date must be greater than the date indicated in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "P" schedule mandatory for the date indicated in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "P" schedule is prohibited on any date other than the date indicated in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS if the interest basis is "S" (Special). "P" and "I" schedule dates must be greater than process date + liquidation currency delivery days, if a Nostro payment for Deposits/Syndicated Loans is required. "P" and "I" schedule periods must be greater than the liquidation currency delivery days, if a Nostro payment for Deposits/Syndicated Loans is required. "I" schedule for a discounted contract is prohibited on the date indicated in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule for a discounted contract is mandatory on the date indicated in field VALUE.DATE on LD.LOANS.AND.DEPOSITS if VALUE.DATE is greater than the process date. "I" schedule on an interest bearing contract is prohibited for the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule on a Commitment contract is prohibited for the value in field INT.START.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule on an interest bearing contract is mandatory for the value in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule on a Commitment contract is mandatory for the value in field FIN.MAT.DATE OR the field INT.END.DATE if earlier. "C" schedule on a discounted contract is mandatory for the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS if value date is a forward date. "C" schedule on an interest bearing contract is prohibited for the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "C" schedule on an interest bearing commitment is prohibited for the value in the field INT.START.DATE on LD.LOANS.AND.DEPOSITS. "C" schedule on an interest bearing commitment is mandatory for the value in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS, OR the field INT.END.DATE whichever is least. "D" schedule is prohibited after the value in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "D" schedule is prohibited before the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "D" schedule is prohibited before the process date. "R" schedule is prohibited for a value prior to or equal to the process date. "R" schedule must be after the value in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "R" schedule must be after the value in the field INT.START.DATE on LD.LOANS.AND.DEPOSITS. "R" schedule is prohibited on or after the value in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "R" schedules on non-working days prohibited for PERIODIC.AUTOMATIC. "R" schedules prohibited on or after the value in the field INT.END.DATE on LD.LOANS.AND.DEPOSITS. "M" schedules are prohibited on the value in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "M" schedules must be after the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "F" schedules are prohibited for the value in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "F" and "N" schedules are prohibited prior to process date. "I" & "C" dates must be defined together if both interest and commission are discounted. Annuity contracts with inclusive commission must have "C" schedules date frequencies matching the "A" frequency.

      • getSchType

        public TField getSchType()
        Returns the SchType

        Field description: This field indicates the schedule type that can be created by the user. The user can input a single schedule or a valid combination of schedules eg PIC except the A schedule. Validation Rules: A combination of up to 3 schedule types each represented by a single alphabetic character of the form: A = Annuity (principal + interest) P = principal B = Principal Increase I = interest C = commission R = revision M = margin F = fees N = capitalised charge D = diary K = Asset Leasing schedule. If a schedule type is input this must be of the form A, P, B, I, C, M, D, F, N, K, or R, or a valid combination of up to three e.g. PIC. "P" schedules are only allowed for commitment contracts where the COMMIT.DRAWING field is set to NO. "B" schedules are prohibited for Drawing commitment contracts and if interest basis is set to 'S'. "N" shedules are prohibited for Commitment contract. "C" schedules are prohibited for Deposit contracts. "R" and "I" schedules are prohibited if no interest is due on the Contract. "C" schedules are prohibited if no commission is due on the contract. "P" & "I" & "C" schedules cannot be defined together as "PIC" when interest and commission is/are discounted. "R" schedules cannot be combined with "I" schedules when interest is discounted as "R" schedules are prohibited for system date. Multiple "I" schedules must be defined if the value in field CAPITALISATION = (Y)es on LD.LOANS.AND.DEPOSITS. "I" schedules must be defined alone if interest basis = "S". "C" schedules must be defined alone if commission basis = "S". "R" schedules are mandatory for PERIODIC.AUTOMATIC Interest Rate type. "R" schedules are mandatory for PERIODIC.MANUAL Interest Rate type. "R" schedules are prohibited for PERIODIC.STRAIGHT Interest Rate type. "R" schedules are prohibited from amending the next "R" date for PERIODIC.AUTOMATIC Interest Rate type. "R" schedules are prohibited for FLOATING Interest Rate type contracts. "R" schedules are prohibited if Interest Basis is "S". "R" - for PERIODIC AUTOMATIC Interest Rate type only; a new "R" Schedule can only be input if an "R" Schedule is still outstanding. The new Schedule must be later than the existing Schedule. "M" schedules are prohibited when no Interest due on the Contract. "M" schedules are prohibited for Fixed Interest Rate type. "M" schedules are prohibited for Commitments. "K" schedules are only allowed for finance leasing annuity contracts, where a financial charge ( depreciation) is required and must be input after the "A" schedule is defined. Input can only take place for the following financial charge types: - "RB" - reducing balance method, the dates and frequency must conform to the "A" schedule type, a depreciation rate must be input. "FS" - flexible schedule method, using this method different rates can be applied over the life of the contract, and therefore a new "K" schedule must be input for each rate change. "AN" and "SL" type financial charges do not require "K" schedules to be input as the schedule dates will conform to the "A" schedules, and no rate input is required.

        Set the specified value for Date
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field indicates the date of the event. If a multiple period contract is involved and the date inserted in this field does not represent the maturity date this field will represent the end date of the period and the starting date for the next period. Validation Rules: Standard date format. A valid date is mandatory for each schedule type. If a date is input it must be a valid date format. An override will be required if the date is not defined on the HOLIDAY table. An override will be required if the date, as defined on the HOLIDAY table, is a non-working day. Input prohibited for dates prior to the value in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. Input prohibited for dates later than the value in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. Schedules can only be defined for dates greater than the process date except for I + C schedules when the process date may equal an interest period end date as in the case of back valued contracts. "P" schedule date must be greater than the date indicated in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "P" schedule mandatory for the date indicated in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "P" schedule is prohibited on any date other than the date indicated in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS if the interest basis is "S" (Special). "P" and "I" schedule dates must be greater than process date + liquidation currency delivery days, if a Nostro payment for Deposits/Syndicated Loans is required. "P" and "I" schedule periods must be greater than the liquidation currency delivery days, if a Nostro payment for Deposits/Syndicated Loans is required. "I" schedule for a discounted contract is prohibited on the date indicated in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule for a discounted contract is mandatory on the date indicated in field VALUE.DATE on LD.LOANS.AND.DEPOSITS if VALUE.DATE is greater than the process date. "I" schedule on an interest bearing contract is prohibited for the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule on a Commitment contract is prohibited for the value in field INT.START.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule on an interest bearing contract is mandatory for the value in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "I" schedule on a Commitment contract is mandatory for the value in field FIN.MAT.DATE OR the field INT.END.DATE if earlier. "C" schedule on a discounted contract is mandatory for the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS if value date is a forward date. "C" schedule on an interest bearing contract is prohibited for the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "C" schedule on an interest bearing commitment is prohibited for the value in the field INT.START.DATE on LD.LOANS.AND.DEPOSITS. "C" schedule on an interest bearing commitment is mandatory for the value in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS, OR the field INT.END.DATE whichever is least. "D" schedule is prohibited after the value in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "D" schedule is prohibited before the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "D" schedule is prohibited before the process date. "R" schedule is prohibited for a value prior to or equal to the process date. "R" schedule must be after the value in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "R" schedule must be after the value in the field INT.START.DATE on LD.LOANS.AND.DEPOSITS. "R" schedule is prohibited on or after the value in the field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "R" schedules on non-working days prohibited for PERIODIC.AUTOMATIC. "R" schedules prohibited on or after the value in the field INT.END.DATE on LD.LOANS.AND.DEPOSITS. "M" schedules are prohibited on the value in field FIN.MAT.DATE on LD.LOANS.AND.DEPOSITS. "M" schedules must be after the value in field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "F" schedules are prohibited for the value in the field VALUE.DATE on LD.LOANS.AND.DEPOSITS. "F" and "N" schedules are prohibited prior to process date. "I" & "C" dates must be defined together if both interest and commission are discounted. Annuity contracts with inclusive commission must have "C" schedules date frequencies matching the "A" frequency.

        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field indicates the schedule type that can be created by the user. The user can input a single schedule or a valid combination of schedules eg PIC except the A schedule. Validation Rules: A combination of up to 3 schedule types each represented by a single alphabetic character of the form: A = Annuity (principal + interest) P = principal B = Principal Increase I = interest C = commission R = revision M = margin F = fees N = capitalised charge D = diary K = Asset Leasing schedule. If a schedule type is input this must be of the form A, P, B, I, C, M, D, F, N, K, or R, or a valid combination of up to three e.g. PIC. "P" schedules are only allowed for commitment contracts where the COMMIT.DRAWING field is set to NO. "B" schedules are prohibited for Drawing commitment contracts and if interest basis is set to 'S'. "N" shedules are prohibited for Commitment contract. "C" schedules are prohibited for Deposit contracts. "R" and "I" schedules are prohibited if no interest is due on the Contract. "C" schedules are prohibited if no commission is due on the contract. "P" & "I" & "C" schedules cannot be defined together as "PIC" when interest and commission is/are discounted. "R" schedules cannot be combined with "I" schedules when interest is discounted as "R" schedules are prohibited for system date. Multiple "I" schedules must be defined if the value in field CAPITALISATION = (Y)es on LD.LOANS.AND.DEPOSITS. "I" schedules must be defined alone if interest basis = "S". "C" schedules must be defined alone if commission basis = "S". "R" schedules are mandatory for PERIODIC.AUTOMATIC Interest Rate type. "R" schedules are mandatory for PERIODIC.MANUAL Interest Rate type. "R" schedules are prohibited for PERIODIC.STRAIGHT Interest Rate type. "R" schedules are prohibited from amending the next "R" date for PERIODIC.AUTOMATIC Interest Rate type. "R" schedules are prohibited for FLOATING Interest Rate type contracts. "R" schedules are prohibited if Interest Basis is "S". "R" - for PERIODIC AUTOMATIC Interest Rate type only; a new "R" Schedule can only be input if an "R" Schedule is still outstanding. The new Schedule must be later than the existing Schedule. "M" schedules are prohibited when no Interest due on the Contract. "M" schedules are prohibited for Fixed Interest Rate type. "M" schedules are prohibited for Commitments. "K" schedules are only allowed for finance leasing annuity contracts, where a financial charge ( depreciation) is required and must be input after the "A" schedule is defined. Input can only take place for the following financial charge types: - "RB" - reducing balance method, the dates and frequency must conform to the "A" schedule type, a depreciation rate must be input. "FS" - flexible schedule method, using this method different rates can be applied over the life of the contract, and therefore a new "K" schedule must be input for each rate change. "AN" and "SL" type financial charges do not require "K" schedules to be input as the schedule dates will conform to the "A" schedules, and no rate input is required.
