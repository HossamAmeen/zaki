      • getConValDate

        public TField getConValDate()
        Returns the ConValDate

        Field description: Multi-valued field associated with the previous five fields. Used in the consolidation of forex amounts for theSEC.TRADE transaction. Input only allowed if the CONSOLIDATE.FX field on the SC.STD.SEC.TRADE file is set to Y and there are accountsused in the SEC.TRADE that have a currency that is different from the trade currency of SEC.TRADE. Allows input of the value date associated with the forex movement specified in the provious five fields. Willdefault to the settlement date of the transaction. There will be one set of values in these six fields for each currency one of more customers are settling in ifthe settlement currency is different from the trade currency. Automatically updated by the system during input. Validation Rules: Input only allowed if the CONSOLIDATE.FX field on SC.STD.SEC.TRADE is set to Y.

        Returns:

        Returns the ConsolAmt

        Field description: Multi-valued no input field associated with the previous field and the next four fields. Used in the consolidation of FOREX amounts in the SEC.TRADE transaction.       • getConsolRate

        public TField getConsolRate()
        Returns the ConsolRate

        Field description: Multi-valued field associated with the previous three fields and the next two fields. Used in the consolidationof forex amounts for the SEC.TRADE transaction. Input only allowed if the CONSOLIDATE.FX field on the SC.STD.SEC.TRADE file is set to Y and there are accountsused in the SEC.TRADE that have a currency that is different from the trade currency of SEC.TRADE. Input of an exchange rate to be used between the associated currencies in the TRADE.CURR and SETTLE.CURR field.This exchange rate will be used to calculate the amount to be debited from accounts settling the associatedSETTLE.CURR. Input in this field will cause the SETTLE.AMT field to be updated as the equivalent to the amount inthe associated CONSOL.AMT field at this exchamge rate. There will be one set of values in these six fields for each currency one of more customers are settling in ifthe settlement currency is different from the trade currency. Automatically updated by the system during input. Validation Rules: Mandatory field if the CONSOLIDATE.FX field on SC.STD.SEC.TRADE is set to Y. Otherwise a no input field.

        Returns:
