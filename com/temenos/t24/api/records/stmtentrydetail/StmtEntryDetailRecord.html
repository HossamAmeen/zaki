        Field description: A multivalue field linking together all the Statement Entry records and Category Entry records generated at the same time by the same transaction/deal. If a transaction generates several Entries, all the Entries have the same Statement Number, the Entries on the same file are distinguished from each other by their Sequence Numbers. The Statement Number consists of: 1. 4 digit date expressed as the number of days since 31st December 1967. 2. Time in seconds since midnight, consisting of 5 integers and (usually) 2 decimals. If the time is an exact number of seconds since midnight, the decimals are omitted. The Sequence number is a 4 digit number from 0001 to 9999. Every Entry generated by the same transaction/deal has the same values in Fields 26.1 to 26.3. Field 26.1 contains the Statement Number which is the first part of the key for all the Statement Entries and Category Entries. Field 26.2 contains the range of Sequence Numbers used for the Statement Entries. If there are no Statement Entries, this field will be blank. Field 26.3 contains the range of Sequence Numbers used for the Category Entries. If there are no Category Entries, this field will be blank. Example: If a transaction processed on 3rd June 1986 at 30.5 seconds after 11.15 a.m. generates 3 Statement Entries and 2 Category Entries - The Statement Number is 672940530.50 made up as follows - Date 6729 Time 40530.50 The Entries generated have the following Ids - Statement Entries Category Entries 672940530.500001 672940530.500001 672940530.500002 672940530.500002 672940530.500003 All the Entries have the following values in Fields 26.1 to 26.3 - Field 26.1 672940530.50 Field 26.2 1-3 (3 Statement Entries) Field 26.3 1-2 (2 Category Entries)
