      • getBaseRoundType

        public TField getBaseRoundType()
        Returns the BaseRoundType

        Field description: This field associated with the field CURRENCY may be used when interest, commission, charges etc on which this tax is applicable are required to be rounded. The type specified in this field along with the value in the associated field BASE. ROUND. UNIT would be used to round the base amount for tax calculation. The Types that may be used are: 1.NATURAL 2.LOWER 3.HIGHER Natural Natural-rounding rules will be applied depending upon the rounding unit. E.g. If rounding unit is 10 and interest/ commission calculated is 246 then commission is rounded off to 250 for calculation of tax. Lower Always rounded off to the next lowest rounding unit E.g. if rounding unit is 10 and interest/commission calculated is 246 then commission is rounded off to 240 for calculation of tax. Higher Always rounded off to the next highest rounding unit. E.g.. If rounding unit is 10 and interest/commission calculated is 246 then commission is rounded off to 250 for calculation of tax. Validation Rules: HIGHER, LOWER or NATURAL or any abbreviation thereof. Optional input This field is mandatory if BASE. ROUND. UNIT is entered.

        Parameters:
        Returns:
        TField - The BaseRoundType

      • getCurrency

        public TField getCurrency()
        Returns the Currency

        Field description: Specifies the currency of the associated banded tax specifications. Multi-valued field associated with CALC.TYPE, BANDED.RATE, UPTO.AMT, MINIMUN.AMT, MAXIMUM.AMT, OVERALL.MIN.AMT and OVERALL.MAX.AMT. If a specific currency is not represented within the multivalue set, the System will default to the details associated to the DEFAULT.CCY. The currency equivalent amount will be derived at the currency table MID.REVAL.RATE. Validation Rules: 3 characters (uppercase alpha) - type SSS or 1-3 numeric characters Currency Code. The currency code entered must appear on the Currency table. Mandatory input for banded tax specification. No input if a flat RATE is specified.

        Parameters:
        Returns:
        TField - The Currency

      • getOverallMaxAmt

        public TField getOverallMaxAmt()
        Returns the OverallMaxAmt

        Field description: Defines the Overall Maximum Amount to be charged on tax involving the Currency defined. This amount represents the maximum to be charged irrespective of any special conditions which could exist for a Customer or group of Customers except if these have been specified as a Flat amount. This amount could represent the maximum amount the bank would ever conceivably charge a customer for a transfer regardless of the size of the Transfer Amount. Validation Rules: Up to 14 AMT type characters (standard amount format). Associated multivalue field. (Optional input.) It must be greater than the Overall Minimum Amount. It must be greater or equal to the last Maximum Amount specified within a range for the Currency.
