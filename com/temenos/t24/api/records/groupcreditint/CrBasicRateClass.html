      • getCrBasicRate

        public TField getCrBasicRate()
        Returns the CrBasicRate

        Field description: Specifies a floating Basic Rate which is held on the Basic Interest Rate table. Rates can be fixed or linked to Basic Rates plus or minus a Margin. Note : Fields CR.BASIC.RATE to CR.LIMIT.AMT are used to specify rates applicable to different balance limits. Validation Rules: 1-4 numeric characters. Multivalue field associated with Fields CR.BASIC.RATE to CR.LIMIT.AMT. (Optional input. Not allowed if corresponding CR.INT.RATE, is input or INTEREST.DAY.BASIS contains 'NONE'.) There must be a record in the BASIC.INTEREST table (Ref: General Tables) with a key consisting of this code, the Currency of the Account and an effective date which is less that or equal to the date specified in Field 0, (Record ID). Examples: The rate or rates of interest which will be applied are defined by the following multivalue group of fields: BASIC RATE INT RATE MARGIN OPER MARGIN RATE LIMIT AMOUNT Examples of their use is as follows: 1) If the interest rate is to be equal to a floating Basic Rate, only enter the following field: BASIC RATE - Code of the applicable Basic Rate. 2) If the interest rate is to be fixed, only enter the following field: INT RATE - the actual rate of interest. 3) If the interest rate is to be equal to a floating Basic Rate minus a Margin; enter the following fields: BASIC RATE - Code of the applicable Base Rate MARGIN OPER - 'SUBTRACT' MARGIN RATE - the Marginal Rate. 4) If the interest rate is to be equal to a floating Basic Rate plus a Percentage of the floating Basic Rate; enter the following fields: BASIC RATE - Code of the applicable Basic Rate MARGIN OPER - 'MULTIPLY' MARGIN RATE - Percentage to be added 5) If the interest rate is to be equal to a floating Basic Rate, minus a Margin up to a limiting amount and a floating Basic Rate, minus a different Margin above that limiting amount, enter the following fields: BASIC RATE - Code of the applicable Basic Rate MARGIN OPER - 'SUBTRACT' MARGIN RATE - Rate to be subtracted LIMIT AMOUNT - The limit amount for the above defined rate BASIC RATE - Code of the applicable Basic Rate MARGIN OPER - 'SUBTRACT' MARGIN - Rate to be subtracted.

        Parameters:
        Returns:
        TField - The CrBasicRate

      • getCrMarginOper

        public TField getCrMarginOper()
        Returns the CrMarginOper

        Field description: Determines the method by which the Rate specified in CR.BASIC.RATE or CR.INT.RATE, is to be adjusted by the Margin specified in CR.MARGIN.RATE. If the Rate specified in CR.BASIC.RATE or CR.INT.RATE, is to be adjusted by a Margin, this field specifies whether the Margin Rate is to be added to, subtracted from or multiplied by the Basic Rate to arrive at the effective interest rate. Note : Fields CR.BASIC.RATE to CR.LIMIT.AMT are used to specify rates applicable to different balance limits. If 'Multiply' is specified the final rate is (100 + M) percent of the Rate from CR.BASIC.RATE or CR.INT.RATE where M is the Margin Rate percentage from CR.MARGIN.RATE. Note : For examples see CR.BASIC.RATE. Validation Rules: 'ADD', 'SUBTRACT' or 'MULTIPLY', or any abbreviations thereof. Multivalue field associated with fields CR.BASIC.RATE to CR.LIMIT.AMT. (Optional input. Only allowed if corresponding CR.BASIC.RATE or CR.INT.RATE, is input.)

        Parameters:
        Returns:
        TField - The CrMarginOper

        public TField getCrMaxRate()
