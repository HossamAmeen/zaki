      • getRelField

        public TField getRelField()
        Returns the RelField

        Field description: Specifies the field in the main file record whose contents are to be used to build the keys to the related file. This field begins a multivalue set designed to build a key to the related file. This key can therefore be constructed using parts of field contents from the main file or bits of text. The key so built is then used to select the related file under a defined condition (see REL.SELECTION). If the "ALL" keyword is specified then all fields from the main file record will be used as keys to the related file. If "0" is entered the key to the main file record will be used as the key to the related file. If a text string is entered it is appended to what existed from previous extractions within the subvalue set. A field number if entered causes the contents of that field from the main file record to be used as a key to the related file. This may include specific value numbers or subvalue numbers from the field specified e.g. 23.2.3. Validation Rules: 1-3 numeric field number or text "ALL" or 1-35 alphanumeric text in quotes "aaaaaa" No input without related file defined. If "ALL" keyword is specified it must be first occurrence in the subvalue set. Text input must be contained in quotes.

        Parameters:
        Returns:
        TField - The RelField

      • getRelModCall

        public TField getRelModCall()
        Returns the RelModCall

        Field description: Specifies the name of a routine which may be called to modify the contents of the field from the main file. Or may specify a string or field extraction. String extraction, e.g. [2,8]23L - extracts characters 2 to 2+8 from a string length 23 left justified. Field extraction, e.g. [.2] - extracts portion of field contents between the 1st and 2nd occurrence of "." Validation Rules: 1-35 alphanumeric characters. May not be entered without related file defined. Must be a string extraction in format [x,y]nnA or a field extraction in format [c,n] otherwise must begin "@" and exist in the PGM.FILE as a defined subroutine.
