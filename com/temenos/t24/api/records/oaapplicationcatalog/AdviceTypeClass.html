      • getConstraintRule

        public TField getConstraintRule()
        Returns the ConstraintRule

        Field description: Entry in this field should be a valid record from OA.RULE.DEFINITION, wherein a Boolean rule can be specified. It should be defined by using the status codes which are given in STATUS.CODE field. The rule will always be an expression. Here, the first word should be a status code, Second word should be an operand and third word should be a literal with quotes. When an advice is triggered automatically or manually (Through AA.GENERATE.ADVICE) the rule specified in this constraint Rule field will be evaluated , and if the rule pass and result is TRUE then only advice will be generated. If the rule is failed then system will raise an error message and no advice will be generated, if advice is triggered automaticallythen this Constraint Rule will be evaluated at application status level itself, if fail, system will not post any advice, just current transaction will commit. Examples: APP.COMPLETE = "YES" AND QUOTATION = "YES" STAGE >= 3 It is similar field like CONSOLIDATION.RULE. The examples provided for CONSOLIDATION.RULE will apply to this field as well. The input in this field is optional.

      • getStatusTriggerRule

        public TField getStatusTriggerRule()
        Returns the StatusTriggerRule

        Field description: Entry in this field should be a valid record from OA.RULE.DEFINITION, wherein a Boolean rule can be specified. It should be defined by using the status codes which are given in STATUS.CODE field. The rule will always be an expression. Here, the first word should be a status code, Second word should be an operand and third word should be a literal with quotes. This rule will be evaluated only for automatic generation of advice (Whenever user status value is modified). The rule will be evaluated each time when a status code in the rule is changed. The advice will be triggered if the rule evaluates as True when prior evaluation is False. Examples: APP.COMPLETE = "YES" CREDIT.SCORE != "" It is similar field like CONSOLIDATION.RULE. The examples provided for CONSOLIDATION.RULE will apply to this field as well. The input in this field is optional.
