      • getIntKey

        public TField getIntKey()
        Returns the IntKey

        Field description: This is an optional input field. The default mechanism (see below) will happen automatically along with other fields ( INT.SPREAD,INT.RATE) Default Mechanism 1. The system would first look for an appropriate BTTC record and take the values defined therein. Then it would use the values defined in appropriate BGC record for modification/override purposes. Finally it could use the values from BL.PARAMETER. It is possible that some of the fields in BTTC or BGC records are not defined (null values). As a general rule, the system would skip these fields and use the values defined at the next level of parameter records. 2. If Bill-Type of all Bill Registers attached to a Bill Contract are same then the system would first look for a BTTC record with key "DISCOUNT*NNNN" where "NNNN" is the value of the common Bill-Type. If there is no such record then the system would look for a record with key "DISCOUNT". 3. If Bill-Type of all Bill Registers attached to a Bill Contract is not the same, then the system would only look for a BTTC record with key DISCOUNT. 4. The system would first update the Contract Group field in Bills Contract; it would determine which is the first Contract Group defined in APPL.GEN.CONDITION file (Record Key: BL)which satisfy the Bill Contract conditions. If no contract group defined therein satisfies the Bill Contract conditions, the system would not update the Contract Group field in the Bills Contract. 5. The BGC file could have records with key values of either "C-Customer No." or a valid Bill Contract Group. In BGC, the system would first look for a record with key "C-Customer No.". If the record does not exist, then the system would look for a record with key equal to the Contract Group field in the Bills Contract. 6. The values in BGC record fields could in some cases override the BTTC values are in some cases could modify the BTTC values. It is possible that both BTTC and BGC records are not available or that the values for some fields are not defined in these records. In such cases, the values from the BL.PARAMETER file would be used by the system to default the values. Validation Rules: Must be a valid record in BASIC.INTEREST or PERIODIC.INTEREST Either Interest key or Interest rate is allowed. To specify PERIODIC.INTEREST - specify interest key prefixed with "P" (ie. To specify PERIODIC.INTEREST key "01" - it is "P01")

        Parameters:
        Returns:
        TField - The IntKey
