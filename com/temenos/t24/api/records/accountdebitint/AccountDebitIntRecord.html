        public TField getDr2CalculType()
        Returns the Dr2CalculType

        Field description: Determines whether the calculation will be performed up to the debit balance Level or in Bands (where the interest calculation employed is dependent upon the size of debit balance). (DR2 fields specify a second interest charge which may be levied.) If more than one set of field range DR2.BASIC.RATE to DR2.LIMIT.AMT is input, this indicates that a different interest rate is applicable depending on the size of the debit balance. DR2.CALCUL.TYPE specifies whether separate calculations are to be made for each Band of the debit balance or one is to be made on the entire balance, dependent on the Level of the balance. Example: Assume a debit balance of 15,000 and an Account Debit Interest record which specifies an applicable rate of 10 per cent up to 10,000 and 15 per cent above 10,000. If Band is specified, 2 calculations will be performed, i.e. 10 per cent on 10,000 15 per cent on 5,000 If Level is specified, 1 calculation will be performed, i.e. 15 per cent on 15,000. Validation Rules: BAND or LEVEL or any abbreviation thereof. (Mandatory input if more than one set of field range DR2.BASIC.RATE to DR2.LIMIT.AMT is specified, otherwise optional. Not allowed if INTEREST.DAY.BASIS contains NONE or GENERAL.)
