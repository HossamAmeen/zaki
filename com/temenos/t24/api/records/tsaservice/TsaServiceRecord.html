      • getCorrelationId

        public TField getCorrelationId()
        Returns the CorrelationId

        Field description: Holds the Unique key generated by IRIS API and sent via OFS request to T24. It is applicable for both TSM and service record. TSA.SERVICE records if it has this correlation id value then corresponding service start or stop stage will be able to emit an Integration framework event from T24 Service Framework. T24 Service Framework stages which emits IF event are 1. TSM start or stop. 2. Service's start or stop process by TSM. 3. If any change in the agents allocated to that service. 4. Critical fatal outs from any of that service's agents. For example, 1. When TSM is successfully launched in a server, then that TSM agent will emit IF event with event name as START and Service name as TSM. When TSM is stopped manually(SERVICE.CONTROL = STOP) then TSM emits STOP event with service name as TSM. 2. When TSM launched all the agents for a service, then TSM agent will emit IF event with event name as START, corresponding service id and agent details to indicate that the service has been started with these many agents. Event will be emitted only once per service in a server. When Service agents are stopping for any reason such as either its completed all ids process or SERVICE.CONTROL marked for STOP in its TSA.SERVICE record, then TSM will capture these details and emits Service STOP event with corresponding service agent details. 3. When there is any change in already allocated agents, either AGENT.REQUIRED increased or decreased in TSA.WORKLOAD.PROFILE, then TSM after allocating agents it emits CURRENT.AGENTS event with service name and current agent details. 4. In case any of that service agent goes for critical fatal out, then that agent itself emits service STOP event with eventNature as CRITICAL and service name. Validations: Value expected for the TSA.SERVICE records which are operates through IRIS API.

        Parameters:
        Returns:
        TField - The CorrelationId

      • getFrequency

        public TField getFrequency()
        Returns the Frequency

        Field description: This field accepts a value in the format Date Time Frequency The date can be entered in the T24 date format(system date). The time in the format HH:MM(24 hour format). The frequency can be entered in the format 1D,2D,etc Example 20151231 15:00 1D TSM would build the service profile only for the services which are START or AUTO. But when the frequency is specified in the service and SERVICE.CONTROL set as STOP , the TSM would check whether the specified frequency has been reached taking into account the time as well and would start the service. When the service is stopped, the frequency would be cycled to the next date. Validation Rules: The frequency field is not allowed for an AUTO service. The service control cannot be changed if the frequency is present.

        Returns:
        TField - The Frequency

      • getServiceControl

        public TField getServiceControl()
        Returns the ServiceControl

        Field description: This field holds the control the user wishes to exercise upon the current service. Currently it can hold the following values. START - The Service is ready to start and can be kicked off. STOP - The Service has to be stopped. The service if it had been running will be terminated. AUTO - If AUTO is specified, then the service is ready for kicking off and does not require manual input of START . The service will be available for starting and can be stopped at the end of the process. Validation Rules: Up to 10 Characters. Valid values are START, STOP and AUTO.

        Parameters:
        Returns:
        TField - The ServiceControl

      • getStopCount

        public TField getStopCount()
        Returns the StopCount

        Field description: Number of crashes allowed for a service in a time period as specified in the field STOPPAGE.TIME. When not specified, the value is taken from TSA.PARAMETER 1.Non-mandatory field, accepts numeric value 2.Value can be given in this field only when STOPPAGE.TIME is specified. e.g. STOPPAGE.TIME = 80 STOP.COUNT = 2 Here the Service can have 2 crashes in a time period of 80 seconds. At the third crash within 80 seconds, Service will be stopped.

        Parameters:
        Returns:
        TField - The StopCount

      • getStoppageTime

        public TField getStoppageTime()
        Returns the StoppageTime

        Field description: Time interval specified for a service to be monitored for the number of crashes allowed as specified in the field STOP.COUNT. When not specified, the value is taken from TSA.PARAMETER Validation Rules: : 1. Non-mandatory field, accepts numeric value 2. STOPPAGE.TIME should be greater than the time specified in TIME.OUT 3. The Value given should be greater than the time specified in DEATH.WATCH time in TSA.PARAMETER when no TIME.OUT is specified. 4. When no TIME.OUT and DEATH.WATCH time in TSA.PARAMETER is given, then STOPPAGE.TIME should be greater than default death watch time 300s.

        Returns:

      • setCorrelationId

        public void setCorrelationId(TField value)
        Set the specified value for CorrelationId
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Holds the Unique key generated by IRIS API and sent via OFS request to T24. It is applicable for both TSM and service record. TSA.SERVICE records if it has this correlation id value then corresponding service start or stop stage will be able to emit an Integration framework event from T24 Service Framework. T24 Service Framework stages which emits IF event are 1. TSM start or stop. 2. Service's start or stop process by TSM. 3. If any change in the agents allocated to that service. 4. Critical fatal outs from any of that service's agents. For example, 1. When TSM is successfully launched in a server, then that TSM agent will emit IF event with event name as START and Service name as TSM. When TSM is stopped manually(SERVICE.CONTROL = STOP) then TSM emits STOP event with service name as TSM. 2. When TSM launched all the agents for a service, then TSM agent will emit IF event with event name as START, corresponding service id and agent details to indicate that the service has been started with these many agents. Event will be emitted only once per service in a server. When Service agents are stopping for any reason such as either its completed all ids process or SERVICE.CONTROL marked for STOP in its TSA.SERVICE record, then TSM will capture these details and emits Service STOP event with corresponding service agent details. 3. When there is any change in already allocated agents, either AGENT.REQUIRED increased or decreased in TSA.WORKLOAD.PROFILE, then TSM after allocating agents it emits CURRENT.AGENTS event with service name and current agent details. 4. In case any of that service agent goes for critical fatal out, then that agent itself emits service STOP event with eventNature as CRITICAL and service name. Validations: Value expected for the TSA.SERVICE records which are operates through IRIS API.

        Parameters:
        value - The value to set
        Returns:
        nothing

        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field accepts a value in the format Date Time Frequency The date can be entered in the T24 date format(system date). The time in the format HH:MM(24 hour format). When the service is stopped, the frequency would be cycled to the next date. Validation Rules: The frequency field is not allowed for an AUTO service.
