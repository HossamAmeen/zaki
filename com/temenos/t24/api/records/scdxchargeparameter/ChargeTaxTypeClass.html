      • getDerivedAmount

        public TField getDerivedAmount()
        Returns the DerivedAmount

        Field description: If the charge is same as another already calculated charge and does not have to recalculated, the name of the other charge field can be defined here. Example: Entry load on broker side of trade is same on entry load on customer side. In such a scenario, for the ENTRY.LOAD.BR record (entry load on the broker leg of the trade), the field can be set as ENTRY.LOAD.CU (entry load for the customer leg) and the amount calculated for entry load in the customer side will be updated in this field. Validation Rules: Accepted Values - Valid charge/tax from SCDX.CHARGE.PARAMETER Input to derived amount is applicable for Fund charges only Input to derived amount is not applicable for Tax type Applicable only for Broker side charges

        Parameters:
        Returns:
        TField - The DerivedAmount
