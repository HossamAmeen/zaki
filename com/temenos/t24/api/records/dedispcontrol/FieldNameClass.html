        public TField getCondition()
        Returns the Condition

        Field description: Specifies a value that the FIELD.NAME is to be compared with. OPERAND and CONDITION are entered to describe how FIELD.NAME is to be tested. The following values will be interpreted by the system and should only be used with date checking. "!TODAY" The system run date will be used. "!NEXT.WORKING.DAY" The NEXT WORKING DAY from the DATES file will be used. "!LAST.WORKING.DAY" The LAST.WORKING DAY from the DATES file will be used. Validation Rules: 1-30 type A (alphanumeric) characters. Either: a character string (no delimeters - eg. SMITH not 'SMITH') or a numeric value (with sign and decimal point if required) or a date (8 digits: 4-digit year, 2-digit month, 2-digit day.) FIELD.NAME, OPERAND and CONDITION form a multivalue set. (Optional input)

      • getFieldName

        public TField getFieldName()
        Returns the FieldName

        Field description: Specifies the name of a Field or a user defined routine to be used to control the Status and/or Priority. Example: The multivalue sets below will be used to specify the Status and/or Priority of SWIFT messages which are Message Type 100: FIELDNAME CARRIER OPERAND EQUAL CONDITION SWIFT FIELDNAME MESSAGE.TYPE OPERAND EQUAL CONDITION 100 Example: To specify a routine the value in this field should contain the name of the routine to call prefixed by the '@' character. FIELDNAME @EXAMPLE.NAME OPERAND EQUAL CONDITION EXAMPLE.CONDITION The routine is passed the current DE.O.HEADER record in argument one, the OPERAND in argument two and the CONDITION in argument three. The return argument is argument four and should evaluate to true (1) or false (0 or null). The routine itself should be specified as EXAMPLE.NAME(HEADER.REC, OPERAND, CONDITION, RETURN.FLAG) Validation Rules: One of the following: 'COUNTRY.CODE' 'LANGUAGE' 'VALUE.DATE' 'AMOUNT' 'TRANS.REF' 'CURRENCY' 'MESSAGE.TYPE' 'APPLICATION' 'ACCOUNT' 'CUSTOMER' 'COMPANY' 'CARRIER' 'TO.ADDRESS' 'MESSAGE.PRIORITY' 'FORMAT' 'APP.FORMAT' 'IN.PAY.TYPE' 'TEST.KEY.REQ' 'TEST.KEY.INP' 'TEST.KEY.AUTH' 'TELEX.BATCH.NO' 'BANK.DATE' OR @ROUTINE.NAME FIELD.NAME, OPERAND and CONDITION form a multivalue set. (Mandatory input)
