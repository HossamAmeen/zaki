        public TField getDefltInactive()
        Returns the DefltInactive

        Field description:

        The FATCA statuses (Account classifications) are user defined. The classifications are defined in the FATCA TAX STATUS application. The default FATCA TAX STATUS pertaining to Inactive accounts or owners is specified in this field.

        Consider an account with joint or beneficial owners. If there is a need to change the list of beneficial or joint owners by removing the existing owners, one of the options could be to remove the details from FATCA record (deleting the multi-value). Another option could be to mark the status as INACTIVE (the status defined in this field) and leave the multi-values untouched. So, for any due diligence or documentation checks, the owners marked inactive will be ignored.

        Validation rules

        A valid status as defined in FATCA TAX STATUS

        public TField getDocStatusFreq()
        Returns the DocStatusFreq

        Field description: This field is used to set a frequency specific to companies based on which the close of business job (FATCA.DOC.BASED.STATUS) will identify the expired documents in FATCA supplementary info and process the FATCA status as RECALCITRANT/DORMANT. This field is a standard frequency field. The processing of expired documents for a company will be done based on the frequency set in this field. For example, If FATCA status update based on expired documents to be processed every business day for a company starting from date 01-july-2013, the DOC.STATUS.FREQ need to be set as 01 JUL 2013 BSNSS Validation Rules:

        This field is in two parts. Next Run Date: 1-9 Date characters, Application Frequency: 1-5 type SS.

        D(aily) (Every day) Tomorrow.

        B(snss) (Every business day) Next working day after today

        T(wmth) (Twice a month, 15th and last day) The next 15th or month end from today, whichever is sooner.

        Wn (Every n weeks) n weeks from today.

        Mnndd (Every nn months on the dd day). If todays date is less than dd, the next date is day dd in nn months from last month, otherwise it is day dd in nn months form the current month. If this date does not exist (e.g. 30th February) the previous day is assumed (28th or 29th February).

        public TField getDormIdentApp()

        Field description:

        The dormant accounts will be identified based on the value specified in the field (identified based on DORM IDENT FIELD) in the application specified in this field.

        DORM IDENT VALUE : 32

        Any client with CUSTOMER STATUS of 32 will be considered Dormant as per the above setting.
