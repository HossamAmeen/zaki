      • getBackValueMaximum

        public TField getBackValueMaximum()
        Returns the BackValueMaximum

        Field description: Specifies the maximum period of Back Value/Exposure acceptable for this type of transaction without an override being required. If Y, N or NO is entered the inputs (a) to (f) are not allowed. (a), (bb), (c) specify the number of calendar or working days to be subtracted from the Processing Date in order to obtain the Maximum Back Value Date/Exposure date as follows: (a) - If days are to be subtracted from the Run Date. Only the '-' sign will be allowed for this field. (bb) 01-99 Indicates the number of days to be subtracted. (c) C If (bb) are calendar days, or W If (bb) are working days. If it is necessary to subtract any other days to those specified in (bb) before obtaining the Maximum back Value Date/Exposure date then (d), (ee), (f) are used as follows: (d) As for (a) previously. (ee) As for (bb) previously. (f) C If (c) is 'C' or W If (c) is 'W' Note : Processing Date is the date of the data being processed which may not always be today's date. If different, the Processing Date is displayed next to the User ID on the screen. This field allows you to specify a Maximum back Value or Exposure Date for this Transaction Type being either the Processing Date, or a number of working or calendar days to be subtracted from the Processing Date. This prevents the operator from inputting a Value Date or an Exposure Date which exceeds an acceptable period for this type of transaction. Any Funds Transfer payment input for the Transaction Type with a Back Value date greater than that specified in this field must be overridden. Example: A Default Value Date of -03W would be 3 working days backwards. or -01W -02C would be 1 working day plus 2 calendar days backwards. Note : The date specified in this field may not be earlier than the Back Value Maximum as specified in the Dates file. Validation Rules: Y If the Maximum Back Value or Exposure Date is to be the Processing Date. N or NO Where no special maximum is required. The Back Value minimum as specified in the Dates file will apply. or (a) (bb) (c) (d) (ee) (f) As defined below. (Optional input; Default = None).

        Returns:
        TField - The BackValueMaximum

      • getDrChequeTxnCode

        public TField getDrChequeTxnCode()
        Returns the DrChequeTxnCode

        Field description: Defines the debit Transaction Code to be assigned when a cheque issued by a customer and drawn on ourselves, has been presented to us for payment by the bank who has negotiated it.         public void setForwValueMaximum(TField value)
        Set the specified value for ForwValueMaximum
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies the maximum period of Forward Value/exposure acceptable for this type of transaction without an override being required. If Y, N or No is entered the inputs (a) to (f) are not allowed. (a), (bb), (c) specify the number of calendar or working days to be added to the Processing Date in order to obtain the Maximum Forward Value Date/Exposure date as follows: (a) + If days are to be added to the Run Date. Only the '+' sign will be allowed for this field. (bb) 01-99 Indicates the number of days to be added. (c) C If (bb) are calendar days. or W If (bb) are working days. If it is necessary to add any other days to those specified in (bb) before obtaining the Maximum Forward Value Date/Exposure date then (d), (ee), (f) are used as follows: (d) As for (a) previously. (ee) As for (bb) previously. (f) C If (c) is 'C' or W If (c) is 'W' Note : Processing Date is the date of the data being processed which may not always be today's date. If different, the Processing Date is displayed next to the User ID on the screen. This field allows you to specify a Maximum Forward Value or Exposure Date for this Transaction type being either the Processing Date, or a number of working or calendar days to be added to the Processing Date. This prevents the operator from inputting a Value Date or an Exposure Date which exceeds an acceptable period for this type of transaction. Any Funds Transfer payment input for this Transaction Type with a Forward Value Date greater than that specified in this field must be overridden. Example: A Default Value Date of +03W would be 3 working days forward. or +01W+02C would be 1 working day plus 2 calendar days forward. Note : The date specified in this field may not be later than the Forward Value Maximum as specified in the Dates file. Validation Rules: Y If the Maximum Forward Value or Exposure Date is to be the Run Date. N or NO Where no special maximum is required. The Forward Value minimum as specified in the Dates file will apply. or (a) (bb) (c) (d) (ee) (f) As defined in the Details. (Optional input; Default = None).
