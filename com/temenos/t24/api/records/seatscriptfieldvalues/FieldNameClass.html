        public TField getExpectedValue()
        Returns the ExpectedValue

        Field description: It should be in any of the following formats: 1. Accepts values within brackets i.e. (150000) means that the expected result is the difference of the previous value and the current value of that field. Example, CLOSING.BAL.NOM field in SECURITY.POSITION which is a cumulative value and not an absolute one. 2. Values enclosed by "~" For example: FIELD.NAME: TRANSACTION.IDS EXPECTED.VALUE: ~DXTRA0033500004.100322.1 250~ What ever is present between ~ needs to be located in the FIELD.NAME This field can have a maximum of 35 characters.

      • getFieldName

        public TField getFieldName()
        Returns the FieldName

        Field description: A Valid field name, which hold the script in OFS Syntax. Refers to the field name and the field, multi-value or sub-value position. Multi value or sub value positions can be in the format FIELD.NAME:FIELD.VALUE For example: FIELD.NAME: TX.PRICE:(TRANSACTION.IDS:DXTRA0033500004.100322.1 250):1 Which means that the multi-value position for TX.PRICE needs to be picked up by locating DXTRA0033500004.100322.1 250 in the field TRANSACTION.IDS. The bracket is only for clarity Up to 65 characters of input will be accepted, leading spaces, trailing spaces and duplicated embedded spaces will be removed.
