        public TField getProtectionClause()
        Returns the ProtectionClause

        Field description: The field can hold value YES or NO. If the field is flagged to YES, the system derived exchange rate for the event date for a particular currency would be compared with historic exchange rates applied for the currency and the best exchange rate (either current day's rate or one of the historic rates, whichever is more beneficial to the Bank) would be considered for liquidation. If the field is flagged to NO, either the user input rate or the system derived rate (for no input by user) would be used for liquidation.
