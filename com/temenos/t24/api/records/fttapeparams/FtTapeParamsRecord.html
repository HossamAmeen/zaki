      • getCheckPrevTapes

        public TField getCheckPrevTapes()
        Returns the CheckPrevTapes

        Field description: This field defines whether for incoming or outgoing tapes the system should check the FT.TAPE.HIST file for tapes with the same number of records, totals and checksums for possible duplication. An input of 'YES' means that the tape, whether incoming or outgoing, will be checked against tapes in the FT.TAPE.HIST file for tapes with the same number of records, totals and checksums within the number of days specified in field NO.DAYS.FOR.TAPE. This is a measure to prevent possible duplication of a tape in case it has been processed before. If a matching tape is found then the user will be given an override message with the matching tape number and the date/time on which it was processed. Validation Rules: Y(ES) - the tape should be checked for possible duplication. N(O) - the tape should not be checked. (Mandatory input).

        Parameters:
        Returns:
        TField - The CheckPrevTapes

      • getVerification

        public TField getVerification()
        Returns the Verification

        Field description: Specifies whether or not the downloaded file can be run in VERIFY mode as well as UPDATE. In VERIFY mode the application file is processed and only reports are generated, no accounting entries are raised. This mode may be used to enquire on the contents of a file, and is also useful when first defining the interace to check that parameters are correctly set. When using the generic accounting interface (UPDATE.ROUTINE = AC.INWARD.FILE), this value should be set to YES. N.B. If this field is set to yes it is important that the UPDATE.ROUTINE is capable of processing the inward file only for reporting purposes. Validation Rules: Input must either be Y or NO. Optional.

        Parameters:
        Returns:
        TField - The Verification

        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field defines whether for incoming or outgoing tapes the system should check the FT.TAPE.HIST file for tapes with the same number of records, totals and checksums for possible duplication. An input of 'YES' means that the tape, whether incoming or outgoing, will be checked against tapes in the FT.TAPE.HIST file for tapes with the same number of records, totals and checksums within the number of days specified in field NO.DAYS.FOR.TAPE.
