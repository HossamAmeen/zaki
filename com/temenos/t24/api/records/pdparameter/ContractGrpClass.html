      • getAdvPdStart

        public TField getAdvPdStart()
        Returns the AdvPdStart

        Field description: If the value of this field is set to 'YES', then the generation of advices will be scheduled from the start date of the PD contract. If set to 'NO' then the advice generation will relate to the date specified in ADVICE.FREQ field in the PD.PARAMETER. According to the settings done here, the relevant frequency will be automatically populated by the system, in the field ADVICE.FREQUENCY of PD.PAYMENT.DUE record

        As per PD.PARAMETER, the frequency is monthly and the day is set to 31st, and a PD contract starts on 24th. Then, if 'YES' is defined in ADV.PD.START, the advices will be scheduled for every month from 24th. If 'NO' is defined in ADV.PD.START, then the advices will be scheduled on 31st,or last day of every month.

        1-3 alphabetical characters 'YES', 'NO' or ' ' (Null) ' ' (Null ) will default to 'YES'

        Parameters:
        Returns:

      • getAdviceFreq

        public TField getAdviceFreq()
        Returns the AdviceFreq

        Field description: Defines the start date and frequency for sending the overdue advices for all the PD contracts within a group, on a regular basis, as an alternative to sending the overdue advices based on the change of status or the chaser advices set by the user for every PD contract through the schedule option. The type of advice like RA (Regular Advice) can be defined in PD.SCHEDULE.TYPE. This advice frequency will be populated in PD.PAYMENT.DUE record as a system default, for producing advices on the scheduled dates. The start date of the frequency can be the Parameter date defined under this field or the start of the PD record, as defined in ADV.PD.START. This field is a part of the associated multi value set defined under contract group.

        16 character standard date/frequency code format (associated Multi Value field)- type FREQ

        Parameters:
        Returns:
        TField - The AdviceFreq

      • getContractGrp

        public TField getContractGrp()
        Returns the ContractGrp

        Field description: Defines the PD contract group with which the customer is associated. Customers can be classified into different contract groups like Corporate, Private, and Large etc. Based on the contract group, various options like frequency of sending PD advices within a particular status, blocking the advices produced by the system on change of status, collecting charges for sending PD advices etc. can be defined. For defining a CONTRACT.GRP code, a record for PD.PAYMENT.DUE should be created in APPL.GEN.CONDITION. A series of Customer group codes can be defined in this record, using CONTRACT.GRP field, and can be classified according to the decisions set by various criteria (user defined field specified in Standard Selection of PD as J descriptor, which may be used to link a decision field in CUSTOMER application, like SECTOR can also be used as decision criteria)

        The value should have been already defined in PD.PAYMENT.DUE record of APPL.GEN.CONDITION, as a code. A default contract group (DFLT) should be defined in PD.PAYMENT.DUE record of APPL.GEN.CONDITION and this group should be specified in PD.PARAMETER, to take care of producing normal chaser advices, in addition to status based advices.

        Parameters:
        Returns:
        TField - The ContractGrp
