      • getAuthorizationprinciple

        public TField getAuthorizationprinciple()
        Returns the Authorizationprinciple

        Field description: Holds the number of Authorizations required for a payment. Can be set to any values 1 or 2. A payment that needs 1 level of Authorization would go through 4 eye principle. A payment with 2 level of Authorization would go through 6 eye principle. 1 - 4 eye principle 2 - 6 eye principle

        Validation Rules: Input is mandatory for this field.

        For payments processed using Order Entry application, the necessity to use this table to determine levels of authorization is based on the value set in AuthPrincipleOption field in PP.COMPANY.PROPERTIES table. If this table is set to be used for retreiving levels of authorization and if a Zero Authoriser Version is used to process an Order Entry payment, then the payment will get auto authorised if the USER setup allows.

        Parameters:
        Returns:
        TField - The Authorizationprinciple
