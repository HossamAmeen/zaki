      • getNewRate

        public TField getNewRate()
        Returns the NewRate

        Field description: Defines the new rate to be applied for this key. The new rate can come from either a user input or the overnight automatic updates of the file whenever the rate for this key changes on the BASIC.INTEREST file. Example : The interest key 01 represents a banks base mortgage rate. It is announced on the 12 December 1993 that this rate is to be changed from 10.5% to 11% with effect from 1st January 1994. This information is loaded on the BASIC.INTEREST table as follows: ID 01GBP19940101 RATE 11% The record on this file after the overnight update will be: NEW.RATE 11% EFFECTIVE.DATE 12 DEC 1993 APPLICATION.DATE ADVICE.DATE CONTRACT.UPD.DATE OLD.RATE 10.5% 10% OLD.EFF.DATE 15 MAY 92 01 JAN 92 OLD.APPL.DATE 01 JUN 92 01 FEB 92 The history of prior changes for this key is shown in the multivalued history fields. The first multivalue of these history fields show the current interest rates. From now on the new rate of 11% will be defaulted for any new contracts entered with a value date on or after 12 December 1993. However existing contracts linked to this rate will not be updated with the new rate as the application date has not yet been entered. The user can then input the remaining date fields as follows: NEW.RATE 11% EFFECTIVE.DATE 12 DEC 93 APPLICATION.DATE 01 JAN 94 ADVICE.DATE 20 DEC 93 CONTRACT.UPD.DATE 15 DEC 93 OLD.RATE 10.5% 10% OLD.EFF.DATE 15 MAY 92 01 JAN 92 OLD.APPL.DATE 01 JUN 92 01 FEB 92 This means that on the 15 December 1993 the contracts will be updated with the new rate details. Advices will be sent on the 20th December 93 and from 1st January 94 all contracts linked to this interest key will accrue interest at 11%. Note: At the end of day processing on the 1st January 1994, the new rate fields will be cleared out and moved to the first multivalue set in the old rate fields. Validation Rules: 1-8 numeric characters.

        Returns:
        TField - The NewRate

      • setNewRate

        public void setNewRate(java.lang.CharSequence value)
        Set the specified value for NewRate
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the new rate to be applied for this key. The new rate can come from either a user input or the overnight automatic updates of the file whenever the rate for this key changes on the BASIC.INTEREST file. Example : The interest key 01 represents a banks base mortgage rate. It is announced on the 12 December 1993 that this rate is to be changed from 10.5% to 11% with effect from 1st January 1994. This information is loaded on the BASIC.INTEREST table as follows: ID 01GBP19940101 RATE 11% The record on this file after the overnight update will be: NEW.RATE 11% EFFECTIVE.DATE 12 DEC 1993 APPLICATION.DATE ADVICE.DATE CONTRACT.UPD.DATE OLD.RATE 10.5% 10% OLD.EFF.DATE 15 MAY 92 01 JAN 92 OLD.APPL.DATE 01 JUN 92 01 FEB 92 The history of prior changes for this key is shown in the multivalued history fields. The first multivalue of these history fields show the current interest rates. From now on the new rate of 11% will be defaulted for any new contracts entered with a value date on or after 12 December 1993. However existing contracts linked to this rate will not be updated with the new rate as the application date has not yet been entered. The user can then input the remaining date fields as follows: NEW.RATE 11% EFFECTIVE.DATE 12 DEC 93 APPLICATION.DATE 01 JAN 94 ADVICE.DATE 20 DEC 93 CONTRACT.UPD.DATE 15 DEC 93 OLD.RATE 10.5% 10% OLD.EFF.DATE 15 MAY 92 01 JAN 92 OLD.APPL.DATE 01 JUN 92 01 FEB 92 This means that on the 15 December 1993 the contracts will be updated with the new rate details. Advices will be sent on the 20th December 93 and from 1st January 94 all contracts linked to this interest key will accrue interest at 11%. Note: At the end of day processing on the 1st January 1994, the new rate fields will be cleared out and moved to the first multivalue set in the old rate fields. Validation Rules: 1-8 numeric characters.

        Parameters:
        value - The value to set
        Returns:
        nothing
