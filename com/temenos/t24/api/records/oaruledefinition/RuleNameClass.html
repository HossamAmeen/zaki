      • addRuleExpression

        public void addRuleExpression(TField param)
        Appends the specified RuleExpression to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Boolean expression dealing with either Context attribute or Status code. EX : NATIONALITY != "US" PURCHASE.PRICE >= VALUATION.LEVEL SCORE > 0 PERSONAL.DATA = "COMPLETED"

        Expression can also be defined by comparing two context attributes or status codes. Eg - NATIONALITY!=RESIDENCE or HOMEADDRESSTATUS == WORKADDRESSSTATUS Here, NATIONALITY and RESIDENCE are fields from formlet and HOMEADDRESSTATUS and WORKADDRESSSTATUS are user defined status codes.

        Here NATIONALITY, PURCHASE.PRICE and SCORE are fields from the Formlet class application. And PERSONAL.DATA is user defined status Code.

        eg1 : ADDRESS.TYPE == 'OWNED'

        Here "ADDRESS.TYPE" is a valid field name in the SS record of the primary table OA.APP.ADDRESS which is defined in the context "OA.APP.ADDRESS*ADDRESS"

        "==" is an operand

        'OWNED' is a literal

        eg2 : MARKET.VALUE >= MARKET.VALUE.AAA

        Here "MARKET.VALUE" is a valid field name of the SS record of the primarytable OA.APP.COLLATERAL which is defined in the context "OA.APP.COLLATERAL*COLLATERAL"

        ">=" is an operand

        "MARKET.VALUE.AAA" is the Rule Variable which is defined in the same Status Rules application.

        The list of comparison operands are here.

        Comparison List = "<", "<=", "==", "!=", ">=", ">"

        Join List = "||", "&&", "OR", "AND"

        Here is the explanation for Comparison List operators

        Equals Operator ==

        Not Equals operator !=

        Less than operator "<"

        Greater than operator ">"

        Less than or Equals to operator "<="

        Greater than or Equals to operator ">="

        Some of the Rule Examples :

        1. COUNTRY != "US"

        2. CURRENCY == "USD"

        3. (GENDER == "MALE") AND (AGE >= 35)

        Parameters:
        param - the RuleExpression to append
        Returns:
        nothing

        public void setRuleExpression(java.lang.CharSequence param, int idx)
        Set the specified RuleExpression to a specific position in collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Boolean expression dealing with either Context attribute or Status code. EX : NATIONALITY != "US" PURCHASE.PRICE >= VALUATION.LEVEL SCORE > 0 PERSONAL.DATA = "COMPLETED"

        Expression can also be defined by comparing two context attributes or status codes. Eg - NATIONALITY!=RESIDENCE or HOMEADDRESSTATUS == WORKADDRESSSTATUS Here, NATIONALITY and RESIDENCE are fields from formlet and HOMEADDRESSTATUS and WORKADDRESSSTATUS are user defined status codes.

        Here NATIONALITY, PURCHASE.PRICE and SCORE are fields from the Formlet class application. And PERSONAL.DATA is user defined status Code.

        eg1 : ADDRESS.TYPE == 'OWNED'

        Here "ADDRESS.TYPE" is a valid field name in the SS record of the primary table OA.APP.ADDRESS which is defined in the context "OA.APP.ADDRESS*ADDRESS"

        "==" is an operand

        'OWNED' is a literal

        eg2 : MARKET.VALUE >= MARKET.VALUE.AAA

        Here "MARKET.VALUE" is a valid field name of the SS record of the primarytable OA.APP.COLLATERAL which is defined in the context "OA.APP.COLLATERAL*COLLATERAL"

        ">=" is an operand

        "MARKET.VALUE.AAA" is the Rule Variable which is defined in the same Status Rules application.

        The list of comparison operands are here.

        Comparison List = "<", "<=", "==", "!=", ">=", ">"

        Join List = "||", "&&", "OR", "AND"

        Here is the explanation for Comparison List operators

        Equals Operator ==

        Not Equals operator !=

        Less than operator "<"

        Greater than operator ">"

        Less than or Equals to operator "<="

        Greater than or Equals to operator ">="

        Some of the Rule Examples :

        1. COUNTRY != "US"

        2. CURRENCY == "USD"

        3. (GENDER == "MALE") AND (AGE >= 35)
