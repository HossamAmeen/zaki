        public void addCountry(TField param)
        Appends the specified Country to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the destination Country(ies), of the associated payment, to which the Zone charge will apply. To minimize the input, the User should not include any Country code for the default Zone and only define the exception countries in the other Zone(s). The destination of the payment will always be determined from the residence of Credit A/C Customer. Example: If the telex charges to all countries except Japan and Australia are GBP 2.00 the following should be input: Field Name Field Input Default Zone 1 Zone 1 Country - Currency GBP Amount 2.00 Zone 2 Country JP (Japan) Country AU (Australia) Currency GBP Amount 5.00 Validation Rules: 2 type SSS (Uppercase alpha) character Country code. Associated sub-value field. (Mandatory input unless defining charges for Default Zone when it is optional.)
