        public TField getLocCapChkRtn()
        Returns the LocCapChkRtn

        Field description: This field is to facilitate the application or local routines to decide if capture of a transaction is allowed or not Holds a local routine(EB.API record with TYPE = 'BASIC') or an EB.API record of type METHOD which implements an interface defined in the EB.API record HOOK.RC.TYP.CAP.CHK.RTN. The hook name must be in the form of "RC.CAPTURE.CHK.":RC.TYPE.ID:".LOC". YES/NO field optional When 'YES' during RC capture processing ,system would check if a local routine exists in the system If exists, then, call that routine to decide if the transaction is eligible for capture or not. If this returns CAPTURE.FLAG=FALSE, then, this would override core's capture flag and the transaction should not be captured When this is set to NO or null, no local capture check routine would be called

        public TField getLocPreProcess()
        Returns the LocPreProcess

        Field description:

        This field defines whether a specific local routine needs to be called to pre process a message. For example; to get the current due amount for transactions where the amount due is not fixed and can change with time.

        YES /NO Null

        YES indicates that a core pre processor routine needs to be called. The routine name will be built up as "RC.PREP":RC.TYPE:".LOC" key

        e.g. RC.PREP.AC.FIXED.LOC for fixed amounts such as charges or debit interest against the settlement account

        RC.PREP.AA.DUE.LOC for overdue AA bill processing

        Optional input

        YES /NO Null
