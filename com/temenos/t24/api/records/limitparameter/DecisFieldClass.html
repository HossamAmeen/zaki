      • getDecisField

        public TField getDecisField()
        Returns the DecisField

        Field description: Specifies, together with fields 16-18, which condition(s) have to be met by a financial transaction to be verified and checked under the limit PRODUCT.NO (field 14.1) specified for the corresponding APPLICATION (Field 13.1). For all Financial Applications (eg Loans, Money Market,Accounts, Foreign Exchange etc), T24 allows the user to define the Limit products according to his own requirements. The definition of these various Limit products will be done by Application using the association of the multivalue Fields 13 to 18. Whenever Market conditions change or the Bank's management decides to reorganise their Limit control, a simple maintenance in this table will automatically allow and take care of such changes in the Limit verification process. Note: The LIMIT PARAMETER table allows the user to define both 'AND' and 'OR' conditions: . The 'AND' condition will be specified by the expansion of the associated sub-value fields 15.1.1 - 18.1.1. If, for example, the user wants to define a specific Limit checking for Money Market transactions with a category code in the range 21075-21079 AND the currency equal to USD, the following input will then be performed in this table: Field Number Input 13.1 Application MM.MONEY.MARKET 14.1 Product 201 15.1.1 Decis Field CATEGORY 16.1.1 Decision EQ 17.1.1 Decision FR 21075 18.1.1 Decision TO 21079 15.1.2 Decis Field CURRENCY 16.1.2 Decision EQ 17.1.2 Decision FR USD 18.1.2 Decision TO - . The 'OR' condition will be specified by the expansion of the complete set of multivalue fields (13.1 to 18.1.1). If, for example, the user wants to define the same specific Limit checking for Money Market transactions which have either the category code in the range 21075 to 21079 OR the currency code equal to USD, the following input will then be performed in this table: Field Number Input 13.1 Application MM.MONEY.MARKET 14.1 Product 201 15.1.1 Decis Field CATEGORY 16.1.1 Decision EQ 17.1.1 Decision FR 21075 18.1.1 Decision TO 21079 13.2 Application MM.MONEY.MARKET 14.2 Product 201 15.2.1 Decis Field CURRENCY 16.2.1 Decision EQ 17.2.1 Decision FR USD 18.2.1 Decision TO - Validation Rules: Up to 40 alpha numeric characters in the format: "FIELD NAME / FIELD NAME" 'Field Number "-" Multi Value "." Sub value /Field Number "-" Multi Value "." Sub Value' Multivalue field associated with Fields 13 to 18. Input mandatory when APPLICATION has been defined; otherwise not allowed. The field name input must correspond to a valid field name within the STANDARD.SELECTION record for the APPLICATION defined in field 13.1. Field names may be defined as SYS or USR fields in STANDARD.SELCTION and must be of TYPE "D" The field name must be input with the multi value/sub-number when the user wants to specify a special multi value/sub-field. Multi and sub values should be entered in the format FIELD NAME - Multi value . Sub Value E.g LOCAL.REF-13.1 LOCAL.REF-1 Input of 2 field names separated by a '/' is possible when both fields are of 'D' type or the 2nd field is numeric (to define call/notice maturity). When 2 field names are input, the lst field has to be a Takedown value date, the 2nd one a Maturity date and the following input fields can deal with the difference of these 2 fields in calendar days. If input, the 2 date fields must be different (e.g. VALUE.DATE/VALUE.DATE will not be accepted).

      • getDecisionFr

        public TField getDecisionFr()
        Returns the DecisionFr

        Field description: Specifies the value, or start of a range of values, with which the contents of the field indicated in DECIS.FIELD (Field 15.1.1) are to be compared. For all Financial Applications (eg Loans, Money Market, Accounts, Foreign Exchange etc), T24 allows the user to define the Limit products according to his own requirements. The definition of these various Limit products will be done by Application using the association of the multivalue Fields 13 to 18. Whenever Market conditions change or the Bank's management decides to reorganise their Limit control, a simple maintenance in this table will automatically allow and take care of such changes in the Limit verification process. Note: The values input in this field and the next must be in agreement with the Format rules of the element specified in Field 15.1.1 DECIS FIELD. If, for example, the input in DECIS FIELD corresponds to a category code, a maximum input of 5 numeric characters will be allowed in this field. Validation Rules: Up to 35 alphanumeric characters Multivalue field associated with Fields 13 to 18. Input mandatory when DECISION has been defined. Otherwise input not allowed. Default value: None Input must begin and/or end with 3 dots when DECISION = 'LK' or 'UL'. Input for the other DECISION values (eg GE, EQ,LE, etc) will be checked according to the rules defined in the corresponding APPLICATION. It must be noted, however, that no check will be performed for the Local reference fields.

        Returns:
