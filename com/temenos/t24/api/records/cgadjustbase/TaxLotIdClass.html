      • getCgTrdCost

        public TField getCgTrdCost()
        Returns the CgTrdCost

        Field description: This field will be fetched from CG.TXN.BASE record. This field contains the residual nominal cost should any transaction have been partiallynetted against one or more other transactions. For example, using the FIFO (First in, first out) basis : Purchase 100,000 @ 101, CG.TRD.COST would equal 40,400 Purchase 100,000 @ 102, CG.TRD.COST would equal 102,000 Sale 60,000, CG.TRD.COST would equal 0. In the above example, the sale of 60,000 has resulted in the first transaction being partially sold, theCG.NOMINAL has been reduced from the original 100,000 to 40,000, in line with this the original TRD.VALUE of101,000 has been reduced to 40,400 (40000/100000 * 101000). Subsequent sales are compared against the initialpurchase cost until it is exhausted and then the next candidate purchase is located. Suppose a further sale of 80,000 took place, then the following CG.NOMINAL balances will exist : Purchase 100,000 @ 101 CG.TRD.COST would equal 0 Purchase 100,000 @ 102 CG.TRD.COST would equal 61,200 Sale 60,000, CG.TRD.COST would equal 0. Sale 80,000, CG.TRD.COST would equal 0. Noinput field

        Returns:
        TField - The CgTrdCost

        Field description: This field will be fetched from CG.TXN.BASE record. This field contains the residual nominal cost should any transaction have been partiallynetted against one or more other transactions. For example, using the FIFO (First in, first out) basis : Purchase 100,000 @ 101, CG.TRD.COST would equal 40,400 Purchase 100,000 @ 102, CG.TRD.COST would equal 102,000 Sale 60,000, CG.TRD.COST would equal 0. In the above example, the sale of 60,000 has resulted in the first transaction being partially sold, theCG.NOMINAL has been reduced from the original 100,000 to 40,000, in line with this the original TRD.VALUE of101,000 has been reduced to 40,400 (40000/100000 * 101000). Subsequent sales are compared against the initialpurchase cost until it is exhausted and then the next candidate purchase is located. Suppose a further sale of 80,000 took place, then the following CG.NOMINAL balances will exist : Purchase 100,000 @ 101 CG.TRD.COST would equal 0 Purchase 100,000 @ 102 CG.TRD.COST would equal 61,200 Sale 60,000, CG.TRD.COST would equal 0.
