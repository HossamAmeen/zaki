        public void addOvrAddress(TField param)
        Appends the specified OvrAddress to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field enables the user to divert a particular message type to another Address. The user has a number of options: Simple insertion of a name or address. This will re-route the message in PRINT format to the named party. The code 'TX-' followed by a telex number. This will re-route the message by TELEX to the number detailed. The code 'SW-' followed by a S.W.I.F.T. code. This will re-route the message by SWIFT to the respective bank. The code 'C-' followed by a valid customer number or mnemonic. This will re-route the message in PRINT format to the customer indicated. Validation Rules: 50 Alphanumeric character field.

      • getMsgClass

        public TField getMsgClass()
        Returns the MsgClass

        Field description: MSG.CLASS An integral part of the 'DELIVERY' multi-value set, this field contains a message class Id (defined in EB.ADVICES). There may be many classes specified, but only classes relating to the operation will be triggered. For example, the EB.ADVICES number MD-1101 (MD New Deal - CA Type) may have the following classes and messages. Message No: 2320 EB.MESSAGE.CLASS: OPERATIONS Message No: 760 EB.MESSAGE.CLASS: BANKADVICE Message No: 900 EB.MESSAGE.CLASS: DEBITPROVISION. Now assuming that the EB.MESSAGE.CLASS.ID has been mapped to MD internal class specified in MD.PARAMETER as follows: BASIC.MESSAGE = OPERATIONS BANK.ADVICE=BANKADVICE PROVISION.DEBIT=DEBITPROVISION In the MD Transaction, the system will always populate the OPERATIONS and NOTIFICATION class when MD is issued. The PROVISION.DEBIT class will only be triggered, when action on Provision debit account has been completed. Validation Rules: SWIFT related field.

        Parameters:
        Returns:
        TField - The MsgClass
