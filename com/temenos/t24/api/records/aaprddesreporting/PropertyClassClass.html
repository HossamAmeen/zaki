      • getPosClassCapitalise

        public TField getPosClassCapitalise()
        Returns the PosClassCapitalise

        Field description: Where a Charge payable/Charge receivable and Interest Payable/receivable is capitalised, the Position class corresponding to the nature of item i.e. charge or interest, a position class is defined here. For example, in case of Charge, it can be AAPMC and in case of interest it can be AAPMN. To show expected movements for these capitalisation events in case of undisbursed/unfunded contracts the 4th Character will be replaced by "E", i.e AAPEC or AAPEN. Validation: Must exist on the table PM.POSN.CLASS and mandatory for Property class/Property other the Principal Property class/Property.

        Parameters:
        Returns:
        TField - The PosClassCapitalise

      • getPosClassDecrease

        public TField getPosClassDecrease()
        Returns the PosClassDecrease

        Field description: PM Position class for Principal Decrease event under GAP is specified here. From the Position class defined here, the Position class for Principal Decrease event under CAS is derived by replacing the 3rd and 4th characters with those of the Position class defined for the category code of the settlement Account in PM.AC.PARAM. Where the timing for Disbursal or funding event is not certain, the Position classes built will have the 4th Character as "E" . For example, the User has defined AAPMD for GAP event, the CAS position class is derived as AAXXD, where XX are taken from PM.AC.PARAM corresponding to the category code of the settlement Account. The Position class for expected event is then built as AAPED for GAP and AAXED for CAS ( X is from PM.AC.PARAM) Validation Rules: Must exist in PM.POSN.CLASS and mandatory for Principal Property class/Property

        Parameters:
        Returns:
        TField - The PosClassDecrease
