      • getCrIntCodes

        public TField getCrIntCodes()
        Returns the CrIntCodes

        Field description: Defines which Transaction codes are used to post Credit Interest entries. The transaction code in this field has to be present in the file TRANSACTION. When the extract of the Interest Movements is done the system will check the Transaction code of the entry with this field and move the entry details as a Credit Interest movement, if it matches the transaction code in this field. The transaction code defined in this field must not be duplicated in eithre the Debit or the Tax transaction code. Validation Rules: Optional field 1-10(Max Value) numeric characters Transaction Code. The Maximum value is specified in EB.OBJECT for TRANSACTION. Sub valued to enable input of more than one transaction code.
