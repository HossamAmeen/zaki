      • getAutoReorderType

        public TField getAutoReorderType()
        Returns the AutoReorderType

        Field description: Auto reorder type NO.HELD or CHEQUE.NUMBER or Null can be entered here. If Auto reorder type is NO.HELD, then auto reorder will be made if the No of cheque held in CHEQUE.REGISTER is less than or equal to the Minimum holding for the Cheque.Type. If Auto reorder type is CHEQUE.NUMBER, then during the cheque issue, based on the Minimum holding for the cheque type, list of auto cheque number gets created in CHEQUE. REGISTER If any of these Cheque number gets presented, then auto reorder will be done for that account. If Auto reorder type if "", then reordering in COB shall happen if the sum of presented, returned and stopped cheques is lesser than the Minimum holding for the cheque type. However, if a record is available in Cheque.Issue for that account with Request Status Less than 90 (Cheque issue), then reorder will not be done. Validation Rules: Can be specified only when the Auto. Request is "Yes" Accepted values NO.HELD or CHEQUE.NUMBER or Null.

        Parameters:
        Returns:
        TField - The AutoReorderType

      • getAutoRequest

        public TField getAutoRequest()
        Returns the AutoRequest

        Field description: This field is used for indicating automatic re-order of cheque book required or not. Re-ordering will be done based on the Auto.Reorder.Type. Auto reorder type can be No.Held (Re-ordering will be done if the Cheque unused by the customer goes below the minimum holding) or Cheque.Number (On presenting a particular cheque reorder will be done) If Re-ordering is mentioned as "YES" then next field Request.Status is mandatory. During auto re-order, system creates a record in Cheque.Issue with Request Status as specified in the next field. However, if a record is available in Cheque.Issue for that account with Request Status Less than 90 (Cheque issue), then reorder will not be done. Null can be used if re-ordering is not required for this cheque type. Validation Rules: Valid values "YES" or null
