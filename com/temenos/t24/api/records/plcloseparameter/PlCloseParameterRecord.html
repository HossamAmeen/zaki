      • addAlGrouping

        public void addAlGrouping(java.lang.CharSequence param)
        Appends the specified AlGrouping to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The accounting entries raised for each APPLIC.ID/PLCATEGORY will be grouped by default. However, the groups can be further sub-divided based on the elements making up the PL CRF key. As default one accounting entry will be raised for each currency by APPLIC.ID/PLCATEGORY. The user can further specifiy the parts of the key which are used for grouping based on their settings in the PROFIT&LOSS record in CONSOLIDATE.COND. So it will be possible to group the entries with the same key elements (such as NATIONALITY, SECTOR, INDUSTRY etc) but in so doing the number of accounting entries will increase. Validation Rules: Must be a valid name from PROFIT&LOSS record in CONSOLIDATE.COND No duplicates allowed

        Parameters:
        param - the AlGrouping to append
        Returns:
        nothing

        public TField getCloseFreqDate()
        Returns the CloseFreqDate

        Field description: The year end and subsequent cycle frequency for this particular company. When this is set the field FINANCIAL.YEAR.END on the COMPANY record is changed to match this date. This field is in two parts: 1) Next Financial Year End: 1-9 Date characters. Default value calculated by the system from the Frequency. 2) Frequency: 2-5 type FQU (standard frequency format). Validation Rules: Mandatory input Must be a month end date. Frequency must be Monthly, Quarterly, Half-yearly or Yearly. The frequency cycle must start with M12. The day in the date and the day in the frequency (last two digits) must be the same. The date cannot be more than twelve months in front of the run date.
