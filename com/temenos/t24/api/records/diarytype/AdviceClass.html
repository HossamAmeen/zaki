      • getAdvice

        public TField getAdvice()
        Returns the Advice

        Field description: This field is now NO INPUT, with the data transferred to the MESSAGE.TYPE field of EB.ADVICES. The EB.ADVICES record is created as follows: Record key composed of Activity Code-DIARY.TYPE.ID Where Activity Code is one of the following SC-0102 - Confirmation SC-0103 - Entitlement Reversal SC-0104 - Diary Reversal The following additional EB.ADVICE records can also be created manually for pre confirmation advices SC-0100 - Confirmation Pre Advice (unauthorised entitlement creation) SC-0101 - Confirmation Pre Advice amendment (unauthorised entitlement amendment) The message type information will be transferred to the MESSAGE.TYPE field and the ADVICE.FORMAT informationwill be transferred to the PRINT.FORMAT field. If existing message types are set up the delivery process will function as now. Currently the following specific confirmation message types are supported by Corporate Actions: 1505 - Dividend Tax Certificate 1510 - Capital Increase Advice 1520 - New Issue Advice 1530 - Coupon /DividendAdvice 1540 - Stock Dividend Advice 1560 - Redemption Advice Additionally if new message types are required SOFT DELIVERY processing will be invoked as described in theDELIVERY section of the T24 user guide. The information handed off to delivery for new message types will be as follows Record 1 equates to data defined in DE.MESSAGE 1500, Corporate action generic message Record 2 comprises the Entitlement record. Record 3 comprises the Diary record. Record 4 comprises the Security Account Master record. Record 5 comprises the Security Master record. Associated with the ADVICE.FORMAT field to define a particular message and message format. For example if thisfield contained 1560 and the ADVICE.FORMAT field associated with it contained 3 then for a French speaking customerthe DE.FORMAT.PRINT record used would be 1560.3.1.FR. Validation Rules: 1-4 numeric characters. No input.
