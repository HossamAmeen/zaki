      • getBandAmount

        public TField getBandAmount()
        Returns the BandAmount

        Field description: This field is used to define the allocation amount from the incoming record. This allocation amount is used to calculate the amounts from which the allocation to the Departments / Account Officers specified in the field TO.DAO is made. The specification of the allocation amount is set as a series of bands and a related percentage for each band. The final band (that is all of the P & L entry not in the bands previously specified) may be specified using the keyword REST. The simplest method is to set the allocation amount to all of the amount on the incoming P & L entry. This is set as ... 7.1.1 BAND.AMOUNT. REST 8.1.1 BAND.PERCENT 100 ... and is the default. Specifying bands. ================= If more complex banding is required then this is specified as a series of amounts, with a related percentage entered in the BAND.PERCENT field. In the followiung examples it is assumed that the local currency is US dollars. Example One. There may be a requirement that each Department / Account Officer keeps the first $100 dollars from any profit they earn, and after that keeps half this profit. This would be set as 7.1.1 BAND.AMOUNT. 100.00 8.1.1 BAND.PERCENT 0 7.1.2 BAND.AMOUNT. REST 8.1.2 BAND.PERCENT 50 Example Two The Department / Account Officer has to allocate all of the first $100, half of the next $1,000 but then keeps the remainder. 7.1.1 BAND.AMOUNT. 100.00 8.1.1 BAND.PERCENT 100 7.1.2 BAND.AMOUNT. 1,000.00 8.1.2 BAND.PERCENT 50 In this example, as there is no REST keyword, only the first $1100 of the incoming amount is re allocated. If the REST keyword is required it must be in the last sub value of the group, but it is not mandatory. This field defaults to a value of REST if it is the last sub value within a multi value group. Fields SELECTION (field 2) to FROM.DAO (field13) form a related multi value set. Fields BAND.AMOUNT and BAND.PERCENT form a related sub value set. Validation Rules: Up to 19 numeric characters or 'REST' Mandatory input.
