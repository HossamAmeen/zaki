      • getEffectiveDate

        public TField getEffectiveDate()
        Returns the EffectiveDate

        Field description: Specifies a future date on which the rates defined in the associated new margin rate fields will take effect and the rates defined in the current margin rate fields will be discarded. Thus at the end of day of the effective date, the new margin rates will replace the current margin rates defined in the record and this effective date and the associated new margin rates will be deleted. The current margin rates will continue to apply on existing and new contracts until the new rates become effective on the effective date. This field is optional and if new margin rates are defined without any effective date, the rates will take effect immediately.

        Validation Rules: Standard T24 Date Field. Only future dates can be defined. Duplicate values are not allowed. The rates that are to take effect immediately (new margin rates defined without effective date) must be defined as the first multivalue set.

        Parameters:
        Returns:
        TField - The EffectiveDate

      • getNewAdjMargin

        public TField getNewAdjMargin()
        Returns the NewAdjMargin

        Field description: Specifies the Adjusted Margin Rate that will take effect from the associated Effective Date defined. If left blank then the Adjusted Margin Rate will become null on the Effective Date and the rate will be considered from subsequent levels.

        This field can either be used to specify the Low Advance Ratio for collateral calculations in Advance Collateral process or to specify the Diversified Margins for portfolios flagged for Diversification.

        If the field "CO.MV.CHECK" in SC.PARAMETER is set to "YES" to enable Preferential LTV functionality, Adj Margin will be used only as Low Advance Ratio in Advance Collateral process and will not be used for calculating Diversified Margin. In this case ideally New Adj Margin should be lesser than the New Margin Rate. Thus an override will be raised if New Adj Margin is greater than New Preferential Margin Rate/New Standard Margin Rate for the effective date.

        Validation Rules: Standard T24 Rate field with values ranging from 0 to 100.

        Parameters:
        Returns:
        TField - The NewAdjMargin

      • getNewPrefntlMarginRate

        public TField getNewPrefntlMarginRate()
        Returns the NewPrefntlMarginRate

        Field description: Specifies the Preferential Margin Rate that will take effect from the associated Effective Date defined. If left blank then the Preferential Margin Rate will become null on the Effective Date and the rate will be considered from subsequent levels.

        Validation Rules: Standard T24 Rate field. This field is allowed for input only when "CO.MV.CHECK" field in SC.PARAMETER is set to "YES" that enables the Preferential LTV functionality. Value can be specified as an absolute value ranging from 0 to 100 or as a variance with "+" sign. A variance indicates the incremental value to be added to the Standard Margin Rate to arrive at the enhanced rate. Thus New Margin Rate becomes mandatory if New Preferential Rate is defined as a variance. The sum of Preferential Rate defined as a variance and New Margin Rate cannot exceed 100. Ideally New Preferential Rate should always be greater than the New Margin Rate. Thus an override will be raised if the New Preferential Rate defined as an absolute value is lesser than the New Margin Rate.

        Parameters:
        Returns:
        TField - The NewPrefntlMarginRate
