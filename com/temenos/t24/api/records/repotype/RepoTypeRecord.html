INTERNAL: An internal trade moving stock from a bank portfolio to a customerportfolio. STOCK.BORROW.LEND: To be used when using Stock Borrowing/Lending.         Field description: The default suffix that will be applied counterparty's customer number to determine the default margin portfolio for use in the REPO contract.       • getProductCategory

        public TField getProductCategory()
        Returns the ProductCategory

        Field description: This field is used to specify the default product category that will be used for the type of Repo contract to which this record refers. The product category is defaulted into the Repo contract, and is used to determine whether the cash side of the contract is a loan or a deposit. It is this information that broadly governs if the contract is a Repo or a Reverse Repo. Where the product category indicates a DEPOSIT, the repo type refers to a REPO contract. Where the product category indicates a LOAN, the repo type refers to a REVERSE REPO contract. This field is closely linked to the TRANSACTION.INDEX field that holds the text "LOAN" or "DEPOSIT" to clarify the product category. The TRANSACTION.INDEX and the product category must agree. Refer to the Rules. Validation Rules: Mandatory input unless DEAL.TYPE is BUY/SELL.BACK, SELL/BUY.BACK or INTERNAL A maximum of 5 characters may be entered. Values in the range 21200 to 21299 are allowed. For repo contracts, input must be in the range 21200 - 21249 and the TRANSACTION.INDEX must be DEPOSIT. For Reverse repo contracts, input must be in the range 21250 - 21299 and the TRANSACTION.INDEX must be LOAN. Must be the key to a valid entry on the CATEGORY file.

        Parameters:
        Returns:
        TField - The ProductCategory
