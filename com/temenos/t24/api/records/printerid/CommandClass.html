      • add

        public void add(java.lang.CharSequence param)
        Appends the specified Command to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Instead of sending all output to a particular printer, T24 can execute another command specified here. Typically this would be used to send the report to a file. This command can be any command executable from Universe. Unix commands may be executed by enclosing the command in quotation marks and prefixing with SH -c This command can contain variable information and can be broken down into 3 component parts. i) the statement of execution. e.g. the Unix commands cat , cp for example. ii) the source. Reports will always be found in the &HOLD& file, the file HOLD.CONTROL contains details about the report and has the same key. iii) the selection/destination criteria. e.g. all Forex deals that were input today, a specified subdirectory. Note that when Unix commands are specified the Unix path name relative to the current location must be specified. For Example, i) It is possible to copy a report to another area such as 'REPORTS' which is a subdirectory of 'FOREX.ARCHIVES', a command should be entered to copy the specified report from &HOLD& to FOREX.ARCHIVES. The format of a possible Unix command if executed from universe would be SH -c "cat ../bnk.data/eb/'&HOLD&'/id > FOREX.ARCHIVES/REPORTS" Since the key to &HOLD& will be different for each report generated, the key cannot be specified directly in the command, so an element which can be evaluated to generate the required key can be specified. This should be defined in the format (e.g. &filename > field&). In this case the key to the &HOLD& record to be copied can be found in the field @ID of the file HOLD.CONTROL, so this should be specified as &HOLD.CONTROL > @ID& giving a command to be defined as : SH -c "cat ../bnk.data/eb/'&HOLD&'/&HOLD.CONTROL>@ID& > FOREX.ARCHIVES/REPORTS" NOTE : The Unix cat command with > option will append the contents of the source file to the target file. This can be made more flexible. ii) For example it is possible to specify a variable target file name based on the type of report produced. To expand on the previous example, consider only copying any reports in &HOLD& to a file name (e.g. OUTPUT.FILE) defined in the corresponding entry on REPORT.CONTROL in the field LOCAL.REFERENCE.FILE. SH -c "cat ../bnk.data/eb/'&HOLD&'/&HOLD.CONTROL>@ID& > LINKUP/&REPORT.CONTROL>OUTPUT.FILE&" The value of 'OUTPUT.FILE' in the REPORT.CONTROL record related to the &HOLD& and HOLD.CONTROL record will be evaluated to give the file name within the LINKUP directory that the report could be copied to. In order to execute the above, the following should be defined. i) An entry on the LOCAL.TABLE should be created for the 'OUTPUT.FILE'. ii) An entry for the 'OUTPUT.FILE' local table should be included in the LOCAL.REF.TABLE REPORT.CONTROL record. iii) In the STANDARD.SELECTION REPORT.CONTROL record an I-descriptor should be included for the LOCAL.TABLE item (i.e. 'OUTPUT.FILE'). iv) Ensure that the STANDARD.SELECTION record for HOLD.CONTROL contains REPORT.CONTROL in the field SYS.REL.FILE or USR.REL.FILE in an item defined for SYS.FIELD.NO or USR.FIELD.NO. NOTE : An additional feature has been included which, in relation to the example will search for the path of &HOLD& and assign it to the variable 'HOLD.UNIX.PATH'. Therefore, SH -c "cat ../bnk.data/eb/'&HOLD&'/&HOLD.CONTROL>@ID& > REPORTS" could be replaced with SH -c "cat &HOLD.CONTROL>HOLD.UNIX.PATH&/&HOLD.CONTROL>@ID& > REPORTS" This requires an I-descriptor called 'HOLD.UNIX.PATH' to be entered into USR.FIELD.NAME in the STANDARD.SELECTION HOLD.CONTROL record. This I-descriptor will call the UNIX.FILE.PATH subroutine and pass &HOLD& as a parameter. For example, FILE.NAME............HOLD.CONTROL -------------------------------------------------------- 15.1 USR.FIELD.NAME. HOLD.UNIX.PATH 16.1 USR.TYPE....... I 17.1.1 USR.FIELD.NO. SUBR("UNIX.FILE.PATH", "&HOLD&") -------------------------------------------------------- This subroutine may be used as an I-descriptor for any file. COMMAND (Optional Input) Can be multivalued to allow for multiple commands, or subvalued to allow for a lengthy command. Validation Rules: The source must be a valid entry on FILE.CONTROL. Must be followed by '>' (greater than sign). To be followed by a valid multivalue entry in SYS.FIELD.NAME on STANDARD.SELECTION. The whole to be contained within '&' (ampersand).

        Parameters:
        param - the Command to append
        Returns:
        nothing

        public void insert(TField param, int pos)
        Insert the specified Command to the specified position in the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Instead of sending all output to a particular printer, T24 can execute another command specified here. Typically this would be used to send the report to a file. This command can be any command executable from Universe. Unix commands may be executed by enclosing the command in quotation marks and prefixing with SH -c This command can contain variable information and can be broken down into 3 component parts. i) the statement of execution. e.g. the Unix commands cat , cp for example. ii) the source. Reports will always be found in the &HOLD& file, the file HOLD.CONTROL contains details about the report and has the same key. iii) the selection/destination criteria. e.g. all Forex deals that were input today, a specified subdirectory. Note that when Unix commands are specified the Unix path name relative to the current location must be specified. For Example, i) It is possible to copy a report to another area such as 'REPORTS' which is a subdirectory of 'FOREX.ARCHIVES', a command should be entered to copy the specified report from &HOLD& to FOREX.ARCHIVES. The format of a possible Unix command if executed from universe would be SH -c "cat ../bnk.data/eb/'&HOLD&'/id > FOREX.ARCHIVES/REPORTS" Since the key to &HOLD& will be different for each report generated, the key cannot be specified directly in the command, so an element which can be evaluated to generate the required key can be specified. This should be defined in the format (e.g. &filename > field&). In this case the key to the &HOLD& record to be copied can be found in the field @ID of the file HOLD.CONTROL, so this should be specified as &HOLD.CONTROL > @ID& giving a command to be defined as : SH -c "cat ../bnk.data/eb/'&HOLD&'/&HOLD.CONTROL>@ID& > FOREX.ARCHIVES/REPORTS" NOTE : The Unix cat command with > option will append the contents of the source file to the target file. This can be made more flexible. ii) For example it is possible to specify a variable target file name based on the type of report produced. To expand on the previous example, consider only copying any reports in &HOLD& to a file name (e.g. OUTPUT.FILE) defined in the corresponding entry on REPORT.CONTROL in the field LOCAL.REFERENCE.FILE. SH -c "cat ../bnk.data/eb/'&HOLD&'/&HOLD.CONTROL>@ID& > LINKUP/&REPORT.CONTROL>OUTPUT.FILE&" The value of 'OUTPUT.FILE' in the REPORT.CONTROL record related to the &HOLD& and HOLD.CONTROL record will be evaluated to give the file name within the LINKUP directory that the report could be copied to. In order to execute the above, the following should be defined. i) An entry on the LOCAL.TABLE should be created for the 'OUTPUT.FILE'. ii) An entry for the 'OUTPUT.FILE' local table should be included in the LOCAL.REF.TABLE REPORT.CONTROL record. iii) In the STANDARD.SELECTION REPORT.CONTROL record an I-descriptor should be included for the LOCAL.TABLE item (i.e. 'OUTPUT.FILE'). iv) Ensure that the STANDARD.SELECTION record for HOLD.CONTROL contains REPORT.CONTROL in the field SYS.REL.FILE or USR.REL.FILE in an item defined for SYS.FIELD.NO or USR.FIELD.NO. NOTE : An additional feature has been included which, in relation to the example will search for the path of &HOLD& and assign it to the variable 'HOLD.UNIX.PATH'. Therefore, SH -c "cat ../bnk.data/eb/'&HOLD&'/&HOLD.CONTROL>@ID& > REPORTS" could be replaced with SH -c "cat &HOLD.CONTROL>HOLD.UNIX.PATH&/&HOLD.CONTROL>@ID& > REPORTS" This requires an I-descriptor called 'HOLD.UNIX.PATH' to be entered into USR.FIELD.NAME in the STANDARD.SELECTION HOLD.CONTROL record.
