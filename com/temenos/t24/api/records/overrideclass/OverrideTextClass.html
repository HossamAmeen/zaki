        public TField getDefaultClass()
        Returns the DefaultClass

        Field description: Format; 4 type A (alphanumeric) characters. Multi-value field. (Optional input; default value = none.) This is the class that will be used if there is no match returned from OVERRIDE.CLASS.DETAILS (or where OVERRIDE.CLASS.DETAILS is not used at all). The value(s) used are compared with those found on the USER profile, the field is optional as the class can be derived from the settings found in the OVERRIDE.CLASS.DETAILS table.

      • getOverrideDetail

        public TField getOverrideDetail()
        Returns the OverrideDetail

        Field description: This is the key to the OVERRIDE.CLASS.DETAILS record that can be used to make the override check more conditional. In that application you can check the contents of the override message and set the level of authority that must be used to finalise the override. For example where there is a currency and amount returned in the override you can convert these to a local currency amount in order to set the level of priveleges required to authorise the override. Another is a simple literal check which could be used to set levels of authority accoriding to the presence of specific words in the override. Validation Rules: Must be a valid record on OVERRIDE.CLASS.DETAILS.

        Parameters:
        Returns:
        TField - The OverrideDetail
