      • getAppRetentionPeriod

        public TField getAppRetentionPeriod()
        Returns the AppRetentionPeriod

        Field description: Contract specific retention period for erasure can be defined in this field For example, a purpose CONSENT that is marked as allowed for active erasure is included under CONT.ERASURE.PURPOSES and DE.ADDRESS is included under CONT.ERASURE.APPLN. A retention period of 1Y is given in the field APP.RETENTION.PERIOD. The CZ.CDP.PURPOSE record of CONSENT carries a retention period of 6Y by default. Personal data from DE.ADDRESS application are linked to the CONSENT purpose and if a contract erasure is requested for DE.ADDRESS, the retention period used will be 1Y. If the purpose is not mentioned under CONT.ERASURE.PURPOSE or no retention period is mapped, the system will consider the default 6Y retention period. Validation Rule:

        Valid values are from 0D to 999Y. Either retention period or retention period API or retention period rule is allowed.

        Parameters:
        Returns:
        TField - The AppRetentionPeriod

      • getContErasureAppln

        public TField getContErasureAppln()
        Returns the ContErasureAppln

        Field description: Applications that allowed for contract erasure requests are to be mentioned in this field. These applications 1.Need to have a valid data definition record. 2.Should be a master application with MASTER.APPLICATION field in CZ.CDP.DATA.DEFINITION set as null. 3.Should be a table in ST.CUSTOMER.ACTIVITY, that is the USE.IN.ACTIVITY field in ST.CUSTOMER.ACTIVITY.PARAMETER needs to be either LINK or ACTIVITY. Validation Rule:

        This field is allowed input only if ALLOW.CONTRACT.ERASURE is set to YES.
