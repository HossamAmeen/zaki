      • addChgCode

        public void addChgCode(TField param)
        Appends the specified ChgCode to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Identifies the type of charge(s) to be levied as defined on the FT.CHARGE.TYPE/ FT.COMMISSION.TYPE file. This table contains a record for each charge type available. Each record contains a description, a default charge amount and the P&L category which is to be credited whenever the charge is levied. The description from the FT.CHARGE.TYPE/FT.COMMISSION.TYPE will be displayed as an enrichment to this field. Whenever a charge is processed the account entered in the associated CHARGE.ACCOUNT field will be debited. The P&L category defined on the FT.CHARGE.TYPE/FTCOMMISSION.TYPE record for the charge code will be credited. In BL module charges will be calculated and debited to charge account during BL.BILL application processing. Validation Rules: 1 - 2 alphanumeric characters. Multi-value field Must be a valid charge code on the FT.CHARGE.TYPE/ FT.COMMISSION.TYPE file. The same combination of charge code cannot be duplicated.

        Parameters:
        param - the ChgCode to append
        Returns:
        nothing
