      • getIncomeRecog

        public TField getIncomeRecog()
        Returns the IncomeRecog

        Field description: Indicates whether income should be accrued for this class of asset. If this field is set to "NO" system would switch from accrual basis to cash basis i.e. Income would be suspended and memo accruals would be posted. STATUS would be updated as "NAB" in PD and in underlying contract and POST.END.DATE would be updated as null. Defaults from LN.ASSET.CLASS and could be overridden. Validation Rules: Valid values are NO/YES NULL/YES would mean that Income would continue to accrue

        Returns:

      • getWriteOff

        public TField getWriteOff()
        Returns the WriteOff

        Field description: Indicates whether WRITE.OFF is applicable for this class of asset. When the underlying contract (Like LD, MG) is fully liquidated, then the PD contract would be automatically written off (FWOF). STATUS would be updated as FWOF in PD and in the underlying contract as well. When the underlying contract is not fully liquidated, system writes an exception log in PD.PROV.EXCEPTION.LOG file and the user may manually WRITE-OFF the contract later on. Defaults from LN.ASSET.CLASS and could be overridden. Validation Rules: Valid values are YES/NULL

        Returns:
