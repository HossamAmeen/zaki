      • getAccOwnerRef

        public TField getAccOwnerRef()
        Returns the AccOwnerRef

        Field description: This field contains the reference for the Account Owner. At least one valid character other than a blank must be present. For debit entries, the purpose of this field is to identify, to the Account Owner, the instruction which caused the debit. Therefore, the content of this field is the Senders Transaction Reference Number of the original instruction. Credit entries may be the result of one of the following situations: 1. The Account Servicing Institution is identifying, to the Account Owner the receipt of funds for its account as a result of a related transaction. In this case, the content of is the Reference for the Beneficiary of the related transaction. 2. The Account Servicing Institution has issued a payment instruction to the Account Owner and the credit identified in this field is for that payment. The content of the field is the Transaction Reference Number of the payment instruction issued by the Account Servicing Institution. If no reference is available for this field, the code NONREF will be used. The Account Servicing Institution must then supply, in the Supplementary Details field, what it considers to be the best alternative information. Validation Rules: Multi-valued field

        Returns:

      • getTransType

        public TField getTransType()
        Returns the TransType

        Field description: This field contains transaction type identification code for this item. When received via S.W.I.F.T. the content may exist in one of three ways : 1. For entries related to S.W.I.F.T. transfer instructions and subsequent charge messages: Format: Snnn The last three characters will indicate the message type of the S.W.I.F.T. message causing the entry (for debit entries) or the message type of the S.W.I.F.T. message used to advise the Account Owner (for credit entries). 2. For entries related to payment and transfer instructions, including related charges messages, not sent through S.W.I.F.T. or where an alpha description is preferred. Format: Naaa The last three characters must be one of the following codes, indicating the reason for the entry: BOE Bill of Exchange BRF Brokerage Fee CHG Charges and Other Expenses CHK Cheques CLR Cash Letter/Cheques Remittance COL Collections (used when entering a principal amount) COM Commission DCR Documentary Credit (used when entering a principal amount) DIV Dividends Warrants EQA Equivalent Amount ECK Eurocheques FEX Foreign Exchange INT Interest LBX Lock Box1. LDP Loan Deposit MSC Miscellaneous RTI Returned Item SEC Securities (used when entering a principal amount) STO Standing Order TCK Travellers Cheques TRF Transfer VDA Value Date Adjustment 3. For entries being first advised by the statement (items originated by the Account Servicing Institution): Format: Faaa The last three characters must be an appropriate code indicating the reason for the entry (i.e. the codes listed above). Validation Rules: Multi-valued field

        Parameters:
        Returns:
        TField - The TransType
