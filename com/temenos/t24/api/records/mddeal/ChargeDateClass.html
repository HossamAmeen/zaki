      • getChargeAccount

        public TField getChargeAccount()
        Returns the ChargeAccount

        Field description: The account that is to be debited with the charges. Charges can be defined on any date provided it is not before the deal VALUE.DATE or after the MATURITY.DATE. All charges for a given date must be taken in the same currency and debited to the same account. Charges can be taken in any currency provided that the account to be debited is in the CHARGE.CURR. Any taxes on the charges will be debited to the CHARGE.ACCOUNT. The charges will be debited during the end of day of the CHARGE.DATE except when the CHARGE.DATE is before the ext working day in which case they will be taken on-line. Once this has happened, the charge details will be removed from the 'live' record but can be seen by looking into the balances file. If a deal is reversed, relevant charges taken will be reversed. Validation Rules: 1 - 14 numeric characters relating to a customer account or a Nostro account. or 3 - 10 type A (alphanumeric) account mnemonic. Multi-value field associated with fields CHARGE.DATE to CHARGE.CURR and fields CHARGE.CODE to CHARGE.AMT. The CHARGE.ACCOUNT must be in the same currency as the CHARGE.CURR. Must be a valid ACCOUNT number or mnemonic.

        Parameters:
        Returns:

        public TField getChargeCurr()
        Returns the ChargeCurr

        Field description: The currency in which the charges detailed in the fields CHARGE.DATE, CHARGE.ACCOUNT, CHARGE.CODE and CHARGE.AMT are taken. Charges can be defined on any date provided it is not before the deal VALUE.DATE or after the MATURITY.DATE. All charges for a given date must be taken in the same currency and debited to the same account. Charges can be taken in any currency provided that the account to be debited is in the CHARGE.CURR. Any taxes on the charges will be debited to the CHARGE.ACCOUNT. The charges will be debited during the end of day of the CHARGE.DATE except when the CHARGE.DATE is before the next working day in which case they will be taken on line. Once this has happened the charge details will be removed from the 'live' record but can be seen by looking into the balances file. NB, If a deal is reversed, relevant charges taken will be reversed. Validation Rules: Multi-value field containing the currencies in which charges are to be processed and debited to the customer. 3 alphabetic currency mnemonic or 1 - 3 numeric character currency code. (Optional input but must be input in conjunction with the fields CHARGE.DATE, CHARGE.ACCOUNT, CHARGE.CODE and CHARGE.AMT). The currency code or mnemonic must exist on the CURRENCY table. The CHARGE.ACCOUNT must be in the same currency as the CHARGE.CURR.
