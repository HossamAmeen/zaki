        public TField getExpFundsType()
        Returns the ExpFundsType

        Field description: (Added in G14.1) The types of EXPECTED FUNDS that are allowed is to be given in this field. Allowed values are A) ERR - Expected Regular Receipts B) ER - Expected Receipts C) EP - Expected Payments Note : Above functionality is not applicable for if the ID of the application is "COVER",and the details given in fields 13 - Match.Field & 14 -Tolerance is alone taken for processing Cover Limit. Hence incase the values from 5 to 14 is multivalued then for Cover Processing, the information as available in the First Multi value is alone taken for processing. Validation Rules: Mandatory Field Multivalue field associated from field 5 to 14 Allowed values are ERR, ER and EP only The field PAY.FUNDS.TYPE should be filled in with the opposite of the value filled in this field. The values are given below. EXP.FUNDS.TYPE PAY.FUNDS.TYPE ERR RR ER RECEIPT EP PAYMENT
