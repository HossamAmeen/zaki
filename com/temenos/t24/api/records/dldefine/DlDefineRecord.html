      • getOperation

        public TField getOperation()
        Returns the Operation

        Field description: Whether data is to be "S" (saved) or "R" (restored). This field is only used when the record is "V"erified. When the program is run in "V"erify mode, if the operation is set to "S" (save), a unit is created in the data library, with an id as specified in UNIT.NAME. The records specified in fields FILE.NAME and RECORD.NAME are copied from the live files into the unit in the data library. The fields SAVED.FROM, SAVED.RELEASE and SAVED.DATE on the DL.DEFINE record are updated with details of when the data was saved. When the program is run in "V"erify mode with the operation set to "R" (restore), the following actions are performed: i) Any new files (specified by releasing the FILE.CONTROL record) are created ii) Records contained in the unit are copied into the unauthorised file, with a status of "IHLD" iii) Programs are copied into DL.BP, but are not compiled (use EB.COMPILE to do this when required) iv) The fields RESTORED.USER, RESTD.COMPANY, RESTD.RELEASE and RESTD.DATE on the DL.DEFINE record are updated with details of when the data was restored. Defaulted to "R" if the data unit has previously been saved (i.e. if there is a date in SAVED.DATE); defaults to "S" otherwise. Validation Rules: "S" (save) or "R" (restore) Optional input Defaults to "R" if the data has previously been saved. Defaults to "S" otherwise.

      • getTopLevelItem

        public TField getTopLevelItem()
        Returns the TopLevelItem

        Field description: TOP.LEVEL.ITEM is used to create a list of all data items associated with a particular record. When the TOP.LEVEL.TYPE and TOP.LEVEL.ITEM are entered, the fields FILE.NAME and RECORD.NAME will be cleared, then populated with the names of all the associated data items; then TOP.LEVEL.TYPE and TOP.LEVEL.ITEM will be cleared. For DELIVERY, TOP.LEVEL.TYPE should the message type (the id of the record on DE.MESSAGE). FILE.NAME and RECORD.NAME will be populated with all DE.MAPPING, DE.FORMAT.SWIFT and DE.FORMAT.PRINT records for this message type, as well as the DE.MESSAGE record. For ENQUIRY, FILE.NAME and RECORD.NAME will be populated with any routines used by the enquiry, as well as the ENQUIRY record. For REPGEN, FILE.NAME and RECORD.NAME will be populated with the REPGEN.SORT, REPGEN.CREATE and REPGEN.OUTPUT records. Validation Rules: 1-35 Type "A" (alphanumeric) character Optional input Can only be entered if operation is "S" Must reference a record on the corresponding TOP.LEVEL.TYPE file.

        Returns:
