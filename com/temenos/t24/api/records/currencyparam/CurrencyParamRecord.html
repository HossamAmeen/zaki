      • getBaseCcyRank

        public TField getBaseCcyRank()
        Returns the BaseCcyRank

        Field description: Indicates whether a currency should always be the base currency for currency conversions, in the event that a base currency has not been supplied. Each currency can optionally be assigned a unique base currency rank in the same manner as currency rank. The values are stored in the application RANK.BASE.CURRENCY. If two currencies are involved in a conversion and both have a base ccy rank defined then the currency with the lowest rank will take preference. N.B. It has been stated that rates to the EURO (EUR) will always be quoted as the number of other currency units to 1 EUR, even when 1 unit of the other currency is worth more than 1 EUR. For example assuming that 0.71 GBP were worth 1 EUR then the effect of not supplying a base currency or rate to the conversion routine would result in the base currency being set to GBP and the rate expressed as the inverse of 0.71 i.e 1.408451. Setting the EUR to have a base currency rank of 1 would result in the EUR being the base currency and the rate to be 0.71. Great care should be taken when setting the currency rank, there are very few instances where a rank needs to be set, for example the EURO as described. Another example would be a local currency conversion from a currency where rates are quoted as number of units of foreign to 1 local (even when the rate is less than 1) to the EURO, e.g. IEP to EUR. The system will default the base currency for cross rates to be that currency which has a rate greater than 1, e.g. local ccy = EUR CCY1 = IEP, 0.8 to 1 EUR CCY2 = GBP, 0.7 to 1 EUR If no base currency is supplied then the system will default the base to be GBP and the cross rate to be 0.8 / 0.7 = 1.142857142857. If IEP is set to have a base currency rank and GBP is not then the base currency will be set to IEP and the rate to be 0.875. Validation Rules: 3 character numeric. Optional input. If this field is changed then the relevant currency record must be input and authorised for the change to take effect.

      • getNoOfDecimals

        public TField getNoOfDecimals()
        Returns the NoOfDecimals

        Field description: Defines the Number of Decimal places of the Currency. Used to verify that amounts are not entered with decimal places exceeding the number entered in this field. All calculations, e.g. interest, charges, revaluation etc., will be rounded to the Number of Decimal places specified in this field. When the amount fields are displayed, they will also be shown with their corresponding decimals. Validation Rules: 1 numeric character in the range 0 to 8. (Optional input; Default value = 2 decimals.) When the user is creating a new Currency code, this field will already indicate the default value of 2. If the Currency you are creating does not have 2 decimals (e.g. Italian Lira, Japanese Yen), this default value must be changed to the appropriate number of decimals. Once the Currency code has been authorised, the value of this field cannot be changed.

      • getNumericCcyCode

        public TField getNumericCcyCode()
        Returns the NumericCcyCode

        Field description: Specifies the numeric code which corresponds to the alpha ID. This code must be unique and provides an alternative means of referencing the Currency details in T24 Applications. The code may be set by banks currently using Numeric Currency Codes thus allowing these same codes to continue in use (in effect acting as a translation table). It could, alternatively, be set to standard codes used in a particular Country for central bank reporting, etc. Another use is to set these codes according to the frequency of use of the various Currencies. If, most transactions are, for example, in US dollars, Deutsche Marks and Belgian francs you could the Numeric Currency Codes could be allocated as follows: 1 : to US Dollar 2 : to Deutsche Mark 3 : to Belgian Franc These short codes can then be used in Transaction Processing Applications to minimise the input required. Validation Rules: 1-3 numeric characters, including 0. This will allow a maximum number of 999 currency codes. (Mandatory input)

        Returns:
        TField - The NumericCcyCode

      • setBaseCcyRank

        public void setBaseCcyRank(TField value)
        Set the specified value for BaseCcyRank
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Indicates whether a currency should always be the base currency for currency conversions, in the event that a base currency has not been supplied. Each currency can optionally be assigned a unique base currency rank in the same manner as currency rank. The values are stored in the application RANK.BASE.CURRENCY. If two currencies are involved in a conversion and both have a base ccy rank defined then the currency with the lowest rank will take preference. N.B. It has been stated that rates to the EURO (EUR) will always be quoted as the number of other currency units to 1 EUR, even when 1 unit of the other currency is worth more than 1 EUR. For example assuming that 0.71 GBP were worth 1 EUR then the effect of not supplying a base currency or rate to the conversion routine would result in the base currency being set to GBP and the rate expressed as the inverse of 0.71 i.e 1.408451. Setting the EUR to have a base currency rank of 1 would result in the EUR being the base currency and the rate to be 0.71. Great care should be taken when setting the currency rank, there are very few instances where a rank needs to be set, for example the EURO as described. Another example would be a local currency conversion from a currency where rates are quoted as number of units of foreign to 1 local (even when the rate is less than 1) to the EURO, e.g. IEP to EUR. The system will default the base currency for cross rates to be that currency which has a rate greater than 1, e.g. local ccy = EUR CCY1 = IEP, 0.8 to 1 EUR CCY2 = GBP, 0.7 to 1 EUR If no base currency is supplied then the system will default the base to be GBP and the cross rate to be 0.8 / 0.7 = 1.142857142857. If IEP is set to have a base currency rank and GBP is not then the base currency will be set to IEP and the rate to be 0.875. Validation Rules: 3 character numeric. Optional input. If this field is changed then the relevant currency record must be input and authorised for the change to take effect.

        Parameters:
        value - The value to set
        Returns:
        nothing
