        Field description: Specify the routine which will be supplying the values for this field and it has the precedence over the field details given here. This field is used to call a user defined routine to map additional data which is not normally available. The routine should return back the masked output. The parameters to the routine is INPUT, OUTPUT, ERROR Where INPUT : the FIELD.NAME should be passed as incoming argument OUTPUT : the field content returned back from the routine ERROR : If any error Is encountered, then the same should be displayed and do not proceed further. Validation Rules: Should be a valid record in PGM.FILE with "S" type. Up to 35 alphanumeric characters, of which the first must be an '@' character. The routine must be catalogued and in the VOC file. Optional input.

      • getDataLenMask

        public TField getDataLenMask()
        Returns the DataLenMask

        Field description: The content returned for this field is padded / truncated using the length mark as given here. This is useful for padding Zeros / Spaces or special characters before the field value. Can be used to control the formatting of a data item. Bn Right adjust into a length of n by inserting blanks on the left. (n must not be zero.) *n As for Bn but place asterisks before the first significant digit. Zn As for *n but placing zeros before the first significant digit. , Insert commas to separate thousands. . Change '.' to ','. . , or , . Insert points to separate thousands, and use a comma for decimals. - Put the sign after the amount. (The sign normally precedes the amount.) '-' can only be used in conjunction with other masks. A Print the absolute value without a sign, even if it is negative. D Print DR after the amount if it is negative. C Print CR after the amount if it is positive and DR if negative. % Print % after the data. 00x00x000x Digits will be substituted for the zeros. Any other characters (x represents 1 or more characters) will be printed wherever they were in the mask. 'L'00x00x000x ) 'R'00x00x000 As above except that user specifies that format is Left or Right justified. If not specified the default is Right justified. 'L'00x00x000x 'R'00x00x000xX More than one masking parameter may be specified, separated by a space. However, 00x00x000x may not be combined with other masking parameters. Note : Zero amounts will always be printed unless the name of the field is entered in DEPENDENT.ON. Examples: Mask Entered Field contains 1234.67 Field contains -1234.67 *10 ***1234.67 **-1234.67 Z10 0001234.67 -001234.67 B10 - 1234.67 1234.67- , - 1,234.67 1,234.67- . 1234.67 -1234.67 B10 . , C 1.234,67CR 1.234,67DR B10 , D 1,234.67 1,234.67DR The above examples show that various Masking options may be combined. The length of 10 specified includes the comma, point and a preceding sign, but not '%', 'CR' 'DR', or a sign that follows the field. Mask Entered Field Contains: 123456 00-(00-00) 12-(34-56) "L"000- 000012-3456 "R"000- 0000123-456 Validation Rules: Where n is 1 or more digits and x is 1 or more characters: 'Bn' '*n' 'Zn' ',' '.' '-' 'C' 'D' 'A' '%' '00x00x000x..etc. ' (Optional input)

        Parameters:
        Returns:
        TField - The DataLenMask

      • getDataSplText

        public TField getDataSplText()
        Returns the DataSplText

        Field description: Specify the free text details which has to be populated in outgoing file incase of static information. There are chances that the field may represent line break, end of line etc. Hence the equivalent ASCII character can be defined in the format of <NNN>. During formation of the outward file, the ASCII character will be converted using CHARX (NNN) to LINE For example: for carriage return can be represented by CHARX (013) To represent this, this field can be input as <013>. The <013> will be converted to CHARX (013) during formation of outward file. No validations will be required & the content of the field will be mapped as such. The FIELD.LENGTH is also ignored. Validation Rules: Optional field 1-35 Alphanumeric Characters

        Parameters:
        Returns:
        TField - The DataSplText
