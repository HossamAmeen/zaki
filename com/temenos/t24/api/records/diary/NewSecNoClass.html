        public TField getBookCost()
        Returns the BookCost

        Field description: This is the book cost with which the new securities will update the SECURITY.POSITION file. It is different fromthe price of the securities as in Corporate Actions such as Call Payments the the cost to the portfolio of theresultant securities will not be solely the price paid in the Corporate Action but will also include the cost ofthe partly paid securities the portfolio already holds. This field should also be used for events such as Scrip Dividends where the shares are free though they do havea cost assciated with them as they are received in lieu of cash. If an amount in entered in both the BOOK.COST and PRICE fields then this field will take precedence whenupdating the cost of the securities to the portfolio. Any amount input into this field will not effect the accounting. If accounting entries are required then anamount should be entered into the PRICE field. The field will only allow input when security positions are being updated by the Corporate Action. If input is allowed it will be optional. The cost is in the security of the Diary record. Validation Rules: Optional input, numeric field. Input only allowed if the associated NEW.SEC.NO field is populated.

        Returns:

      • getSellNewSec

        public TField getSellNewSec()
        Returns the SellNewSec

        Field description: Flag to indicate whether to Sell New securities which are being offered or not. If this field is set to Y then the securities received by the portfolio in the associated NEW.SEC.NO, NEW.RATIOand OLD.RATIO fields will be sold by the corporate action at the price input in the NEW.PRICE field. This willresult in two securities movements (and hence two SECURITY.TRANS records) being generated for the portfolio by thecorporate action; one crediting the portfolio with the new securities and one debiting the portfolio with thesecurities it has received being sold. Effectively this field allows Enhanced Scrip Dividends to be processed as a single event. Input allowed only if NEW.SECURITIES field in DIARY.TYPE record is set to YES. Validation Rules: Only input of YES or NO allowed. Input only allowed if the NEW.SECURITIES field is set to Y on the DIARY.TYPE record that defines this event. Will default to NO.

        Parameters:
        Returns:
        TField - The SellNewSec
