      • getApInactivityLink

        public TField getApInactivityLink()
        Returns the ApInactivityLink

        Field description: If AP.INACTIVITY.TYPE is mentioned, then this field becomes mandatory.While AP.INACTIVITY.TYPE field decides by what method the contract is deemed inactive,AP.INACTIVITY.LINK decides what values to be validated against for determining the activity. If AP.INACTIVITY.TYPE is DATE then a valid field from the application that contains the date of expiry is to be mentionedin this field.The system validates the date present in the field against period end date to decide the inactivity. If STATUS is mentioned as TYPE, then this LINK field contains a valid field name from the application that determines the statusfollowed by the expected value separated by >.For example, CONTRACT.STATUS>MAT, where CONTRACT.STATUS is a field from the application. If AP.INACTIVITY.TYPE is API, then field should carry a valid EB.API record. The arguments of this API are as follows: ApplicationName (In), ContractRecordDetails (In), InactivityFlag (Out)

        Returns:

        public TField getApInactivityType()
        Returns the ApInactivityType

        Field description: A contract can be deemed inactive by different methods. For example, a record may exist in LIVE table but the status of it may be marked as EXPIRED The field allows configuration for identifying how a contract is deemed inactive.
