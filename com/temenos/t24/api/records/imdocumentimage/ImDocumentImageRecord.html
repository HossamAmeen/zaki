      • getImageReference

        public TField getImageReference()
        Returns the ImageReference

        Field description: This field can be used as a reference to the image being stored. The reference can be free format, but if input is made in IMAGE.APPLICATION and/or COMPANY.MNEMONIC the value will be checked as a valid record of that application.Examples of values related to an application are:ACCOUNT NUMBER where the application is ACCOUNT CUSTOMER ID where the application is CUSTOMERExamples of unrelated values are:DOCUMENT 1634562 PASSPORT PHOTO OF MR ALWYN P JONESThe value from this field is used to provides the key for the IM.REFERENCE cross reference file. This is used to link all related images so that they can be accessed speedily by other T24 applications.Where a related IMAGE.APPLICATION has been entered, it records instances of image records related to that application/reference combination.For example there may be several image records for one specific ACCOUNT.When in any application, except IM.DOCUMENT.IMAGE, the request for an image will use the application field content or the application id to search for related images. If only one is found it is displayed immediately, if more than one exists an Enquiry display will show the records for the user to choose the appropriate image. If none are found a message will advise the user of the search criteria used.It is therefore possible to check for images related to an ACCOUNT when you are in any valid field containing that ACCOUNT number. Validation Rules Up to 18 alphanumeric characters. Values will be checked against the T24 application specified in IMAGE.APPLICATION. ACCOUNT & CUSTOMER mnemonics may be entered as well as the numeric equivalents.

        Parameters:
        Returns:
        TField - The ImageReference

        public void setImageReference(TField value)
        Set the specified value for ImageReference
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field can be used as a reference to the image being stored. The reference can be free format, but if input is made in IMAGE.APPLICATION and/or COMPANY.MNEMONIC the value will be checked as a valid record of that application.Examples of values related to an application are:ACCOUNT NUMBER where the application is ACCOUNT CUSTOMER ID where the application is CUSTOMERExamples of unrelated values are:DOCUMENT 1634562 PASSPORT PHOTO OF MR ALWYN P JONESThe value from this field is used to provides the key for the IM.REFERENCE cross reference file. This is used to link all related images so that they can be accessed speedily by other T24 applications.Where a related IMAGE.APPLICATION has been entered, it records instances of image records related to that application/reference combination.For example there may be several image records for one specific ACCOUNT.When in any application, except IM.DOCUMENT.IMAGE, the request for an image will use the application field content or the application id to search for related images. If only one is found it is displayed immediately, if more than one exists an Enquiry display will show the records for the user to choose the appropriate image. If none are found a message will advise the user of the search criteria used.It is therefore possible to check for images related to an ACCOUNT when you are in any valid field containing that ACCOUNT number. Validation Rules Up to 18 alphanumeric characters. Values will be checked against the T24 application specified in IMAGE.APPLICATION. ACCOUNT & CUSTOMER mnemonics may be entered as well as the numeric equivalents.

        value - The value to set
        Returns:
