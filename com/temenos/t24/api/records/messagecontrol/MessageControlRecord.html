      • getStatus

        public TField getStatus()
        Returns the Status

        Field description: Specifies whether a delivery is to be immediate or held on the delivery system queue for later release. It may be that a User wishes to refrain from sending a delivery message to a customer immediately in which case he may take advantage of this hold facility. Thus when a contract is authorised no delivery is sent automatically (as would be the case under non-hold status) but rather the User has to manually release the message (done within the delivery system) when he decides he wants the message to be sent. Validation Rules: Alpha field - 4 characters allowed. (Only HOLD may be input otherwise the field is blank.) (Optional input; default is blank which means 'do NOT hold')

        Parameters:
        Returns:
        TField - The Status
