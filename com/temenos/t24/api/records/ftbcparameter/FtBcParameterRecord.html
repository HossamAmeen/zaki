      • getCheckSortCode

        public TField getCheckSortCode()
        Returns the CheckSortCode

        Field description: This field defines whether incoming FT transactions from the local clearing system should have the SORT.CODE of the sender bank validated against the BC.SORT.CODE file. If the input to this field is 'YES' then the sort code from the incoming transaction will be checked against the BC.SORT.CODE file. If the sort code is not present then the FT transaction will be placed on 'HLD' and an error message produced which will be placed on the EXCEPTION.LOG.FILE. Validation Rules: Y(ES) The sort code will be checked. N(O) The sort code will not be checked. (Mandatory input).

        Field description: This field defines the maximum and minimum length allowed for the sort code on the BC.SORT.CODE file. This field denotes the maximum and minimum length allowed for sort codes on the BC.SORT.CODE file. If input then the maximum length must be defined with the minimum length being optional. Some examples of valid input are: a) 10 - this means that the sort code may be up to 10 characters in length with no minimum defined. b) 05 - this means that the sort code may be up to 5 characters in length with 0 being a valid input. c) 006.1 - this means that the sort code may be up to 6 characters in length with any number of leading zeroes allowed. The minimum length is defined as 1 therefore at least one character must be entered. Validation Rules: Input is in two parts, the second of which is optional: (i) Maximum length - this may be preceeded by either one or two zeroes. One zero denotes that zero is a valid input, two zeroes denotes that any number of leading zeroes are allowed. (ii) Minimum length - this is optional but if input must be delimited by a '.'.
