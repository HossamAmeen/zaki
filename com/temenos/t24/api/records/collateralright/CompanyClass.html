      • getCompany

        public TField getCompany()
        Returns the Company

        Field description: The company in which the LIMIT.REFERENCE record exists. This field is intended for use in multi-company environments only. It indicates the company to which any contract id or limit reference given in the next field belongs. Input must be a valid company mnemonic defined on the company file. The field is defaulted in a single-company environment to the value of the company mnemonic in use. In a multi-company environment, the default value is determined by the appropriate field on the (signed-on) company record according to the content of the corresponding LIMIT.REFERENCE. Multivalue field associated with LIMIT.REFERENCE. Validation Rules: Up to 3 alphabetic characters (company mnemonic). Multivalue field. (Optional input; default value determined by signed-on company)

      • getLimitRefCust

        public TField getLimitRefCust()
        Returns the LimitRefCust

        Field description: Specifies the customer of the contract in the LIMIT.REFERENCE field or the customer number to be covered when no LIMIT.REFERENCE is defined. A collateral right record is created using the customer number of the provider of collateral. A particular collateral right may cover an entire customer&#146;s risk, or particular contracts / limits. The customer being covered may be the customer or another customer. This field indicates the beneficiary of the contract / limit covered, or if no specific contracts are to be covered it can be used to indicate the beneficiary of the collateral. The value is defaulted when an old limit or a contract is attached in LIMIT.REFERENCE, and can be entered when no contracts are specified. Defaulting will not happen for new limits Validation Rules: Optional Input Must be the Customer of the associated LIMIT.REFERENCE if defined. Must be a valid CUSTOMER number Input allowed only when old limits and contracts are defined in the associated LIMIT.REFERENCE.

      • getLimitReference

        public TField getLimitReference()
        Returns the LimitReference

        Field description: The account, contract id or limit reference against which the collateral right is valid. This field is used to link the collateral at deal or limit level. If left blank, it is assumed by the system that the collateral is valid against all outstandings of the customer. Valid input is any contract, account number or limit idas follows: account number; the account number from the account file (mnemonic not accepted) foreign exchange transaction reference letter of credit transaction reference loans (& deposits) contract number MD.DEAL contract number mortgage contract number, money market contract number limit ; actual id of the actual limit record. More than one of the above references may be entered and the references need not all belong to the same application. In each case, the reference given should belong to the customer mentioned in the item-id to the collateral right record. If this is not the case, an override condition must be accepted. Note that any link to multiple transactions may be made by the use of a product-level limit and that, similarly, a link to multiple limits may be made by the use of a global limit reference. Multivalue field in association with COMPANY field. Note : A mixture of limit and contract/account references is permitted. Validation Rules: Up to 32 alphanumeric characters; exact format determined by the application to which the reference belongs. Multivalue field. (Optional input) Limit reference can be specified with Main group limits. Sub group limits are not allowed.

      • getPercentAlloc

        public TField getPercentAlloc()
        Returns the PercentAlloc

        Field description: If a percentage value is entered then Collateral can be allocated to a limit at any level i.e. product, sub product or below sub product level. If no value or zero value is entered then collateral will be allocated at the top level of the limit structure for the limit defined in the associated LIMIT.REFERENCE field. If a percentage value is entered then that percentage of the execution value of the relevant collateral item will be allocated to the SECURED.AMT of the limit record defined in the associated LIMIT.REFERENCE field. Example 1: - No percentage values entered Collateral execution value = 8m. Limit1 then limit2 are defined in the LIMIT.REFERENCE field. Limit 1 defined as MAXIMUM.SECURED of 8m on limit record. Limit 2 defined as MAXIMUM.SECURED of 8m on limit record. Limit 2 is utilised by a transaction of 5m, which results in a SECURED.AMT of 5m. Limit 1 is then utilised by a transaction of 5m, which results in a SECURED.AMT of 3m, the remainder of the collateral. Then the batch processing takes place. The limits will be modified so that limit 1 (the first limit defined on COLLATERAL.RIGHT) will have a SECURED.AMT of 5m while limit 2 now has a secured amount of 3m, (the remainder). Example 2: - Percentage values entered The limit structure is as follows. Limit 100 is at product level. Limit 101 is at sub product level. Limit 102 is at sub product level Limit 103 is at sub product level Collateral with an execution value of 8m is assigned as follows via LIMIT.REFERENCE and PERCENT.ALLOC 75% to limit 101. 25% to limit 102. In this case all of the collateral is assigned to the limit structure 100. The secured amount, which will be allocated to limit 101, is 6m (75% of the execution value of 8m) and 2m for limit 102 (25% of the collateral execution value). If limit 101 is utilised for more than 6m and limit 102 is not used then the secured amount for 101 will still be 6m. If a transaction takes place which impacts limit 103 then the secured amount will be zero, as the limit has not been defined on COLLATERAL.RIGHT. Validation Rules: Optional input, up to 3 numeric characters. If a value is entered then the sum of all the PERCENT.ALLOC fields must equal 100. Multi valued field in the association beginning at COMPANY
