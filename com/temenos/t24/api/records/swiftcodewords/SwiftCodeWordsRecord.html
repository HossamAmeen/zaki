      • addSuppPrompt

        public void addSuppPrompt(java.lang.CharSequence param)
        Appends the specified SuppPrompt to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Used in connection with the Queries and Answer message types (n95 and n96). These message types use code numbers which have a special meaning. Where the message needs additional information such as dates or text, the code number is appended with the details. This field shows the explanation of the item, and where required the type of detail needed for the values in the SUPPLEMENT.INFO field. In order to identify these special codes, the id is entered as SWN95/16/ or SWN96/16/ to allow the values to be readily used with context work based Enquiries for each message series. Note that the data used in EB.QUERIES.ANSWERS application expects only the /16/ format. Example Codes 23 and 30 are used.

        /23/Please send us a copy of the truncated cheque with transaction reference (1)
        /30/ Amount protested by the Issuer
        Code 23 needs some extra information, indicated by the (1), so the user enters /23/abc1003. This would appear in the message as:
        :75:/23/abc1003
        /30/
        These would be entered in the fields as:
        Record id SUPPLEMENT.INFO SUPP.PROMPT
        SWN95/23/(1) Please send us a copy of the truncated cheque with transaction reference (1)
        SWN95/30/Amount protested by the Issuer
        Validation Rules: Free format text

        Parameters:
        param - the SuppPrompt to append
        Returns:
        nothing

In order to identify these special codes, the id is entered as SWN95/16/ or SWN96/16/ to allow the values to be readily used with context work based Enquiries for each message series. Note that the data used in EB.QUERIES.ANSWERS application expects only the /16/ format. Example Codes 23 and 30 are used.

        /23/Please send us a copy of the truncated cheque with transaction reference (1)
        /30/ Amount protested by the Issuer
        Code 23 needs some extra information, indicated by the (1), so the user enters /23/abc1003. This would appear in the message as:
        :75:/23/abc1003
        /30/
        These would be entered in the fields as:
        Record id SUPPLEMENT.INFO SUPP.PROMPT
        SWN95/23/(1) Please send us a copy of the truncated cheque with transaction reference (1)
        SWN95/30/Amount protested by the Issuer
