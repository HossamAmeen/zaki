      • getChargeStatus

        public TField getChargeStatus()
        Returns the ChargeStatus

        Field description: Code to identify the required status of the charge. Used to identify if the charge has been taken, claimed, or to be deferred or waived.User defined field. The codes that are used to identify various states of the charge are as follows: blank - The charge has not been taken yet 2 - The charge has to be taken immediately to the debit of the charge account given above 3 - The charge has been claimed. If status 3 is input then the charge will be debited to CRF (claimed bucket) and profit and loss will be credited. At this stage tax if any associated with the FT.COMMISSION.TYPE or the FT.CHARGE.TYPE will not be debited to the CRF bucket. 4 - A claimed charge has been settled. This will be allowed to be input only in LC.ACCOUNT.BALANCES. 5 - A claimed charge has been written off to the debit of profit or loss. Input allowed only in LC.ACCOUNT.BALANCES file. 15 - The charge has been waived. Notes: (i) A claim advice is sent to the charge party when charge status is input as 3. (ii) When the charge is paid the operator will manually access the LC.ACCOUNT.BALANCES file, where all the charge details are presented after authorization of the L/C, update the file from 3 to 4 to indicate that the payment has been received, and the CRF Claimed bucket reimbursed. (iii) When claimed charge is not paid, the operator will manually access the LC.ACCOUNT.BALANCES file and change the charge status from 3 to 5. This will reverse the profit and loss and credit to CRF bucket. (iv) When the charge status is null, only the charge amount will be calculated and written in LC.ACCOUNT.BALANCES. The related tax will be calculated at the applicable rate on the day the charge status is set to 2. (v) When the charge status is 15, the related charge is considered waived and written in LC.ACCOUNT.BALANCES. This charge cannot becollected in the future and the related tax will not be stored in LC.ACCOUNT.BALANCES. Validation Rules: User Input. 1 digit. Associated mulivalue field

        Parameters:
        Returns:
        TField - The ChargeStatus

        public void setChargeStatus(TField value)
        Set the specified value for ChargeStatus
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Code to identify the required status of the charge. Used to identify if the charge has been taken, claimed, or to be deferred or waived.User defined field. The codes that are used to identify various states of the charge are as follows: blank - The charge has not been taken yet 2 - The charge has to be taken immediately to the debit of the charge account given above 3 - The charge has been claimed. If status 3 is input then the charge will be debited to CRF (claimed bucket) and profit and loss will be credited. At this stage tax if any associated with the FT.COMMISSION.TYPE or the FT.CHARGE.TYPE will not be debited to the CRF bucket. 4 - A claimed charge has been settled. This will be allowed to be input only in LC.ACCOUNT.BALANCES. 5 - A claimed charge has been written off to the debit of profit or loss. Input allowed only in LC.ACCOUNT.BALANCES file. 15 - The charge has been waived. Notes: (i) A claim advice is sent to the charge party when charge status is input as 3. (ii) When the charge is paid the operator will manually access the LC.ACCOUNT.BALANCES file, where all the charge details are presented after authorization of the L/C, update the file from 3 to 4 to indicate that the payment has been received, and the CRF Claimed bucket reimbursed. (iii) When claimed charge is not paid, the operator will manually access the LC.ACCOUNT.BALANCES file and change the charge status from 3 to 5. This will reverse the profit and loss and credit to CRF bucket. (iv) When the charge status is null, only the charge amount will be calculated and written in LC.ACCOUNT.BALANCES. The related tax will be calculated at the applicable rate on the day the charge status is set to 2. (v) When the charge status is 15, the related charge is considered waived and written in LC.ACCOUNT.BALANCES. This charge cannot becollected in the future and the related tax will not be stored in LC.ACCOUNT.BALANCES. Validation Rules: User Input. 1 digit. Associated mulivalue field

        value - The value to set
        Returns:
