        public TField getFurtherPooling()
        Returns the FurtherPooling

        Field description: This field details whether or not further orders can be pooled into this pool group or not. When this field is set to YES this group record will be referenced on the FD.POOL.GROUP.XREF file and as such further orders may be pooled into the group. When this field is set to NO then this group is not referenced on the FD.POOL.GROUP.XREF file and as such will not be available to the pooling process. Validation Rules: Must be either YES or NO.

        Parameters:
        Returns:
        TField - The FurtherPooling

      • getGroupStatus

        public TField getGroupStatus()
        Returns the GroupStatus

        Field description: The status of the pooling group is held in this field and will hold one of the following values: OPEN ==== When all the orders in the pool group are open (i.e. they have not yet been approved) the GROUP.STATUS is set to OPEN. At this stage further orders may be pooled into this group. APPROVED ======== When at least one, but not all, orders have been approved the GROUP.STATUS is set to APPROVED. At this stage further orders may NOT be pooled into this group. CLOSED ====== When all the orders in the group have been approved and the placement record created the GROUP.STATUS is set to CLOSED, and no further orders may be pooled in to this group. EXIST ===== For notice contracts, where it is possible to pool orders into an existing placement (as set in FD.PARAMETER), the GROUP.STATUS is set to EXIST and further orders may be pooled into the group providing the order is not backvalued and that the value date of the order is greater than, or equal to, the value date of the group. Validation Rules: Must be one of OPEN, APPROVED, CLOSED or EXIST.

        Parameters:
        Returns:
        TField - The GroupStatus
