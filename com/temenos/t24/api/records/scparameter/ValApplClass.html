        public TField getUsrPorSelRtn()
        Returns the UsrPorSelRtn

        Field description: Specifies the user defined subroutine for the respective valuation application entered in field VAL.APPL. This form amultivalue set with field VAL.APPL USR.POS.RTN Purpose Invoked This routine Output array contains list of contracts which should be included in portfolio valuation. If the output array is empty the calling routine searchs for the list of contract Ids as it does usually (i.e.if array is empty SC.OL.VAL.FX reads CUSTOMER.FOREX file and looks for deals attached to this portfolio). Routine usage example EX.USR.PORT.SEL.RTN ( IN.PORTFOLIO, OUT.ID.LIST1, OUT.ID.LIST2, OUT.ID.LIST3) Where OUT.ID.LIST1 is list of contract Ids separated by Field marks (for example for application FOREX it will look asFX0110100001:@FM:FX0110100003.). OUT.ID.LIST2 This is used only in case if LC and represents the contract ids from LC.BENEFICIARY separated byField marks. OUT.ID.LIST3 This is used only in case of LC and represents the contract ids from LC.ISSUING.BANK separated byField marks. Validation Rules:

        public TField getUsrPosRtn()
        Returns the UsrPosRtn

        Field description: Specifies the user defined subroutine for the respective valuating application entered in field VAL.APPL. Forms amultivalue set with field VAL.APPL and USR.PORT.SEL.RTN Purpose Invoked This user routine will be invoked just prior to the write statement on SC.POS.ASSET. The routine is passed the SC.POS.ASSET record, After processing in the calling routine and just before updatingthe SC.POS.ASSET file. The user can make the modifications to this record to his requirement and pass back thearray to the calling routine. So the changes can be reflected in the SC.POS.ASSET file. The routine should be specified as EXAMPLE.ROUTINE(IN.SC.POS.ASSET.REC) Where IN.SC.POS.ASSET.REC contains the SC.POS.ASSET record. Validation Rules:

      • getValAppl

        public TField getValAppl()
        Returns the ValAppl

        Field description: The valuation application is used to define routines called to evaluate, modify and enter selected portfoliopositions in the SC.POS.ASSET file. This field specifies the module code for which the associated routines areintended. This forms a multivalue set with fields USR.PORT.SEL.RTN and USR.POS.RTN. Purpose - These user defined routines are intended to give the flexibility to include or exclude contracts froma particular portfolio when updating the SC.POS.ASSET file for valuation purpose. In order to amend or make changesto the SC.POS.ASSET file, the user defined routines will be called from existing routines. USR.PORT.SEL.RTN routine will be invoked just prior standard search of customer contracts which are included inportfolio. USR.POS.RTN routine will be invoked just prior to the write statement on SC.POS.ASSET. The routine is passed therecord of the SC.POS.ASSET after processing in the calling routine and just before the updating the SC.POS.ASSETfile, the user can make the modifications to this record to his requirement and pass back the array to the callingroutine. So the changes can be reflected in the SC.POS.ASSET file. If the VAL.APPL is SC then the USR.POS.RTN routine is passed the record of SC.POS.ASSET, after processing in thecalling routine and just before the updating the SC.POS.ASSET file. But this routine doesnt overwrite the valuecalculated by the System. The USR.PORT.SEL.RTN routine is not triggered for SC application. Validation Rules: 2 alpha character Module Code. Input must be one of the following AC, FD, FR, FX, DP(Deposits), MD, LC, RP,SW,SC (Optional input)

        Parameters:
        Returns:
        TField - The ValAppl
