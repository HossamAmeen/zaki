      • getCustomerFloat

        public TField getCustomerFloat()
        Returns the CustomerFloat

        Field description: Specifies the default number of days to be applied for any Customer Value Date applicable to the Payment Type for this type of transaction. This field allows the User to define special Customer Value Date conditions at the transaction level by specifying within Payment Type a number of days between the Value Date applied to Customer and the Value Date applicable to the bank. If zero '0' is input, the Customer will receive the Same day value as the bank itself. Example: INCOMING TRANSFER DEBIT NOSTRO ACCOUNT (VALUE 15/1) CUSTOMER FLOAT = 3 (15 plus 3 = 18) CREDIT CUSTOMER ACCOUNT (VALUE 18/1) OUTGOING TRANSFER CREDIT NOSTRO ACCOUNT (VALUE 15/1) CUSTOMER FLOAT = 2 (15 less 2 = 13) DEBIT CUSTOMER ACCOUNT (VALUE 13/1) Validation Rules: 0-9 = Number of working days to be added to the debit Value Date to make the credit Value Date for inward transfers and deducted from the credit Value Date to make the debit Value Date for outward transfers. or P = Processing date irrespective of the other Value Date. Associated multivalue field. (Mandatory input if Payment Type field has been input.)

        Returns:

      • getPaymentType

        public TField getPaymentType()
        Returns the PaymentType

        Field description: Defines the Type of Payment for which the Value Date conditions defined in the next three fields are applicable. If ALL is specified then no other Payment Types may be input. If either CONV or NOCONV is input, both must be specified, separately as one is not valid without the other. Note : This field provides the facility to define whether or not conversion is involved in the transaction in order to determine if different Value Date conditions exist: ALL The conditions specified in the next three fields are applicable to all transactions irrespective of whether the transactions involve conversion of Currencies or not. CONV The conditions specified in the next three fields are only applicable to transactions involving conversion of Currencies. NOCONV The conditions specified in the next three fields are only applicable to transactions not involving conversion of currencies. Validation Rules: ALL = All transactions CONV = Only transactions involving conversion of currencies NOCONV = Transactions not involving conversion of currencies. Associated multivalue field with fields PAYMENT.VALUE, CUSTOMER.FLOAT and SAME.CUSTOMER.FLOAT. (Optional input.)

      • getPaymentValue

        public TField getPaymentValue()
        Returns the PaymentValue

        Field description: Specifies the default Payment/Transfer Value date applicable to the Payment Type for this type of transaction. If 'Y', 'N' or 'No' is entered the inputs (a) to (f) are not allowed. (a), (bb), (c) specify the number of calendar or working days to be added to or subtracted from the Processing Date in order to obtain the default Value Date as follows: (a) +/- If days are to be added to or, subtracted from, the Processing Date. (bb) 01-99 Indicates the number of days to be added/subtracted. (c) C If (bb) are calendar days, or W If (bb) are working days. If it is necessary to add or subtract any other days to those specified in (bb) before obtaining the default Payment Transfer Value Date then (d), (ee), (f) are used as follows: (d) As for (a) previously. (ee) As for (bb) previously. (f) C If (c) is 'C' or W If (c) is 'W' Note : Processing Date is the date of the data being processed which may not always be today's date. If different, the Processing Date is displayed next to the User ID on the screen. This field allows you to specify a default payment transfer Value Date for this Transaction type being either the Processing Date, or a number of working or calendar days to be added to or subtracted from the Processing Date. For inward payments the value applies to the debit value date and for all other payments the value applies to the credit value date. Example: A Default Value Date of +03W would be 3 working days forward. or +01W+02C would be 1 working day plus 2 calendar days forward. Validation Rules: Y If the default value date is to be the Run Date. N or NO Where no default is applicable, this will allow the User to input the Value date. or (a) (bb) (c) (d) (ee) (f) As defined below. Associated multivalue field. (Mandatory input.)

        Returns:
