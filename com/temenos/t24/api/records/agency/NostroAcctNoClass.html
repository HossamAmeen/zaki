      • getNostroAcctNo

        public TField getNostroAcctNo()
        Returns the NostroAcctNo

        Field description: Defines the Account Number in our books of our Nostro Account. This field will be input when the host bank maintains a Nostro Account with the Agent being defined. This field is used to relate this account number to the associated OUR.EXT.ACCT.NO which is the account identifier used by the correspondent bank. The default Nostro to be used in any Transaction Processing will always be defaulted from the Nostro Account files and never taken from this file. Note: The customer of the Nostro account defined in this field must have a class of 'Bank' in accordance with the conditions defined in the ACCOUNT.CLASS table. Validation Rules: 2-16 numeric character Account Number or 3-10 type MNE (uppercase alpha or numeric or '.') character mnemonic. Multivalue field to define all Nostro Accounts for this Agent. Associated with field OUR.EXT.ACCT.NO. (Optional input if Agent is defined as a bank and the Account is a Nostro as per ACCOUNT.CLASS file. Otherwise not allowed.) The Currency of the Nostro Account will be checked to ensure that it is the local currency in the Country of residence of the Agent, as Nostros are normally held in the appropriate local currency of the Agent. Where the Currency/Country do not match an override will be required, providing a reason narrative. The customer number on the NOSTRO.ACCOUNT record must correspond to the ID of the Agent being defined. This field becomes NO INPUT if the records id is a BIC.CODE (begins with SW-)

        Parameters:
        Returns:
