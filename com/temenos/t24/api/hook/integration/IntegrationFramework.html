      • getJoinIds

        public List<String> getJoinIds(String sourceApplicationName, String sourceFieldName, List<String> sourceRecordIds, TStructure sourceRecord, String sourceFieldValue, String targetApplicationName, IntegrationContext integrationContext)
        This interface enables the implementer to return join Ids where the data from two related tables need to be fetched but does not have common key fields
        This interface is invoked by integration framework API T24IntegrationFrameworkServiceImpl.modifyJoinId.
        This interface provides access to all the IN parameters of the T24IntegrationFrameworkServiceImpl.modifyJoinId.

        T24 Details: The EB.API hook used by this interface is IF.JOIN.ID.HOOK.
        The T24 field specifying this hook is the CALLBACK.IMPL field in IF.CALLBACK.IMPL.CATALOG.
        There is no specific exception need to be raised. If the return value is NULL or empty, then Integration Framework will skip this join.
        Parameters:
        sourceApplicationName - The IN.FILE.NAME, either placed in the directory or read from the hook/@routine This is the application from which the join definition starts.
        sourceFieldName - This is the field name from the application from which the join definition starts
        sourceRecordIds - This represents the list of incoming ID's that is used to extract the record ID's to get the related data
        sourceRecord - This parameter holds the current transaction record
        sourceFieldValue - This field represents the value of the field that is used to get the join ID
        targetApplicationName - This is the application from which the related field values need to be obtained.
        integrationContext - Integration Framework Context related variables
        Returns:
        List<String> - A list of string representing the record ID's.
