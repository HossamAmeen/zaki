      • getCapRate

        public TField getCapRate()
        Returns the CapRate

        Field description: This field will correspond to the capitalisation rate to be used for bonds that capitalise a portion of thecoupon interest payable. The effective interest rate on the bond will therefore always be equal to this rate plusthe rate in the INTEREST.RATE field. This rate will be used to calculate that portion of the coupon interest thatwill not be paid out to the customer and instead will be capitalised to the principal amount. As a result of thecapitalisation the factor on the bond will increase and hence the value of the nominal holding will increase inturn. The factor in this case will be calculated using the following formula : FACTOR = (LCV + CI)/NOMINAL Where :- LCV - Last Capitalised Value CI - Current Capitalised Interest Validation Rules: Optional Input Change allowed only if associated with a future date

        Returns:

        public TField getCpnDate()
        Returns the CpnDate

        Field description: This field will specify the coupon rate effective date. If this field is entered then the field INTEREST.RATERATE.CH.DATE will be made No input fields. The usage of this field has been specified with an example given below: CPN.DATE 14 APR 2000, CPN.RATE 8.00, CAP.RATE, FACTOR CPN.DATE 14 APR 1998, CPN.RATE 5.00, CAP.RATE 3.00, FACTOR CPN.DATE 14 APR 1996, CPN.RATE 4.40, CAP.RATE 4.60, FACTOR CPN.DATE 14 APR 1994, CPN.RATE 4.00, CAP.RATE 4.00, FACTOR 1.10 Therefore the coupon rate on the 01 sept 98 would be 5%. Validation Rules: Must be descending No Duplicate dates allowed

        public TField getFactor()

        Field description: This field corresponds to the multiplication factor used for bonds that have factors associated with them. Validation Rules: Optional Input Change to this field will be allowed only for those factors associated with effective dates in the future 0.00 will be a valid rate If the CAP.RATE field has been entered then this field may only be entered during initial setup of the security for dates in the past. The factors in the future in this case will be automatically calculated by the system.
