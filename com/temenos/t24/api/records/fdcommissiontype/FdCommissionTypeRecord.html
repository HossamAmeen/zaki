        public TField getTaxCode()
        Returns the TaxCode

        Field description: The tax code, or TAX.TYPE.CONDITION ID to be used to take additional tax from the customer based on the commission amount. Tax can be applied to the commission calculated, and will be debited from the customer at the same time as the commission. Validation Rules: 1 or 2 digit Tax code. (Optional input) A tax code with a date earlier than or equal to Today's date must exist. A valid TAX.TYPE.CONDITION ID defined in the format *tax.type - sub type where tax type is a valid TAX.TYPE on the TAX.TYPE table, and sub type is a sub division of the TAX.TYPE. for example *VAT, *VAT-001.

      • getUsesCustBasis

        public TField getUsesCustBasis()
        Returns the UsesCustBasis

        Field description: This field , when set to "YES" will ensure that the file 'FD.CUST.VALUE' will be taken into consideration when calculating the commission rate. 'FD.CUST.VALUE' is a file that holds the total of the customer's banking portfolios, excluding 'ASSET.TYPE' and 'SUB.ASSET.TYPE,' as defined on the "FD.PARAMETER.FILE". In summary, this field specifies if the commission is based upon customer, rather than deal value. .Validation Rules: Optional Input. "Yes," "No," or null are valid If this field is set to "Yes," then input to the field CALC.TYPE, must be "Level." It cannot be either "Band," or "Flat. Input to this field cannot be altered, once the record has been authorised.

        Returns:
        TField - The UsesCustBasis
