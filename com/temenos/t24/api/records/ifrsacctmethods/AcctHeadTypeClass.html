      • getAcctHeadType

        public TField getAcctHeadType()
        Returns the AcctHeadType

        Field description: ACCT.HEAD.TYPE forms part of a multi-value set to specify the asset types, which can be FAIRVALUE, AMORTISED, DISCLOSURE, IMPAIR.AMORTISED, UNWIND, IMPAIR.AMC.ADJUST, IMPAIR.FAIRVALUE, AMORTISED.UNDER.IMP, IMPAIR.FV.ADJ.INC or IMPAIR.FV.ADJ.DEC , IMPAIR.INTEREST. Also holds the hedge asset types HEDGED.EFFECTIVE, HEDGING.EFFECTIVE, HEDGED.INEFFECTIVE, HEDGING.INEFFECTIVE, HEDGED.CLOSED.RELATION, HEDGING.CLOSED.RELATION For example for Fair value in equity - the idea is to post EIR accrual to PL and the difference between the EIR accruals and the fair value estimation to Equity. So we need a asset type called AMORTISED - where EIR accrual will be calculated as the NPV at EIR rate and the difference between the contract posting and the calculated posting will need to be posted as an adjustment. After this we need to have a fair value estimation say FAIRVALUE - the calculation would be the NPV at market rate - and the posting would be the difference between contract posting, EIR posting and the calculated amount. So in order to ensure this happens asset type for AMORTISED needs to be defined before FAIRVALUE. The Acct methods AMORTISED-STAGE1, AMORTISED-STAGE2, FAIRVALUE-STAGE1, FAIRVALUE-STAGE2 to be specified as a part of IFRS9 standards.

        Validation Rules: : Hedge related asset types are allowed only when 'IH' product is installed in the company. If HEDGED.EFFECTIVE is defined, then system validates for HEDGING.EFFECTIVE also to be defined

        Asset types related to IFRS 9 are allowed only when 'I9' product is installed in the company, ST.REGULATORY.PARAMETER configured for IFRS record and IFRS.PARAMETER is being set for the current company.

        Mandatory Input.

      • getBalToExld

        public TField getBalToExld()
        Returns the BalToExld

        Field description: A Valid record from the table AC.BALANCE.TYPE with REPORTING.TYPE as VIRTUAL. This is to define the balance types to exclude from the T24 balance calculation. Validation: The normal accounting heads AMORTISED, FAIRVALUE and DISCLOSURE will have the same AC.BALANCE.TYPE record attached to the new field BAL.TO.EXLD. All normal accounting heads will uniformly follow a single T24 balance. When the field BAL.TO.EXLD is populated for AMORTISED accounting head and left blank for FAIRVALUE or DISCLOSURE, system will default the BAL.TO.EXLD from AMORTISED to other normal accounting heads. The impair accounting heads IMPAIR.AMORTISED, UNWIND, IMPAIR.AMC.ADJUST, AMORTISED.UNDER.IMP, IMPAIR.FAIRVALUE, IMPAIR.FV.ADJ.INC, IMPAIR.FV.ADJ.DEC will have a common BAL.TO.EXLD value. Any value which is different will invoke error.

        Parameters:
        Returns:
        TField - The BalToExld
