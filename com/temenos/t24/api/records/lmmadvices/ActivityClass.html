        public TField getActivity()
        Returns the Activity

        Field description: Specifies the reference number which uniquely identifies a particular activity. If neither the 'ALL' nor the 'NONE' option are taken in field ALL NONE ADVICES then each activity number must be entered in this multivalue field - along with a Y(es) or N(o) in the field REQUIRED Y N stating whether a confirmation/advice is required for this activity or not. Note: The activities input whether required or not will be deemed to be the only relevant activities for the appropriate category. Validation Rules: Numeric field - 4 characters. Multivalue field. (Optional input; default value is blank) Input prohibited when ALL NONE ADVICES is set to "NONE". Input prohibited when ALL NONE ADVICES is set to "ALL".

      • getDaysPriorPost

        public TField getDaysPriorPost()
        Returns the DaysPriorPost

        Field description: Indicates the number of days prior to or after an event when an advice will be despatched. This field indicates the number of days prior to or after an event when an advice will be despatched. A minus sign preceding the no. of days will result in the advice being sent after the event date. Validation Rules: Input must be of the following form:- NNN or XNNN or "blank" where N = a number of days. X = a minus sign. "blank" = same day. (Optional input. Default = "blank".) (For LD.LOANS.AND.DEPOSITS only.) Input must be of a valid form. Input restricted to pre-advice type activities. (see overview for full list).

      • getRequiredYN

        public TField getRequiredYN()
        Returns the RequiredYN

        Field description: Specifies whether the activity in the previous field should produce a confirmation/advice or not. If neither the 'ALL' nor the 'NONE' option are taken in field 1 then each activity number must be entered in the field ACTIVITY and a Y(ES) or N(O) entered in this field. This states whether a confirmation/advice is to be sent (YES) or not (NO) whenever the activity in the previous field occurs. Validation Rules: Alpha field up to 3 characters. Only input allowed is Y(ES) ; N(O) or blank. Multivalue field. (Optional input; default value is blank which is only allowed when field 2 is blank) Input prohibited when field ALL NONE ADVICES is set to 'NONE'. Input prohibited when field ALL NONE ADVICES is set to "ALL". Input mandatory if a value in field ACTIVITY.
