      • addFileSuffix

        public void addFileSuffix(TField param)
        Appends the specified FileSuffix to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Identifies for ONLINE updates the type of file (identified in FILE.NAME) for which record updates are to be extracted. There are three types of T24 file from which records can be extracted, unauthorised, authorised and history. NAU indicates that extracts are required from the Unauthorised file. HIS indicates that extracts are required from the History file LVE indicates that extracts are required from the Live file If no input is entered then the default will be the Live file. Input is not allowed for the PERIODIC record. For the purpose of OFAC extracts the unauthorised file would normally be defined so that checking can be made before authorisation takes place. Validation Rules: Multi valued field of 3 characters. Input is only allowed for the ONLINE record. Must be one or more of the following, if no input is made then LVE will be defaulted: NAU - for unauthorised records LVE - for authorised records HIS - for history records

        Parameters:
        param - the FileSuffix to append
        Returns:
        nothing

      • getExtractRtn

        public TField getExtractRtn()
        Returns the ExtractRtn

        Field description: EXTRACT.RTN A local routine to process and write transactional records to a centralized file for all companies can be attached here. This is a user exit field and if a routine is attached in this field, this routine will be called instead of EB.EXTRACT.ONLINE to extract the records. A local routine can be specified for each FILE.NAME. This user exit is available only for ONLINE record. Even if journaling is switched off, the local routine is called. The routine must have two arguments, txn id and the txn record (R.JOURNAL) If local routine is to be attached it should be specified for every FILE.NAME in the record. Validation Rules: The routine name must be prefixed with an @ symbol The routine must have a record in PGM.FILE with PGM.TYPE set to 'S' and APPL.FOR.SUBR set to EB.EXTRACT.PARAMETER

        Parameters:
        Returns:
        TField - The ExtractRtn

Input is not allowed for the PERIODIC record. For the purpose of OFAC extracts the unauthorised file would normally be defined so that checking can be made before authorisation takes place. Validation Rules: Multi valued field of 3 characters. Input is only allowed for the ONLINE record. Must be one or more of the following, if no input is made then LVE will be defaulted: NAU - for unauthorised records LVE - for authorised records HIS - for history records
