      • getDDEndDate

        public TField getDDEndDate()
        Returns the DDEndDate

        Field description: The drawdown end date. This indicates the date after which drawdowns will be prohibited; an override will be required for drawdowns starting later than this date. For commitment ion tranches, for value R or N value in LCU.TYPE, the undrawn balance of a tranche is reversed out on the expiry of D.D.END.DATE and hence, no withdrawal is possible after the expiry of the final D.D.END.DATE, even if the commitment is not fully drawn. Validation Rules: Standard date format. (Optional input. Default = FIN.MAT.DUE.) Must be greater than the value in VALUE.DATE. Must be equal to or less than the value in FIN.MAT.DATE. Must be greater than the value in D.D.START.DATE. Must be a valid date. Valid for Commitment contracts only. An override will be required if the date is not defined on the HOLIDAY table. An override will be required if the date as defined on the HOLIDAY table is a non working day. Rules applicable for commitment in tranches are as follows; If process date is between D.D.START.DATE & D.D.END.DATE, only D.D.END.DATE and its corresponding TRANCHE.AMT are permitted for input if drawings have already been made under the tranche If D.D.END.DATE is prior to process date, no input in any of the fields for the corresponding set is allowed Between D.D.START.DATE & D.D.END.DATE if the corresponding TRANCHE.AMT is increased, accruals will be re-worked based on new value in COMMT.AVAIL.AMT effective the start date. Decrease in TRANCHE.AMT is permitted to the extent that COMMT.AVAIL.AMT does not become negative D.D.START.DATE & D.D.END.DATE should be sequential and no overlapping periods are allowed. That is, the subsequent set of D.D.START.DATE should be later than the previous D.D.END.DATE.

        Parameters:
        Returns:

        Field description: The drawdown start date. This indicates the date from which drawdowns will be allowed; default will be VALUE DATE. For LCU.CALC.BASE of TO and TU, no drawings are permitted before D.D.START.DATE of the first tranche, or if drawings were to result in COMMIT.AVAIL.AMT becoming negative. If commitment were defined in tranches this field may be multi valued and amount allocated to each period may be defined in TRANCHE.AMT. Validation Rules: Standard date format. (Optional input. Decrease in TRANCHE.AMT is permitted to the extent that COMMT.AVAIL.AMT does not become negative D.D.START.DATE & D.D.END.DATE should be sequential and no overlapping periods are allowed.       • getTrancheAmt

        public TField getTrancheAmt()
        Returns the TrancheAmt

        Field description: Represents the amount of commitment allotted for the relevant period covered by D.D.START.DATE & D.D.END.DATE. Validation Rules: Associated multivalue with D.D.START.DATE & D.D.END.DATE. Input mandatory, if D.D.START.DATE & D.D.END.DATE are multivalued. Value in this field cannot be changed if process date is greater than D.D.END.DATE for relevant set. Value in this field should be equal to value in AMOUNT field. If D.D.START.DATE & D.D.END.DATE are multivalued, aggregate of TRANCHE.AMOUNT will have to be equal to value in AMOUNT field . If AMOUNT field is also multivalued to define banded fee, aggregate of TRANCHE.AMOUNT should equal the highest value in AMOUNT.

        Parameters:
        Returns:
