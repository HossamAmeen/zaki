        public TField getContractType()
        Returns the ContractType

        Field description: Identifier for the different types of Miscellaneous Deals that can be used by T24. These main types can be sub-divided into specific types as required by the bank to meet their business requirements. This field contains a list of valid types of miscellaneous deal. A Miscellaneous Deal can be one of four types: CA - Contingent Asset CL - Contingent Liability MA - Memorandum Asset ML - Memorandum Liability Validation Rules: 2 alphanumeric characters Mandatory Input Must be one of the following, CA, CL, MA or ML.

        public TField getEndCategory()
        Returns the EndCategory

        Field description: The end of the range of category codes covered by the CONTRACT.TYPE input in the corresponding CONTRACT.TYPE field. Each CONTRACT.TYPE corresponds to a range of categories to be defined locally. This field together with the START.CATEGORY field define this range. Validation Rules: 1 - 5 numeric characters.       • getStartCategory

        public TField getStartCategory()
        Returns the StartCategory

        Field description: For each of the four main CONTRACT.TYPE there needs to be a start and an end category to define the range of CATEGORY codes which can be used. The range needs to be large enough to cater for the different sub-types that will be created. The START.CATEGORY and END.CATEGORY define the range. Validation Rules: 5 numeric characters. Mandatory Input Once the record has been authorised this field cannot be changed. Must be present on the CATEGORY table. Category code ranges cannot overlap each other.
