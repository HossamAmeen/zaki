      • getSystemId

        public TField getSystemId()
        Returns the SystemId

        Field description: This field will have the EB.SYSTEM.ID of the transaction. This is an optional field and need to be defined only if different mapping of ID fields are to be done depending on the main application. EX: If the ID of Tax reporting contains the loan reference then the TXN.REFERENCE can be directly mapped for EB.SYSTEM.ID 'LD' for STMT.ENTRY raised from LD.LOANS.AND.DEPOSITS In case of STMT.ENTRY raised from PD.PAYMENT.DUE only part of the TXN.REFERENCE need to be mapped to get the loan reference. Hence this can be achieved by the combination of SYSTEM.ID as “PD“and ID.STMT.FLD with an extract function. Max 6 Alphanumeric character.

        Parameters:
        Returns:
        TField - The SystemId
