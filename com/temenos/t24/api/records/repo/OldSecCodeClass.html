      • getOldDepo

        public TField getOldDepo()
        Returns the OldDepo

        Field description: The depository where the security defined in the SEC.CODE field is held is defined in this field. The securities that currently form a part of REPO or Reverse REPO contract are defined in the field OLD.SEC.CODE, OLD.NOMINAL, OLD.DEPO and OLD.CU.ACCT.NO. On initial input these fields will be blank. Once the REPO contract is authorised, any securities defined in the new security fields (NEW.SEC.CODE, NEW.NOMINALL, NEW.DEPO and NEW.CU.ACCT.NO) are moved into the old fields. The addition, removal, increase of nominal, decrease of nominal and subsitution of securities is effected by input to the new fields. These fields should be seen as an after image for the securities that will form the REPO contract. Fields OLD.SEC.CODE to OLD.CU.ACCT.NO form a related multi value set. Validation Rules: This is a no input, no copy field. Must be the key to a valid entry on the CUSTOMER.SECURITY file. Must be defined as a DEPOSITORY in the CUSTOMER.SECURITY file.

        Returns:
