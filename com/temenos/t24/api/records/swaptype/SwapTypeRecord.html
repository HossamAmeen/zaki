      • getBalanceSheet

        public TField getBalanceSheet()
        Returns the BalanceSheet

        Field description: This field details whether the SWAP contracts using this swap type are to be treated as an ON balance sheet instrument or an OFF balance sheet instrument. This value is defaulted into the SWAP contract. When this field contains the value OFF the contracts will undergo the currency revaluation processing. Each night each leg of the contract id revalued against the local currency if 1) The leg is not notional (ie the NOTIONAL field on the SWAP.BALANCES file is set to NO). 2) The currency of the leg is not the local currency. And any difference between this value and the value as of last night is posted as unreleasied profit and loss to the category specified on the SWAP.PARAMETER record. When this field contains the value of ON the contracts will not be revalued. Validation Rules: Input must either be ON or OFF. Mandatory Input.

        Parameters:
        Returns:
        TField - The BalanceSheet

      • getSodMat

        public TField getSodMat()
        Returns the SodMat

        Field description: This field specifies whether the SWAP contracts using this swap type are to be processed at the start of day or at the end of day. Input Yes to make the swap mature or take value during the start of day procedure ( SW.START.OF.DAY ) on the morning of the maturity or value date, instead of being processed in the end of day procedure (SW.END.OF.DAY ) after the system has been closed down on the maturity or value date. Limit exposures for the swap are also updated during the start of day procedure ( SYSTEM.LIMIT.SOD ) on the morning of the maturity or value date, instead of being updated in the end of day procedure (SYSTEM.LIMIT.EOD ) after the system has been closed down on the maturity or value date. All accounting entries, payments, payment messages and limit updates for deals maturing or taking value are raised during the start of day procedure. There is a SOD.MAT field on SWAP . If it is left blank, the value of SWAP.TYPE SOD.MAT is used. If SWAP SOD.MAT is set, it overrides SWAP.TYPE SOD.MAT . Validation Rules: Y or N (default = N)

        Parameters:
        Returns:
        TField - The SodMat

      • setSodMat

        public void setSodMat(java.lang.CharSequence value)
        Set the specified value for SodMat
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field specifies whether the SWAP contracts using this swap type are to be processed at the start of day or at the end of day. Input Yes to make the swap mature or take value during the start of day procedure ( SW.START.OF.DAY ) on the morning of the maturity or value date, instead of being processed in the end of day procedure (SW.END.OF.DAY ) after the system has been closed down on the maturity or value date. Limit exposures for the swap are also updated during the start of day procedure ( SYSTEM.LIMIT.SOD ) on the morning of the maturity or value date, instead of being updated in the end of day procedure (SYSTEM.LIMIT.EOD ) after the system has been closed down on the maturity or value date. All accounting entries, payments, payment messages and limit updates for deals maturing or taking value are raised during the start of day procedure. There is a SOD.MAT field on SWAP . If it is left blank, the value of SWAP.TYPE SOD.MAT is used. If SWAP SOD.MAT is set, it overrides SWAP.TYPE SOD.MAT . Validation Rules: Y or N (default = N)
