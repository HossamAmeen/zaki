      • getTransaction

        public TField getTransaction()
        Returns the Transaction

        Field description: This field links AA to the other applications. When an external (non-AA) application raises a debit or credit movement to an arrangement account this needs to be translated to an activity on the arrangement. The transaction code of the underlying entry will be checked here and the associated TXN.ACTIVITY will be used. If the transaction code is not specified the DEFAULT.CR.ACTIVITY or DEFAULT.DB.ACTIVITY will be used based on the sign of the movement.

        This field is part of a multi value set with TXN.ACTIVITY.

        Input must be a valid record from TRANSACTION.

        Parameters:
        Returns:
        TField - The Transaction
