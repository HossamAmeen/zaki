      • getContractType

        public TField getContractType()
        Returns the ContractType

        Field description: Identifies the type of the loan throughout the contract. This field records the contract type of the loan. If this type is changed, i.e. from LINEAR to NONREDEM, then this will be recorded here. The date from which the contract type applies to the loan is specified in the associated multivalue of the TYPE.EFF.DATE field. Information is stored in these fields in reverse chronological order, i.e. the current contract type will always be found in the first multivalue. It should be noted that this field will only contain data relating to the contract after the date held in the RECORD.START.DATE field. Any data prior to this may be found on the MG.BALANCES.HIST file. Validation Rules: 1 - 12 alphanumeric characters Multivalue field associated with TYPE.EFF.DATE

        Parameters:
        Returns:

      • getTypeEffDate

        public TField getTypeEffDate()
        Returns the TypeEffDate

        Field description: Stores the dates on which the loan types in the CONTRACT.TYPE field became effective. The contract type of a loan may vary throughout the life of the contract. The types that have applied to the current loan are held in the associated field CONTRACT.TYPE. This field identifies the dates on which these types became effective. Information is stored in these fields in reverse chronological order, i.e. the most recent effective date will always be found in the first multivalue. It should be noted that this field will only contain data relating to the contract after the date held in the RECORD.START.DATE field. Any data prior to this may be found on the MG.BALANCES.HIST file. Validation Rules: Standard date format Multivalued field associated with CONTRACT.TYPE.
