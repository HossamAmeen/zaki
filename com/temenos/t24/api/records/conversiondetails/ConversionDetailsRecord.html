      • getNewCoCodePos

        public TField getNewCoCodePos()
        Returns the NewCoCodePos

        Field description: This field is used to define the expected position of the company code on the records after they have been converted. By checking the value in this field for each incoming record it is determined whether the conversion should be run on this record. If the record has a company code in the field specified, the record has already been converted and the record is not processed. The number of records which were not converted for this reason is detailed in the RUN.INFO field as ALREADY CONVERTED. N.B. This is a numeric only field, as using field names may cause problems when multiple upgrades are run at the same time and the same field is converted more than once. Validation Rules: Up to three numeric characters. Mandatory input for file types H, U and W.

        Parameters:
        Returns:
        TField - The NewCoCodePos

      • getOldCoCodePos

        public TField getOldCoCodePos()
        Returns the OldCoCodePos

        Field description: This field is used to define the expected position of the company code on the records before they are converted. By checking the value in this field for each incoming record it is determined whether the conversion should be run on this record. This is a numeric only field, as using field names may cause problems when multiple upgrades are run at the same time. Validation Rules: Up to three numeric characters. Mandatory input for file types H, U and W.

      • getReRunFlag

        public TField getReRunFlag()
        Returns the ReRunFlag

        Field description: This field is used to define whether the routine may be rerun or not. Each conversion should be able to be rerun and hence the default value for this field is YES when there is a value present in the NEW.CO.CODE.POS field, as records in the new format will be checked against the expected position to see if the record has already been converted. When there is no value present in the NEW.CO.CODE.POS field (ie the file is not of type H, U or W) then the default value of this field is NO. However, it is possible to set this value to YES to indicate that the routine is able to be re run (for instance a FILE.ROUTINE is specified which sets the value of a given field to null). Validation Rules: Input must be either YES or NO. Defaults to YES when data is present in the NEW.CO.CODE.POS field, otherwise defaults to NO.

        Parameters:
        Returns:
        TField - The ReRunFlag

      • getRecordRoutine

        public TField getRecordRoutine()
        Returns the RecordRoutine

        Field description: This field is used to define a routine which will be called after EACH record has gone through the processing which adds and/or removes fields but BEFORE the record is written. For example, such a routine may be used to null a given field or other tasks which would best be performed at record level. If file pointers or parameter records are required as part of this routine they should be opened / read into a COMMON area (the loading of the COMMON area should be done in a routine specified in the PRE.ROUTINE field) to avoid repetive and slow processing. 12 RECORD.ROUTINE.... @RECORD.ROUTINE.NAME This routine is called with three arguments, being the record id, dynamic record array and the file name which is currently being processed, and should be specified as SUBROUTINE RECORD.ROUTINE.NAME(YID, R.RECORD, FN.FILE) Validation Rules: Must be a valid catalogued subroutine, preceded by the '@' character.
