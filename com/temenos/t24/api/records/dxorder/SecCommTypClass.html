      • getSecCommCde

        public TField getSecCommCde()
        Returns the SecCommCde

        Field description: If SCDX.CHARGE.METHOD is enabled in DX.PARAMETER, This will be defaulted with Commission code / tax code with which CHARGE.TAX.AMT is derived. In case of commission , FT.COMMISSION.TYPE code is prefixed with 'COM~'. In case of Tax , TAX code is prefixed with 'TAX~'. If SCDX.CHARGE.METHOD is not enabled in DX.PARAMETER, Commission code-'OVERRIDE' for fully manual, 'AUTO' (defaulted by DX.GET.COMMISSION) for full automatic processing or'*<FT.COMMISSION/CHARGE.TYPE.ID>'for semi-automatic. Validation Rules: No-input until the order has been authorised. Up to 20 characters &#150; input must exist on FT.COMMISSION.TYPE or FT.CHARGE.TYPE No Input unless there is input to field SEC.COMM.TYPE and SEC.COMM.CDE = &#145;OVERRIDE&#146;

        Parameters:
        Returns:

      • getSecCommTyp

        public TField getSecCommTyp()
        Returns the SecCommTyp

        Field description: The Commission Type defaulted by DX.GET.COMMISSION. Validation Rules: If SCDX.CHARGE.METHOD is enabled in DX.PARAMETER, then values will be defaulted from charge or Tax type asdefined in SCDX.CHARGE.PARAMETER (from CHARGE.TAX.TYPE field) based on the customer type No-input until the order has been authorised. Up to 10 alpha characters &#150; If not enabled, then input must be one of the following : COMMISSION/EXECUTION/CLEARING/REGULATORY/AUTOMATIC/MISC Multi-value field (No Duplicates allowed) If field is left Null, it will default to AUTOMATIC. If SEC.COMM.TYP is set to AUTOMATIC then Multi-values not allowed This field should not allow the input of AUTOMATIC if it was set to ANY of the other selections

        Parameters:
        Returns:
