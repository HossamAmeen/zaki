      • getOutputFormat

        public TField getOutputFormat()
        Returns the OutputFormat

        Field description: Defines the format of the report. The output can either be in report format, intended for printing, or can be in file format which can be used as an input file for spreadsheet or WP analysis. The report format is exactly as defined on the ENQUIRY, with headers, correct column positions, page breaks, etc. The file format is simply the data produced by the ENQUIRY output as rows and columns. Columns are separated by a TAB, ascii character 9, and rows are separated by a LF, ascii character 10. All data is enclosed in quotes (double quotations) and any multi/subvalue fields are delimited by LF and SPACE respectively. In addition the field names can be inserted as the first row as a header (used by spreadsheets and WPs as column headers) by specifiying the column headers in the HEADER field of the ENQUIRY record. Note : The data output in file format is in the order in which the ENQUIRY fields are processed and is not based on the column settings. Note : For file format the TOP.MARGIN and BOTTOM.MARGIN in the related DE.FORM.TYPE record must be set to zero. Validation Rules: Must be one of the following: REPORT Output will be in report format (default) FILE Output will be in file format with TAB separated columns null Same as REPORT (default)

        Returns:
        TField - The OutputFormat

        public TField getReportControl()
        Returns the ReportControl

        Field description: The ID of the REPORT.CONTROL record which controls the form and ultimate output destination of the report. Where FILE output is required the REPORT.CONTROL record should specify a "form" suitable for file output. The appropriate form, specified in the field FORM.NAME on the REPORT.CONTROL record and defined in DE.FORM.TYPE, should have TOP.MARGIN and BOTTOM margin set to zero (not null) to prevent spacing lines and form feeds in the output file. Validation Rules: 30 alphanumeric characters. (Mandatory)

        public void setOutputFormat(TField value)
        Set the specified value for OutputFormat
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the format of the report. The output can either be in report format, intended for printing, or can be in file format which can be used as an input file for spreadsheet or WP analysis. The report format is exactly as defined on the ENQUIRY, with headers, correct column positions, page breaks, etc. The file format is simply the data produced by the ENQUIRY output as rows and columns. Columns are separated by a TAB, ascii character 9, and rows are separated by a LF, ascii character 10. All data is enclosed in quotes (double quotations) and any multi/subvalue fields are delimited by LF and SPACE respectively. In addition the field names can be inserted as the first row as a header (used by spreadsheets and WPs as column headers) by specifiying the column headers in the HEADER field of the ENQUIRY record. Note : The data output in file format is in the order in which the ENQUIRY fields are processed and is not based on the column settings. Note : For file format the TOP.MARGIN and BOTTOM.MARGIN in the related DE.FORM.TYPE record must be set to zero. Validation Rules: Must be one of the following: REPORT Output will be in report format (default) FILE Output will be in file format with TAB separated columns null Same as REPORT (default)

        value - The value to set
