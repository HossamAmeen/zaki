      • addPropertyType

        public void addPropertyType(TField param)
        Appends the specified PropertyType to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The options are NULL, PRODUCT.ONLY, SUSPEND, SUSPEND.OVERDUES, ACCRUAL.BY.BILLS, CREDIT,RESIDUAL.ACCRUAL, FORWARD DATED, VARIATION, COMMISSION and REBATE UNAMORTISED. PRODUCT.ONLY - If this is selected, the details for this Property are defined only at product level andcannot be viewed or edited at arrangement level. For example for properties which use ACCOUNTING class, it ishighly unlikely that the user would need to define or modify accounting rules for each arrangement. Hence definingthem at the product would suffice. SUSPEND - If this is selected, it would mean that the property would be allowed to suspend. OnlyINTEREST and CHARGE properties can be suspend. SUSPEND.OVERDUE - If this is defined, it would mean that any overdues in the property will also besuspended when property is suspend. To define SUSPEND.OVERDUE, the property should already be set to SUSPEND.

        ACCRUAL.BY.BILLS – If selected,it performs accrual update in bills. CREDIT - This is to state that the property Needs to raise Credit entry or Debit entry. Default type isDebit.If CREDIT is specified for charge then it will be considered as bonus. RESIDUAL ACCRUAL - This is to make difference between the TOT.ACCR.AMT and the Amount made due to moveunder RES balance type. FORWARD DATED - This is introduced to display future dated conditions. VARIATION - This is stated so that proofing would check for the existence of variations(GOLD/PLATINUM/INTERNET).Rules to check whether customers are satisfied these variations will bedefined in eligibility condition. REBATE UNAMORTISED - Normal charges without property type as “REBATE.UNAMORT” can be set to amortize.When an arrangement with such a charge definition is paid off, it should recognise the unamortised amountimmediately, without depending on the configuration in payment schedule COMMISSION - This is introduced to identify charges those are used as commissions. PRODUCT.QUALIFIER - Allowed only for ACTIVITY.RESTRICTION and when set type should be PRODUCT.ONLY.Rule set of fields in activity restriction are mandatory and will be validated during proofing stage

        Parameters:
        param - the PropertyType to append
        Returns:
        nothing
