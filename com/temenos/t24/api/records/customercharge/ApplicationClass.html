      • getActualGroup

        public TField getActualGroup()
        Returns the ActualGroup

        Field description: Defines the Customer's actual charge group for the application. This field has the same value as the default group when the customer charge record is first created. If a customer level record in XX.GROUP.CONDITION files is authorized (For FT, FD and LC), it's ID (C-CUSTOMER.NUMBER) automatically gets defaulted here. In that case, its value could not be changed. When the customer level record in XX.GROUP.CONDITION files (For FT, FD and LC) is reversed then value of DEFAULT.GROUP field is automatically defaulted here. For the FT, FD and LC applications, the processing checks for the customer level record in XX.GROUP.CONDITION first.

        1. Cannot change if the value is C-Customer No. 2. Can be changed only to a value in XX.GROUP.CONDITION.

        Parameters:
        Returns:
        TField - The ActualGroup
