      • getChargeCode

        public TField getChargeCode()
        Returns the ChargeCode

        Field description: This field indicates the charge code applicable to the "F", "N", "P", "I" or "A" schedules. This forms an association with the next field CHG.BASE.AMT to provide a flexible charge calculation mechanism. Contains the id to the LMM.CHARGE.CONDITIONS record. The charge type may be RECEIVEable by the bank (i.e paid by the customer), or PAYable (paid by the bank). A charge code must be defined with a type F schedule together with an AMOUNT. A charge code may also be linked to a P or I schedule, providing the LMM.CHARGE.CONDITIONS record is linked to a FT.CHARGE/ COMMISSION.TYPE. When A, P or I schedule with a linked charge is processed, the charge amount will be calculated according to the FT charge definition, based on the amount specified in CHG.BASE.AMT field. Charges are debited or credited as follows: Type F schedule RECEIVE debited from CHARG.LIQ.ACCT, PAY credited to FEE.PAY.ACCOUNT. Type P schedule RECEIVE debited from PRIN.LIQ.ACCT, PAY credited to PRIN.LIQ.ACCT. Type I schedule RECEIVE debited from INT.LIQ.ACCT, PAY credited to INT.LIQ.ACCT. Receivable charges may be amortised if requried. This is denoted by the AMORTISATION field on the LMM.CHARGE.CONDITIONS record. Example: CHARGE.CODE 1 is a PAYABLE charge linked to a FT.COMMISSION.TYPE of 0.25% CHARGE.CODE 2 is a PAYABLE charge linked to a FT.COMMISSION.TYPE of 0.5% CHARGE.CODE 3 is a RECEIVABLE charge. The following deposit schedules are defined for 31/03/96 Type P 25,000 USD Charge Code 1 Type I Charge Code 2 (Interest Amount 1,000 USD) Type F 50 USD Charge Code 3 On 31/03/96 the following entries will be raised: Credit PRIN.LIQ.ACCT 25,000 USD Credit PRIN.LIQ.ACCT 25,000 * 0.25% = 62.50 USD (linked CHARGE.CODE 1) Credit INT.LIQ.ACCT 1,000 USD Credit INT.LIQ.ACCT 1,000 * 0.5% = 5 USD (linked CHARGE.CODE 2) Debit CHRG.LIQ.ACCOUNT 50 USD (Charge Code 3) This must be a valid charge code as defined on the LMM.CHARGE.CONDITIONS table. Validation Rules: 1-3 numeric characters. Must be a valid entry on LMM.CHARGE.CONDITIONS. For "N" type schedules the value entered here must not have a value of YES in the MTHLY.AMORTISATION field of the LMM.CHARGE.CONDITIONS record. For "A", "P" or "I" schedules the value entered must be defined in LMM.CHARGE.CONDITIONS with a linked CHARGE.CODE.KEY

      • getChgBaseAmt

        public TField getChgBaseAmt()
        Returns the ChgBaseAmt

        Field description: This field specifies the amount that the charge calculation is based on. This forms an association with the previous field CHARGE.CODE to provide a flexible charge calculation mechanism. Valid input is: IP - Initial Principal OP - Outstanding Principal PR - Principal Repayment IA - Interest Amount AP - Annuity Payment (Interest + Principal) Validation Rules: Mandatory field if corresponding CHARGE.CODE is defined. Otherwise, no input is allowed. Input must be either IP, OP, PR, IA or AP.
