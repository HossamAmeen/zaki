        public TField getCurrency1()
        Returns the Currency1

        Field description: Indicates the currency for the posting line which will always be same as the account currency. Example: If the posting line is for debiting or crediting an USD account then currency1 will be matched with USD. In case posting line is for debiting charges from separate client charge account then it is the currency of client charge account. Validation Rules: Mandatory field.3 alphanumeric characters.The value links to the field 'CurrencyCode' in PPT.CURRENCY and cannot be the same as field 'Currency2'.

      • getCurrency2

        public TField getCurrency2()
        Returns the Currency2

        Field description: Indicates the currency of the other leg of transaction, which could be different in case of FX payment. That can also influence the Dealer Desk to be used. Example: If the posting line is for principal debit then currency2 will be the currency of principal credit account. And in case of posting line for debiting charges then this shall be the corresponding Fee currency(which shall be same as P&L account currency). Validation Rules: Mandatory field.3 alphanumeric characters.The value links to the field 'CurrencyCode' in PPT.CURRENCY and cannot be the same as field 'Currency1'.
