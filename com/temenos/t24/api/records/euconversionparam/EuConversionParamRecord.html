      • addSelection

        public void addSelection(TField param)
        Appends the specified Selection to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines either the WITH clause of a uniVerse SELECT statement for the file being converted, or a subroutine to perform the selection. For example the STMT.ENTRY record contains the following SELECTION statement: WITH @ID UNLIKE 'F...' This means select all statement entries which are not forward. The actual statement executed would be to select the actual files per company, e.g. SELECT FBNK.STMT.ENTRY WITH @ID UNLIKE 'F...' It is possible that on sites with very large numbers of statement entries that the select list may be so large that available memory is exceeded or that it is simply more efficient to process several smaller lists as opposed to one very large list. If during the test conversion process these problems arise then one possible solution would be to split the processing as follows RECORD SELECTION ====== ========= STMT.ENTRY1 WITH BOOKING.DATE GE 19980101 STMT.ENTRY2 WITH BOOKING.DATE GE 19970101 AND LE 19971231 STMT.ENTRY3 WITH BOOKING.DATE LE 19961231 The actual dates chosen should have the effect of splitting the entries into roughly equal groups. A group size of 200,000 has been found to be effective. If a subroutine is required then it must be prefixed with the @ character. The select subroutine must have one argument which will contain a list of record keys separated by field marks. In addition to including the normal inserts I_COMMON and I_EQUATE the routine should also include I_EU.COMMON. The common variable C$EU.MORE.TO.PROCESS can be set to 1 to indicate that the routine will split the list of keys into groups, the parent routine will continue calling the select routine after processing each list until C$EU.MORE.TO.PROCESS is reset to ''. Validation Rules: Optional input. Multivalued field. If a select statement clause is entered which is greater than 35 characters then the multi value can be expanded. If a conversion routine is being called then it must be prefixed with @. If no input is made then the entire file being converted will be selected.

        Parameters:
        param - the Selection to append
        Returns:

        public void setSelection(TField param, int idx)
        Set the specified Selection to a specific position in collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines either the WITH clause of a uniVerse SELECT statement for the file being converted, or a subroutine to perform the selection. For example the STMT.ENTRY record contains the following SELECTION statement: WITH @ID UNLIKE 'F...' This means select all statement entries which are not forward. The actual statement executed would be to select the actual files per company, e.g. SELECT FBNK.STMT.ENTRY WITH @ID UNLIKE 'F...' It is possible that on sites with very large numbers of statement entries that the select list may be so large that available memory is exceeded or that it is simply more efficient to process several smaller lists as opposed to one very large list. If during the test conversion process these problems arise then one possible solution would be to split the processing as follows RECORD SELECTION ====== ========= STMT.ENTRY1 WITH BOOKING.DATE GE 19980101 STMT.ENTRY2 WITH BOOKING.DATE GE 19970101 AND LE 19971231 STMT.ENTRY3 WITH BOOKING.DATE LE 19961231 The actual dates chosen should have the effect of splitting the entries into roughly equal groups. A group size of 200,000 has been found to be effective. If a subroutine is required then it must be prefixed with the @ character. The select subroutine must have one argument which will contain a list of record keys separated by field marks. In addition to including the normal inserts I_COMMON and I_EQUATE the routine should also include I_EU.COMMON. The common variable C$EU.MORE.TO.PROCESS can be set to 1 to indicate that the routine will split the list of keys into groups, the parent routine will continue calling the select routine after processing each list until C$EU.MORE.TO.PROCESS is reset to ''. Validation Rules: Optional input. Multivalued field. If a select statement clause is entered which is greater than 35 characters then the multi value can be expanded. If a conversion routine is being called then it must be prefixed with @. If no input is made then the entire file being converted will be selected.

        Parameters:
        param - the Selection to append
        idx - the position in collection
        Returns:
