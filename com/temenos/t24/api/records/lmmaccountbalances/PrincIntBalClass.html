        public void addAcIntRate(TField param)
        Appends the specified AcIntRate to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The actual interest rate used in the calculation of interest in this set of accrual fields (fields 52 to 57). Whenever an interest accrual takes place, fields 52 to 57 inclusive are updated with the components used in the accrual calculation. These fields are PRINC INT BAL (the balance of principal on the Contract); AC INT RATE (the actual interest rate used in the calculation); AC INT SPREAD (the interest spread, if applicable); DATE INT ACC TO (the date interest is accrued to); and the resulting interest amount fields - INT AMT TODATE (rounded) and ACTUAL ACC AMT (not rounded). So as to keep a full history of the interest accrual during the current interest period, these fields are multivalue associated and each time one of the calculation components changes, a new set of values is added. This field, AC INT RATE, contains the interest rate that was used in the calculation of interest in the multi set of accrual fields of which this field is part. An example of how this field and the other accrual fields are used is shown in the details of PRINC INT BAL (Field 52). Validation Rules: Standard rate format.

      • getPrincIntBal

        public TField getPrincIntBal()
        Returns the PrincIntBal

        Field description: The outstanding principal balance as of the DATE INT ACC TO field in this multivalue set of accrual fields (fields 52 to 57). Whenever an interest accrual takes place, fields 52 to 57 inclusive are updated with the components used in the accrual calculation. These fields are PRINC INT BAL (the balance of principal on the Contract); AC INT RATE (the actual interest rate used in the calculation); AC INT SPREAD (the interest spread, if applicable); DATE INT ACC TO (the date interest is accrued to); and the resulting interest amount fields - INT AMT TODATE (rounded) and ACTUAL ACC AMT (not rounded). So as to keep a full history of the interest accrual during the current interest period, these fields are multivalue associated and each time one of the calculation components changes, a new set of values is added. This field, PRINC INT BAL, is calculated by summing the OUTS CURR PRINC and OUTS OD PRINC fields to arrive at a net outstanding principal balance. It is signed negative for asset contracts and positive for liability contracts. An example of how this field, and the other accrual fields are used is shown below: Example: Asset contract, valued 11th February 1987, Maturing 11th March 1987. Principal is one million pounds, interest rate is fixed at 10%, accruals are performed daily and the first day accrual method is employed. State of fields prior to first accrual: CURRENCY : GBP DATE FROM : 11th February 1987 EFFECTIVE DATE : 11th February 1987 TRANS PRIN AMT : - 1,000,000 OUTS FWD PRINC : 0 OUTS CURR PRINC : - 1,000,000 OUTS OD PRINC : 0 OUTS ACCRUED INT : " " START PERIOD INT : 11th February 1987 END PERIOD INT : 11th March 1987 COMMITTED INT : - 7,671.23 PRINC INT BAL : " " INT AMT TODATE : " " AC INT RATE : " " AC INT SPREAD : " " DATE INT ACC TO : " " ACTUAL ACC AMT : " " NB: " " = NULL OR BLANK State of fields after first accrual: CURRENCY : GBP DATE FROM : 11th February 1987 EFFECTIVE DATE : 11th February 1987 TRANS PRIN AMT : - 1,000,000 OUTS FWD PRINC : 0 OUTS CURR PRINC : - 1,000,000 OUTS OD PRINC : 0 OUTS ACCRUED INT : - 273.97 START PERIOD INT : 11th February 1987 END PERIOD INT : 11th March 1987 COMMITTED INT : - 7,671.23 PRINC INT BAL : - 1,000,000 INT AMT TODATE : - 273.97 AC INT RATE : 10% AC INT SPREAD : " " DATE INT ACC TO : 11th February 1987 ACTUAL ACC AMT : - 273.9726 Note: One days interest has been calculated, the fields in the calculation have been updated and the final field OUTS ACCRUED INT has been updated. State of fields after second accrual: CURRENCY : GBP DATE FROM : 11th February 1987 EFFECTIVE DATE : 11th February 1987 TRANS PRIN AMT : - 1,000,000 OUTS FWD PRINC : 0 OUTS CURR PRINC : - 1,000,000 OUTS OD PRINC : 0 OUTS ACCRUED INT : - 547.95 START PERIOD INT : 11th February 1987 END PERIOD INT : 11th March 1987 COMMITTED INT : - 7,671.23 PRINC INT BAL : -1,000,000 INT AMT TODATE : - 547.95 AC INT RATE : 10% AC INT SPREAD : " " DATE INT ACC TO : 12th February 1987 ACTUAL ACC AMT : - 547.94521 Note: Another days interest has been calculated and added to OUTS ACCRUED INT. Because the components in the interest calculation were no different to the first accrual, a new set of accrual details were not added but the amount and date fields were changed. If, prior to the third accrual, a prepayment of 250,000 principal was made, valued 13th February, the state of the fields after the third accrual would be as follows: State of fields after third accrual: CURRENCY : GBP DATE FROM : 11th February 1987 EFFECTIVE DATE : 11th February 1987 TRANS PRIN AMT : - 1,000,000 OUTS FWD PRINC : 0 OUTS CURR PRINC : - 1,000,000 OUTS OD PRINC : 250,000 OUTS ACCRUED INT : - 753.43 START PERIOD INT : 11th February 1987 END PERIOD INT : 11th March 1987 COMMITTED INT : - 5,890.42 PRINC INT BAL : -1,000,000- 750,000 INT AMT TODATE : - 547.95 - 205.48 AC INT RATE : 10% 10% AC INT SPREAD : " " " " DATE INT ACC TO : 12th February 1987 13th February 1987 ACTUAL ACC AMT : - 547.94521- 205.47945 1ST SET 2ND SET Note: This time, another set of accrual fields have been written because the principal balance has changed. Another result of the principal change is that the COMMITTED INT field has been re-calculated. An example for a deposit contract would be similar to the one above with the following exceptions: 1. Amounts would have the opposite sign. 2. The OUTS ACCRUED INT field would be replaced by the OUTS CUR ACC I PAY field. The field AC INT SPREAD is only used when the contract is tied to a floating interest rate. In this case the AC INT RATE is still the actual interest rate used in the calculation but this time it includes the interest spread as shown in AC INT SPREAD. Validation Rules: 1 to 14 type AMT (Standard amount format) characters plus a decimal point.
