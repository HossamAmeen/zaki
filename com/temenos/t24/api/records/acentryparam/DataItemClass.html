CONTROL.TOTAL The number of transactions in the file. FILE.REF The Tape/File number from the header. CHECKSUM The supplied CHECKSUM from the header. CALC.CHECKSUM The calculated CHECKSUM in the header. CR.TOTAL The supplied Credit Txn total in the header. DR.TOTAL The supplied Debit Txn total in the header.       • getDataValRtn

        public TField getDataValRtn()
        Returns the DataValRtn

        Field description: The name of the validation routine to be applied to the extracted data item. The routine may be a Standard system validation routine beginning IN2..., see Regional Development guide for futher details, or a user customised routine. A user routine must have no arguments, the value is supplied in the variable COMI, and the value may be modified as required, errors should be returned in the variable ETEXT. Some of the main IN2 routines likely to be used are: IN2 Validates numbers, removes leading zeros. IN2ANT Validates internal and customer accounts. Will convert MNEMONIC and ALTERNATE.ACCOUNT ids to account number where valid. IN2ACC As IN2ANT except that only customer accounts are allowed. IN2CUS Validates customer numbers, converts mnemonics to valid numbers where supplied. IN2CCY Validates currency codes. Will convert numeric currency code to a standard code. IN2A Ensures input is alpha-numeric. IN2S Ensures input is SWIFT standard. User Routines which may be used which are generally available include: FT.LOCAL.TXN This will compare an external transaction code supplied against the file FT.LOCAL.TXN.CODES. This may be translated into a Globus transaction code, and may also be used to ignore certain types of transaction. FT.CHECK.SORT.CODE This routine validates the SORT.CODE supplied against the FT.LOCAL.CLEARING record, to see if the code is defined as one of our clearing codes. If not not an error is returned. Fields DATA.ITEM to ENQ.FILE.LOC form a related sub value set. Fields RECORD.TYPE to ENQ.LOC.FILE form a related multi value set. Validation Rules: Any routines specified in this field must be catalogued. Optional.

        Parameters:
        Returns:
        TField - The DataValRtn

      • getEnqFileLoc

        public TField getEnqFileLoc()
        Returns the EnqFileLoc

        Field description: This field is used to specify the location in the extract entry file, AC.INWARD.ENTRY, for the associated DATA.ITEM to be mapped to. When the interface file is processed a reporting entry is built in the file AC.INWARD.ENTRY. The key to this file is: FILE.NAME "." FILE.SEQ.NO "-" record in file (DATA items) FILE.NAME "." FILE.SEQ.NO "-HEADn" (header items) FILE.NAME "." FILE.SEQ.NO "-TRAIL" (trailer) FILE.NAME "." FILE.SEQ.NO "-ERROR" (header/trailer errors) Data will be mapped to the same field as the corresponding accounting entry to be raised, and additionally to the location defined in this field. Data is always mapped to the reporting file, even if the entry is not raised, the field ENTRY.POSTED denoted whether or not the entry was raised. To define specific multi values and subvalues a "-" character should be specified and the multi/sub value set, e.g. FIELD.NAME-1 FIELD.NAME-1-1 N.B. If the field name specified is LOCAL.REF then the multi value indicator must be present, and a subvalue indicator may be set if required. Fields DATA.ITEM to ENQ.FILE.LOC form a related sub value set. Fields RECORD.TYPE to ENQ.LOC.FILE form a related multi value set. Validation Rules: Must be a valid data field as defined for the AC.INWARD.ENTRY file.

        Parameters:
        Returns:
        TField - The EnqFileLoc

      • setEnqFileLoc

        public void setEnqFileLoc(java.lang.CharSequence value)
        Set the specified value for EnqFileLoc
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field is used to specify the location in the extract entry file, AC.INWARD.ENTRY, for the associated DATA.ITEM to be mapped to. When the interface file is processed a reporting entry is built in the file AC.INWARD.ENTRY. The key to this file is: FILE.NAME "." FILE.SEQ.NO "-" record in file (DATA items) FILE.NAME "." FILE.SEQ.NO "-HEADn" (header items) FILE.NAME "." FILE.SEQ.NO "-TRAIL" (trailer) FILE.NAME "." FILE.SEQ.NO "-ERROR" (header/trailer errors) Data will be mapped to the same field as the corresponding accounting entry to be raised, and additionally to the location defined in this field. Data is always mapped to the reporting file, even if the entry is not raised, the field ENTRY.POSTED denoted whether or not the entry was raised. To define specific multi values and subvalues a "-" character should be specified and the multi/sub value set, e.g. FIELD.NAME-1 FIELD.NAME-1-1 N.B. If the field name specified is LOCAL.REF then the multi value indicator must be present, and a subvalue indicator may be set if required. Fields DATA.ITEM to ENQ.FILE.LOC form a related sub value set. Fields RECORD.TYPE to ENQ.LOC.FILE form a related multi value set. Validation Rules: Must be a valid data field as defined for the AC.INWARD.ENTRY file.

        Parameters:
        value - The value to set
        Returns:
        nothing
