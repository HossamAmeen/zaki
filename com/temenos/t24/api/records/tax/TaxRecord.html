      • getCalcRoutine

        public TField getCalcRoutine()
        Returns the CalcRoutine

        Field description: Specify either a jBC subroutine(exist on PGM.FILE as a type 'S'), it should be prefixed with '@' or an EB.API record of type METHOD which implements an interface defined in the EB.API record HOOK.TAX.CALC.RTN. The calculated tax amount is returned by this routine. Validation Rules: Input to this field is Optional. The jBC routine should accept the following parameters : ( PASS.CUSTOMER, PASS.DEAL.AMOUNT, PASS.DEAL.CCY, PASS.CCY.MKT, PASS.CROSS.RATE, PASS.CROSS.CCY, PASS.DWN.CCY, PASS.DATA, PASS.CUST.CDN,R.TAX,TAX.AMOUNT ) Where PASS.CUSTOMER: The customer number for whom the tax is calculated. PASS.DEAL.AMOUNT: The deal amount used for calculations. PASS.DEAL.CCY: The currency of the deal amount. PASS.CCY.MKT: The market to use for currency conversions. PASS.CROSS.RATE: The cross rate that need to be used. PASS.CROSS.CCY: The other currency used in transaction. PASS.DWN.CCY: The currency to return the tax amount in. PASS.DATA: It contains all charges and tax details. PASS.CUST.CDN: It contains the customer condition. R.TAX: It is the array having the Tax record. TAX.AMOUNT: It contains the calculated tax amount. Note: PASS.CUSTOMER, PASS.DEAL.AMOUNT, PASS.DEAL.CCY, PASS.CCY.MKT, PASS.CROSS.RATE, PASS.CROSS.CCY, PASS.DWN.CCY, PASS.DATA, PASS.CUST.CDN and R.TAX are Input parameters. TAX.AMOUNT is the Output parameter.

        Returns:

        public void setUniqueSuffix(java.lang.CharSequence value)
        Set the specified value for UniqueSuffix
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies if the tax account number to be credited is the same tax account for all postings, or a different account for each transaction. The tax account number is constructed in the format CCY ccccc nnnn where: CCY is the currency of the transaction, or the local currency (depending on the TAX.IN.LOCAL field) Ccccc is the 5 digit CATEGORY code. Nnnn is a 4 digit suffix, either fixed as 0001 if this field is not set to Y, or a random number formatted with leading zeroes if this field is set to Y. In high volume environments where tax is taken on many transactions it may be desirable to set this field to Y for performance reasons.         Set the specified value for UniqueSuffix
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies if the tax account number to be credited is the same tax account for all postings, or a different account for each transaction. The tax account number is constructed in the format CCY ccccc nnnn where: CCY is the currency of the transaction, or the local currency (depending on the TAX.IN.LOCAL field) Ccccc is the 5 digit CATEGORY code. Nnnn is a 4 digit suffix, either fixed as 0001 if this field is not set to Y, or a random number formatted with leading zeroes if this field is set to Y. In high volume environments where tax is taken on many transactions it may be desirable to set this field to Y for performance reasons.
