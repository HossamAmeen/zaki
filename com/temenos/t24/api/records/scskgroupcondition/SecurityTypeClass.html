      • getSecurityType

        public TField getSecurityType()
        Returns the SecurityType

        Field description: Identifies the level for calculating the basis amount. This field will be used to decide on the Commission type to calculate the safekeeping charges based on the holdings of the portfolio. Where a variety of levels of parameter are set, the system will select a charge parameter in the following order of priority: Priority Classification ID format Example 1 Security Number '.' Depository nnnnnn-nnn.depository 123456-000.100001 2 Security Number nnnnnn-nnn 123456-000 3 SUB.ASSET.TYPE '.' Depository S-nnnn.depository S-300.100001 4 SUB.ASSET.TYPE S-nnnn S-300 5 ASSET.TYPE '.' Depository A-nnn.depository A-10.100001 6 ASSET.TYPE A-nnn A-10 7 Default '.' Depository ALL.depository ALL.100001 8 Default ALL ALL Validation Rules: 1. Security Number : 6N-3N or 2. Sub-Asset type prefixed with "S-". or 3. Asset Type prefixed with "A-". or 4. "ALL" - all others. Any of these four levels can be suffixed with a depository id after the separator '.' Input is Mandatory Must exist as an SCSF.CHARGE.PARAMETER record.

        Parameters:
        Returns:
