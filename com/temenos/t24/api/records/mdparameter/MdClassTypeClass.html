      • getMdClassType

        public TField getMdClassType()
        Returns the MdClassType

        Field description: MD.CLASS.TYPE These message classes are raised by the MD Application for production of messages. This along with EB.CLASS.TYPE are used to formulate the class mapping. The main driving concept of ACTIVITY and CLASS lends itself to facilitate the user to trigger the messages related to the operation. The system will try to default a sensible CLASS tiding to the corresponding activity, populate the information in MD Transaction. The user may however select the apt CLASS to choose production of appropriate messages. Valid inputs are: ACKNOWLEDGEMENT ADDITIONAL.MSG ADVICE.THRU BANKADVICE BASIC.MESSAGE CANCELLATION CREDIT.CUST CSN.CREDIT CSN.DEBIT DEBIT.CUST NOTIFICATION PARTICIPATION PROVISION.CREDIT PROVISION.DEBIT USER.DEFINE1 USER.DEFINE2 ... USER.DEFINE9 Validation Rules: Only the aforesaid list of values are permitted.
