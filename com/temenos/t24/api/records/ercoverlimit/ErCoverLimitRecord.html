      • getUtilisedLimit

        public TField getUtilisedLimit()
        Returns the UtilisedLimit

        Field description: System updates the amount used by the Sender of MT103 as identified with the ID. This is the difference between Original.Amount and Available.Amount. When "EC" record is created in AC.EXPECTED.RECS while processing an Inward MT103 message, then the Utlised.Amount is reduced to that extent of Debit amount. When "RC" record is received for the corresponding payment message then Utilised.Amount is restored to that extent. When utilised.amount equal to Original.Amount, then FT is created with Hold status and resepctive AC.EXPECTED.RECS is created with status as UAL-Unavailable Limit. Validation Rules: \No input field Updated by System during MT103 Inward processing.

        Returns:
