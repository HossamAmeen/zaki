      • getFieldName

        public TField getFieldName()
        Returns the FieldName

        Field description: The name of the field to be amended should be entered in FIELD.NAME. The field can be either entered as a field name defined on the STANDARD.SELECTION record or as a field number, which will be translated to the corresponding field name. If the field is multi- or sub-valued, a particular value or sub-value can be specified, or the whole field. Validation Rules: Optional input 1-40 type "A" (alphanumeric) characters Cannot be entered if ID is "ALL.FILES". If the id is not "ALL.FILES", either FILE.ROUTINE or field name must be present. If entered, must be in one of the following formats: Field name or Field name-xx where xx is the multivalue number or Field name-xx.yy where yy is the subvalue number or nn where nn is the field number or nn.xx or nn.xx.yy The field name must exist on the STANDARD.SELECTION record

        Parameters:
        Returns:
        TField - The FieldName

      • getFieldRoutine

        public TField getFieldRoutine()
        Returns the FieldRoutine

        Field description: Fields to be amended can be specified on the DB.SCRAMBLE.DEFINE record. Alternatively, a routine can be written to amend the field specified. Instead of the field being replaced with the characters specified in PREFIX, INCLUDE.ID and SUFFIX, the routine specified in FIELD.ROUTINE is called and the amended field is passed back. The routine does not need to update the field - that will be done DB.SCRAMBLE.DEFINE$RUN. Variables for the routine are as follows: ORIGINAL.FIELD, REPLACE.CHARS, NEW.FIELD, YERROR ORIGINAL FIELD - Present contents of the field REPLACE.CHARS - Replacement characters, as specified in PREFIX, INCLUDE.ID and SUFFIX. NEW.FIELD - Amended field to be written back YERROR - Error message Validation Rules: Optional input If entered, a VOC entry for the subroutine must exist. Cannot be entered if ID is "ALL.FILES".

        Parameters:
        Returns:
        TField - The FieldRoutine

      • getIncludeId

        public TField getIncludeId()
        Returns the IncludeId

        Field description: If the field specified is present, it will be replaced by the text specified in PREFIX and SUFFIX. The id of the record being processed can also be used in the replacement characters. For example, PREFIX is specified as "ADDRESS LINE 1 ", INCLUDE.ID is specified as "Y" and SUFFIX is specified as " ***". If the field named in FIELD.NAME is present, it will be replaced by: "ADDRESS LINE 1 xyz ***" where xyz is the id of the record being processed. Validation Rules: Optional input Y or NO Default NO (if ID is not "ALL.FILES") Cannot be entered if ID is "ALL.FILES".

        Parameters:
        Returns:
        TField - The IncludeId
