      • selectRecords

        public List<String> selectRecords(String companyMnemonic, String tableName, String fileSuffix, String filterAndSort)
        Executes the given jQL SELECT statement and returns a list of IDs of matching records.
        Selects on some high volume tables (such as STMT.ENTRY) are blocked by the framework and will generate an exception.
        The list of tables blocked in this way is as follows:
        • AA.ACCOUNT.DETAILS
        • AA.ACTIVITY.HISTORY
        • AA.ARRANGEMENT.ACTIVITY
        • AA.PROCESS.DETAILS
        • ACCT.ENT.TODAY
        • CATEG.ENTRY
        • CATEG.ENTRY.DETAIL
        • DE.HANDOFF
        • DE.O.HEADER
        • DE.O.MSG
        • FUNDS.TRANSFER$HIS
        • HOLD.CONTROL
        • POR.TRANSACTION
        • RE.CONSOL.SPEC.ENTRY
        • RE.SPEC.ENTRY.DETAIL
        • SC.SETTLEMENT$HIS
        • SEC.OPEN.ORDER$HIS
        • SEC.TRADE
        • SECURITY.TRANS
        • STMT.ENTRY
        • STMT.ENTRY.DETAIL
        • STMT.ENTRY.DETAIL.XREF
        • STMT.PRINTED
        • TELLER$HIS

        T24 Details: : Uses EB.READLIST to select records from the database with the given jQL statement.
        Parameters:
        companyMnemonic - Optional. The MNEMONIC of the COMPANY record, eg 'BNK', or 'CO2'. If the empty string is provided then the system will use the currently logged in COMPANY where necessary.
        tableName - The name of the table, eg "CUSTOMER" or 'FUNDS.TRANSFER". No 'F.' prefix is necessary, just the name of the application.
        fileSuffix - The file suffix. Use empty string for the live table, otherwise use '$NAU', '$HIS', '$SIM', '$DEL', etc as per the FILE.CONTROL > SUFFIXES field.
        filterAndSort - The jQL filter and sorting instructions, eg 'WITH SHORT.NAME NE "" BY-DSND CUSTOMER.CODE'.
        Returns:
        List<String> - The record IDs of matching records, or an empty list if no matches are found.
        Throws:
        T24CoreException if there is an error with the select statement, or if selects on the table have been blocked.

      • updateLocalfields

        public TStructure updateLocalfields(String tableName, String recordId, TStructure record)
        Updates the local reference fields of the live record with the values in the local reference field of the given record. Values in the live record will be overwritten with the given values. Core fields cannot be modified with this method.
        T24 Details: : Uses F.LIVE.WRITE to write the record to file if it has changed.
        Parameters:
        tableName - The name of the table, eg "CUSTOMER" or 'FUNDS.TRANSFER". No 'F.' prefix is necessary, just the name of the application.
        recordId - The ID of the required record, eg '1212'.
        record - The record containing the required local reference values.
        Returns:
        TStructure - Record object.
        Throws:
        T24CoreException if the file does not exist or the record does not exist.
