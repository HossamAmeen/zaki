      • getDSlipFmt

        public TField getDSlipFmt()
        Returns the DSlipFmt

        Field description: If the value in this field is not present in the corresponding field in VERSION record, the value in this field and in D.SLIP.FUNC are appended to the VERSION record.If there is no value D.SLIP.TRIGGER of the VERSION record, it is defaulted to RQ. The value(s) defined here are used to produce Deal Slips according to the function and whether the user wants to have them produced automatically or when a 'hot key' is used. Deal Slips can be printed to a connected local printer or directed to a Network device. Refer to the REPORT CONTROL section of the Systems Administration User Guide for details of setting up printers. Deal slips can be linked to a function such as Input, Reversal or Authorisation and specific Deal Slips can be made to relate to those functions. Either an ENQUIRY record or a D.SLIP.FMT record can be entered. If a valid D.SLIP.FMT then the system will provide an enrichment from the record so the user will know thay have the correct record. Alternatively, an ENQUIRY record can be used to provide a Deal Slip, in this case the enrrichment will inform the user that an ENQUIRY record has been selected. The user can drilldown to the related table D.SLIP.FMT to view or select the record to use. The combination of the associated fields is used to define when the Deal Slip can be produced. To trigger them automatically you need to define:

        Field Content Required
        D.SLIP.FMT TELLER.SLIP.INPUT
        D.SLIP.FUNC I
        This will only print the Deal Slip 'TELLER.SLIP.INPUT' when the deal is validated at I(nput) time. To trigger them via a 'hot key' or the Desktop Deal Slip icon you need
        Field Content Required
        D.SLIP.FMT TELLER.SLIP.INPUT
        D.SLIP.FUNC I
        This will only print the Deal Slip 'TELLER.SLIPPUT' at I(nput) time when the 'hot key' is used Note: ENQUIRY linked Deal Slips are valid for Desktop (GUI) use only. Since the ENQUIRY will be based on either the live or unauthorised file then only data that is available on those files can be accessed, whereas D.SLIP.FMT can utilise the new data being entered on a deal. ENQUIRY deal slips therefore need to be based on the live or unauthorised record depending on when the deal slip is being produced. For example a new deal would probably be based on the unauthorised record whereas a reversal would be based on the live record. Validation Rules: Input to this field must be a valid record in either ENQUIRY or D.SLIP.FMT files. Must have values in fields D.SLIP.FUNC. If the version record has no value for DEAL.SLIP.TRIGGER, then it is defaulted to RQ. Subvalue field associated with fields 39 to 43.( D.SLIP.VALUE to D.SLIP.IN.EX).

        Parameters:
        Returns:
        TField - The DSlipFmt

      • getDSlipFunc

        public TField getDSlipFunc()
        Returns the DSlipFunc

        Field description: This field is used in conjunction with D.SLIP.FMT. The Deal Slips can be produced at various times according to the function (or codeword FINISH) and whether they are produced automatically or in conjunction with a 'hot key' FUNCTIONS Valid functions are:

        Data Description
        I Input
        A Authorise
        C Copy
        R Reverse
        D Delete
        H History |Restore
        FINISH Where FINISH is specified, this processing is dependent on the application and may not be triggered for all applications. Refer to the User Guide for the application for further details. Currently, only the TELLER application makes use of this facility. Validation Rules: Valid input is I, A, C, R, D, H or FINISH. Must have input in fields D.SLIP.FMT. Subvalue field associated with fields 39 to 43.( D.SLIP.VALUE to D.SLIP.IN.EX).

        Parameters:
        Returns:
        TField - The DSlipFunc

      • getDSlipInEx

        public TField getDSlipInEx()
        Returns the DSlipInEx

        Field description: Given that the condition in D.SLIP.VALUE and D.SLIP.TYPE is satisfied. If this field is set to INCLUDE then the field specified in D.SLIP.FMT will be set as deal slip format in the Version. If this field is set to EXCLUDE then the field specified in D.SLIP.FMT will not be set as deal slip format in the Version. Validation Rules: INCLUDE or EXCLUDE Subvalue field associated with fields 39 to 43.( D.SLIP.VALUE to D.SLIP.IN.EX).
