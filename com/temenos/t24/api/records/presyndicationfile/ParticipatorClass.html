      • getAmtAllot

        public TField getAmtAllot()
        Returns the AmtAllot

        Field description: This field is used to record that share that is finally allotted to the participant by the Agent. When a mandate is floated, participation interests could be received for more than the amount sought by the borrower, resulting in over-subscription. In consultation with the borrower, the Agent would then allot each participant its share in the syndicate. Amount allotted to each participant may not exceed the amount sought. In order to create the FACILITY, aggregate of AMT.ALLOT should equal SL.AMOUNT within each multi-value set Validation Rules: 1-14 numeric characters plus decimal point Value entered in this field may not exceed AMT.SOUGHT

      • getParticipator

        public TField getParticipator()
        Returns the Participator

        Field description: Identification number or mnemonic assigned to the customer who is willing to participate in the syndicate. This set of fields is used to record details of participation like name of participant, extent of participation sought and allotted and the commitment fee payable to each party. Validation Rules: 3-10 alphanumeric customer mnemonic or 1-10 numeric character customer code Must be a valid record on the CUSTOMER table For value P (only participant) in SL.FUNCTION, only OWN.CUST.REF in the SL.PARAMETER is allowed as input. Multi-valued field to assist recording participation details of more than one party. When SL.FUNCTION is Participant and available for fronting is defined other participant details is mandatory.

        Parameters:
        Returns:
        TField - The Participator
