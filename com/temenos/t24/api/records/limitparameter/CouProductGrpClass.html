      • addCouProdNo

        public void addCouProdNo(TField param)
        Appends the specified CouProdNo to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Identifies which country limit products are part of the group defined in the previous field. Note: The check of duplicates and no mix is only done at the sub-value level. It is possible, for example, to define LOANS, MONEY MARKET and as a 3rd Product Group 'LOANS + MONEY MARKET'. Validation Rules: 3 to 4 numeric (or) alpha-numeric with maximum length of 12 LIMIT REFERENCE ID. Sub value field. Mandatory input. Input must be a valid ID on the LIMIT.REFERENCE file. Duplicates will not be allowed. Mix of Product No and Sub Product No will not be allowed i.e. it is possible to define 100 or 101, 102 etc... but not both.

        Parameters:
        param - the CouProdNo to append
        Returns:
        nothing

      • getCouProductGrp

        public TField getCouProductGrp()
        Returns the CouProductGrp

        Field description: Identifies a mnemonic to group together various country limit products. These groups will then be input in the LIMIT.COUNTRY file to specify the content of the country exposure report. In the LIMIT.COUNTRY file, the user will be requested to enter one or various of these groups to specify what type of limit product must be included in the country exposure report. Validation Rules: 3-15 characters type MNEMONIC. Multivalue field. Mandatory input. The user will be forced to define at least one group. Duplicates are not allowed.

        Returns:
