        public TField getConsolidationRule()
        Returns the ConsolidationRule

        Field description: Input allowed only for Consolidated Status Type. Entry in this field should be a valid record from OA.RULE.DEFINITION, wherein a Boolean rule can be specified. It should be defined by using the status codes which are defined prior to the Rule. Here, the first word (Field Name) should always be a status code. Second word should be an operand and third word will be a literal with quotes.

        eg1 : (FOREIGNAPPLICATION == 'YES') Here, "FOREIGNAPPLICATION" is a Status Code which is defined priorly. "==" is an operand "'YES'" is a literal

        eg2 : (HIGHESTDEGREE == 'PG' AND AGESTATUS == 'ADULT') Here, "HIGHESTDEGREE" and "AGESTATUS" are the status codes which are defined prior to this rule definition. "==" is an operand 'PG' and 'ADULT' are Literals.

        The list of comparison operands are here. Comparison List = "<", "<=", "==", "!=", ">=", ">" Join List = "||", "&&", "OR", "AND" Here is the explanation for Comparison List operators Equals Operator == Not Equals operator != Less than operator "<" Greater than operator ">" Less than or Equals to operator "<=" Greater than or Equals to operator ">="

        Some of the Rule Examples : 1. COUNTRY != "US" 2. CURRENCY == "USD" 3. (GENDER == "MALE") AND (AGE >= 35)

      • getStatusType

        public TField getStatusType()
        Returns the StatusType

        Field description: The list of pre-defined status types are Formlet Class, Form Data, Form Evidence, Consolidated, Table, Form Verification, Form Formlet data, Form Formlet Evidence, Form Formlet verification, Form Evidence Requirement and Stage. Each status code should link with status type. This will determine the behaviour of the status code. 1. Formlet Class – a status code which is determined based upon the runtime data of a single formlet record. In order know which Form to get the data from the domain must be specified.

        2. Form Data – OA will automatically store the data status of each form of the application. However, this type of code can be configured to allow PW to more easily reference the status of a specific form. This will also allow for the status to be used to calculate a consolidation status. Again, the domain must be specified.

        3. Form Evidence – like Form Data, the system will maintain a status for each evidence requirement but the user can create a code of this type to mirror the system status for a specific domain.

        4. Consolidated – a consolidated status code is defined by a rule based on multiple user defined codes. The user defined codes used within the rule can contain other consolidation type codes and validation is required to ensure that there is no circular reference.

        5. Table – the source data to determine the status code is a simple T24 table record which is associated to the application through the PW Process.

        6. Form Verification - Reflects the overall verification status of a single form. Reflects the worst status of "Form Formlet verification" . Domain must be specified to fetch the correct Form

        7. Form Formlet data - Reflects the data status of a single formlet within a form. In order to know which Formlet to get the data from the domain and Formlet must be specified.

        8. Form Formlet Evidence - Reflects the evidence status of a single formlet within a form. Domain and Formlet needs to be specified

        9. Form Formlet verification - Reflects the verification status of a single formlet within a form. Formlet and Domain needs to be specified.

        10. Form Evidence Requirement - Reflects the status of a specific evidence requirement of a single form. Domain and Evidence Requirement needs to be specified

        11. Stage - Reflects the current stage of the Application.

        12. Decision - Reflects the decision result of a decision type specified in field Entity type.

        13. Fulfilment- Reflects the fulfilment result of a fulfilment type specified in field Entity type

        Parameters:
        Returns:
        TField - The StatusType
