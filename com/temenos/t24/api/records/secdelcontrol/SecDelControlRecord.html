      • getCounterpartyLimit

        public TField getCounterpartyLimit()
        Returns the CounterpartyLimit

        Field description: A value of Y indicates that counterparty (broker) limit checking is to take place. If limit checking is to take place then even for deals which are free of payment the TOTAL.AMOUNT.DUE and TOTAL.AMOUNT.OUSTANDING (which indicates the limit exposure) will be updated. When delivery takes place and the amounts received fields are updated then these values will be used to update the counterparty limit, i.e. the amount will be backed out of the current counterparty limit. Partial deliveriees can be catered for in this way. Until the amount oustanding is reduced to zero the limit will remain. If the delivery is set for automatic settlement then during the batch relating to the value date of the SEC.TRADE deal then delivery will be automatically settlled and the amount oustanding backed out of limits. Validation Rules: No input field.

        Parameters:
        Returns:
        TField - The CounterpartyLimit

      • setCounterpartyLimit

        public void setCounterpartyLimit(TField value)
        Set the specified value for CounterpartyLimit
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: A value of Y indicates that counterparty (broker) limit checking is to take place. If limit checking is to take place then even for deals which are free of payment the TOTAL.AMOUNT.DUE and TOTAL.AMOUNT.OUSTANDING (which indicates the limit exposure) will be updated. When delivery takes place and the amounts received fields are updated then these values will be used to update the counterparty limit, i.e. the amount will be backed out of the current counterparty limit. Partial deliveriees can be catered for in this way. Until the amount oustanding is reduced to zero the limit will remain. If the delivery is set for automatic settlement then during the batch relating to the value date of the SEC.TRADE deal then delivery will be automatically settlled and the amount oustanding backed out of limits. Validation Rules: No input field.
