      • addOverride

        public void addOverride(java.lang.CharSequence param)
        Appends the specified Override to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Contains all the override messages which the user agreed to during input of the Drawing. The field will be updated by the system and no operator input will be allowed. During the validation of a Drawing contract, the system may provide the user with a series of screen override messages to indicate an anomaly, for instance, that there is no limit set up for the customer. In the event that the user confirms the override, the full text of the actual override message will be stored, for reference purposes only, in this multivalue field. For example, if the user inputs a deal, then when the Drawing is validated a warning bell will sound and a message will appear at the foot of the screen which reads : 'NO LIMIT DEFINED FOR CUSTOMER' This will draw the users attention to the override entry(s). After checking and making any necessary amendments the user then approves the transaction. Validation Rules: This is a multi valued field. Standard T24 alphanumeric field. A maximum of 35 characters may be entered.

      • getActivityCode

        public TField getActivityCode()
        Returns the ActivityCode

        Field description: This System Generated field contains the activity number triggered by the program. The activity is a 4 digit number, which is composed of three main parts: First digit stands for Operation 1 represents O or P operation in LETTER.OF.CREDIT 2 represents A, C or T operation in LETTER.OF.CREDIT 3 represents Sight Drawing Operation in DRAWINGS 4 represents Usance Drawing Operation in DRAWINGS Second digit stands for Type of Instruments 1 - represents Outward Collection 2 - represents Inward Collection 4 - represents Export Letter of Credit 8 - represents Import Letter of Credit And the last two digit stands for detail activity related to that operation. e.g. For the L/C Export Sight Payment activity, the first digit will be 3. The second digit will be 4 and the last two digits will be 01 (to indicate basic INPUT/AUTHORIZE operation). Thus the activity for Export Sight Drawing will be LC-3401. For the complete list of activity code, please consult Users Manual. Validation Rules: This is a no-input field

        Returns:

      • getDocumentAmount

        public TField getDocumentAmount()
        Returns the DocumentAmount

        Field description: Contains the amount to be drawn under this drawing expressed in the currency above (DRAW.CURRENCY). Separates the amount of the drawing. However, should the applicant/opening bank/Beneficiary wish to effect reimbursement in a currency different to that of the drawing, the market foreign exchange rate will be input and the system will automatically convert the draw currency into that of the reimbursement currency. The relevant drawdown account will be in the same currency as that the drawing currency has been converted too. Validation Rules: 16 type AMT (Standard amount format) characters. (Mandatory input) This amount will be checked against the L/C liability amount to ensure this is not exceeded.

        Parameters:
        Returns:
        TField - The DocumentAmount

      • getDrawCurrency

        public TField getDrawCurrency()
        Returns the DrawCurrency

Validation Rules: Mandatory Input CCY (Currency code e.g. GBP must be 3 characters) - Default to LC.CURRENCY In the event that both the drawdown account and payment account are different currencies to that of the drawing, the Foreign exchange rate applicable to the drawdown account currency and the DRAW.CURRENCY will be entered in RATE.BOOKED. With regards to the payment account a seperate Foreign Exchange rate will be entered in TREASURY.RATE field with TREASURY.SPREAD. The rate entered here will be used between DRAW.CURRENCY and PAYMENT.ACCOUNT.CURRENCY.

        public void setOverride(TField param, int idx)
        Set the specified Override to a specific position in collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Contains all the override messages which the user agreed to during input of the Drawing. The field will be updated by the system and no operator input will be allowed. During the validation of a Drawing contract, the system may provide the user with a series of screen override messages to indicate an anomaly, for instance, that there is no limit set up for the customer. In the event that the user confirms the override, the full text of the actual override message will be stored, for reference purposes only, in this multivalue field. For example, if the user inputs a deal, then when the Drawing is validated a warning bell will sound and a message will appear at the foot of the screen which reads : 'NO LIMIT DEFINED FOR CUSTOMER' This will draw the users attention to the override entry(s). After checking and making any necessary amendments the user then approves the transaction. Validation Rules: This is a multi valued field. Standard T24 alphanumeric field.
