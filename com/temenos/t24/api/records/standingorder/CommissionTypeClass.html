        public TField getCommEndDate()
        Returns the CommEndDate

        Field description: Identifies the Current End Date for the Commission frequency if specified in the previous field. Commission will be applied as per frequency specified in the previous field upto this date. Validation Rules: Associated multivalue field-Input allowed only when Commission frequency is specified along with Date. Up to 9 type D Date characters (date format in range 1950-2049). The date input must be greater than or equal to the first date input or calculated by the commission frequency. This field should only be input by the User when the bank wants to collect commission upto a specific date.

      • getCommFrequency

        public TField getCommFrequency()
        Returns the CommFrequency

        Field description: Specify commission Frequency for this Standing Order and based on this frequency, commission will be collected during Funds.Transfer creation and this is valid till the end date if any specified in the next field. Frequency can be specified as date (for one time execution) or date with valid frequency. If Commission frequency is specified, then it should be same as STO's Current.Frequency. Ie. STO's Current.Frequency is specified as BSNSS, then Comm.Frequency should be specified as BSNSS only. But the start date of Commission Frequency can be different from Current.Frequency. Ie. STO's Current.Freuqnecy is given as 20030109BSNSS & Comm.Frequency can be given as 20030110BSNSS to indicate charges to be collected from 10th Jan 2003. To Collect commission one time, in Commission frequency specify the date on which it should be collected. Ie. For above STO, if Commission to be collected on 12th Jan 2003 , then specify 20030112 in Comm.frequency field. Over rides are generated-When Commission frequency & end date specified without commission type, When Commission frequency start date not same as Current.frequency date, Commission frequency date is Non-working day. When Commission type or Charge type is specified with or without charge frequency/End date, after each STO execution irrespective of whether the Funds transfer contract created or not (due to override / Exception ), Last.Chg.Date will be updated with that run date. Incase of STO's created during retry, Commission /Charge details will be copied except the Commission/Charge frequency & Commission/Charge end date. Validation Rules: Associated multivalue field-Optional Input Comm.Frequency if entered should be same as STO's current frequency This field is sub-divided into two elements: 1. Date 1-9 type D (date format in range 1950-2049) characters. 2. Frequency 1-5 type SS (uppercase alpha or numeric, first character alpha) characters.

        Parameters:
        Returns:
        TField - The CommFrequency

      • getCommissionFor

        public TField getCommissionFor()
        Returns the CommissionFor

        Field description: This field is used to specify whether the corresponding commission amount entered is due to the sending bank or the receiving bank. If the standing order pay method is OT and the system has been set to produce SWIFT message MT103 - Single Customer Credit Transfer Sender - the corresponding charge amount entered is due to the sending bank. Receiver &#8211; the corresponding charge amount entered is due to the receiving bank. Validation Rules: Permitted values are Sender or Receiver, for pay method of OT with a message type of MT103 the default is Sender. If the standing order pay method is set to OT and the system is set to produce message type MT103 there must at least one senders commission or charge present.

        Parameters:
        Returns:
        TField - The CommissionFor
