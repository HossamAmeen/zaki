      • getChargeFor

        public TField getChargeFor()
        Returns the ChargeFor

        Field description: This field is used to specify whether the corresponding charge amount entered is due to the sending bank or the receiving bank. If the standing order pay method is OT and the system has been set to produce SWIFT message MT103 - Single Customer Credit Transfer Sender - the corresponding charge amount entered is due to the sending bank. Receiver &#8211; the corresponding charge amount entered is due to the receiving bank. Validation Rules: Permitted values are Sender or Receiver, for pay method of OT with a message type of MT103 the default is Sender. If the standing order pay method is set to OT and the system is set to produce message type MT103 there must at least one senders charge or commission present.

        public TField getChgEndDate()
        Returns the ChgEndDate

        Field description: Identifies the Current End Date of the Charge frequency if specified in the previous field. Charges will be calculated as per frequency specified in the previous field upto this date. Validation Rules: Associated multivalue field-Input allowed only when Charge frequency is specified along with Date. Up to 9 type D Date characters (date format in range 1950-2049). The date input must be greater than or equal to the first date input or calculated by the Charge frequency. This field should only be input by the User when the bank wants to collect Charges upto a specific date.

      • getChgFrequency

        public TField getChgFrequency()
        Returns the ChgFrequency

        Field description: Specify Charge Frequency for this Standing Order and based on this frequency, charges will be collected during Funds.Transfer creation and this is valid till the end date if any specified in the next field. Frequency can be specified as date (for one time execution) or date with valid frequency. If Charge frequency is specified, then it should be same as STO's Current.Frequency. Ie. STO's Current. Frequency is specified as BSNSS, then Chg.Frequency should be specified as BSNSS only. But the start date of Charge Frequency can be different from Current Frequency. Ie. STO's Current.Frequency is given as 20030109BSNSS & Chg.Frequency can be given as 20030110BSNSS to indicate charges to be collected from 10th Jan 2003. To Collect charge one time, in Charge frequency specify the date on which it should be collected. Ie. For above STO, if Charge to be collected on 12th Jan 2003 , then specify 20030112 in Chg.frequency field. Over rides are generated-When Charge frequency & end date specified without charge type, When Charge frequency start date not same as Current.frequency date, Charge frequency date is Non-working day Validation Rules: Associated multivalue field-Optional Input Chg.Frequency if entered should be same as STO's current frequency This field is sub-divided into two elements: 1. Date 1-9 type D (date format in range 1950-2049) characters. 2. Frequency 1-5 type SS (uppercase alpha or numeric, first character alpha) characters.

        Parameters:
        Returns:
        TField - The ChgFrequency
