      • getCalcFormula

        public TField getCalcFormula()
        Returns the CalcFormula

        Field description: Formula by which the effective rate is to be calculated. System supports three types of calculations HIGH - Highest rate quoted (amongst the recorded figures) would be reckoned as the effective rate LOW - Lowest rate quoted would be construed as the effective rate MEAN - Simple average of the rates quoted will be the effective rate The bank could attach a its own local routine to calculate the effective rate in this field, if the calculation is to be handled differently. In this case the input in this field would be @ followed by the routine name (with no space in between) Ex @CALC if CALC is the name of the routine Validation Rules: Only allowed values are HIGH, LOW or MEAN @ followed by the local routine name may be attached.

        Parameters:
        Returns:
        TField - The CalcFormula

      • getUpdtBiTable

        public TField getUpdtBiTable()
        Returns the UpdtBiTable

        Field description: Decision to update the BASIC.INTEREST table is provided in this field. When quotes are obtained from different banks the user may desire to calculate and see the effective rate or may decide to update the BASIC.INTEREST table with the effective rate. When quotes have been obtained from all the banks, the user may fill in YES in this field in order to update the BASIC.INTEREST table with the effetctive rate for the effective date in field 1. Once YES is defined in this field, only SEE function is allowed after authorisation. Validation Rules: Option of YES or NO Default value is NO

        Parameters:
        Returns:
