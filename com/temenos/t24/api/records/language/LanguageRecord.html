      • getAmountFormat

        public TField getAmountFormat()
        Returns the AmountFormat

        Field description: Specifies the separators to be used when formatting amounts for Enquiry, Repgen & screen input. This field specifies the separators to be used when formatting amounts. Input of ' ,. ' will use commas to separate the thousands and a full stop for the decimal place. Whilst input of '., ' will use a comma decimal place separator and full stop for thousands. These two methods are the most common ',. ' is used in the United Kingdom whilst on the Continent the '., ' is preferred. For certain uses the separators can be set to use an apostrophe with either the comma or full stop. Here are some examples:

        SettingUnformatted AmountFormatted Amount
        ,. 1234512,345.00
        ,. 1234567.891,234,567.89
        ., 1234512.345,00
        ., 1234567.891.234.567,89
        '. 1234512'345.00
        '. 1234567.891'234'567.89
        ', 1234512'345,00
        ', 1234567.891'234'567,89
        Validation Rules: 2 alpha, comma, full stop or apostrophe. Any combination of 2 can be used. (Optional input) Must be either comma, full stop or apostophe. The same character cannot be used twice. The default is ' ,. '

        Parameters:
        Returns:
        TField - The AmountFormat

        Field description: Allows the specification of the minimum length to be used in validation for manadatory language fields of the specified language code.
