        public TField getProfitCntrCr()
        Returns the ProfitCntrCr

        Field description: The profit centre which will receive the charge for this type of outstanding record is defined in this field, and is used as the contents for the PROFIT.CNTR.DR field on the resulting MI.STAT record. The DEPT.ACCT.OFFICER may be defined in one of two ways ; 1) As a constant. At the time that the associated process is run, this field name is evaluated from the record being processed and the result is set on the MI.STAT record. Validation Rules: Either the key to a valid DEPT.ACCT.OFFICER record enclosed in double quotes, OR a field name. In the latter case no validation is provided. Mandatory

        public TField getShareMultiCust()
        Returns the ShareMultiCust

        Field description: This field indicates whether the cost should be equally shared among customers when multiple customers are specified in a transaction. For eg if a cost of 10$ is allocated to a transaction and a transaction between 3 customers A,B,C is processed in the system. If this field holds a value a. YES it indicates that a cost of 3.33$ will be allocated to A, B and C. b. NO will indicate each customer will be allocated a cost of 10$. Validation Rules: YES or NO or NULL Default is NULL Optional input
