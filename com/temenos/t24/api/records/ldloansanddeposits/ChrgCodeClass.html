        Field description: Shows the flat amount of charges to be debited or credited to the Customer on the value date for a new contract, or on-line for an existing contract. The charges amounts will always be defined in contract currency. When a charge code defined in LMM.CHARGE.CONDITIONS contains a CHARGE.CODE linking it to FT.CHARGE.TYPE/FT.COMMISSION.TYPE, the charge amount will be defaulted when no input is made to this field. The charge amount will be based on the outstanding principal of the contract at input time, according to the rates defined in the FT charge tables. Charges may be be payable or receivable, as defined in LMM.CHARGE.CONDITIONS (field PAY.RECEIVE). Payable charges will be credited to the FEE.PAY.ACCOUNT, receivable charges will be debited from the CHRG.LIQ.ACCT. In the case of bill rediscounting transactions this field is system written by the amount of profit/loss on the transaction. The straight rediscount rate is to be arrived at as follows: Straight re-discount rate = Where d: number of days according to the base in the rediscount transaction (ii) If in the discount transaction, a discount to yield rate has been used, (ie) Profit from re-discounting = (Original Bill Transaction Amount (from first transaction)* yield Rate (from first transaction) Bill Transaction amount (from re-discount transaction) * yield rate (from re-discount transaction))*number of days to maturity from rediscounting transaction /number of days in year according to common base.       • getChrgBookedOn

        public TField getChrgBookedOn()
        Returns the ChrgBookedOn

        Field description: The date on which charges are booked. The date indicated in this field is the date on which the associated charges in fields CHRG.CODE to CHRG.CLAIM.DATE were booked by the System. The date will be the value date of the contract if the deal is current or the forward value date if it is a 'FWD' contract. If this field is blank then it means that the charges have not yet been taken. If Fee collections are defined on the schedules, then any charges taken will be updated in the set of associated charge fields CHRG.CODE to CHRG.BOOKED.ON on the actual schedule date defined. Validation Rules: 11 type D (date) characters. (No input field. Updated by the System.) Multivalue field.

      • getChrgCapitalise

        public TField getChrgCapitalise()
        Returns the ChrgCapitalise

        Field description: Identifies whether the associated CHRG.CODE is to be Capitalised (i.e taken from the outstanding principal) or debited from the CHRG.LIQ.ACCT. Entry of YES indicates the charge is to be capitalised, NO means the charge will be liquidated from the customer charge liquidation account. Charges may be capitalised at any time during the life of the contract, with the exception of the contract start and maturity date. A capitalised charge will INCREASE the outstanding principal for a LOAN contract, and DECREASE the principal for a DEPOSIT contract. Capitalised charges may only be taken using theis field where the CHRG.CLAIM.DATE is less than or equal to the current system date. Future charges must be entered via the Schedule definition screen, using a type 'F' schedule for charges to be liquidated to the Charge liquidation account, and type 'N' for charges to capitalised. Charges may be combined with early maturity or partial repayment events to allow a penalty to be taken for breaking the contract. Validation Rules: Input of YES not allowed if CHRG.CLAIM.DATE = FIN.MAT.DATE. Input of YES not allowed if CHRG.CLAIM.DATE = VALUE.DATE. Input of YES only allowed for charges with MONTHLY.AMORTISATION set to NO. Input of YES not allowed for commitments. Input of YES not allowed for PAYable charges.

      • getChrgClaimDate

        public TField getChrgClaimDate()
        Returns the ChrgClaimDate

        Field description: The date on which charges are to be claimed. In the case of forward valued contracts , the date indicated in this field must equal the value date of the contract as specified in VALUE DATE, except for back-valued contracts where there is the option to override the default and select the current date. Validation Rules: 1-9 type A (alphanumeric) characters. (Optional input. Multivalue field. Default value = VALUE.DATE.) Must be a valid date. For forward dated contracts this field must be equal to the value in VALUE.DATE. An override will be required if charge claim date differs from drawdown date for back valued contracts only when first input. An override will be required if the date as defined on the HOLIDAY table is a non working day. An override will be required if the date is not defined on the HOLIDAY table. Once contract has been authorised, this field will be cleared when the next input is made to the contract.

      • getChrgCode

        public TField getChrgCode()
        Returns the ChrgCode

        Field description: Represents a reference on the LMM.CHARGE.CONDITIONS table which holds the Profit and Loss Category code where the charge amount will be credited. Charges have not been restricted to any product but can be defined for all the products covered by the Loans and Deposits Module; such charges can be applied on the drawdown date or process date only. Charges or fees can be handled by the system either on a cash basis or on an accrual basis. Contracts with charges on an accrual basis must straddle at least one month end.The accrual in this case is performed at month end, the number of month ends involved are determined by the parameters on LMM.CHARGE.CONDITIONS. Charges can be entered using these fields at any time during the life of the contract. It is possible to take a charge which will be deducted from from the contract princiapl in the case of a deposit, or added to the principal in the case of a loan contract. Charges taken from the contract principal are indicated by setting CHRG.CAPITALISE set to YES. Charge code can now be defaulted from the LD.TXN.TYPE.CONDITION record set for the particular CATEGORY. User is allowed to Delete/Modity such charge code defaulted at contract level. Validation Rules: 1-3 numeric characters. (Optional input. Multivalue field. Default value on LMM.INSTALL.CONDITIONS). Must be a valid code on the LMM.CHARGE.CONDITIONS table. Charge code for accrual invalid where no month end period is included in the contract. Charge code invalid if MTHLY.AMORTISATION is set to YES and associated CHRG.CAPITALISE value is YES.

        Returns:
