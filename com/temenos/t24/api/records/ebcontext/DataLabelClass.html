        public void addSelectionProcess(java.lang.CharSequence param)
        Appends the specified SelectionProcess to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This is a field similar to that of a jdescriptor in T24. This can accept any valid T24 applications of type H,U,L,T,W or D and their respective data or local reference fields. Example: COMPANY>LOCAL.COUNTRY>COUNTRY>GEOGRAPHICAL.BLOCK The above will fetch the GEOGRAPHICAL.BLOCK of the COMPANY ID passed. CUSTOMER>RESIDENCE>COUNTRY>GEOGRAPHICAL.BLOCK The above will fetch the GEOGRAPHICAL.BLOCK of the CUSTOMER ID passed. For the SYSTEM record, The applications used in the SELECTION.PROCESS must be either of the three applications COMPANY, USER or DATES. For other CONTEXT records, The first application in the field must be one among the T24 applications defined in the PRIMARY.TABLE or LINKED.TABLE fields of the respective CONTEXT record. Example: CONTEXT.ID - CUSTOMER*SIMPLE PRIMARY.TABLE - CUSTOMER LINKED.TABLE - CUSTOMER.CHARGE - CUSTOMER.ACCOUNT . . SELECTION.PROCESS - FILE1>field1>FILE2>field2 Here, the first application "FILE1" must be either CUSTOMER , CUSTOMER.CHARGE or CUSTOMER.ACCOUNT. Validation rules It can accept a maximum of 50 characters. Expandable Multi value field.

        Parameters:
        param - the SelectionProcess to append
        Returns:
        nothing

      • getMultiValueAction

        public TField getMultiValueAction()
        Returns the MultiValueAction

        Field description: The allowed options for this field are FIRST - applicable for all DATA.TYPEs LAST - applicable for all DATA.TYPEs HIGHEST - applicable for 'date' and 'number' DATA.TYPEs LOWEST - applicable for 'date' and 'number' DATA.TYPEs TOTAL - applicable for 'number' DATA.TYPE alone. AVERAGE - applicable for 'number' DATA.TYPE alone. A DATA.LABEL is derived through a DATA.ROUTINE or SELECTION.PROCES and if the resultant output has multiple values then the value shall be processed based on the option given in the respective MULTI.VALUE.ACTION field. If nothing is mentioned in this field then always the first value from the output will be taken and sent to the rules engine for processing the rule. Validation rules This is an optional field and can accept only the above predefined values.

        Returns:

        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This is a field similar to that of a jdescriptor in T24. This can accept any valid T24 applications of type H,U,L,T,W or D and their respective data or local reference fields. Example: COMPANY>LOCAL.COUNTRY>COUNTRY>GEOGRAPHICAL.BLOCK The above will fetch the GEOGRAPHICAL.BLOCK of the COMPANY ID passed. CUSTOMER>RESIDENCE>COUNTRY>GEOGRAPHICAL.BLOCK The above will fetch the GEOGRAPHICAL.BLOCK of the CUSTOMER ID passed. For the SYSTEM record, The applications used in the SELECTION.PROCESS must be either of the three applications COMPANY, USER or DATES. For other CONTEXT records, The first application in the field must be one among the T24 applications defined in the PRIMARY.TABLE or LINKED.TABLE fields of the respective CONTEXT record. Example: CONTEXT.ID - CUSTOMER*SIMPLE PRIMARY.TABLE - CUSTOMER LINKED.TABLE - CUSTOMER.CHARGE - CUSTOMER.ACCOUNT . . SELECTION.PROCESS - FILE1>field1>FILE2>field2 Here, the first application "FILE1" must be either CUSTOMER , CUSTOMER.CHARGE or CUSTOMER.ACCOUNT. Validation rules It can accept a maximum of 50 characters. Expandable Multi value field.

        Parameters:
        param - the SelectionProcess to append
        idx - the position in collection
        Returns:
        nothing
