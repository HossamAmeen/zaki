      • getUserBlock

        public TField getUserBlock()
        Returns the UserBlock

        Field description: Specify the User Header Block Name for mapping into User Header (Block 3) of outgoing Swift Message. Possible Codes in User Header Block -3 (SWIFT Standards Release 2002) and their Globus Block Name in DFS are

        Swift Field Swift Field details Values Globus Block Name
        103 Fin Copy Service Code 3!a SERVICE.ID
        113 Banking Priority (Optional) 4!x
        108 Message User Reference (Optional) 16x
        119 Validation Flag (Optional) 8!c
        115 Payment-Release-Information-Receiver (Optional) 32x PAY.REL.INF
        121 Unique End-to-End Transaction Reference (Optional) 3x UETR.REFR
        111 service-type-identifier (Optional) 3x SERVICE.TYPE.ID
        User Block name has to be mapped to valid user name in the next field and same is mapped using application handoff which in turn generates Swift message with the given user header Block 3 Details. System will populate STP for MT103+ and REMIT for MT103 Extend messages (MT103 with tag 77T) in field 119 (Validation Flag). User Block and User field allowed only for 103. For other message type it is not allowed. Validation Rules: Allowed only for DE.FORMAT.SWIFT with record id-103. Valid values SERVICE.ID, BNK.PRIORITY, MUR, VALID.FLAG, PAY.REL.INF. If entered then next field User.Field is mandatory Duplicate or Null values not allowed

        Parameters:
        Returns:
        TField - The UserBlock

        public TField getUserField()

        Field description: Specifies the name of a field being placed in the SWIFT message as defined on the DE.MESSAGE table.
