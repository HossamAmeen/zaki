        public TField getRestValue()
        Returns the RestValue

        Field description: This field stores the value that will be used in the comparison of the data in the REST.APPLI and REST.FIELD fields: This field can also be used to specify an API/local routine to pre-process the contents of RSTVAL.FIELD before comparison. Validation Rules: Any type of data. Mandatory if the REST.APPLI field is populated and the RSTVAL.FIELD field is not populated. If the RSTVAL.FIELD is populated, this field must contain an API/local routine pre-fixed with '@' symbol or be left blank. When using the NE or EQ operands multiple data can be entered by separating the data with the character from the SEPARATOR field.

        Returns:

      • getRstvalField

        public TField getRstvalField()
        Returns the RstvalField

        Field description: Specifies field which which contains the value that will be used in the comparison of the data in the REST.APPLI and REST.FIELD fields For TRANSACTION restrictions the data stored here will be used to determine if a transaction meets the criteria. If the criteria is true then the restriction is passed otherwise an error message will be displayed. For HOLDING Restrictions (Only available if ASSET.MANAGEMENT module is installed), the data stored here will be used to determine if a portfolio matches certain holdings criteria. If the criteria is true then the restriction is passed otherwise an error message will be displayed. This must be a valid field of REST.APPLI (or RSTVAL.FILE if specified) Mandatory if RSTVAL.FILE or RSTVAL.KEY is specified.

        Returns:
        TField - The RstvalField
