      • addRepayOrder

        public void addRepayOrder(TField param)
        Appends the specified RepayOrder to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies the Repayment order corresponding to the Repayment Method for Settlement Priority rules. The values in this field will be validated based on the value in REPAY.METHOD field. Validation: If Repay Method is AZ.AMOUNT.TYPE then allowed values are PR, IN, OD.PR, OD.IN, OD.CH, PE, PS. If Repay Method is AZ.PRODUCT.PARAMETER then only valid Loan type APP id's where LOAN.REPAY transaction code defined are allowed. If Repay Method is DATE then input in this field is not allowed. Input allowed only if REPAY.METHOD field is defined. Input not allowed when LINK.TO.SETT.PRIOR is defined. Mandatory input other than DATE type REPAY.METHOD. Field Properties: Standard T24 alphanumeric field. A maximum of 20 characters may be entered.

        Parameters:
        param - the RepayOrder to append
        Returns:
        nothing
