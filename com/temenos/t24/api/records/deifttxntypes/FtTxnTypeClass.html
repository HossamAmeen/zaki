      • getFtTxnType

        public TField getFtTxnType()
        Returns the FtTxnType

        Field description: FT.TXN.TYPE.CONDITION type which has to be used while processing Incoming Swift message through OFS.GLOBUS.MANAGER can be specified here. For each message type the criteria may be different. For Income MT200, it will create always OT transaction. Hence one FT.TXN.TYPE can be given here. But Incoming MT103 can have different criteria's as below and FT.TXN.TYPE can be given up to 8 Multivalue set and which ever is matching that FT.TXN.TYPE will be used.

        Option No-Multivalue Acct with bank present (Tag 57) Receiver Correspondent (Third party) bank present (Tag 54) Receiver Charges present (Tag 71G)
        8 NO YES YES
        7 NO YES NO
        6 NO NO YES
        5 NO NO NO
        4 YES YES YES
        3 YES YES NO
        2 YES NO YES
        1 YES NO NO
        Case 5 is considered as default. Example: If inward Swift delivery message has tag 57 and 71G which satisfies the criteria 2 and TXN.TYPE-MV-2 is set as OT02 in DE.I.FT.TXN.TYPES, then FT.TXN.TYPE-OT02 will be used as FT.TXN.TYPE in Inward Funds transfer and details pertaining to OT02 like outward delivery message and charge code will be used while creating a Funds Transfer Contract For criteria related to other Inward Message Types MT200, MT205, MT210 refer to User guide. Incase the Txn.type is not specified for the above combination, error is raised during FT creation. Validation Rules: Mandatory for this application. Multi-value field Should be a valid record from FT.TXN.TYPE.CONDITION.

        Parameters:
        Returns:
        TField - The FtTxnType
