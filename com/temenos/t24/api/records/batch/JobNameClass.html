      • getFrequency

        public TField getFrequency()
        Returns the Frequency

        Field description: Defines the frequency at which the job is processed next. The next run date entered in field NEXT.RUN.DATE for the job is automatically cycled when the system returns to on-line mode. The value that gets updated is based on the frequency defined in this field. Following frequency formats are supported: D - Every working day. Dnn - Every nnth working day. W - Weekly every Friday. M - Last working day of every month. Mnn nn represents the offset added to the first day of every month to arrive at Next Run date. When the resultant date is a holiday, previous Working day is considered. For example, say bank date is 15 MAY 2011 and following is the representation of the Holiday table for JUNE 2011 01 02 03 04 05 06 07 08 W W W H H H W W Frequency M01 will default Next Run date as 02 JUNE 2011 and Frequency M05 will default Next Run date as 03 JUNE 2011 since 6th is holiday A - The job is picked on the any ad-hoc date. The next run date has to be updated manually. Y - Last working day of the year. Ynn - Last working day of the nn'th month. Jxxx - Always run with job name xxx. Validation Rules: Mandatory input.

        Parameters:
        Returns:
        TField - The Frequency

      • getNextRunDate

        public TField getNextRunDate()
        Returns the NextRunDate

        Field description: Specifies bank date on which the job is to be next run. When the end of day is initiated by the batch control system, it only runs those jobs whose next run date is the same as the current bank date. Hence this field allows the user to control when the job is next run. Note : If no date is entered in this field then the next run date is automatically calculated (when the record is updated) using the FREQUENCY specified and the LAST.RUN.DATE (field maintained by batch control system). However if no last run date exists (i.e new job) then the current bank date and frequency is used to calculate the next run date. For jobs with the frequency D, the NEXT.RUN.DATE is not taken into account and is run during every Close of Business. Validation Rules: 0-11 date characters. Year must be in the range 1950 - 2049

        Parameters:
        Returns:
        TField - The NextRunDate

        public void setFrequency(TField value)
        Set the specified value for Frequency
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the frequency at which the job is processed next. The next run date entered in field NEXT.RUN.DATE for the job is automatically cycled when the system returns to on-line mode. The value that gets updated is based on the frequency defined in this field. Following frequency formats are supported: D - Every working day. Dnn - Every nnth working day. W - Weekly every Friday. M - Last working day of every month. Mnn nn represents the offset added to the first day of every month to arrive at Next Run date. When the resultant date is a holiday, previous Working day is considered. For example, say bank date is 15 MAY 2011 and following is the representation of the Holiday table for JUNE 2011 01 02 03 04 05 06 07 08 W W W H H H W W Frequency M01 will default Next Run date as 02 JUNE 2011 and Frequency M05 will default Next Run date as 03 JUNE 2011 since 6th is holiday A - The job is picked on the any ad-hoc date. The next run date has to be updated manually. Y - Last working day of the year. Ynn - Last working day of the nn'th month. Jxxx - Always run with job name xxx. Validation Rules: Mandatory input.

        value - The value to set
        Returns:

        public void setNextRunDate(TField value)
        Set the specified value for NextRunDate
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies bank date on which the job is to be next run. When the end of day is initiated by the batch control system, it only runs those jobs whose next run date is the same as the current bank date. Hence this field allows the user to control when the job is next run. Note : If no date is entered in this field then the next run date is automatically calculated (when the record is updated) using the FREQUENCY specified and the LAST.RUN.DATE (field maintained by batch control system). However if no last run date exists (i.e new job) then the current bank date and frequency is used to calculate the next run date. For jobs with the frequency D, the NEXT.RUN.DATE is not taken into account and is run during every Close of Business. Validation Rules: 0-11 date characters. Year must be in the range 1950 - 2049

        Parameters:
        value - The value to set
        Returns:
