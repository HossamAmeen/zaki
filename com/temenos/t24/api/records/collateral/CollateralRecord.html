      • addExclusionId

        public void addExclusionId(TField param)
        Appends the specified ExclusionId to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Id of COLLATERAL.EXCLUSION record under which the collateral falls Collateral can be marked for exclusion to exclude the collateral value from liability allocation The exclusion criteria can be Country, Currency, Security code (Security master id in APPLICATION.ID field), Industry and Counterparty If the collateral matches with the exclusion criteria defined in COLLATERAL.EXCLUSION and the EXCLUDE.ALL field in COLLATERAL.EXCLUSION is marked as "Yes", then while committing the collateral record, the collateral exclusion id will be defaulted in this field.

        For a Non-Discretionary portfolio, exclusion can be defined for individual assets based on the underlying asset's properties. For example, if the Collateral currency is USD and the Asset's currency is GBP, then only GBP currency exclusion can be defined for the Asset.

        Similarly currency based exclusion for a Discretionary Portfolio will be based on the portfolio currency (as defined in the SEC.ACC.MASTER) instead of the Collateral currency.

        Only when all the assets in a Non-Discretionary Portfolio are exluded, the Collateral status will be changed to "EXC".

        Can also be manually input.

        Validation Rules: 60 digit multi-value field Values allowed are: Exclusion ID <Exclusion ID>!<SC.POS.ASSET.ID>*<SECURITY.NO> - For Asset based exclusions Exclusion ID should be a valid record in COLLATERAL.EXCLUSION application The Collateral's criteria or the Asset's criteria must match with the exclusion's criteria.

        Parameters:
        param - the ExclusionId to append
        Returns:
        nothing
