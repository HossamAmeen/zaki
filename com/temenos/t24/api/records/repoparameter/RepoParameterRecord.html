      • getUpdateScPosSod

        public TField getUpdateScPosSod()
        Returns the UpdateScPosSod

        Field description: A no change field. YES / NO field. Null has the same effect as NO. If NO or NULL is selected, The SECURITY.TRANSFER record for both legs of the REPO contract will get generated n days before the value date or maturity date of the repo contract - where n is a) if DAYS.DELIVERY field is input by the user b) otherwise the value input in DAYS.PRIOR.EVENT for activity RP-0140 in EB.ACTIVITY. The SECURITY.POSITION gets updated immediately on creation of the SECURITY.TRANSFER. If YES is selected, 1. Irrespective of the value date, the SECURITY.TRANSFER record for the near leg will get generated immediately upon authorisation of the REPO contract. The SECURITY.POSITION gets updated on creation of the SECURITY.TRANSFER. 2. The security transfer record for the far leg of the REPO contract will get generated n days before the actual maturity date of the repo contract - where n is a) if DAYS.DELIVERY field is input by the user b) otherwise the value input in DAYS.PRIOR.EVENT for activity RP-0140 in EB.ACTIVITY. 3. If the far leg SECURITY.TRANSFER record is generated prior to maturity date, (as would happen where n has a value) the SECURITY.POSITION will not get updated immediately on generation of the SECURITY.TRANSFER, but it will get updated in the SOD of the maturity date of the REPO. Validation Rules: Valid input either YES, NO or NULL. Null has the same effect as NO.

        Parameters:
        Returns:
        TField - The UpdateScPosSod
