      • getFwdBillSettle

        public TField getFwdBillSettle()
        Returns the FwdBillSettle

        Field description: When an early repayment of a bill component has to be included for updating delinquency amount of subsequent bills (as there is no outstanding bill whose delinquency amount could be reduced), this attribute should be configured to YES. It should be noted that this attribute will be meaningful only when the overdue condition has the required setup for delinquency processing.

        When the option 'YES' is selected, on making an early repayment, the repayment amount will be updated in activity balances. During subsequent make due processing of any scheduled bill, system will reduce the delinquency amount to the extent of repayment amount. Any ageing processing will continue to happen based on the outstanding delinquency amount. In case the outstanding delinquency amount is 0 or within tolerance, ageing status of the bill will be marked as SETTLED during make due activity of the bill,

        When the option is not selected, during early repayment, no details will be captured in activity balances. During subsequent make due processing of any scheduled bill, both the outstanding total amount and outstanding delinquency amount will have the same values.

        Allowed Options - YES or Null

        Parameters:
        Returns:
        TField - The FwdBillSettle

      • getPaymentRuleType

        public TField getPaymentRuleType()
        Returns the PaymentRuleType

        Field description: This field is used to specify how the payment rules will apply to an arrangement.

        This can have any of the below five options.

        Current: Payment will be applied on arrangement's current balances. (e.g. current principal, accrued interest, etc.). This will settle the current balances and doesn’t look at application order. Input to application order will not be allowed for Current Balance repayments.

        Bill Property: For product with bills and due amounts, payment to the due amounts will be Property wise(e.g. all Principal amounts followed by all Interest amounts, etc.)

        Bill Date: For product with bills and due amounts, payment to the due amounts will be bill date wise (e.g., all amount on bill 1 followed by all amounts of bill 2, etc.).

        Pay.In.Advance : Indicates the amount should go into the UNC balance.

        Advance: If chosen, the payment amount will be applied only on the current balances of Interest, account properties and advance due balances for the charge properties During this settlement, System will raise future bills in advance and these bills will either be in "SETTLED" or "ADVANCED" status. Advance field should be set only for the "Payment" Bill types and valid only for the "DUE" Payment method. Valid for future schedules if the properties belong to the system calculated payment types CONSTANT, PROGRESSIVE and ACTUAL/OTHER (Any charges). Also valid for any bills in Issued status(even for LINEAR/INTEREST.ONLY type). Otherwise, the definition will be void and the repayment amount will be passed to the remainder activity.

        Parameters:
        Returns:
        TField - The PaymentRuleType

      • setPaymentRuleType

        public void setPaymentRuleType(java.lang.CharSequence value)
        Set the specified value for PaymentRuleType
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field is used to specify how the payment rules will apply to an arrangement.

        This can have any of the below five options.

        Current: Payment will be applied on arrangement's current balances. (e.g. current principal, accrued interest, etc.). This will settle the current balances and doesn’t look at application order. Input to application order will not be allowed for Current Balance repayments.

        Bill Property: For product with bills and due amounts, payment to the due amounts will be Property wise(e.g. all Principal amounts followed by all Interest amounts, etc.)

        Bill Date: For product with bills and due amounts, payment to the due amounts will be bill date wise (e.g., all amount on bill 1 followed by all amounts of bill 2, etc.).

        Pay.In.Advance : Indicates the amount should go into the UNC balance.

        Advance: If chosen, the payment amount will be applied only on the current balances of Interest, account properties and advance due balances for the charge properties During this settlement, System will raise future bills in advance and these bills will either be in "SETTLED" or "ADVANCED" status. Advance field should be set only for the "Payment" Bill types and valid only for the "DUE" Payment method. Valid for future schedules if the properties belong to the system calculated payment types CONSTANT, PROGRESSIVE and ACTUAL/OTHER (Any charges). Also valid for any bills in Issued status(even for LINEAR/INTEREST.ONLY type). Otherwise, the definition will be void and the repayment amount will be passed to the remainder activity.

        Parameters:
        value - The value to set
        Returns:
        nothing
