        public TField getApplicId()
        Returns the ApplicId

        Field description: This field is used to identify an application for which the fields associated to it have being set up. Validation Rules: No Input Allowed Must be a valid application set up for the company indicated in the ID or In the case of Forex it can be set to the following values : FX.SP - Signifying all Forex contracts that have been set up with a revaluation type of 'SP' FX.RB - Signifying all Forex contracts that have been set up with a revaluation type of 'RB' FX.SL - Signifying all Forex contracts that have been set up with a revaluation type of 'SL' FX.IN - Signifying all Forex contracts that have been set up with a revaluation type of 'IN' FX.IH - Signifying all Forex contracts that have been set up with a revaluation type of 'IH' FX - Signifying all forex contract otherwise not defined specifically as above in other words a catch-all set. Special value of 'ALL' signifies a catch all for the entire system that will be revalued on an individual contract basis. Special value of 'REST' signifying a catch all which will not be revalued individually but rather in a consolidated manner.

      • getRevalType

        public TField getRevalType()
        Returns the RevalType

        Field description: This field signifies the revaluation method was being used for revaluing deals arising out of the application in the APPLIC.ID field. The are only two ways in which revalution is performed, these being : 'SP' - Spot revaluation that uses the spot rate. 'RB' - Rebate revaluation that uses the forward rates table. Validation Rules: Must be either 'SP' or 'RB' If APPLIC.ID = FX.RB then must be 'RB' If APPLIC.ID # FX.RB then must be 'SP' If APPLIC.ID = REST then must be 'SP' No Input Allowed
