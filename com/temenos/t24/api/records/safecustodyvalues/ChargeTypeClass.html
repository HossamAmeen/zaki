      • getCalcZeroBasis

        public TField getCalcZeroBasis()
        Returns the CalcZeroBasis

        Field description: Controls whether to calculate fees for portfolios for which the charge basis amount is zero. The method of deriving the charge basis amount is controlled by the settings within SCPM.GROUP.CONDITION and SCPM.CHARGE.PARAMETER(For Management Fees) and SCSK.GROUP.CONDITION and SCSF.CHARGE.PARAMETER(For Safe Custody Fees). This parameter will also become relevant for flat positions, where all holdings for the relevant asset(s) have been sold. If set to NO then no calculation will be performed where the charge basis amount is zero. As a consequence, any OVERALL.MIN.AMOUNT defined in the application FT.COMMISSION.TYPE will be disregarded. This is the default behaviour if no value is defined in this field. If set to YES then a calculation will be performed for the relevant asset(s) where the charge basis amount is zero. As a consequence, any OVERALL.MIN.AMOUNT defined in the application FT.COMMISSION.TYPE will become effective. Note that where the fee calculation method is set to TOTAL, then a single charge basis amount is calculated for all assets, whereas for the DETAIL method each asset is considered separately. The new parameter described here is referenced for both of these calculation methods. If nothing is entered NO will be assumed. Special note regarding portfolios with no positions: It will only be possible to impose a minimum fee via FT.COMMISSION.TYPE>OVERALL.MIN.AMOUNT in such cases for MONTHLY accrual where the calculation method defined on SCxx.GROUP.CONDITION is TOTAL. In this instance, the calculation will be performed using the TOT.COMM.CODE and TOT.PERCENT specified on SCxx.GROUP.CONDITION. If DETAIL method is specified then T24 will be unable to establish the required Commission Code/Percent where no assets at all are held. DAILY accrual does not support FT.COMMISSION.TYPE>OVERALL.MIN.AMOUNT functionality for such portfolios.

        public TField getCompanyCalc()
        Returns the CompanyCalc

        Field description: Multi-valued field associated with the CHARGE.TYPE field to indicate whether the fees are calculated on a Company Wide basis or at an individual portfolio basis. The bank has a choice of defining a single fees calculation frequency on the SAFECUSTODY.VALUES record (in the CHARGE.FREQ field) for each type of fees or defining an individual fees frequency for each type of fees on the SEC.ACC.MASTER for each portfolio (using the SAFECUSTODY.FREQ and ADVISORY.FREQ fields) to which fees are to be charged. If this field is set to Y then the SAFECUSTODY.FREQ and ADVISORY.FREQ fields on the SEC.ACC.MASTER application will be no input fields and will always be equal to the CHARGE.FREQ field from the SAFECUSTODY.VALUES record for the Company. Thus the calculation of Safecustody or Advisory Fees will take place for all the portfolios in the company on the same date. If this field is set to NO then the user will have to input a charging frequency for each individual portfolio in the SAFECUSTODY.FREQ and ADVISORY.FREQ fields of the SEC.ACC.MASTER for each portfolio to which fees are to be charged. Validation Rules: Only input of Y or NO allowed. Will default to Y.

        Field description: This field is associated with the CHARGE.TYPE field. The field indicates whether or not the system will perform an extract to the corresponding extract files SC.ASSET.BAL or SAFECUSTODY.EXTRACT on a daily basis or not.
