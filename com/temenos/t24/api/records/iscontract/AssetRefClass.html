      • getRebateAction

        public TField getRebateAction()
        Returns the RebateAction

        Field description: Define expected action on calculated rebate amount. This field is used to setup whether: Rebate amount is credited immediately into the PL category/ Customer Account. Rebate amount is amortized into the PL category during the finance tenor This field always works in conjunction with Rebate Allowed field. If Rebate Allowed is not set as YES then it is not required to capture Rebate Action. If Rebate Allowed is not set, then this field value should be None. Meaning rebate amount is not required to be calculated for the contract. If chosen value is Payment then ORIG.PAYMENT.DUE and PAYMENT.OUTSTANDING field values in IS.PAYMENT.BALANCES table are updated after deducting rebate amount from purchase price. The Rebate amount can be paid to Customer or PL category by using Islamic Payment application or by using External applications. If chosen value is Finance then ORIG.PAYMENT.DUE and PAYMENT.OUTSTANDING field values in IS.PAYMENT.BALANCES table are updated after deducting rebate amount from purchase price. The Rebate amount can be amortized to PL category from Finance arrangement throughout the Tenor of finance.

        Validation Rules: 1. Allowed values are None, Payment and Finance. By default it should be set as None. It is part of sub value set starting with Asset Reference and a part of Commodity multi value field.

        Parameters:
        Returns:
        TField - The RebateAction
