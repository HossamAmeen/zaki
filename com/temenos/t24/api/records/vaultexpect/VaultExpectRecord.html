      • getNomOutstanding

        public TField getNomOutstanding()
        Returns the NomOutstanding

        Field description: Total nominal still outstanding on the expectation. If the DIRECTION is RECEIPT this field contains the number of securities the bank is still waiting to receive to satisfy this expectation. If the DIRECTION is DELIVERY then this field contains the number of securities the bank still has to deliver from the Vault to satisfy this expectation. When this field goes to zero then the VAULT.EXPECT record automatically moves to the authorised fiel. Validation Rules: No input field automatically updated by the system.
