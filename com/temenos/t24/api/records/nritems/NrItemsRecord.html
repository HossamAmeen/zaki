      • getAccOwnerRef

        public TField getAccOwnerRef()
        Returns the AccOwnerRef

        Field description: This field contains the reference for the Account Owner. At least one valid character other than a blank must be present. For debit entries, the purpose of this field is to identify, to the Account Owner, the instruction which caused the debit. Therefore, the content of this field is the Senders Transaction Reference Number of the original instruction. Credit entries may be the result of one of the following situations: 1. The Account Servicing Institution is identifying, to the Account Owner the receipt of funds for its account as a result of a related transaction. In this case, the content of is the Reference for the Beneficiary of the related transaction. 2. The Account Servicing Institution has issued a payment instruction to the Account Owner and the credit identified in this field is for that payment. The content of the field is the Transaction Reference Number of the payment instruction issued by the Account Servicing Institution. If no reference is available for this field, the code NONREF will be used. The Account Servicing Institution must then supply, in the Supplementary Details field, what it considers to be the best alternative information. Validation Rules: No Input allowed (Populated by the system when the item is created)

        Returns:
        TField - The AccOwnerRef
