      • addOverride

        public void addOverride(java.lang.CharSequence param)
        Appends the specified Override to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field would contain a valid record from OA.RULE.DEFINITION. Wherein a Boolean Rule can be defined for comparing the field values to either literals or with variables. It should be defined with the field names of the SS record of the primary table defined in the corresponding Context record. Here, the first word should be a valid field name of the context primary table. Second word should be an operand and third word can be a literal (one with quotes) or it can be a Rule variable which is defined in the same application. eg1 : ADDRESS.TYPE == 'OWNED' Here "ADDRESS.TYPE" is a valid field name in the SS record of the primary table OA.APP.ADDRESS which is defined in the context "OA.APP.ADDRESS*ADDRESS" "==" is an operand 'OWNED' is a literal

        eg2 : MARKET.VALUE >= MARKET.VALUE.AAA Here "MARKET.VALUE" is a valid field name of the SS record of the primarytable OA.APP.COLLATERAL which is defined in the context "OA.APP.COLLATERAL*COLLATERAL" ">=" is an operand "MARKET.VALUE.AAA" is the Rule Variable which is defined in the same Status Rules application.

        The list of comparison operands are here. Comparison List = "<", "<=", "==", "!=", ">=", ">" Join List = "||", "&&", "OR", "AND" Here is the explanation for Comparison List operators Equals Operator == Not Equals operator != Less than operator "<" Greater than operator ">" Less than or Equals to operator "<=" Greater than or Equals to operator ">="

        Some of the Rule Examples : 1. COUNTRY != "US" 2. CURRENCY == "USD" 3. (GENDER == "MALE") AND (AGE >= 35)

        Parameters:
        param - the Override to append
        Returns:
        nothing

        public void insertOverride(TField param, int pos)
        Insert the specified Override to the specified position in the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field would contain a valid record from OA.RULE.DEFINITION. Wherein a Boolean Rule can be defined for comparing the field values to either literals or with variables. It should be defined with the field names of the SS record of the primary table defined in the corresponding Context record. Here, the first word should be a valid field name of the context primary table. Second word should be an operand and third word can be a literal (one with quotes) or it can be a Rule variable which is defined in the same application. eg1 : ADDRESS.TYPE == 'OWNED' Here "ADDRESS.TYPE" is a valid field name in the SS record of the primary table OA.APP.ADDRESS which is defined in the context "OA.APP.ADDRESS*ADDRESS" "==" is an operand 'OWNED' is a literal

        eg2 : MARKET.VALUE >= MARKET.VALUE.AAA Here "MARKET.VALUE" is a valid field name of the SS record of the primarytable OA.APP.COLLATERAL which is defined in the context "OA.APP.COLLATERAL*COLLATERAL" ">=" is an operand "MARKET.VALUE.AAA" is the Rule Variable which is defined in the same Status Rules application.

        The list of comparison operands are here.
