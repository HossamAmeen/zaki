      • getReportToRun

        public TField getReportToRun()
        Returns the ReportToRun

        Field description: Defines the PM report to be printed, there is a specific range of reports see below. Each of the reports has been given a 'name' and it is that name that must be entered in this field. The valid names are - CASH.FLOW GAP GAP2 INTEREST.SENSITIVITY A brief description of each report is given. CASH.FLOW - This report shows amounts only and has the following columns - Time period/Date. Outgoing Funds (Takings maturing) Incoming Funds (Placings maturing) Net Difference. Cumulative Balance. GAP - This report gives details of balances and interest rates and has the following columns - Time period/Date. Takings maturing principal. Takings average interest rate. Placings maturing principal. Placings average interest rate. Net Difference principal. Break even rate (this is the rate at which the net difference must be lent or borrowed to breakeven i.e to not make a loss or a profit). Cumulative Balance. This cumulative amount starts from the first time period and works towards the last time period. Such that the cumulative amount for the last time is equal to the total outstanding amount.         Set the specified value for ReportToRun
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the PM report to be printed, there is a specific range of reports see below. Each of the reports has been given a 'name' and it is that name that must be entered in this field. The valid names are - CASH.FLOW GAP GAP2 INTEREST.SENSITIVITY A brief description of each report is given. CASH.FLOW - This report shows amounts only and has the following columns - Time period/Date. Outgoing Funds (Takings maturing) Incoming Funds (Placings maturing) Net Difference. Cumulative Balance. GAP - This report gives details of balances and interest rates and has the following columns - Time period/Date. Takings maturing principal. Takings average interest rate. Placings maturing principal. Placings average interest rate. Net Difference principal. Break even rate (this is the rate at which the net difference must be lent or borrowed to breakeven i.e to not make a loss or a profit). Cumulative Balance. This cumulative amount starts from the first time period and works towards the last time period. Such that the cumulative amount for the last time is equal to the total outstanding amount. GAP2 - This report gives details of balances and interest rates and has the following columns - Time period/Date. Takings maturing principal. Takings average interest rate. Placings maturing principal. Placings average interest rate. Net Difference principal. Break even rate (this is the rate at which the net difference must be lent or borrowed to breakeven i.e not make a loss or profit). Cumulative Balance. This cumulative amount is the total amount outstanding at that period. This means that for the first time period the cumultaive amount is equal to the total of the Net difference principal column (total outstanding amount). INTEREST.SENSITIVITY. This report has the following columns - Time period/Date. Average No of Days for outstanding takings. Average No of Days for outstanding placings. Absolute Difference in the average days. Net principal position for the period. The effect of increase in the interest rates by following percentages - .25% .50% .75% 1% 2% 3% Validation Rules: One of the allowed report names - CASH.FLOW, GAP, GAP2 or INTEREST.SENSITIVITY (Mandatory input)

        Parameters:
        value - The value to set
        Returns:
        nothing

      • setStartBreak

        public void setStartBreak(TField value)
        Set the specified value for StartBreak
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Indicates at which point in the calendar record the time periods are to start from in the report. On a report there is a line for each time period. The time periods are taken from the appropriate record on the PM.CALENDAR file. This field contains the period from which reporting is to start, thus allowing reporting to start from Spot date rather than Calendar date. Validation Rules: 1 to 11 alphanumreic characters (Mandatory input) Must be one of the following - Value Meaning CAL Today (Call). TOM Next Working day (Tomorrow). SPT Two working days from Today (Spot). nnD Number of Calendar Days from Spot date. nnW Number of Weeks from Spot date. nnM Number of Months from Spot date. nnY Number of Years from Spot date. * Remainder that is anything after last time period. Note in the above nn equals 01 - 99.

        Parameters:
        value - The value to set
        Returns:
        nothing
