        public TField getAddPay2Rcvd()
        Returns the AddPay2Rcvd

        Field description: Records the amount of the second additional payment element, defined in the ADD.PAY.TYPE field on the MG.PARAMETER, paid on each individual repayment received. This field is part of an associated multivalue set of fields, PAYMENT.DATE to ADD.PAY.5.RCVD, that record all details relating to individual payments. These fields record both automatic scheduled repayments and manual redemptions entered via the MG.PAYMENT application. However, it is only possible to repay additional payment elements via an automatic repayment as there is no facility for this via the MG.PAYMENT application. Therefore, this field will only record an amount if the payment details in the group in question relate to an automatic repayment. In the case of a manual redemption this field will contain a null value. If nothing is received for this additional payment type on an automatic repayment then this field will contain a zero value. Additional payment elements are defined on the MG.PARAMETER table. Up to five different types may be created. This field records the amount of the second additional element defined in the ADD.PAY.TYPE field of the MG.PARAMETER table, received on each repayment. It should be noted that the name of this field will be modified to incorporate the name of the additional payment as defined on the MG.TYPE.TEXT file. For example, if the second additional payment element defined on the MG.PARAMETER table has a description of "SAVING" on the MG.TYPE.TEXT file then this field will have the name TOT.SAVING.RCVD. Information is stored in this group of fields in reverse chronological order, i.e. the most recent payment details will always be found in the first multivalue of each field. This field will only contain data relating to the contract after the date held in the RECORD.START.DATE field. Any data prior to this may be found on the MG.BALANCES.HIST file. Validation Rules: 1 - 19 numeric characters. Multivalue field associated with fields PAYMENT.DATE to ADD.PAY.5.RCVD.

      • getMgPaymentNo

        public TField getMgPaymentNo()
        Returns the MgPaymentNo

        Field description: Identifies the contract payment number from the MG.PAYMENT application if the multivalue group in question details a manual redemption. This field is part of an associated multivalue set of fields, PAYMENT.DATE to ADD.PAY.5.RCVD, that record all details relating to individual payments. Although these fields record both automatic repayments and manual redemptions entered via the MG.PAYMENT application, the REPAYMENT.NO field will only contain a value when the multivalue group relates to a manual redemption entered via the MG.PAYMENT application. In the case of automatic repayments this field will contain a null value. If the payment detailed in the multivalue group in question relates to a manual redemption, then the contract payment number of that redemption is recorded in this field. This enables a link to be maintained back to the original payment record on the MG.PAYMENT file. In order to access this record the MG.PAYMENT application should be used with a record ID of the contract number and this payment number separated by a full stop. For example, if the MG.BALANCES record for the contract MG9012300012 records the details of a manual redemption with a payment number of 1 then ID to the relevant MG.PAYMENT record is MG901230012.1. Information is stored in this group of fields in reverse chronological order, ie. the most recent payment details will always be found in the first multivalue of each field. It should be noted that these fields will only contain data relating to the contract after the date held in the RECORD.START.DATE field. Any data prior to this may be found on the MG.BALANCES.HIST file. Validation Rules: 1 - 3 numeric characters. Multivalue field associated with fields PAYMENT.DATE to ADD.PAY.5.RCVD.

        Field description: Multi-valued repayment type field. Mortgage account repayments may be of three types, and the values held here indicate the type of payment for the associated multi-values as follows: i) REDEMPTION, the repayment is for an amount of principal only. ii) AUTO.REPAY, the repayment is for an amount of interest only. iii) AUTO.REDEM, the repayment is for both principal and interest.
