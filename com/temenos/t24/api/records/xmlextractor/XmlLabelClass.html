      • getFldConvrsn

        public TField getFldConvrsn()
        Returns the FldConvrsn

        Field description: Specifies conversion to be performed on the data in GLOBUS.FLD. Can take the format: LINK XXX,YYY This will read file XXX using the data in GLOBUS.FLD as the key and return field name YYY as defined in the STANDARD.SELECTION record for XXX. Do not specify F. or company mnemonic in file name. Field Name YYY may be a data field or an I or J descriptor defined in the Standard Selection record of file XXX Note that if a multi or sub valued item is returned from a LINK conversion, the multi and sub values are converted to spaces, so the result cannot be displayed in separate lines. Can also define the routine to be executed for special formatting of the data specified in GLOBUS.FLD. When a valid basic routine is used in FLD.CONVRSN, the output of the routine will be value stored in the XML.LABEL element . The routine mentioned in this field should be written with 4 parameters as given below: Example : TEST.FLD.CONVRSN.RTN (IN. FLD.NAME, RET.VALUE, IN.REC.ID, IN.DATA.REC) Incoming Parameters: IN.FLD.NAME The field name for which the process is made. RET.VALUE - Current value of the field. IN.REC.ID Id of the record for the multivalue that is being processed IN.DATA.REC Contains full record of the file Outgoing Parameters: RET.VALUE Validation Rules: : Optional Input. The routine should be a valid basic routine preceded by an ampersand(&). 35 character alphanumeric.
