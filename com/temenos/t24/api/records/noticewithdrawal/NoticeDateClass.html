        public TField getDateAvailFrom()
        Returns the DateAvailFrom

        Field description: The date when the funds will be made available. Calculated based on the date when notice was given (NOTICE.DATE) and the notice period defined on the SAVINGS.COND file. Eg. If the notice date given for a 500.00 withdrawal is 13/04/90 and the notice period is 3M (3 months) then the DATE.AVAIL.FROM will be set to 13/07/90. Validation Rules: 11 Date NOINPUT

      • getDateAvailUpto

        public TField getDateAvailUpto()
        Returns the DateAvailUpto

        Field description: The date upto which the funds will be made available. This is calculated based on the DATE.AVAIL.FROM and the availablity set in the SAVINGS.COND file. Eg. If the date available from is set to 13/07/90 and the AVAILABLITY field in the SAVINGS.COND file is 1M then the DATE.AVAIL.UPTO will be set to 13/08/90. If a withdrawal is attempted outside the FROM and UPTO date then a override: ACCOUNT & - NOTICE CONDS FOR WITHDRAWAL NOT MET is displayed. Validation Rules: 11 Date NOINPUT

        public TField getNoticeAmount()
        Returns the NoticeAmount

        Field description: The amount the customer intends to withdraw. This amount will determine the notice period to be used from the SAVINGS.COND file. Eg. 3months notice may be required for amounts upto 500.00 whereas 6months notice is required for larger amounts. NOTE: Onc a notice amount is specifed the customer will be allowed to make ANY NUMBER of withdrawals upto this amount, without an override, until the notice is removed from this file or the UPTO date is reached. Validation Rules: 19 Numeric characters.
