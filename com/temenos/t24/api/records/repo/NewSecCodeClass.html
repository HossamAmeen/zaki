      • getNewCuAcctNo

        public TField getNewCuAcctNo()
        Returns the NewCuAcctNo

        Field description: The margin account, defaulted from the MARGIN.PORTFOLIO for regular REPO or from the CUST.PORTFOLIO for Customer a REPO according to the repo currency is held here. The securities that will form a part of REPO or Reverse REPO contract, are defined in the field. If the securities that currently form the REPO contract, defined in the old fields (OLD.SEC.CODE, OLD.NOMINAL, OLD.DEPO and OLD.CU.ACCT.NO), match the contents of the new fields (NEW.SEC.CODE, NEW.NOMINAL, NEW.DEPO and NEW.CU.ACCT.NO) then no transfers are raised. Once the REPO contract is authorised, any securities defined in the new security fields (NEW.SEC.CODE, NEW.NOMINALL, NEW.DEPO and NEW.CU.ACCT.NO) are moved into the old fields. The addition, removal, increase of nominal, decrease of nominal and subsitution of securities is effected by input to the new fields. These fields should be seen as an after image for the securities that will form the REPO contract. If BANK.PORTFOLIO is a DEALER.BOOK and the REPO.TYPE field GENERATE.FT is YES then this field will default to the internal account with currency as security currency and category as set in REPO.TYPE field DEALER.BOOK.SUSP. Validation Rules: A maximum of 16 characters may be entered. Must be the key to a valid entry on the ACCOUNT file.

        Returns:
