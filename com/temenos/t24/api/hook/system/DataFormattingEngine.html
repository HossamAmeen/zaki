      • convertFieldData

        public String convertFieldData(String data, String transactionId)
        This interface enables the implementer to return the converted / formatted value to the attached field.
        This interface is invoked during the COB process through the DFE process
        The EB.API hook used by this interface is DFE.MAP.FIELD.CONV.HOOK.
        The T24 field specifying this hook is the DFE.MAPPING>FIELD.CONV.
        If an exception is thrown in the implementing class will be logged in the EXCEPTION.LOG.FILE.
        Parameters:
        data - The field value, against which the hook has been attached in DFE.MAPPING
        transactionId - This is the id of the current record being processed
        Returns:
        String - the formatted/converted input value of the field.

      • getCompanyCode

        public String getCompanyCode(String sourceData)
        This interface enables the implementer to return a valid company code.This will be defaulted in the OFS message to process the transaction for specific company.
        This interface is invoked by an api DFE.PROCESS.INWARD.DATA, which will process each records based on the mapping given in DFE MAPPING and PARAMETER.
        The EB.API hook used by this interface is DFE.PARAMETER.GET.COMPANY.CODE.HOOK.
        The T24 field specifying this hook is the COMPANY.CODE field in DFE.PARAMETER.
        If an exception is thrown in the implementing class it will be treated as an error.
        Parameters:
        sourceData - The source data with type String
        Returns:
        String - A valid company code.
