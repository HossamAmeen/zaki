        public TField getRtpChargeAcCcyAmount()
        Returns the RtpChargeAcCcyAmount

        Field description: RTP Charges are always collected from requestor only. This field is updated only when this RTP is at requestor's Bank side and charge is applicable for him. Holds the charge to be applied for the corresponding RTP.CHARGE.TYPE for the order in the currency of the RTP.CHARGE.ACCOUNT. Validation Rules: User input allowed only when RTP.CHARGE.ACCOUNT.CCY is defined. When left blank, charge based on charge type will be calculated by the system and amount will be updated in terms of the currency defined in CHARGE.ACCOUNT.CCY field.

      • getRtpChargeAmount

        public TField getRtpChargeAmount()
        Returns the RtpChargeAmount

        Field description: RTP Charges are always collected from requestor only. This field is updated only when this RTP is at requestor'sBank side and charge is applicable for him. Holds the charge to be applied for the corresponding RTP.CHARGE.TYPE. User input allowed only when RTP.CHARGE.CURRENCY isdefined. Both RTP.CHARGE.AMOUNT and RTP.CHARGE.AC.CCY.AMOUNT cannot be defined together. >When left blank, charge based on charge type will be calculated by the system and amount will be updated interms of the currency defined in RTP.CHARGE.CURRENCY field. When this field is left blank and RTP.CHARGE.AC.CCY.AMOUNT hasvalue then the amount will be converted into equivalent RTPChargeAmount in terms of currency in the fieldRTP.CHARGE.CURRENCY. If RTP.CHARGE.CURRENCY is blank then the amount will be converted into equivalent based on value in the field CURRENCY.
