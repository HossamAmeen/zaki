      • getCarrierAddrNo

        public TField getCarrierAddrNo()
        Returns the CarrierAddrNo

        Field description: Specifies the carrier to be used, and the version of the address. Although a record may exist on the Product Table to send all messages to a particular customer by TELEX and PRINT for example, if copies are not allowed for a particular message type (copies on the message table is 'NO'), only the first carrier will be used when the message is formatted. Validation Rules: Can be entered as either carrier.address-no or just carrier Carrier must reference a record on the carrier definition file, DE.CARRIER. The carrier module, as defined in the field CARRIER.MODULE on the DE.CARRIER record, must be in the list of outward carriers on DE.PARM. If address version number is entered, it must be 1-3 numeric characters. Address number defaults to 1. If the Product record is for a customer or a company and the carrier is not Euclid, an address must exist on the Address file for the Carrier Address number. The ADDRESS field from the carrier record (DE.CARRIER) is used for looking-up the address file. Example: If record ID is C-1234.ALL.ALL and TELEXP.2 is specified, there must be an address on file for C-1234.TELEX.2, since the address field for TELEXP on the carrier file is TELEX. CARRIER.ADDR NO, LANGUAGE, FORMAT and COPIES form a multivalue set. Mandatory input

        Parameters:
        Returns:
        TField - The CarrierAddrNo

        public TField getHoldMailOpt()
        Returns the HoldMailOpt

        Field description: Defines the Hold Options Available when Hold Output is set to Y. Valid Values : HELD, DELETED, CLEAR and SEND HELD option is to hold the output delivery message for the carrier address during Hold period. Hold needs to be manually released post the Hold Expiry date DELETED option does not generate output delivery message for the carrier address during Hold Period CLEAR and SEND are applicable only for PRINT Carrier.
