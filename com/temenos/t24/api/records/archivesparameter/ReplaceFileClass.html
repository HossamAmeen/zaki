      • getReplaceFile

        public TField getReplaceFile()
        Returns the ReplaceFile

        Field description: Specifies secondary file to read during the check. When the contents of a field from the current main file record are being checked they can be used as the key to a replace file. The purpose is to replace the contents of the main file field with those of one of the fields on the replace file and then continue the subsequent checks on the new data received. This field begins a subvalue set such that any number of replacements may be made in succession each using the returned field contents from the last replacement to read the next replace file defined. Validation Rules: 1-35 alphanumeric characters. May not be entered without FIELD.NUMBER defined and not allowed for history types 1 or 2.

        Parameters:
        Returns:
        TField - The ReplaceFile

      • getReplaceFld

        public TField getReplaceFld()
        Returns the ReplaceFld

        Field description: Specifies the field number in the replace file to be extracted. The field specified here will be extracted from the replace file record. The contents replace the main file field contents being checked. Subsequent conditions in the parameter file record will now be applied to the new data contents. Validation Rules: 1-3 numeric characters. May not be entered for history types 1 or 2. Must exist as valid field number in the replace file specified.

      • getRplModCall

        public TField getRplModCall()
        Returns the RplModCall

        Field description: Specifies the name of a routine which may be called to modify the contents of the field extracted from the replace file. Or may specify a string or field extraction. String extraction, e.g. [2,8]23L - extracts characters 2 to 2+8 from a string length 23 left justified. Field extraction, e.g. [.2] - extracts portion of field contents from replace file between the 1st and 2nd occurrence of "." Validation Rules: 1-35 alphanumeric characters. May not be entered for history types 1 or 2. Must be a string extraction in format [x,y]nnA or a field extraction in format [c,n] otherwise must begin "@" and exist in the PGM.FILE as a defined subroutine.
