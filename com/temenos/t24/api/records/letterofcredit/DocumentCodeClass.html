      • addDocumentTxt

        public void addDocumentTxt(java.lang.CharSequence param)
        Appends the specified DocumentTxt to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This free text multi valued field provides the user with the ability to either generate or amended pre-coded documents derived from field <DOCUMENT.CODE>. Used in the production of messages/advices, this field in conjunction with <DOCUMENT.CODE> is used to identify the number of each document being sent in the first mailing to the respective party. When used to create an amendment messages using Operation A, any modification of this narrative structured text field identify that the document required of the LC contract is amended. The document details can be modified during the operation A and when AMENDMENT.DEL is set to YES, an MT707 is generated and the amendment details are mapped to tag 46B of the MT707/MT708. When the maximum lines exceeds(250) under this field, the text inputted in this field is auto split and extension message MT708 is generated. The format of the Narrative structured text must begin with the code and must be within slashes (/ADD/, /DELETE/, /REPALL/) for both MT707 (amendment message) and MT708 (extension to amendment message). Multiple codes and multiple combination of codes allowed in the field except for REPALL. No other code is allowed if REPALL is used. ADD - Must be followed by the text to be added (a sentence, paragraph or line item) DELETE - Must be followed by the text to be deleted (a sentence, paragraph or line item) REPALL - Must be followed by the text that replaces all text in same field. No other code may be used. Line 1 (/Code/)(Narrative) Line 2 (Continuation of Narration) Line 3 (/Code/)(Narrative) As per swift, first line of the Narrative structured text in MT708 should begin with the code. User to deduce and input the code manually at the beginning of a MT708 extension message. In case, the user fails to input the code manually, system will automatically add the code at beginning of tag 46B at the time of generation of one or more MT708 extension messages. System will identify the last code evidenced in the previous sequence and prefix this code in the subsequent MT708 messages. If the amendment is handled through LC.AMENDMENTS application and accepted through operation 'D', the user should manually edit the contents to reflect these changes in line with the original instrument (i.e. LETTER.OF.CREDIT application) Validation Rules: Accepts X (Alphanumeric) characters Also, Accepts Z (Alphanumeric) characters when a MT700/701/710/711/720/721/707/708 is generated. Optional input. If input under both DOCUMENT.TXT and DOCUMENTS.REQ, then combined maximum length of the field should not exceed 250 lines. Valid codes to be used are /ADD/, /DELETE/, /REPALL/ in case of amendment.

        Parameters:
        param - the DocumentTxt to append
        Returns:
        nothing

      • getDocumentCode

        public TField getDocumentCode()
        Returns the DocumentCode

        Field description: Used in the production of messages/advices. A list of the various types of documents that are commonly required under L/C's are detailed in the LC.ADVICE.TEXT file and recorded by an ID code. This field can therefore be used to display the documents required for presentation under a letter of credit, or in the case of collections to display the actual documents presented.The full description of the document will be displayed in the field <DOCUMENT.TXT>. Validation Rules: It accepts either Text or Customer Number-Text, If both exists priority is given to Customer Number-Text Customer Number should exists in CUSTOMER table. Text can be maximum of 10 Alphanumeric Characters Must exist in LC.ADVICE.TEXT file If documents required under the LC are not recorded in the LC ADVICE.TEXT file, they can be input separately in DOCUMENTS.REQ.

        Returns:

        public void setDocumentCode(TField value)
        Set the specified value for DocumentCode
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Used in the production of messages/advices. A list of the various types of documents that are commonly required under L/C's are detailed in the LC.ADVICE.TEXT file and recorded by an ID code. This field can therefore be used to display the documents required for presentation under a letter of credit, or in the case of collections to display the actual documents presented.The full description of the document will be displayed in the field <DOCUMENT.TXT>. Validation Rules: It accepts either Text or Customer Number-Text, If both exists priority is given to Customer Number-Text Customer Number should exists in CUSTOMER table. Text can be maximum of 10 Alphanumeric Characters Must exist in LC.ADVICE.TEXT file If documents required under the LC are not recorded in the LC ADVICE.TEXT file, they can be input separately in DOCUMENTS.REQ.

      • setDocumentTxt

        public void setDocumentTxt(java.lang.CharSequence param, int idx)
        Set the specified DocumentTxt to a specific position in collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This free text multi valued field provides the user with the ability to either generate or amended pre-coded documents derived from field <DOCUMENT.CODE>. Used in the production of messages/advices, this field in conjunction with <DOCUMENT.CODE> is used to identify the number of each document being sent in the first mailing to the respective party. When used to create an amendment messages using Operation A, any modification of this narrative structured text field identify that the document required of the LC contract is amended. The document details can be modified during the operation A and when AMENDMENT.DEL is set to YES, an MT707 is generated and the amendment details are mapped to tag 46B of the MT707/MT708. When the maximum lines exceeds(250) under this field, the text inputted in this field is auto split and extension message MT708 is generated. The format of the Narrative structured text must begin with the code and must be within slashes (/ADD/, /DELETE/, /REPALL/) for both MT707 (amendment message) and MT708 (extension to amendment message). Multiple codes and multiple combination of codes allowed in the field except for REPALL. No other code is allowed if REPALL is used. ADD - Must be followed by the text to be added (a sentence, paragraph or line item) DELETE - Must be followed by the text to be deleted (a sentence, paragraph or line item) REPALL - Must be followed by the text that replaces all text in same field. No other code may be used. Line 1 (/Code/)(Narrative) Line 2 (Continuation of Narration) Line 3 (/Code/)(Narrative) As per swift, first line of the Narrative structured text in MT708 should begin with the code. User to deduce and input the code manually at the beginning of a MT708 extension message. In case, the user fails to input the code manually, system will automatically add the code at beginning of tag 46B at the time of generation of one or more MT708 extension messages. System will identify the last code evidenced in the previous sequence and prefix this code in the subsequent MT708 messages. If the amendment is handled through LC.AMENDMENTS application and accepted through operation 'D', the user should manually edit the contents to reflect these changes in line with the original instrument (i.e. LETTER.OF.CREDIT application) Validation Rules: Accepts X (Alphanumeric) characters Also, Accepts Z (Alphanumeric) characters when a MT700/701/710/711/720/721/707/708 is generated. Optional input. If input under both DOCUMENT.TXT and DOCUMENTS.REQ, then combined maximum length of the field should not exceed 250 lines. Valid codes to be used are /ADD/, /DELETE/, /REPALL/ in case of amendment.

        Parameters:
        param - the DocumentTxt to append
        idx - the position in collection
        Returns:
        nothing
