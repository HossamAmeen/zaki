      • getRecIdMap

        public TField getRecIdMap()
        Returns the RecIdMap

        Field description: This should be a valid ID to link to the file name defined above. The record ids can be defined in below manner: 1. The record ID can be the transaction ID itself e.g. SCRIPT035000001. 2. Can be hard coded values within quotes e.g. "SYSTEM" or 'SYSTEM'. 3. Can be formed using a routine e.g. @SSF.SEC.TRADE.POSN.ID. 4. If the field is left blank then the system assumes that only one file is updated and gets the value of RECORD ID from the after image of the transaction 5. Can be valid field of the base application. If it is a multivalue or subvalue field then it needs to be specified for example FIELD.NAME:1:1 This field can have a maximum of 85 characters.

        Parameters:
        Returns:
        TField - The RecIdMap
