      • getBuyRate

        public TField getBuyRate()
        Returns the BuyRate

        Field description: Indicates the Market Buy Rate. Used as the base for calculating (default) rates for Customer Transactions (those less than the negotiable amount). In a Buy/Sell installation this Rate must be input whereas in a Middle Rate environment this Rate is calculated automatically and therefore cannot be input. When installing T24 the user must decide which Currency environment is to be adopted locally. The Company record contains a field which specifies whether a MIDDLE RATE or a BUY/SELL RATE system is to be used: 1. Buy/Sell Rate System This field is mandatory (except for local currency) and in conjunction with SELL.RATE the System automatically calculates the Middle Rate. The resultant difference between Sell and Middle Rates is then taken as the value of DEFAULT.SPREAD. 2. Middle Rate System No input is allowed to this field. Rates are input using the Middle Rate and Default Spread fields and the Market Buy Rate is calculated accordingly. When amending this rate it is checked to be within the specified tolerance of the existing rate. An override is required if out of tolerance. Note : The Buy Rate is the Market Buy Rate and not the final Customer Buy Rate. The latter will be identified by the addition/subtraction of the appropriate Treasury and Customer spreads to this market Buy Rate. Validation Rules: 1-12 type R (Standard Rate Format - up to 8 integer and 6 decimal) characters plus a decimal point or 1-15 type R (Up to 8 integer and 9 decimal) characters plus a decimal point when EXTENDED.PREC in SPF is set to Y (when input allowed). Associated multivalue field. (Mandatory input for Buy/Sell Rate System otherwise no input allowed).

      • getCurrencyMarket

        public TField getCurrencyMarket()
        Returns the CurrencyMarket

        Field description: Identifies to which market the following rates apply in those countries where more than one market exists to determine the exchange rates of foreign currencies. All Transaction Processing Applications contain a data element Currency Market and any input made in this field will be checked against this table to derive or verify the appropriate exchange rate of the transaction. The exchange rates applicable to the Market 1 will always be defaulted if no other input has been done. For the local currency input in this FIELD will also be allowed to permit the user to use, if required, the fields Quotation Suspense and Negotiable Amount. These are the only 2 fields of the association which will be allowed for the local currency. A typical example is Belgium where foreign currencies are quoted on the Convertible/Regular market and also on the Financial/Free market. Different sets of exchange rates will exist according to these two markets and this explains the reason and the purpose of this field. The CURRENCY table allows you to identify, if required, up to nine different markets and consequently to define for each foreign currency up to nine different sets of exchange rates. The System will always force you to define at least one set of rates, i.e. to define the exchange rates applicable to the currency Market 1 which will be the default currency Market used by all Transaction processing applications. It must also be noted that this CURRENCY.MARKET field can also be used within the concept of notional market, i.e. where you would like to define different sets of exchange rates according to the type of product handled. One set of exchange rates, for example, could be defined to identify the exchange rates applicable to your Notes transactions, set of rates for travellers cheques, another set of rates for your transfer transactions, etc. Validation Rules: 2 numeric characters. Market 1 must always be input. Associated multivalue field. For all the countries where more than one market exists for the determination of the exchange rates, input of the various markets in this field will be essential to ensure a correct definition of the exchange rates applicable to the various markets. The Currency Market 1 will always be mandatory. If a market >= 10, the market indicated in the first digit must be already present on the record.

      • getCustMedSpread

        public TField getCustMedSpread()
        Returns the CustMedSpread

        Field description: Defines the spread marketing areas require for medium value transactions. This Spread will be used by the System to calculate the Customer Buy/Sell Rates by modifying the Treasury Buy/Sell Rates applicable to transactions where the amount is greater than the UPTO.SMALL.AMT. When the user does not want to apply the default customer spread on any particular transaction, he will always have the possibility to input the spread at the transaction level. This will then override the customer spread value defined in this table. Different spread conditions can also be defined for groups of customers or directly at the Customer level. This will be done by means of user controlled tables specifically compiled for this purpose, (e.g. the FT.GROUP.CONDITION table or the FT.CUSTOMER.CONDITION table in the FUNDS.TRANSFER Application). This default Customer spread will then be used in any transaction where the amount exceeds the UPTO.SMALL.AMT and will be recognised as the default exchange profit of the transaction attributable to the Marketing area. Validation Rules: Up to 10 type R characters plus a decimal point. (Standard rate format 6,9.) Associated multivalue field. Input not allowed for local currency; Otherwise optional Input. No default value. Quotation or use of a rate, other than the Customer Buy/Sell Rate, will only effect the profit margin of the marketing area. If a Treasury and Customer medium spread have been input while the Treasury and Customer small spreads were not entered, the system will then use the medium spread for all amounts up to the negotiable amount. For any amount above the negotiable amount only the Customer medium spread will be applicable. If the Treasury and Customer medium spreads are not input while both the UPTO.SMALL.AMT and the NEGOTIABLE.AMT have been entered, the system will take no spread on any transaction which exceeds the UPTO.SMALL.AMT. Example: The following example (based on UK environment exchange rate quotation) is an illustration of the concept and the relation of these various spread fields with UPTO.SMALL.AMT and NEGOTIABLE.AMT. Let us suppose that Wonder bank London branch applies the following rules for the transactions involving conversion in US Dollars. 1) UPTO.SMALL.AMT = USD 1,000.00. 2) NEGOTIABLE.AMT = USD 10,000.00, i.e. for any transaction involving a conversion in USD for an amount greater than USD 10,000.00, the exchange rate must be requested from the dealer. 3) TRSY.SMALL.SPRD = 0.0060 4) CUST.SMALL.SPRD = 0.0100 5) TRSY.MED.SPREAD = 0.0030 6) CUST.MED.SPREAD = 0.0050 7) BUY.RATE = 1.2520 8) SELL.RATE = 1.2500 Then the following default values will apply if the bank is selling US Dollars against Pound Sterling to the Customer. Trans Amt Treas Spread Cust Spread Final Sell Rate ========= ============ =========== =============== USD 800.00 0.0060 0.100 1.2340 USD 5,000.00 0.0030 0.0050 1.2420 USD12,000.00 N/A 0.0050 Treas rate input - 0.0050

        Field description: Defines the default spread marketing areas require from small value transactions. This default spread will then be used in any transaction when the amount does not exceed the UPTO.SMALL.AMT and will be recognised as the default exchange profit of the transaction attributable to the Marketing area. The spread will be used by the System to calculate the Customer Buy/Sell rates by modifying the Treasury Buy/Sell rates. Note : Quotation or use of a rate, other than the Customer Buy/Sell rate, will only effect the profit margin of the marketing area. Validation Rules: Up to 10 type R characters plus a decimal point. (Standard rate format 6,9.) Associated multivalue field. Either this field or TRSY.SMALL.SPRD must be input when an UPTO.SMALL.AMT is present. Input not allowed for local currency. Default value = None.

      • getDefaultSpread

        public TField getDefaultSpread()
        Returns the DefaultSpread

        Field description: Indicates the amount to be added to/ subtracted from MID.REVAL.RATE to obtain the Market Buy/Sell Rates. The field reflects the actual market spread between Middle and Buy/Sell Rates. It is important to realise that the addition/subtraction of this spread to/from the middle rate will generate the Market Buy/Sell rates and not the final Customer Buy/Sell rates. The latter will be identified by the addition/subtraction of the appropriate Treasury and Customer spreads to the Market Buy/Sell rates. Validation Rules: Up to 10 type R characters plus a decimal point. Standard rate format 6,9. (when input allowed). Associated Multivalue Field. Input not allowed for local currency. Mandatory input for Middle Rate System, otherwise no input allowed. In a Middle Rate installation this is a mandatory field (except for the local currency) and the rate input is added to/subtracted from MID.REVAL.RATE to obtain the Market Buy/Sell Rate. In a Buy/Sell installation input is not allowed to this field. The system calculates this spread by comparing MID.REVAL.RATE to SELL.RATE. When allowed to be input, the default market spread must be smaller than the Middle Rate.

      • getMidRevalRate

        public TField getMidRevalRate()
        Returns the MidRevalRate

        Field description: Specifies the Market Middle Rate for this Currency market. In a Middle Rate installation this rate must be input whereas in a Buy/Sell environment this rate is calculated automatically and therefore cannot be input. When installing T24 the user must decide which Currency environment is to be adopted locally. The COMPANY record contains a field which specifies whether a MIDDLE RATE or a BUY/SELL RATE System is to be used. This market middle rate will be the default rate used by the revaluation routine to revalue the bank books and the SPOT foreign exchange contracts. All other revaluation types mayuse the REVAL.RATE for revaluation calculations except for rebate revaluations which continue to use this field. 1. Middle Rate System This field is mandatory (except for local currency) and in conjunction with DEFAULT.SPREAD the System automatically calculates the Market Buy/Sell Rates. 2. Buy/Sell Rate System No input is allowed to this field. Rates are input to the Buy and Sell fields and the Middle Rate is calculated accordingly. When amending this Rate it is checked to be within the specified tolerance of the existing rate. An override is required if out of tolerance. Note : (i) Care must be taken inputting this Rate. The banks books will be revalued using the value of this rate. (ii) The Middle Rate will also be used as the reference to check any rate input at transaction level. Variance greater than the Rate Allowance defined for the Currency will require an override. This rate is ignored when an alternative rate is entered in REVAL.RATE except for rebate revaluations. The revaluation processes use the REVAL.RATE if present , overriding any value in the MIDDLE MARKET Rate. Validation Rules: 1-12 type R (Standard Rate Format - up to 8 integer and 6 decimal) characters plus a decimal point or 1-15 type R (Up to 8 integer and 9 decimal) characters plus a decimal point when EXTENDED.PREC in SPF is set to Y. (when input allowed). Associated multivalue field. Mandatory input for Middle Rate System otherwise no input allowed. Input in this field for the local currency is also not allowed.

      • getNegotiableAmt

        public TField getNegotiableAmt()
        Returns the NegotiableAmt

        Field description: Defines the amount above which the exchange rate to be used must be obtained from the dealer. The purpose of this field is to define a maximum amount for which the Rates specified in this record can be used without reference to the dealers. The exchange rate on transactions, below the specified value, need not be input. Based on the applicable Rates, Spreads and Condition tables, the System will apply the appropriate rate, and calculate the local currency equivalent. Transactions which exceed the amount specified should be booked with the dealers, and the rate obtained should be input. Any rate input will be treated as the Treasury rate, and for this reason only the Customer Medium Spread will be added/subtracted to/from it to determine the Customer Buy/Sell rate. For transactions which exceed the negotiable amount, the System will still allow to default the rates applicable to smaller transactions but in this case an override will be required to force the transaction with the existing Rates. Note : Input in this field will be allowed for local currency and will be used in the case, for example, where the Customer asks to pay in a Foreign currency the equivalent of 10.000 units of the local currency. Validation Rules: Up to 14 type AMT characters. (Standard amount format.) Associated multivalue field. (Mandatory input)

      • getTrsyMedSpread

        public TField getTrsyMedSpread()
        Returns the TrsyMedSpread

        Field description: Defines the spread Treasury require for Medium value Transactions. This field defines the profit margin attributable to the dealers on any Transaction where the value is less than NEGOTIABLE.AMT but, where specified, greater than UPTO.SMALL.AMT. This default Treasury spread will be used by the System to calculate the Treasury Buy/Sell Rate (by modifying the actual Market Buy/Sell rate Fields 16 and 17) in any transaction where the amount exceeds the Upto Small Amount but is not greater than the Negotiable amount. When the latter is the case, a rate will need to be input in the transaction and this rate will be considered as being inclusive of the Treasury Spread. Validation Rules: Up to 10 type R characters plus a decimal point. (Standard rate format 6,9.) Associated multivalue field. Input not allowed for local currency; otherwise optional input. No default Value.
