        public TField getCategStart()
        Returns the CategStart

        Field description: Allows further definitions by category code or category code range. This will be the start category. For example all customer accounts could be defined as high volume except for a particular range of categories, (i.e. retail customer accounts, or customer savings accounts) If a category code range or a single category code is defined then all accounts in this category code range will default to the opposite of the default value. If the default value is "YES" then these accounts will be non HVT. If the default value is "NO" then accounts defined here will default to HVT. Validation Rules:

        Should accept a valid category code

        Category code must be in the range 1000-19999 This field is made no inputtable if the live record already exits. And also if the new record is created for company level and system record exists already, then this field gets populated from the system record.
