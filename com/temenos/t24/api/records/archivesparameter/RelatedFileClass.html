      • getConcModCall

        public TField getConcModCall()
        Returns the ConcModCall

        Field description: Specifies the name of a routine which may be called to modify the entry to be looked up on the concat file. String extraction, e.g. [2,8]23L. - extracts characters 2 to 2+8 from a string length 23 left justified. Field extraction, e.g. [.2]. - extracts portion of field contents between the 1st and 2nd occurrence of "." Concat related treatments use the related field and related mod call subvalue set to build the key to the concat file. When this key is located the entry to be removed is normally the key to the main file record being processed. If this is not the case this field exists to allow the key to be modified or an entirely different key to be returned from a called routine. The modified key will be the entry looked for in the concat record. Validation Rules: 1-35 alphanumeric characters. May not be entered without related file defined. Only allowed if RELATED.ACTION is "C" for concat removal. Must be a string extraction in format [x,y]nnA or a field extraction in format [c,n] otherwise must begin "@" and exist in the PGM.FILE as a defined subroutine.

        Parameters:
        Returns:
        TField - The ConcModCall

      • getRelSelection

        public TField getRelSelection()
        Returns the RelSelection

        Field description: Specifies the selection condition to be used to select the related file keys to be processed. The subvalue set beginning RELATED.FIELD and ending REL.MOD.CALL builds a key to the related file. This key is then used in a selection of the related file. If "EQ" is entered only the record equal to the key so built is chosen for processing from the related file. If one of the LIKE operands is used the related file is selected using the key built up and a number of records may be chosen from the related file. Each record chosen will be processed according to the action specified in the following field RELATED.ACTION. Validation Rules: 1-8 alphanumeric characters. Not allowed without related file defined. Must be one of the following: LK LiKe (containing) ...LK LiKe with exact in rightmost LK... LiKe with exact in leftmost EQ EQual

        Parameters:
        Returns:
        TField - The RelSelection

      • getRelatedAction

        public TField getRelatedAction()
        Returns the RelatedAction

        Field description: Specifies the action to be performed on records chosen from the related file by the selection criteria built in the previous fields. Validation Rules: 1 alphanumeric character. Not allowed without related file defined. Must be one of the following: A Archive related record to tape and then delete. D Delete related record when main file record is archived. C Remove related file key from a concat record on the related file.

        Parameters:
        Returns:

      • getRelatedFile

        public TField getRelatedFile()
        Returns the RelatedFile

        Field description: Specifies the name of a related file from which one or more records require archiving when the main file record is archived. When a record from the main file is found to satisfy all conditions and is archived this field specifies one or more related files which require treatment as a result. The field begins a multivalue set designed to build up criteria to select related file records and apply the specified treatment to them. Validation Rules: 1-27 alphanumeric characters. Filename entered must exist as an application on the PGM.FILE. Not allowed for history type 5.

        Parameters:
        Returns:
        TField - The RelatedFile
