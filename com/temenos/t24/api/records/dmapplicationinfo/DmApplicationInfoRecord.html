      • addCustFieldName

        public void addCustFieldName(TField param)
        Appends the specified CustFieldName to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies the names of customer fields in the Application. Documents would be tracked only for the customer(s) whose IDs equal the values of the fields defined here. If no field is defined, then documents for this application could be tracked only from a related application. Example.1 Record ID-FUNDS.TRANSFER APPLN.MNEMONIC-FT CUST.FIELD.NAME-DEBIT.CUSTOMER CUST.FIELD.NAME-CREDIT.CUSTOMER When an FT is input, T24 will track required documents for customers whose IDs equal to the values of the fields DEBIT.CUSTOMER and CREDIT.CUSTOMER in the FT. Example.2 Record ID- SEC.TRADE APPLN.MNEMONIC-SETR CUST.FIELD.NAME-CUSTOMER.NO When a SEC.TRADE is input, since CUSTOMER.NO is a multi-value field, T24 will track required documents for all customers whose IDs equal to the values of the field CUSTOMER.NO in the SEC.TRADE. Example.3 Record ID-SECURITY.MASTER APPLN.MNEMONIC-SEMR CUST.FIELD.NAME- Null In the above case, documents will not be tracked when a record is directly input in SECURITY.MASTER. However, if SECURITY.MASTER is defined as a related application for SEC.TRADE (in the application DOCUMENT.REQUIRED), when a record is input in SEC.TRADE, documents for SECURITY.MASTER will be also tracked along with those required for SEC.TRADE. Validation Rules: Optional. Value allowed only if the application specified in the ID is a CUS or FIN type application (Value of the field CLASSIFICATION in FILE.CONTROL is CUS or FIN for the application specified in the ID). Any valid field defined in the Standard Selection of the application. However care must to taken to define only those fields which represent a Customer ID. For CUSTOMER application, only allowed value would be @ID i.e. the ID of the Customer record itself.

        Parameters:
        param - the CustFieldName to append
        Returns:
        nothing
