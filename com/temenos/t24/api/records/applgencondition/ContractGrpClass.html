      • getContractGrp

        public TField getContractGrp()
        Returns the ContractGrp

        Field description: Identifies the contract group based on the conditions defined in the fields from DECIS.FIELD to DECISION.TO. Each contract group can be based on multiple decision checks. All decision checks must evaluate to true for the group code to be allocated. The first group to pass all decision checks will be allocated, and no further processing will take place, so the order in which groups are defined is critical. If a decision check evaluates to false then the next decision group will be evaluated. Each decision check is performed by either: Calling a locally developed subroutine to perform complex evaluation tasks. or: Evaluating the variable returned from the DECIS.FIELD value using the comparison defined in the DECISION field against the values defined in the DECSION.FR and DECSION.TO fields. The variable returned from the DECIS.FIELD value can be one of 3 types as follows and must be defined on the STANDARD.SELCTION record for the application. DATA ITEM The contents of a data field in the application record being processed. LINKED DATA ITEM The contents of a linked record field as defined in the STANDARD.SELECTION record for the application. For example in MM.MONEY.MARKET: 1. 5 SYS.FIELD.NAME. CUST.TAX.GROUP 2. 5 SYS.TYPE....... J 3. 5. 1 SYS.FIELD.NO CUSTOMER.ID>CUSTOMER.CHARGE>TAX.ACT.GROUP Where the customers tax group is derived from the customer charge record. IDESCRIPTOR FIELD Where the result of a subroutine can be allocated to a field name. For example in MM.MONEY.MARKET 1.140 SYS.FIELD.NAME. PRINCIPAL.EQUIV 2.140 SYS.TYPE....... I 3.140. 1 SYS.FIELD.NO SUBR("PCONV",PRINCIPAL,CURRENCY,"GBP") A subroutine PCONV is called which converts the PRINCIPAL in the CURRENCY of the contract to GBP. GBP is passed to the subroutine as a constant, it would be equally possible to use a LOCAL.REFERENCE field value for the "to" currency. Validation Rules: 5 alphanumeric characters.
