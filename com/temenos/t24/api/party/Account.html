com.temenos.t24.api.party

Class Account

Packaged in : AC.AccountApi.jar

Provides methods to retrieve various balances and details of an account. The member variable for this class is accountId, this variable must be set for many of the methods to work correctly.

      • getCapitalisationDate

        public TDate getCapitalisationDate(String groupConditionId, String interestType)
        Returns the next capitalisation date for the ACCOUNT or GROUP.CONDITION using the given interestType. The system will automatically decide which capitalisation to return. For interestType, use the following values:
        • CR = Credit 1 interest
        • CR2 = Credit 2 interest
        • DR = Debit 1 interest
        • DR2 = Debit 2 interest
        • ALL = All of the above

        T24 Details: This invokes the T24 routine AC.GET.NEXT.CAP.DATE. This uses values in the ACCT.CAPITALISATION or GROUP.CAPITALISATION tables.
        Parameters:
        groupConditionId - Required. The condition group of the account.
        interestType - Required. One of 'CH', 'CR', 'CR2', 'DR', 'DR2' or 'ALL'.
        Returns:
        TDate - next capitalisation date of required interest or charge for the account.
        Throws:
        T24CoreException if accountId is not a valid account identifier.

      • getEntries

        public List<String> getEntries(String processType, String forwardValueDatedEntry, String futureValueDatedEntry, String reversalEntry, TDate startDate, TDate endDate)
        Returns the accounting entries posted to account for a given date. It is possible to fetch the accounting entries for specific booking date/ value date/ exposure date/ processing date. The default choice would be to return the accounting entries for a specified booking date. The Accounting entries listed includes STMT.ENTRY, Forward Entries and Future value dated STMT.ENTRY.
        T24 Details: This invokes the T24 routine EB.ACCT.ENTRY.LIST.
        Parameters:
        processType - Required. Must be one of 'PROCESS', 'VALUE', 'EXPOSURE', 'BOOK.VALUE', 'BOOK', 'POSITION.EXPOSURE'.
        forwardValueDatedEntry - Optional. Set this parameter to 'FWD.VD.ENTRY' to include the forward value dated entries. If omitted, forward value dated entries will not be returned.
        futureValueDatedEntry - Optional. Set this parameter to 'FUTURE' to include the future value dated entries. If omitted, future value dated entries will not be returned.
        reversalEntry - Optional. Set this parameter to 'YES' to include the reversal entries. If omitted, reversal entries will not be returned.
        startDate - Required. Starting date. Start date is included in the ouput.
        endDate - Required. End date. End date is included in the output.
        Returns:
        List<String> - A list of keys to the STMT.ENTRY table in T24, which match the query criteria.
        Throws:
        T24CoreException if accountId is not a valid account identifier.

      • getNostroAccount

        public NostroAccount getNostroAccount(String currencyId, TDate effectiveDate, String productId)
        Returns the NOSTRO account details to the appropriate currency and application. If no account is mapped to the given currency then it will return the account mapped to the "ALL" application in the NOSTRO.ACCOUNT table.
        T24 Details: This invokes the T24 routine GET.NOSTRO.
        Parameters:
        currencyId - Required. Key to the CURRENCY and NOSTRO.ACCOUNT tables.
        effectiveDate - Optional. The effective date from when the given account id will be acting as NOSTRO. If omitted then no cross check against the effectiveDate will be performed.
        productId - Required. Key to the EB.PRODUCT table. Product against which the validation has to be performed. Eg FT, LD or a special value "ALL" which will use any matching product.
        Returns:
        NostroAccount - the nostro account information.
        Throws:
        T24CoreException on invalid parameters, or if no matching nostro can be found.

      • getNostroAccountForTransactionType

        public NostroAccount getNostroAccountForTransactionType(String currencyId, TDate effectiveDate, String transactionType)
        Returns the NOSTRO account information which is appropriate to the given currency or accountId and applicable only to the FT application. Either accountId or currencyId must be provided, but not both.
        T24 Details: This invokes the T24 routine GET.NOSTRO.
        Parameters:
        currencyId - Required. Key to the CURRENCY and NOSTRO.ACCOUNT tables.
        effectiveDate - Optional. The effective date from when the given account id will be acting as NOSTRO. If omitted then no cross check against the effectiveDate will be performed.
        transactionType - Required. Key to the FT.TXN.TYPE.CONDITION table. Transaction type for which the NOSTRO.ACCOUNT details are required.
        Returns:
        NostroAccount - the nostro account information. If no account is available specifically for the given transactionType, it will return the account of the "ALL" application in the NOSTRO.ACCOUNT table.
        Throws:
        T24CoreException on invalid parameters or invalid in NOSTRO.ACCOUNT file.

      • getOpenActualBalance

        public Amount getOpenActualBalance()
        Returns the 'Ledger Balance' of an given account at the start of the day. In T24 terms, this is the OPEN.ACTUAL.BALANCE from EB.CONTRACT.BALANCES. All authorised entries in the account with value date today or less than today would contribute to the Actual balance. Being a Start of Day balance, This balance would not be updated by entries booked during the day.
        T24 Details: This invokes the T24 routine AccountService.getOpenActualBalance.
        Parameters:
        Returns:
        Amount - open actual(Ledger) balance of the account in the currency of the account.
        Throws:
        T24CoreException if accountId is not a valid account identifier.

      • getOpenAvailableBalance

        public Amount getOpenAvailableBalance()
        Returns the available balance of an given account at the start of the day. The balance consists of all authorised debits and credits with value date less than or equal to today. It would optionally include unauthorised debits or credits if indicated in ACCOUNT.PARAMETER in AVAILABLE.BAL.UPD. This balance is not updated by entries booked during the day.
        T24 Details: This invokes the T24 routine AccountService.getAvailableBalanceDetails.
        Parameters:
        Returns:
        Amount - open available balance of the account in the currency of the account.
        Throws:
        T24CoreException if accountId is not a valid account identifier.

        T24 Details: This invokes the T24 routine AccountService.getOpenClearedBalance.

        Returns:
        Amount - open cleared balance of the account with currency of the balance.

        T24CoreException if accountId is not a valid account identifier.

        public TBoolean isValidNostroAccount(String productId, String currencyId, TDate effectiveDate)
        Returns true if the given accountId is a NOSTRO account against the given currency and application.
        T24 Details: This invokes the T24 routine GET.NOSTRO.
