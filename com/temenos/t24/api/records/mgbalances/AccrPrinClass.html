        public TField getAccrActAmt()
        Returns the AccrActAmt

        Field description: This field contains the amount of interest, unrounded, accrued for the multivalue accrual group in question. The set of fields ACCR.FROM.DATE to ACCR.ACT.AMT form the multivalue set of interest accrual fields. These fields record all details of interest accrued on the contract. A new group is created each time one of the accrual parameters, i.e. outstanding principal or interest rate, changes, or a month end has been reached. This field records the amount of interest accrued for each of the multivalue accrual groups. This amount is un-rounded and is held to four decimal places. The rounded accrued amount is stored in the ACCR.AMT field. Information is stored in these fields in reverse chronological order, i.e. the most recent accrual details will always be found in the first multivalue of each field. It should be noted that this field will only contain data relating to the contract after the date held in the RECORD.START.DATE field. Any data prior to this may be found on the MG.BALANCES.HIST file. Example : The following shows the accrual details on a contract with repayments on the 28th of each month with an interest rate change from 10.9% to 11.3% on 13/10/90. Accruals are shown from 28/09/90 to 27/10/90 which represents one complete interest period. The currency of the contract is pounds sterling which has two decimal places.

        ACCR.PRINCIPAL 19000.0019000.0019000.00
        ACCR.RATE11.3%10.9%10.9%
        ACCR.FROM.DATE 14/10/9001/10/9028/09/90
        ACCR.TO.DATE27/10/9013/10/9030/09/90
        ACCR.DAYS14133
        ACCR.AMT83.5074.7817.26
        ACCR.ACT.AMT83.494474.786117.2583

        public TField getAccrAmt()
        Returns the AccrAmt

        Field description: This field contains the amount of interest accrued for the multivalue accrual group in question, rounded to the number of decimal places of the currency of the contract. The set of fields ACCR.FROM.DATE to ACCR.ACT.AMT form the multivalue set of interest accrual fields. These fields record all details of interest accrued on the contract. A new group is created each time one of the accrual parameters, i.e. outstanding principal or interest rate, changes, or a month end has been reached. This field records the amount of accrued interest, rounded to the correct number of decimal places for the currency of the loan, for each of the accrual groups. This amount is accumulated for each interest period and recorded in the OTS.INTEREST field which then determines the amount of interest to be paid at the end of the interest period. Interest is accrued to four decimal places and then rounded in such a way that the error in rounding is kept to a minimum. The four decimal place un-rounded amounts are held in the associated ACTUAL.ACCR.AMT field. Any rounding error from a given accrual group is carried forward via the un-rounded accrual amount of the following group. However, this is only true if the following group forms part of the same interest period. As with all other multivalue fields on this file the accrual groups are stored in reverse chronological order. The method of rounding is best illustrated by the following example. Example : The following shows the accrual details on a contract with repayments on the 28th of each month with an interest rate change from 10.9% to 11.3% on 13/10/90. Accruals are shown from 28/09/90 to 27/10/90 which represents one complete interest period. The currency of the contract is pounds sterling which has two decimal places. ACCR.PRINCIPAL : 19000 19000.00 19000.00 ACCR.RATE : 11.3% 10.9% 10.9% ACCR.FROM.DATE : 14/10/90 01/10/90 28/09/90 ACCR.TO.DATE : 27/10/90 13/10/90 30/09/90 ACCR.DAYS : 14 13 3 ACCR.AMT : 83.50 74.78 17.26 ACCR.ACT.AMT : 83.4944 74.7861 17.2583 After the ACTUAL.ACCR.AMT has been calculated for the period 01/10 to 13/10 the following calculation is made to obtain the ACCR.AMT for the group: Total un-rounded accrual this period: 17.2583 + 74.7861 --------- 92.0444 Less the rounded amount this period: 92.0444 - 17.26 --------- 74.7844 This amount is then rounded to give the ACCR.AMT for the group of 74.78. This procedure is repeated for the next group: Total un-rounded accrual this period: 17.2583 + 74.7861 + 83.4944 -------------- 175.5388 Less the rounded amount this period: 92.0444 - 17.26 - 74.78 ------------- 83.4988 This amount is then rounded to give the ACCR.AMT for the group of 83.45 The total rounded amount for the period is then: 17.26 + 74.78 + 83.45 ----------- 175.49 This compares with the un-rounded total of 175.5388. Validation Rules: 1 - 19 numeric characters Multivalue field associated with fields ACC. FROM.DATE to ACCR.ACT.AMT.

        public TField getAccrPrin()
        Returns the AccrPrin

        Field description: Identifies the principal amount on which interest was accrued for the multivalue accrual group in question. The set of fields ACCR.FROM.DATE to ACCR.ACT.AMT form the multivalue set of interest accrual fields. These fields record all details of interest accrued on the contract. A new group is created each time one of the accrual parameters, i.e. outstanding principal or interest rate, changes, or a month end has been reached. This field records the outstanding contract principal on which interest was accrued for each of the accrual groups. Information is stored in these fields in reverse chronological order, i.e. the most recent accrual details will always be found in the first multivalue of each field. It should be noted that this field will only contain data relating to the contract after the date held in the RECORD.START.DATE field. Any data prior to this may be found on the MG.BALANCES.HIST file. Example : The following example shows the accrual details on a contract with repayments of 2000.00 per month to be paid on the 28th of each month: ACCR.PRINCIPAL : 17000.00 17000.00 19000.00 ACCR.RATE : 11.3 11.3% 11.3% ACCR.FROM.DATE : 01/11/90 28/10/90 01/10/90 ACCR.TO.DATE : 27/11/90 31/10/90 27/10/90 ACCR.DAYS : 27 3 27 ACCR.AMT : 144.07 16.01 161.03 ACCR.ACT.AMT : 144.0750 16.0083 161.0250 In this case the first accrual group (the right most column) ends on 27/10/90 due to the change in the principal balance on 28/10/90 caused by the repayment. The second group splits at the October month end and the final group ends with the next repayment on 28/11/90. Validation Rules: 1 - 19 numeric characters Multivalue field associated with fields ACCR.FROM.DATE to ACCR.ACT.AMT.

This field records the interest rates which have been applied for each of the accrual groups. Information is stored in these fields in reverse chronological order, i.e. the most recent accrual details will always be found in the first multivalue of each field. It should be noted that these fields will only contain data relating to the contract after the date held in the RECORD.START.DATE field. Any data prior to this may be found on the MG.BALANCES.HIST file. Example : The following example shows the accrual details on a contract with repayments on the 28th of each month with an interest rate change from 10.9% to 11.3% on 15/10/90 ACCR.PRINCIPAL : 19000 19000.00 19000.00 ACCR.RATE : 11.3 10.9 10.5% ACCR.FROM.DATE : 16/10/90 01/10/90 28/09/90 ACCR.TO.DATE : 27/10/90 15/10/90 30/09/90 ACCR.DAYS : 12 15 3 ACCR.AMT : 71.57 86.29 17.26 ACCR.ACT.AMT : 77.5666 86.2917 17.2583 In this case the first accrual split is due to the September month end. The second split arises when the rate changes 15/09/90. The final group ends on 27/10/90 as the principal balance will change the following day due to the repayment on 28/10/90.
