        public TField getCommsnAmt()
        Returns the CommsnAmt

        Field description: Identifies the Amount of Commission to be applied in respect of the Commission Type specified in the previous field. The contents of this field will be placed in the COMMISSION.AMT field in Funds Transfer. If no input is made in this field, the Application will default to the appropriate Commission Amount (as explained within Commission Type). Input to this field will only be allowed when the associated Commission Type has been input and will override the applicable default value. Validation Rules: 1-18 characters plus a decimal point (standard amount format) - type AMT. Associated subvalue field. (Optional input.)

      • getCommsnType

        public TField getCommsnType()
        Returns the CommsnType

        Field description: Identifies the Commission Types which are applicable to the transaction being processed. The contents of this field will be placed in the COMMISSION.TYPE field in Funds Transfer. For more details refer to the Funds Transfer help text, field COMMISSION.TYPE. Validation Rules: 2 character Commission Type code (alphanumeric) - type A. Associated subvalue field. (Optional input. See Details for default.) The Commission Type input in this field must exist in the Commission Type table. When this field is input, the input of an amount will be allowed. If the type is entered with no amount then the values of the code in the Commission file are used in the associated Commission Amount field.

        Returns:
