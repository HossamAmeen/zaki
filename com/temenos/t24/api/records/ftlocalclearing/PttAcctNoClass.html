      • getPttAcctNo

        public TField getPttAcctNo()
        Returns the PttAcctNo

        Field description: This field defines the PTT account number of a branch of the bank, and together with fields PTT.NOSTRO.BI to ACCT.OFF.PTT, defines the PTT clearing accounts for that branch, and the branch (DEPT.ACCT.OFFICER) code. These fields should be used if the bank receives payments directly from the PTT, and these are processed directly into the Funds Transfer application (for example PTT BED transaction tape in Switzerland). Incoming PTT transactions are treated in a similar way to incoming local clearing payments, and will generate either BI (incoming credit) or BD (direct debit) Funds Transfers. Validation Rules: 1-9 numeric characters, leading zeroes allowed. (Optional input)

        Returns:

      • getPttNostroBc

        public TField getPttNostroBc()
        Returns the PttNostroBc

        Field description: This field defines which NOSTRO account is to be used for the BC transaction type for the associated PTT.ACCT.NO. These fields should be used if the bank generates payments directly to the PTT. This field together with PTT.ACCT.NO and ACCT.OFF.PTT, defines the NOSTRO account to be debited, when processing outward payments. A separate account may be specified for each branch (ACCT.OFF.PTT) of the bank, and or each type of transaction for reconciliation purposes, if required. Validation Rules: 2-16 alphanumeric account number or 3-10 alphanumeric account mnemonic. (Optional input)
