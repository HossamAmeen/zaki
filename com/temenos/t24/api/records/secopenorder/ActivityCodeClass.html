      • getActivityCode

        public TField getActivityCode()
        Returns the ActivityCode

        Field description: System maintained field. This field holds the activity number, which is triggered by the latest activity in thelife-cycle of the transaction. The activity is 4 digit number composed of three main parts: First indicates theoperation 1 -Inputting a SEC.OPEN.ORDER Second indicates the type of instrument 1 - New order 2 - Modified order 3 - Cancelled order And the last two digit indicate the detail activity related to the function For example. : For theinput/authorize function, last two digit will be 01. For Reversal function, last two digits will be 02. Thus, theactivity for new order will be SC-1101. Validation Rules Up to 7 Alpha numeric characters

      • getMsgClass

        public TField getMsgClass()
        Returns the MsgClass

        Field description: An integral part of the Delivery multi-value set, this field contains a message class ID, (defined inEB.ADVICES). There may be many classes specified, but only classes, which are related to the operation, will betriggered. For example, the EB.ADVICES number SC-1101 (New order) may have the following classes and messages: Message No. : 502 EB.MESSAGE.CLASS : NEWORDER And let us assume that the EB.MESSAGE.CLASS ID has been mapped to SC internal class specified in SC.PARAMETER asfollows: NEW. MESSAGE In the SEC.OPEN.ORDER, the NEWORDER class will only be triggered for MT502, when BROKER.TYPE is BROKER andCONFIRMATION is YES. You can choose whether you want to produce the messages by flipping the YES/NO flag - SEND.ADVICE

        Parameters:
        Returns:
        TField - The MsgClass

      • getOvrAddress

        public TField getOvrAddress()
        Returns the OvrAddress

        Field description: This field enables the user to divert a particular message type to another (third) party. The user has a numberof options: Simple insertion of a name or address. This will re-route the message in PRINT format to the namedparty. The code TX- followed by a telex number. This will re-route the message by TELEX to the number detailed. Thecode SW- followed by a S.W.I.F.T. code. This will re-route the message by SWIFT to the respective bank. The code C-followed by a valid customer number or mnemonic. This will re-route the message in PRINT format to the customerindicated. Validation Rules 35 Alphanumeric character field.

        Returns:
