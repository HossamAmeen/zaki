        Field description: Holds the charge to be applied for the corresponding CHARGE.TYPE for the order in the currency of the CHARGE.ACCOUNT.       • getChargeAmount

        public TField getChargeAmount()
        Returns the ChargeAmount

        Field description: Holds the charge to be applied for the corresponding CHARGE.TYPE for the order. Validation Rules: User input allowed only when CHARGE.CURRENCY is defined. Both CHARGE.AMOUNT and CHARGE.AC.CCY.AMOUNT cannot be defined together. When left blank, charge based on charge type will be calculated by the system and amount will be updated in terms of the currency defined in CHARGE.CURRENCY field. When this field is left blank and CHARGE.AC.CCY.AMOUNT has value then the amount will be converted into equivalent CHARGE.AMOUNT in terms of currency in the field CHARGE.CURRENCY. If CHARGE.CURRENCY is blank then the amount will be converted into equivalent PAYMENT.CURRENCY.

        Parameters:
        Returns:
        TField - The ChargeAmount

      • getChargeType

        public TField getChargeType()
        Returns the ChargeType

        Field description: Holds the type of the charges to be collected. Validation Rules: Any number of commission/charge types can be defined by the user. When user defines the type, system will not consider the commission/charge type defined in PAYMENT.ORDER.PRODUCT. When left blank system will update the field based on the set up in the field MULTI.FEES on PAYMENT.ORDER.PRODUCT. The combination of user defined and system defined types is not possible.
