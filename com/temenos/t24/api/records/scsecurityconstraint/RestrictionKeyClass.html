      • getSubroutine

        public TField getSubroutine()
        Returns the Subroutine

        Field description: This field is used to store a subroutine that will get called if all the "restriction" and "holdings" test have passed. The format of the subroutine is as follows: SECURITY.NUMBER - Outgoing - The SECURITY.MASTER ID being traded. Example: 000001-000 PORTFOLIO.NO - Outgoing - The customers portfolio number. Example: 100113-1 TRANSACTION.DATE - Outgoing - The transaction date of the trade. Example: 20020214 TRANSACTION.CODE - Outgoing - The transaction code of the trade. Example: BUY ACCOUNT.NO - Outgoing - The customers account number (maybe null). Example: 18901 AMOUNT - Outgoing - The value of the transaction being traded. Example: 10000 CCY - Outgoing - The currency of the AMOUNT being traded. Example: GBP RESTRICTION - Returned - 1: Subroutine test successful, anything else: Subroutine failure Validation Rules: Subroutine must exist in the PGM.FILE.

        Parameters:
        Returns:
        TField - The Subroutine
