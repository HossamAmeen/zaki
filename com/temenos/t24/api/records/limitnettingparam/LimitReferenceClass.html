        public TField getLimitReference()
        Returns the LimitReference

        Field description: Introduced in G14.1. Specifies the FX Limit Reference for which the parameters in the associated fields would apply while processing the Limit Outstanding using the core routine FX.LIMIT.NETTING.I. If the value is ALL, then the parameters in the associated fields would apply to all remaining FX Limit Sub-Product References, not explicitly specified here. Validation Rules: Valid Fx Limit.Reference ID with field FX.OR.TIME.BAND set to FX. Only sub-product Limit References are allowed. Duplicate values are not allowed. Associated with the fields LIMIT.REFERENCE to RATE.USED. Default value is ALL. The value ALL is mandatory.

        Returns:

      • getRateUsed

        public TField getRateUsed()
        Returns the RateUsed

        Field description: Introduced in G14.1. Specifies whether Forward or Spot rates are to be used for calculating the Profit/Loss, while processing the outstanding of a FX Limit using the Core routine FX.LIMIT.NETTING.I. If FWD or SPOT, then Forward or Spot rates respectively would be used for calculating the Profit/Loss of Forward Contracts. Profit/Loss of Spot Contracts would be always calculated using the Spot Rates irrespective of the value in this field. Validation Rules: Mandatory field. Valid values are FWD or SPOT. Default value is FWD.

        Returns:
