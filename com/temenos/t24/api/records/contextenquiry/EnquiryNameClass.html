      • getAutoLaunch

        public TField getAutoLaunch()
        Returns the AutoLaunch

        Field description: Where the record refers to a specific field, that is a field name has been set in the key, this field determines whether the associated enquiry will automatically be launched once input to this field is complete. Where the record refers to a file, the functions that will trigger the automatic launch of the enquiry are defined here.The launch occurs once any of the functions defined are completed. Validation Rules: For field level records, only input of YES or NO is allowed. Records that refer to applications, or applications and versions allow the definition of functions. Valid Function codes are: A 2 C D E H I L P Q R S V If NOT is entered, plus one or more Function codes, the System converts it to a list of all the functions that are allowed. 0-29 type S (uppercase alpha or numeric) characters: 1. Function codes separated by spaces or 2. NOT + one or more Function codes or 3. NO (No trigger produced) or 4. ALL (All Functions produce the trigger). Autolauching of enquiries is possible when the record is put on HOLD by specifying "O" Function on AUTO.LAUNCH.

        Parameters:
        Returns:
        TField - The AutoLaunch
