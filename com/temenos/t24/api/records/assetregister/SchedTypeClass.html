        public TField getAmount()
        Returns the Amount

        Field description: The schedule amount to be paid on the schedule date as specified in DATE.FREQ.. It will default an amount calculated by type of schedule , schedule time remaining , amount already paid and original cost. Validation Rules: For schedules SL and FS, the amount must be less the COST.OF.ASSET - RESIDUAL.VALUE. For schedule RB, the amount must be less than the CURRENT.VALUE - RESIDUAL.VALUE. For schedule CA, the amount must be less than the MAX.CAP.ALLOWANCE - TOTAL.CLAIMED.TD For schedule WO, the amount must be equal to the CURRENT.VALUE.

        public TField getRate()
        Returns the Rate

        Field description: The rate field is used as part of the depreciation amount calculations for certain schedule types only The rate is used as follows : Reducing Balance Schedule = (CURRENT VALUE - RESIDUAL VALUE) * RATE Flexible Schedule = (COST OF ASSET - RESIDUAL VALUE) * RATE Capital Allowance Schedule = (COST OF ASSET - RESIDUAL VALUE) * RATE Validation Rules: This is mandatory for RB , FS and CA schedule types only. Cannot be entered for an SL schedule.

      • getSchedType

        public TField getSchedType()
        Returns the SchedType

        Field description: There are five schedule types that can be chosen. They are as follows:- SL Straight Line RB Reducing Balance FS Flexible Schedules CA Capital Allowances WO Write Off Only schedule types 'SL', 'RB', and 'FS' are treated as schedules for depreciation and are also allowed for depreciation if DEPRECIATION is set to 'YES'. Validation Rules: For schedule types SL, RB, or FS only one schedule type can be specified at any one time, that is to say multiple schedules must be of the same type. For schedule types SL, RB, or FS a frequency code or multiple occurances must be specified. CA schedule type is only allowed if the field Capital Allowance is set to 'YES' If WO schedule is chosen then no frequency or multiple occurance is allowed.

        Parameters:
        Returns:
        TField - The SchedType

        public TField getTransCodeCr()

        Field description: The credit transaction code for this schedule. Validation Rules: It will default from ASSET.CLASS if one is entered. A mandatory field if a schedule is defined. This must be a valid TRANSACTION code with the DEBIT.CREDIT.IND equal to 'CREDIT' .
