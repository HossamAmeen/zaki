      • getRelParty

        public TField getRelParty()
        Returns the RelParty

        Field description:

        This field is used when defining all the relationships within the group in one CUSTOMER.RELATIONSHIP record. The relationship defined here can be between T24 Customers, Persons or Entities. Allowing the definition of relationships with individuals or entities who are not customers of the bank. Forms part of a multi value set of fields from ORIG.PARTY to OWNING.PER. The fields REL.RELATION to OWNING.PER can be sub valued to define multiple relationships to one originating party (ORIG.PARTY)

        Validation rules

        Optional input

        Valid values are:

        CUSTOMER

        PERSON

        ENTITY

        No input if the fields ORIG.RELATION or REL.RELATION are entered.

        Parameters:
        Returns:
        TField - The RelParty

      • getRelPartyId

        public TField getRelPartyId()
        Returns the RelPartyId

        Field description:

        This field is used to define the CUSTOMER or PERSON.ENTITY record of the relating party.

        Forms part of a multi value set of fields from ORIG.PARTY to OWNING.PER

        Validation rules

        Optional input

        Mandatory input if the field REL.PARTY is entered

        If REL.PARTY is "Customer" must be a valid record on T24 Customer application

        If REL.PARTY is "Person " must be a valid record on the PERSON.ENTITY application which has been classified as a PERSON record

        If REL.PARTY is "ENTITY" must be a valid record on the PERSON.ENTITY application which has been classified as an ENTITY record

        Parameters:
        Returns:
        TField - The RelPartyId
