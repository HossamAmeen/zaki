      • addApplication

        public void addApplication(TField param)
        Appends the specified Application to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines which applications may be referenced by the field APPLICATION.ID on the collateral main file. The values entered in this field determine which applications may be used when a collateral record belonging to this type is linked to a contract or account record elsewhere in the system. When a collateral record belonging to this type is entered, the input to APPLICATION.ID must be made in accordance with the definition in this field as follows: AC - collateral belonging to this type may be represented by a credit balance recorded in the Account module; BL - collateral belonging to this type may be represented by a Bill Contract with OPERATION as COLLATERAL recorded in the Bills module; FD - collateral belonging to this type may be represented by a deposit (order) recorded in the Fiduciary module; LCE - collateral belonging to this type may be a Letter of Credit Export order. LCI - collateral belonging to this type may be a Letter of Credit Import order. LD - collateral belonging to this type may be represented by a deposit recorded in the Loans & Deposits module; MD - collateral belonging to this type may be represented by a record of a contingent asset in the Miscellaneous Deals module; MM - collateral belonging to this type may be represented by a placement recorded in the Money Market module; SC - collateral belonging to this type may be represented by a securities portfolio or by a group of sub-assets within a portfolio. More than one of the above values may be entered in combination. Examples: (a) The collateral type is a credit balance on an account recorded in the Account module (AC). (b) The collateral type is either a deposit or a Money Market placement (LD and MM). Validation Rules: 'AC', 'BL', 'FD', 'LCE', 'LCI', 'LD', 'MD', 'MM' or 'SC'. Multi-value field. Mandatory input when APPLICATION.INPUT is 'M' or 'O', otherwise not allowed. This is a NOINPUT field when APPLICATION.INPUT is 'N'. Otherwise input is mandatory. Values may not be duplicated.

        param - the Application to append
        Returns:

        public void addAssetType(java.lang.CharSequence param)
        Appends the specified AssetType to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The asset type to which the collateral type corresponds. Refers to a record on the ASSET.TYPE file, or to a range of keys on this file. The ASSET.TYPE keys defined in this field are used when a range of assets belonging to a portfolio are pledged as collateral. If a portfolio number is quoted on the collateral record, the system will examine the COLLATERAL.TYPE to which the record belongs. If this record refers to a range of asset type and/or sub-asset types, then the COLLATERAL record will automatically take the value of only those parts of the portfolio which belong to one of the types listed. Validation Rules: 1-3 digit numeric asset type code or a range specified by 1 to 3-digits; '-'; 1 to 3-digits. (Optional input) Multi-value field. Input is not allowed when APPLICATION.INPUT is 'N'. Input is only allowed when APPLICATION is 'SC'. If a discreet value is entered, it must be a valid key to a record on the ASSET.TYPE file.
