      • getRole

        public TField getRole()
        Returns the Role

        Field description: A loan application Purpose requires that one or more entities fills-in the application form or simply that information is recorded about one or more entities. All entities filling-in the form (or about whom information is recorded on the form) do not necessarily assume the same role in the application process. Some may play the more prominent role of Main Applicant while some other will be co-applicant , guarrantor etc. The High level group or category of information recorded on the application is referred to in OA as a Domain Class. It can also be understood as information about an entity group. The entity group assumes a distinct ROLE in the application process. The role assumed by the entity group is recorded in the ROLE field and the distinct entities part of this entity group are indicated in the DOMAIN.TYPE field.

        1)Associated Multi-value Field beginning with DOMAIN.CLASS, including ROLE, TYPE, SEQUENCE, REFERENCE, FORM.PURPOSE, FORM and ending with FORM.REFERENCE 2)Non Mandatory Input 3)Must be a valid record ID in the table AA.CUSTOMER.ROLE

        Parameters:
        Returns:
        TField - The Role

      • getSequence

        public TField getSequence()
        Returns the Sequence

        Field description: When a combination of same domain class, domain type and role is allowed to appear more than once on the same application , the sequence field indicates a count of the occurrence of this combination For example if two guarantors are allowed we might have DOMAIN.CLASS as PARTY for both, DOMAIN.TYPE as INDIVIDUAL for both, ROLE as GUARRANTOR for both but the sequence will be 1 for the first occurrence and 2 for the second Hence the sequence field may allow to apply different rules to a specific occurrence of the combination. For example, only to second guarantor Non Input field and system maintained

        Parameters:
        Returns:
        TField - The Sequence

      • getType

        public TField getType()
        Returns the Type

        Field description: This Field indicates The Domain Type. A Sub-Grouping of a Domain Class and represent a particular type of information included under the broad category that is the Domain Class. The FORM to be used for recording this particular type of information for the current purpose is therefore directly linked to the DOMAIN.TYPE in the associated FORM field

        1)Associated Multi-value Field beginning with DOMAIN.CLASS, including ROLE, TYPE, SEQUENCE, REFERENCE, FORM.PURPOSE, FORM and ending with FORM.REFERENCE 2)Mandatory Input 3)Input in this field should be a valid value in the table OA.DOMAIN.TYPE

        Parameters:
        Returns:
        TField - The Type

      • getUniqueIdentifier

        public TField getUniqueIdentifier()
        Returns the UniqueIdentifier

        Field description: This field indicates the unique references that will be used to distinguish between the forms used for Applicant1 Vs Form used for Applicant2 and so on. It is similar kind of field like Sequence. The sequence would be different for Applicant1 and Applicant2. But unique reference should unique across the domain classes. This field is introduced basically to use it for API message creation. If user has not inputted the value, then system will default it as DomainType-Role-Sequence.

        1. Non mandatory field 2. Duplicate values are not allowed.

        Parameters:
        Returns:
        TField - The UniqueIdentifier
