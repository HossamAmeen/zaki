        public TField getCloseOnline()
        Returns the CloseOnline

        Field description: Field introduced in G13.2 When Online Closure of Account is required, then this field is to be set to Y. If this field is set to Y then the following actions will take place at the time of committing the record: 1. If CLOSE.MODE is FT, the system creates the FT in HOLD and the user has to autorise the FT and made the account balance as zero and then move the account to history by authorising ACCOUNT CLOSURE manually. 2. If CLOSED.MODE is TELLER the system creates the TELLER.DEFAULT and the user has to make the a/c bal zero through TELLER and then move the account to history by autorising the ACCOUNT CLOSURE manually. If the record is subsequently deleted, then the earlier updation in Account application will be restored back to NULL. If the SETTLEMENT.ACCOUNT is supplied, then, at the time of authorization of ACCOUNT.CLOSURE Record, then the final settlement will take place between the account that is to be closed and the SETTLEMENT.ACCOUNT. If the SETTLEMENT.ACCOUNT is not supplied, then a TELLER.DEFAULT record will be created and with this TELLER has to be created by USER. In both the cases above, the interest and charges will be posted to PL online during authorization of ACCOUNT.CLOSURE. 3. If the CLOSE.MODE is Payment, system creates a Payment Order record. If the closure is online, the PO record is created with IHLD status- otherwise, PO record is generated during the COB. Both Account transfer type and Beneficiary Transfer Type Payment orders are allowed. If PAYMENT option is selected as close mode and if the PI module is not installed in the system, a FT is created instead of a PO transaction Validation Rules: Valid Values are Y, N or NULL Default is NULL
