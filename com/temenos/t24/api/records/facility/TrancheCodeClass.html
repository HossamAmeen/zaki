      • getTrancheCode

        public TField getTrancheCode()
        Returns the TrancheCode

        Field description: Unique 3 digit numeric code to identify the tranche reference within the facility. In case disbursement is phased out, the bank may define different tranches, each with distinct validity, amount and purpose. This field is used to store the reference number of tranche and when loans are drawn, the reference in this field is quoted in order to identify the tranche against which the drawing is made. The reference number of a tranche within a presyndication reference (multiple facility records within the same presyn record) has to be unique. Ex - SL0025000012 is the ID of the presyn record that has 3 facility records namely SL0025000012-01- Revolving USD facility (only one tranche) SL0025000012-02 - Non revolving GBP facility with two tranches SL0025000012-03.- Non revolving GBP facility with three tranches The tranche code of all tranches within SL0025000012 have to be unique. Hence, the tranche code could be defined as follows; Facility Tranche codes Alternately * SL0025000012-01 001 100 SL0025000012-02 002, 003 200, 300 SL0025000012-01 004, 005 400, 500 * Such numbering can also be followed if found convenient. In the event that no details are input in this field or if a tranche is created by default, the reference number commencing from 001 are generated sequentially. Validation Rules: 3 digit numeric Tranche.code within a presyn reference has to be unique. Once input, this field cannot be modified. For revolving facility only one tranche is allowed.

        Parameters:
        Returns:
        TField - The TrancheCode

      • getTrancheEndDt

        public TField getTrancheEndDt()
        Returns the TrancheEndDt

        Field description: Date when the tranche validity expires. On the date mentioned in this field, the tranche validity expires and the undrawn balance in the tranche is carried forward based on value in field Tranche.C.Fwd No fresh loans would be permitted after the expiry of this date under the tranche. Also while handling rollovers, principal increases are prohibited after this date. For a revolving facility, the date in this field should be equal to Draw.Mat.Date. If no tranche is defined, a tranche is created by default with Tranche.End.Dt equal to the Draw.Mat.Date For a non revolving facility, atleast one tranche should have an end date equal to the Draw.Mat.Date Validation Rules: 1-9 type A alphanumeric characters Must be less than or equal to Draw.Mat.Date. Must be greater than Tranche.St.Dt For a non revolving facility, atleast one tranche should have an end date equal to the Draw.Mat.Date

        Set the specified value for TrancheCode
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Unique 3 digit numeric code to identify the tranche reference within the facility. In case disbursement is phased out, the bank may define different tranches, each with distinct validity, amount and purpose. This field is used to store the reference number of tranche and when loans are drawn, the reference in this field is quoted in order to identify the tranche against which the drawing is made. The reference number of a tranche within a presyndication reference (multiple facility records within the same presyn record) has to be unique. Ex - SL0025000012 is the ID of the presyn record that has 3 facility records namely SL0025000012-01- Revolving USD facility (only one tranche) SL0025000012-02 - Non revolving GBP facility with two tranches SL0025000012-03.- Non revolving GBP facility with three tranches The tranche code of all tranches within SL0025000012 have to be unique. Hence, the tranche code could be defined as follows; Facility Tranche codes Alternately * SL0025000012-01 001 100 SL0025000012-02 002, 003 200, 300 SL0025000012-01 004, 005 400, 500 * Such numbering can also be followed if found convenient. In the event that no details are input in this field or if a tranche is created by default, the reference number commencing from 001 are generated sequentially. Validation Rules: 3 digit numeric Tranche.code within a presyn reference has to be unique. Once input, this field cannot be modified. For revolving facility only one tranche is allowed.
