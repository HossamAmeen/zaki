      • getAsDateFreq

        public TField getAsDateFreq()
        Returns the AsDateFreq

        Field description: Specifies the date and frequency of the schedule. The date of the schedule is defined along with the frequency of future process dates of the schedule. Schedules can be processed on any day of the month and at a frequency of either daily, monthly, quarterly, half-yearly, or yearly. When a schedule is processed : o If the schedule has no frequency then the schedule is removed from the swap contract. o If the schedule has a frequency then the next process date is updated according to the frequency. Eg. Schedule processed on 01 Jan 95 frequency M0101, then the date.freq field is updated with 01 Feb 95 frequency M0101. The swap balance file, "SWAP.BALANCES", is updated accordingly. Validation Rules: 16 Character standard date/frequency code format (type A FREQ). (Mandatory input if the associated schedule type has been defined). A frequency is disallowed for "PX" and "RX" schedules. For all schedules, the date must be GE the value date of the contract and LE the maturity date of the contract. The date must be equal to the Asset interest effective date for schedule type "PX". The date must be greater than the asset interest effective date and today for schedule type "IP" (Interest payment). If schedule type is IP or AP and Day convention is Following or Modified then daily and business day freq is not allowed.

        Parameters:
        Returns:
        TField - The AsDateFreq

        Field description: Specifies the Interest Settlement Date for the IP schedule for the Asset Leg. The date specified will be considered as the Processing date for the Interest Payment.       • getAsType

        public TField getAsType()
        Returns the AsType

        Field description: Defines the type of the Asset Swap schedule. Can be any valid type as defined on the SWAP.SCHEDULE.TYPE file. The Swap Schedule Type file defines all of the schedules which are permitted on a Swap contract. Details such as the transaction codes to be used on entries, P&L category codes, Charge codes etc., are also defined on the Swap Schedule Type table. The first two characters of the schedule type ID determine the type of schedule. When a schedule is processed the Swap balances file (SWAP.BALANCES) is updated accordingly. Validation Rules: 1-6 Alpha-numeric characters. Must be a valid definition of the SWAP.SCHEDULE.TYPE file. Different schedules can be added but the first two characters must be one of the specified fixed schedule types. The schedules AC, AM, AY, CI, and CM, which are system generated schedules, cannot be input by the user. Only one "PX"/"IS" schedule and one principal re-exchange "RX" schedule can be specified on the Asset leg of the contract. Input of an "RX" scheduled is disallowed if there is no "PX"/"IS" schedule defined (except on flexible currency swaps) On deals with the SWAP.TYPE FLEX.PRIN.PAYMENT

        Returns:
        TField - The AsType

      • setAsType

        public void setAsType(TField value)
        Set the specified value for AsType
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the type of the Asset Swap schedule. Can be any valid type as defined on the SWAP.SCHEDULE.TYPE file. The Swap Schedule Type file defines all of the schedules which are permitted on a Swap contract. Details such as the transaction codes to be used on entries, P&L category codes, Charge codes etc., are also defined on the Swap Schedule Type table. The first two characters of the schedule type ID determine the type of schedule. When a schedule is processed the Swap balances file (SWAP.BALANCES) is updated accordingly. Validation Rules: 1-6 Alpha-numeric characters. Must be a valid definition of the SWAP.SCHEDULE.TYPE file. Different schedules can be added but the first two characters must be one of the specified fixed schedule types. The schedules AC, AM, AY, CI, and CM, which are system generated schedules, cannot be input by the user. Only one "PX"/"IS" schedule and one principal re-exchange "RX" schedule can be specified on the Asset leg of the contract. Input of an "RX" scheduled is disallowed if there is no "PX"/"IS" schedule defined (except on flexible currency swaps) On deals with the SWAP.TYPE FLEX.PRIN.PAYMENT
