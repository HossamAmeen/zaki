      • addActivityType

        public void addActivityType(java.lang.CharSequence param)
        Appends the specified ActivityType to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field indicates how the activity would be triggered and indicate the characteristic of this activity. More than one type indicates the activity would share the characteristic of all types specified.They are classified into the following types:

        ACCOUNTING - The activity deals with accounting entries and may require balances and allocation rules for them to perform this activity correctly. ACCRUAL - The activity deals with accrual of interest. Activity of this type do not generate AA.ARRANGEMENT.ACTIVITY id everytime but still perform their functionality. They have specific AAA id and use the same everytime this activity is run

        NORR - The activity will not trigger reversal / replay. The activity would not be reversed / replayed by other activities but can be forcibly reversed by the user. NOREPLAY - The activity would not be reversed / replayed by any other Activity. NOREVERSE - The activity cannot be manually reversed by the user. SCHEDULED - The activity can be executed as part of a scheduled COB job.(like MAKEDUE) TRANSACTION - The activity is triggered by external transactions(like DISBURSE) USER - The activity can be triggered manually by the user. SECONDARY - The activity can be triggered as a SECONDARY activity. SOD-PROCESS - The activity, when processed as a Scheduled activity could be run from the SOD process as well. If the option is not stated the activity would be executed only as part of COB. NOLOG - Whenever activities are performed on an arrangement, they are recorded onto a file called AA.ACTIVITY.HISTORY. An option of NOLOG indicates that those activities need not be recorded into Activity History. PRE.NOTICE - This activity can be generate pre notice advice BUNDLE.ADJUST - Indicates, if a backdated activity belonging to this type is processed on a donor arrangement, system would check for bundle linkages(notional pooling) and would trigger backdated events on Recipients as well to catch up with accruals lost or gained from this backdated event on donor. CASHFLOW - This activity will update the cash flow information for the contract. POSITION.MANAGEMENT - This activity will update the position management details for the contract. DIRECT.ACCOUNTING - It a mirror activity of the transaction and does not raise any accounting entries in AA. DDA - Transaction activity used for DDA (Demand Deposit Accounts) accounts to Debit\Credit the funds. SIMULATION - This activity will be used primarily for simulation. CLOSURE - Indicates closure related activities which will move the arrangement to closure state. ONLINE-SERVICE - This activity will be triggered from the online service AA.ONLINE.SERVICE for online accrual and capitalisation process. RATE.CHANGE - Indicates rate change activity and will be triggered during Basic rate change or periodic rate change events. INCLUDE.USER.CLASS - Whenever a new user property class is added to the Temenos released product line, system will enable this type of activity classes to access those property classes by rebuilding the Activity class records to include those classes. PRE.LIVE-Indicates this activity is allowed during preliminary stage during orchestration process. AUTO.SETTLEMENT-Indicates whether the current activity is a settlement activity(I.e. Settle-Settlement). RESTRICT.LOG - This activity updates only the context details in AA.ACTIVITY.HISTORY and is currently applicable only for ACCOUNTS credit and debit activities

        Validation rules:

        Valid inputs are ACCOUNTING, ACCRUAL, NORR, NOREPLAY, NOREVERSE, SCHEDULED, TRANSACTION, USER, PRE.NOTICE, SECONDARY, SOD-PROCESS, NOLOG, SIMULATION, CASHFLOW, CLOSURE, RATE.CHANGE, DIRECT.ACCOUNTING, DDA, BUNDLE.ADJUST, ONLINE-SERVICE, POSITION.MANAGEMENT,AUTO.SETTLEMENT Can be multi-valued and more than one type may be stated. Duplicates not allowed. Maximum of 12 characters

        Parameters:
        param - the ActivityType to append
        Returns:
        nothing

        public void setRelatedActClass(java.lang.CharSequence value)
        Set the specified value for RelatedActClass
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field is used ONLY whilst simulating an Arrangement scenario and is used only for LENDING and DEPOSIT product lines at the moment.

        For example, when a NEW arrangement is to be simulated for 2 schedules, the schedules could be run only after the arrangement could be disbursed(either fully or partially). But, disbursement is always transaction driven and may not be captured using a AA.SIMULATION.CAPTURE application.
