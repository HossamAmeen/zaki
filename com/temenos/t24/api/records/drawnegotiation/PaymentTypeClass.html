        public TField getAccptPeriod()
        Returns the AccptPeriod

        Field description: Acceptance period mentioned in LC to which the DRAWING should be adjusted. Applicable for mixed payment LCs. When mixed payment LCs are input, permissable acceptance or deferred periods are set up and portions of the total amount allocated. The type and period indicate from which portion the amount should be drawn. Where the drawing type is SP then this field defaults to the maturity date in the form yyyymmdd.         PAY.PORTION 25000
        ACPT.TM.BAND 30
        DRAW.TYPE AC
        PAYMENT.PCT 25
        PAY.PORTION 25000
        ACPT.TM.BAND 60
        Let us assume that a particular drawing is to be drawn from the third portion. Then the ACCPT.PERIOD should be input as 60. For unmixed payment LCs this will be defaulted to expiry date of the LC Validation Rules: Mandatory Input for mixed payment.

      • getDrawdownAcct

        public TField getDrawdownAcct()
        Returns the DrawdownAcct

        Field description: Identifies the account where funds are to be received from under this drawing. The account which will be debited on either the value date or maturity review date, depending on drawing type, in reimbursement of funds paid to the beneficiary of the drawing. If this field is not entered, then a default will be found from the parent L/C contract. For Import/Opening L/C's, this default will come from the APPLICANT.ACC field of the L/C. For Export/Advising L/C's, the default will come from the ISSUING.BANK.ACC field of the L/C. If these fields were not entered, and this drawing involves an actual payment, (i.e. drawing types 'AC', 'D',or 'SP'), then a mandatory input is required. For drawing types 'AC', 'DP', this account will be debited on the Maturity Review date in respect of reimbursement. For drawing type 'SP', this account will be debited on the Value date in respect of reimbursement. The currency of this account need not be the same as the currency of the drawing. If this is the case, the system will automatically calculate the amount to be debited (see RATE.BOOKED above). Validation Rules: 1-16 Mnemonic characters relating to a Customer Account or a Nostro Account, or any other account. Default from L/C (see below). Mandatory input depending upon type. Mandatory input for drawing types 'AC', 'DP', or 'SP'.

        Parameters:
        Returns:
        TField - The DrawdownAcct

      • getPaymentAcct

        public TField getPaymentAcct()
        Returns the PaymentAcct

        Field description: Identifies the account where funds are to be paid to under this drawing. The account which will be credited on either the value date or maturity review date, depending on drawing type, for the payment in respect of the drawing. If the payment method entered above is either 'BP' (Bankers payment) or 'CH' (Cheque), then this field will be automatically updated by the system to indicate the account to be used. This will be set up on the LC.PARAMETERS file as 'B.P.ISSUED.ACC' and 'CHEQUES.ISSUE.ACC'. For drawing types 'AC', 'DP', this account will be credited on the Maturity Review date in respect of payment, unless the drawing is being discounted, in which case it will be credited on the value date. For drawing type 'SP', this account will be credited on the Value date in respect of payment. For drawing type 'CO' with 'UNDER.RESERVE' set to 'YES', then this account will be credited on the value date. Validation Rules: 1-16 Mnemonic characters relating to a Customer Account or a Nostro Account, or any other account. Default from Payment Method (see below). Mandatory input for Drawing types 'AC', 'DP', 'SP', and 'CO' with UNDER.RESERVE set to 'YES'.

        Parameters:
        Returns:
        TField - The PaymentAcct

      • getPaymentType

        public TField getPaymentType()
        Returns the PaymentType

        Field description: Code specifying what type of drawing this is. This code identifies which kind of Drawing is being made under the L/C, and will impact the kind of processing done. The following types can be entered for a new drawing: AC - Acceptance (Usance credits) DP - Deferred Payment (Usance credits) SP - Sight Payment The following types can only be entered as a change to an existing drawing: For Drawing types of 'AC' and 'DP', a term or usance credit is established whereby the funds will be debited and paid on the MATURITY.REVIEW date specified later. If the funds are required to be paid before this date, the Drawing can be Discounted, whereby payment is made on the Value date specified, but reimbursement is obtained on the MATURITY.REVIEW date. If 'SP' is entered, then both Payment and reimbursement is made on the VALUE.DATE specified. Validation Rules: 2-2 type AAA (Alpha) characters. Valid codes are: 'AC'; 'DP'; 'SP'. (Mandatory Input)

      • getRateBooked

        public TField getRateBooked()
        Returns the RateBooked

        Field description: This is an exchange rate field. This is the rate quoted by the treasury department to the trade finance department (ie excluding department spread). The department spread will be accepted in RATE.SPREAD field. The exchange rate including the rate spread will be customer debit rate and will be shown in DEBIT.CUST.RATE field. The DEBIT.CUST.RATE will be applied between the document currency and the debit account currrency (DRAW.CURRENCY and DRAWDOWN.ACCOUNT.CURRENCY) if they differ. Default calculated from currency table, using current mid market spot rate. Validation Rules: Standard rate format (Only up to 6 integer and decimal characters). (Optional input) Input to this field is only allowed if the drawdown account is in a different currency to the draw currency. Mainly used in import transactions, when the importer's debit account is in a different currency to the document currency.

        public TField getTreasuryRate()
        Returns the TreasuryRate

        Field description: This is an exchange rate between the document currency and the credit account currency. This is the rate quoted by the treasury department to the trade finance department excluding departmental spread. The department spread will be accepted in CUSTOMER.SPREAD field. The exchange rate including the rate spread will be customer credit rate and will be shown in CREDIT.CUST.RATE field. The CREDIT.CUST.RATE will be applied between the document currency and the credit account currency (DRAW.CURRENCY and PAYMENT.ACCOUNT.CURRENCY) if they differ. Input to this field is only allowed if the payment account is in a different currency to the draw currency. Mainly used in export transactions, when the exporter's credit account is in a different currency to the document currency.
