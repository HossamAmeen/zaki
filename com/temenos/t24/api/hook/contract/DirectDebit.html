        public String setClearingReference(String mandateReference, String transactionReference, DirectDebitContext directDebitContext, TString errorReason)
        This Interface enables the implementer to set the clearing reference for the direct debit.
        This interface is invoked during the payments happening in COB or online service.

        T24 details: Interface method for the MANDTE.REF.CHK.RTN field in DD.PARAMETER in T24. The EB.API hook used by this interface is HOOK.DD.PARAM.MANDTE.REF.RTN. If an exception is thrown in the implementing class then the exception message will be logged in the EXCEPTION.LOG file and displayed on screen if executed online.
        Parameters:
        mandateReference - The mandateReference from the Payments DebitAuthoritySerivce interface.
        transactionReference - The transactionReference from the Payments DebitAuthoritySerivce interface.
        directDebitContext - Direct Debit related variables.
        errorReason - Optional: Provide a reason for the transaction failure, this will be returned to the Payments DebitAuthoritySerivce interface.
        Returns:
        String - a clearing reference value will be updated in the CLEAR.SYS.REF field of the DD.DDI table.

      • setMandateId

        public String setMandateId(Mandate ddData, DirectDebitContext directDebitContext, String currentSequenceReference, DdDdiRecord ddDdiRecord, TBoolean isUpdate)
        This Interface enables the implementer to set the mandateId, modify the DD.DDI record and update the table DD.DDI.LIST.
        The currentSequencialReference is stored in the DD.DDI.LIST table and will be incremented by 1. This interface is invoked during the payments happening in COB or online service.

        T24 details: Interface method for the UPDATE.MANDATE.RTN field in DD.PARAMETER in T24. The EB.API hook used by this interface is HOOK.DD.PARAM.UPDATE.MANDTE.RTN. If an exception is thrown in the implementing class then the exception message will be logged in the EXCEPTION.LOG file and displayed on screen if executed online.
        Parameters:
        ddData - Mandate data taken from the AC.ENTRY.PARAM.
        directDebitContext - Direct Debit related variables.
        currentSequenceReference - The sequential DD reference number for the parametersied account number i.e the record of t24 table DD.DDI.LIST.
        ddDdiRecord - The record of t24 table DD.DDI.
        isUpdate - Set to true if the write on DD.DDI file is required.
        Returns:
        String - a mandateId for the DD.DDI table. If isUpdate is set to true, then the ddDdiRecord will be written to the DD.DDI table with the returned mandateId.
