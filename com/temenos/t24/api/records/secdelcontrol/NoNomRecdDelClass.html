        public TField getNoNomDate()
        Returns the NoNomDate

        Field description: The date delivery has taken place. A multi-value to cater for partial deliveries and receipts. Associated with fields NO.NOM.RECEIVED.DEL and NO.NOM.REF.NO Although not a mandatory field, it should be used to record the date on which actual delivery/receipt has been effected. Historical data is retained for the SEC.DEL.CONTROL file and therefore completing this information can provide you with such information as late deliveries when this file is interrogated using a suitable enquiry. Validation Rules: Standard date format. Optional input. (Multi value field.)

        public TField getNoNomRecdDel()
        Returns the NoNomRecdDel

        Field description: Specifies the nominal number of the securities to be received/delivered. Multivalue field to cater for part deliveries and part payment of transactions. Occasionally owing to market conditions it will not be possible for a counterparty to deliver all of an outstanding Stock position. It is Standard practice sometimes in the UK. For example for a Broker upon confirmation from his Client to delivery part of the Stock against a Nominal Consideration. The Delivery Control Record will not be removed until the nominals and amounts to be delivered and paid agree. Validation Rules: 1-14 characters, standard amount format. Optional input.
