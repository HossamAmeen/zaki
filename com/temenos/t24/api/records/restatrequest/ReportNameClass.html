      • addOutputMode

        public void addOutputMode(TField param)
        Appends the specified OutputMode to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The type or format of output required. This field is used to request either the standard printed output or an alternative disk file output format. The word 'PRINT' indicates that the report is to be printed. Any other value in this field indicates that the report output is to be re-directed to a sequential disk file. In this case, the value must identify a record on the CRF report extract file (RE STAT OUTPUT), where the layout and content of the output file is defined. Both printed output and extraction to disk may be specified for the same report. However, multiple extract file formats cannot be requested. The default value is 'PRINT'. Validation Rules: Up to 12 type 'A' (alphanumeric characters). Mandatory Input Multivalue Associated with fields 4 & 5. A maximum of two sub-values are allowed. Duplicate sub-values are not permitted. If the value is not 'PRINT', it must identify a record on the CRF extract format file (RE STAT OUTPUT). Only one extract format may be specified per report.

        Parameters:
        param - the OutputMode to append
        Returns:
        nothing
