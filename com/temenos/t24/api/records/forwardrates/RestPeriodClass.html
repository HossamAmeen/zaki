      • getRestDate

        public TField getRestDate()
        Returns the RestDate

        Field description: Specifies the date of the rest period which is calculated by the system according to the value defined in REST.PERIOD. To calculate the date of the rest period for negative rest period, system will subtract the value defined in REST.PERIOD from the spot date and arrive at the respective working days that fall between Today and Spot date. To calculate the date of the rest period for positive rest period, system will add to the SPOT date the value defined in the field REST.PERIOD. It is important to note that if the Rest period is defined in months and the date which is calculated happens to be a non-working day, the system will automatically move forward to the next working day after this calculated date. If, however, this next working day falls in the next month, the system will instead move BACKWARD to the previous working day. This date calculation is generally based on holiday setups of local country and currency country. However if we need the system to calculate the date with some other country holiday setup instead of local country then it can be done through ST.RATE.PARAMETER table. Validation Rules:-

      • getRestPeriod

        public TField getRestPeriod()
        Returns the RestPeriod

        Field description: Indicates the period to which a rate applies. The period can be defined for days before spot date or after spot date based on the parameterization of REVALUATION.PARAMETER. By default, accepts only the period after spot. The period before spot i.e. ON and TOM can be defined only when the field REVAL.WITHIN.SP of REVALUATION.PARAMETER is set to "YES". The period for days before spot date is prefixed with a negative sign representing ON(-2D) and TOM (-1D) and can be defined only in Days (D). The definition of negative rest period must always start with Today and ends with a day before spot which represents Tomorrow. For example: When the spot market is defined as 2 days, the definition of negative rest dates will be -2D for Today and -1D for Tom. Using the Premium/discount defined for -2D and -1D along with Spot rate, system calculates Today's Rate and Tomorrow's Rate. For deriving Today's Rate the Premium/Discount defined for both -2D and -1D will be summated and added/subtracted from the Spot rate. For Tomorrow's Rate, the system will consider -1D. No interpolation is done while deriving today or Tom rates. Validation rules:-

                    A = M for months; D for days; R for rest.              For example, a 12 month rest period would be written as 12M. R must be the last rest period entered.

        The number of days or months must be entered in logical order i.e. in ascending sequence.

        Days until 31 will be allowed (if both days and months are defined) while for months 01-360 will be accepted. For years the range 01-30 is accepted.

        Days greater than 31 will be allowed when the interest is defined only in terms of days(D) i.e., 01D, 03D, 045D...9999D. In this case days range is from 01 to 9999.

        When 'R' is defined, at least one additional rest period must be defined.

        When more than one rest period is defined, 'R' must be entered on the last entered rest period.

        Parameters:
        Returns:
        TField - The RestPeriod
