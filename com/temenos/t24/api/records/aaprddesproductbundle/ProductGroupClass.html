        public void addProduct(java.lang.CharSequence param)
        Appends the specified Product to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Bundle Definition is enhanced to allow specification of participating arrangements at their Product level. Ex, if a Notional Pooling allows only a certain type of "Checking Account" to be participants, then an existing Normal Current Account might need to be changed to "Checking Account" product before it can be added to a Bundle In the example above, both "Checking Account" and "Normal Current Account" will be part of the ‘Current Accounts’ Product Group. However, in the Bundle condition, instead of specifying this product group, we would instead, specify ‘Checking Account’ in a new field that can accept a valid AA Product. This new field will be mutually exlcusive to the existing Product Group field. Either the Product Group or a Product should be required Allow multiple products to be added for a product group and arrangements to it At arrangement level, validate the participants against these products specified
