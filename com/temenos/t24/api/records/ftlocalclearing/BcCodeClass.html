        public TField getAcctOfficer()
        Returns the AcctOfficer

        Field description: Specifies the account/department code for the associated BC.CODE. This field allows each branch of the bank (BC.CODE) to be associated with an account officer or department. When processing inward payments the appropriate account and acct officer will be used for the BC CODE when generating the funds transfer. Validation Rules: 1-4 numeric account officer code. (Mandatory input) Must exist on the DEPT.ACCT.OFFICER file.

      • getBcCode

        public TField getBcCode()
        Returns the BcCode

        Field description: Defines the BC BANK SORT CODE belonging to the bank. Together with fields BRNCH.NOSTRO.BI, BRNCH.NOSTRO.BD, BRNCH.NOSTRO.BC, and ACCT.OFFICER, this field allows the user to define which Nostro account and account officer correspond to the given sort code and transcation type. All sort codes belonging to the bank should be specified in this field. Validation Rules: 6 numeric characters. (Mandatory input) The code must exist on the BC.SORT.CODE file. No duplicates allowed.

      • getBrnchNostroBc

        public TField getBrnchNostroBc()
        Returns the BrnchNostroBc

        Field description: Defines which NOSTRO account is to be used for the BC transaction type for the associated BC.CODE. This field, together with BC.CODE and ACCT.OFFICER, defines the NOSTRO account to be credited, when processing outward payments. A separate account may be specified for each branch (BC.CODE) of the bank, and or each type of transaction for reconciliation purposes, if required. Validation Rules: 2-16 alphanumeric account number or 3-10 alphanumeric account mnemonic (Mandatory input) Must be an existing ACCOUNT record, with LIMIT.REF of NOSTRO.
