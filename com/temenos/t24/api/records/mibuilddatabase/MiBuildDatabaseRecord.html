      • getRebuildFrDate

        public TField getRebuildFrDate()
        Returns the RebuildFrDate

        Field description: Start of period from which databases will be built. MI databases will normally be built incrementally. This means that each time the database build is run - either on line or in EOD - data will be added from the last time the database was built to bring the database up to date. If the database is built each month, a month's data will be added each time. To build from the last time the database was run, this field should be left blank. You may want to build a database for a specific period. For instance, for a new database, you may only want to see data from the beginning of the year. Or you may need to rebuild a part of a database which has already been built - if you change the definition of the database, or if you have new P&L allocation or backvalued balance information to include. In this case, you should put the start date of the period you want to build in this field. Validation Rules: Must be the start of an MI period for the databases being built.

        Parameters:
        Returns:
        TField - The RebuildFrDate

      • getRebuildPlAlloc

        public TField getRebuildPlAlloc()
        Returns the RebuildPlAlloc

        Field description: Date to rebuild P&L Allocation from. P&L Allocation can be run before building the database. A date input in this field specifies when you wish to build P&L allocation from. If this date is left blank, P&L allocation will be run from the last time it was built. If the date in this field is before the last P&L allocation date, P&L allocation will be rerun for the specified period. You would use this if you changed the criteria for P&L allocation, and wished the new allocation of P&L to be picked up by databases already built. Validation Rules: Must be a valid date Optional input Cannot be later than today's date

        public void setRebuildFrDate(TField value)
        Set the specified value for RebuildFrDate
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Start of period from which databases will be built. MI databases will normally be built incrementally. This means that each time the database build is run - either on line or in EOD - data will be added from the last time the database was built to bring the database up to date. If the database is built each month, a month's data will be added each time. To build from the last time the database was run, this field should be left blank. You may want to build a database for a specific period. For instance, for a new database, you may only want to see data from the beginning of the year. Or you may need to rebuild a part of a database which has already been built - if you change the definition of the database, or if you have new P&L allocation or backvalued balance information to include. In this case, you should put the start date of the period you want to build in this field. Validation Rules: Must be the start of an MI period for the databases being built.
