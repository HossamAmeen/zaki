        public void addFormat(java.lang.CharSequence param)
        Appends the specified Format to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The format mask to be used when printing FIELD. Any valid Universe info basic FMT mask. The more common values used are: 10L 10 characters left justified 20R 20 characters right justified 11R2, With two decimal places & commas D2/E European date eg 30/05/91 There are additional formats which can be used: WORDS Convert an amount into words WORDSCCY*xxxx Convert amount into words and translate currency as specified in field xxxxxx 11R/nnn/, Replaces /nnn/ with number of decimals defined for the currency held in field nnn of this FIELD's file. 24C Centre data in length of 24 D2.E+nn Add nn years to date and print 30.05.95 You can also accumulate the data in a total - to be printed later: 10R, !TOTAL.1 10 chars right justified with commas and accumulate amount into total number 1. The format is defaulted automatically for you from the STANDARD.SELECTION file. A User written Subroutine can be called to do the Formatting for this field. This Subroutine can be defined in this field with the name preceded by the '@' character. The Subroutine should have ONE argument which will contain the value of the Data or Text to be printed and can be changed by the User written Subroutine. Any errors can be flagged by setting the common variable ETEXT. This Subroutine must be defined on the PGM.FILE with a PGM.TYPE of 'S' i.e. Subroutine HEADER: Any data or text can be specified to be printed as part of the Heading while printing the Deal Slip by defining HEADER in this field. If a field is defined as a HEADER type field, the COLUMN for the associated Multivalue MUST contain a Line Number i.e. in the format CC,LL and not as CC or CC,+LL. Validation Rules: 0-40 any characters Subvalued to allow more than one Format specification for this data item. The Format commands will be executed in the order in which they are defined in this field. A subroutine exist on PGM.FILE as a type 'S' or an EB.API record of type METHOD which implements an interface defined in the EB.API record DEAL.SLIP.FORMAT.HOOK. Input should prefix with '@'

        Parameters:
        param - the Format to append
        Returns:
        nothing

      • getField

        public TField getField()
        Returns the Field

        Field description: The name of the field to be printed. The name of the field (and from which file) or free format text can be entered. The format of the field specification is: 1>4 File 1, Field 4 5 File 1, Field 5 ACCOUNT>8 Account file field 8 ACCOUNT>CURRENCY Account file currency field "text" Constant eq "SYSTEM" !TOTAL.1 Total number 1 to be printed "SYSTEM.DATE" The current computer date Where file number is the multivalue number of the FILE definition. If file or field numbers are entered they are immediately translated to their correct names. If the file number is 1, the default, then just the field name is returned. A total can be printed instead of a field - see FORMAT for defining totals. When entering a field from the application then you can specify that the enrichment is printed rather than the field itself. This is entered as COUNTERPARTY ENRICHMENT or 4 E. In this case the 4 E will be immediately translated to COUNTERPARTY ENRICHMENT. Validation Rules: 1-40 any characters. FILE> FIELD, FIELD or "text" or !TOTAL.nn

        Returns:

The format is defaulted automatically for you from the STANDARD.SELECTION file. A User written Subroutine can be called to do the Formatting for this field. This Subroutine can be defined in this field with the name preceded by the '@' character. The Subroutine should have ONE argument which will contain the value of the Data or Text to be printed and can be changed by the User written Subroutine. Any errors can be flagged by setting the common variable ETEXT. This Subroutine must be defined on the PGM.FILE with a PGM.TYPE of 'S' i.e. Subroutine HEADER: Any data or text can be specified to be printed as part of the Heading while printing the Deal Slip by defining HEADER in this field. If a field is defined as a HEADER type field, the COLUMN for the associated Multivalue MUST contain a Line Number i.e. in the format CC,LL and not as CC or CC,+LL. Validation Rules: 0-40 any characters Subvalued to allow more than one Format specification for this data item. The Format commands will be executed in the order in which they are defined in this field. A subroutine exist on PGM.FILE as a type 'S' or an EB.API record of type METHOD which implements an interface defined in the EB.API record DEAL.SLIP.FORMAT.HOOK. Input should prefix with '@'
