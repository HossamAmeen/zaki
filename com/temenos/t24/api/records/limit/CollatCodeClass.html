      • getCollatCode

        public TField getCollatCode()
        Returns the CollatCode

        Field description: The collateral, which is held as security against a facility, is specified in the 'COLLAT' group of fields. These fields are for information only and form the basis for a future Credit Register package. The code entered must relate to an entry on the LIMIT.COLLAT table. Input of the Collateral fields is for information purposes only. If more than one type of Collateral is held, the associated fields can be expanded to record all of the types of Collateral. Validation Rules: 4 numeric character Security Code. This field is associated with all fields prefixed 'COLLAT'. (Optional input.) The Collateral code entered in this field must have been previously defined on the LIMIT.COLLAT table. Duplicates will not be allowed.

        Parameters:
        Returns:
        TField - The CollatCode

      • getCollatCustomer

        public TField getCollatCustomer()
        Returns the CollatCustomer

        Field description: Indicates the Customer in whose name the Collateral is held. Input of all Collateral fields is for information purposes only. Validation Rules: 1 to 6 numeric characters. Associated with all fields prefixed 'COLLAT'. (Optional input; Default = Liability number associated with the Line). It must be the number of a valid Customer on the Customer file. Cannot be input if Collateral Code has not been entered. Only for Customer level Group Limits (For e.g. M0000000020.2000.01.10020) this field will be considered.

        Returns:

      • getCollatGrade

        public TField getCollatGrade()
        Returns the CollatGrade

        Field description: Defines the quality of the collateral. This field is based on CUSTOMER.STATUS and can be used to judge the reliability of the Collateral. For instance, if Customer Status is measured as 'Excellent', 'Good' or 'Undesirable', any of these could be associated with the Collateral to give a similar meaning. Validation Rules: 1 to 4 numeric characters CUSTOMER.STATUS Code. Associated with all fields prefixed 'COLLAT'. (Mandatory input if COLLAT CODE entered; otherwise not allowed.) Input cannot be made if Collateral Code has not been entered. The code entered in this field must be a valid code on the CUSTOMER.STATUS table.

        Parameters:
        Returns:
        TField - The CollatGrade
