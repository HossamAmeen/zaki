      • getSettlementDays

        public TField getSettlementDays()
        Returns the SettlementDays

        Field description: Indicates the number of days forward the value date is it be. Multi value field associated with TRANSACTION.TYPE and TRADE.CURRENCY. Accepts a value when SETT.DAYS.BASIS is BUSINESS or CALENDAR. The number of days entered will be used to calculate the value date (forward) when a security is traded using this Stock Exchange. This field can be used with the combination of TRANSACTION.TYPE and TRADE.CURRENCY. A settlement day can be defined for each combination. Value date will be calculated based on the following priority. If only the settlement date is defined, use the same for calculating the value date. If there is a match with any transaction type and trade currency combination, use the corresponding settlement days for calculating the value date. If there is no exact match with any combination, use the first matched criteria based on TRANSACTION.TYPE for calculating the value date. If there is no match, use the settlement days for which no transaction type or trade currency is linked. Validation Rules: Mandatory when either TRANSACTION.TYPE or TRADE.CURRENCY is input. 1-3 Numeric characters. Input mandatory if SETT.DAYS.BASIS is BUSINESS Input not allowed if SETT.DAYS.BASIS if FIXED. Otherwise input is optional.

        Parameters:
        Returns:
