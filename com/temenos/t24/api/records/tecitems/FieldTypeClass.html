      • getValue

        public TField getValue()
        Returns the Value

        Field description: This field contains the test value for the content of the field defined in FIELD. Typically this is supplied during Subscription, by the customer. But can also be defined at the event definition stage.

        1) When valid field name is given in FIELD,this must be in agreement with the Format rules of the element specified in FIELD.NAME. for example, the input in FIELD.NAME corresponds to a Customer Mnemonic, a minimum input of 3 characters will be required in this field. 2) It can also support Keywords like - !TODAY, - !LAST.WORKING.DAY, - !NEXT.WORKING.DAY, - !OPERATOR (User) - !EXT..., (EXT.XXX variables set during External user login) - !CURRENT..., (Current.XXX variables) - !SUBSCRIPTION.ID (Current subscriber id(Only for SERVICE classification)) 3. Input allowed only when ITEM.CLASSIFICATION is BUSINESS OR SERVICE

        Returns:
