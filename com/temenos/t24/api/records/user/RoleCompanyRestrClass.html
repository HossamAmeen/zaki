      • getRoleCompanyRestr

        public TField getRoleCompanyRestr()
        Returns the RoleCompanyRestr

        Field description: Specifies the Company, to which the user signs on, in order to use specific roles. Used, together with the associated field 74, to specify for each Company, which Role, the user is allowed to login. Defines within each Role precisely, which record and application, the user can access, using which version and function. User who have signed on to a Company, can only access files for which standard or special file mnemonics are specified in the Company record (or have used the PASSWORD Application to access that Company). Validations:

        Permits ALL - to allow the functions of all companies listed in theCOMPANY.CODEfield.

        In a Multi Book company set up, if this field is defined as ALL, - it allows the functions for Lead company and all book companies under that Lead company.

        9 alphanumeric characters CCGGGLLLL where: CC is 2 alpha characters Country code, GGG is 3 numeric characters Company Group and LLLL is 4 numeric local code. or

        2-3 type MNE (uppercase alphanumeric or ) Company Mnemonic characters.

        Multi-valued field associated with Field 75.

        Mandatory input if legacy SMS restriction is not given.

        No Default Value. It must be a Company Code from theCOMPANY.CODEfield of this record.

        Parameters:
        Returns:
        TField - The RoleCompanyRestr
