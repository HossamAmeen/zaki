      • getAmortSchedRr

        public TField getAmortSchedRr()
        Returns the AmortSchedRr

        Field description: Provides an option on how the residual principal amount of the floating rate side of an Amortizing swap needs to be shown in the Interest Rate Gap. The two options are a) each amortizing schedule of a floating rate side contract will be projected in the bucket corresponding to its value date with interest rate same as the current period . b) the entire residual amount will be projected to the end of the current Amortizing schedule as interest rate beyond the current amortizing schedule is still not available. Validation Rules: Allowed values are YES, NO, NULL. When set to YES, the residual principal amount on the floating rating leg side will be shown up to next reset date only in the Interest Rate Gap Report. When set to No each amortizing schedule of a floating rate side contract will be projected in the bucket corresponding to its value date with interest rate same as the current period .

        Returns:
