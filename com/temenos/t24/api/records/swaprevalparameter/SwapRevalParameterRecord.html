      • getImpFutRateApi

        public TField getImpFutRateApi()
        Returns the ImpFutRateApi

        Field description: User can attach an user defined routine to include any margin in to the derived future rate for the future interest payments during valuation. Routine can be attached only when the field NPV.FWD.RATE is set to YES. The routine should have a valid entry in PGM.FILE. The attached routine will be triggered for the floating leg contracts involving local currency. The attached routine accepts the following parameters 1. Swap contract in the form of array 2. Swap balances in the form of array 3. Swap Reval Parameter in the form of array 4. Interest Period Start date 5. Interest Period End date 6. Floating Rate Key as specified in the contract 7. Interest Basis as specified in the contract 8. Leg currency 9. Forward Rate as derived by the system for the future Interest payment. User can include margin to the forward rate based on their requirements. System further considers this modified rate and applies spread (if applicable) on it and will take the resultant rate for processing. User can only modify the forward rate as part of the routine, amendments to other parameters are not allowed; if done, it will not be reflected in the system.

      • getSplNpvProcess

        public TField getSplNpvProcess()
        Returns the SplNpvProcess

        Field description: Indicates the special NPV revaluation methodology in which the Interest payment amount will be included for the valuation on the interest payment date. Value allowed are YES, NO or NONE. Default value is NONE A value of YES can be set only if : a) IA.ACCOUNTING field is set to YES and b) NPV.ACCR.ADJ is set to NO or NONE When set to YES, The core swap module will not raise any entries for interest accrual and NPV valuation but the same will be handled by IA module based on client requirements. This is applicable to SWAP deals set with a Revaluation Type. The interest payment due on that date of revaluation will also be considered for net present valuation. Since accrual entries are suppressed, the entries relating to IP payment will be raised between the Profit and Loss (P&L) and Customer/Nostro/Internal accounts When set to NO or NONE, the core swap module will raise entries for interest accrual and revaluation amount (if revaluation is applicable) and exclude the Interest payment amount during revaluation on the interest payment date.

        Parameters:
        Returns:
        TField - The SplNpvProcess

      • getZeroCouponRate

        public TField getZeroCouponRate()
        Returns the ZeroCouponRate

        Field description: Indicates how the discounting rates are to be used for net present valuation. Values allowed are YES, NO, ALT, NONE. Default value is NONE If set to NO, system does not perform bootstrapping to arrive at the zero coupon rate, it rather uses the rate from PI table directly. If set to YES or ALT, system does bootstrapping to arrive at the zero coupon rate. If set to YES or NONE, The net present value of the future cash flows for the particular period will be discounted by Zero Coupon Rate which is based on the previous period's Coupon Rate(S). If set to ALT, The net present value of the future cash flows for the particular period will be discounted by Zero Coupon Rate which is based on the previous period's Zero Coupon Rate(S).

        Returns:
