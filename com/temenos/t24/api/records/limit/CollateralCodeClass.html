      • getCollReqdAmt

        public TField getCollReqdAmt()
        Returns the CollReqdAmt

        Field description: Collateral required amount is used to control minimum collateral required in fixed type limits. This field is used for defining the collateral required amount for the collateral code specified in the multi-value set. If collateral required amount is less than the collateral a override is generated during the utilization of the limit (Eg .coll.code.301,value of collateral 0, required collateral 100000.00) If multiple collaterals are entered with collateral required amount then multiple overrides are generated. Validation Rules: Rule 1 Input allowed only for a fixed type limit (field 17 to be fixed) Input allowed only if collateral code is given (field 18) No input allowed if collateral required percentage is filled (either collateral required amount or collateral required percentage can be entered, both cannot be entered) Input should be numeric and valid amount

      • getCollReqdPrcnt

        public TField getCollReqdPrcnt()
        Returns the CollReqdPrcnt

        Field description: Collateral required percentage is used to control minimum collateral required in fixed type limits. This field is used for defining the collateral required percentage for the collateral code specified in the multi-value set. If collateral required percentage is defined then, collateral required amount is automatically calculated. Collateral required percentage uses internal amount (field 15) for calculating collateral required amount. If collateral required amount is less than the collateral a override is generated during the utilization of the limit (Eg .coll.code.301,value of collateral 0, required collateral 100000.00) If multiple collaterals are entered then multiple overrides are generated. Collateral required percentage in variable limits alters, the secured amount of the respective collateral right. If collateral value is 10000 and collateral required percentage is defined as 200% then secured amount for the related collateral right will populate 5000. ( ie value of collateral / Collateral required percentage ) Eg (If value of collateral is 10000) Collateral.Code - 301 Coll.Reqd.percnt - 200 Maximum Secured - 2500 Collateral.right - 100001.1 Secured amount - 2500 In above example if maximum secured is 10000 then secured amount is only 5000. Validation Rules: Input allowed only if collateral code is given (field 18) No input allowed if collateral required amount is filled (either collateral required amount or collateral required percentage can be entered, both cannot be entered) Input should be numeric Input is taken as percentage (% sign not required in input) Input not allowed if the related collateral right is allocated to two or more limit references ( In collateral.right record)

        Parameters:
        Returns:
        TField - The CollReqdPrcnt

      • getCollateralCode

        public TField getCollateralCode()
        Returns the CollateralCode

        Field description: Identifies classes of collateral for which a maximum secured amount and collateral required amount underlying a limit is allowed. The collateral code in this field is associated with collateral required amount, collateral required percentage, up to period, period amount, period percentage and maximum secured amount (field 18 to 24). Together these fields define a collateral required amount, collateral required schedules and maximum secured amount per Collateral Code. This secured amount places a maximum value on the amount of a particular class of collateral, which can be used as security for the limit. It enables the definition of a structure to the underlying collateral. Example: A limit is granted up to an overall maximum of GBP 100,000.00. The limit is secured and variable. The current value of the underlying collateral exactly equal to the overall total limit. It is made up of a mortgage, currently valued at GBP 80,000.00, plus guarantees worth GBP 20,000.00. If field 18 (COLLATERAL CODE) is left blank, the limit record appears as follows: 17 FIXED.VARIABLE VARIABLE 26 MAXIMUM.TOTAL 100,000.00 28. 1. 1 COLLAT.RIGHT 12.1 MORTGAGES 29. 1. 1 SECURED.AMT 80,000.00 28. 1. 2 COLLAT.RIGHT 10 GUARANTEES 29. 1. 2 SECURED.AMT 20,000.00 32. 1 ONLINE.LIMIT... 100,000.00 If, however, this field has been used to limit the amount of any guarantees, which are acceptable as collateral to GBP 10,000.00, the limit record appears as follows: 17 FIXED.VARIABLE VARIABLE 18. 1 COLLATERAL.CODE 10 GUARANTEES 19. 1 MAXIMUM.SECURED10,000.00 21 MAXIMUM.TOTAL 100,000.00 23. 1. 1 COLLAT.RIGHT 12.2 GUARANTEES 24. 1. 1 SECURED.AMT 10,000.00 23. 2. 1 COLLAT.RIGHT 12.1 MORTGAGES 24. 2. 1 SECURED.AMT 80,000.00 27. 1 ONLINE.LIMIT... 90,000.00 Validation Rules: 1 to 3-digit numeric Collateral Code. Multivalue field associated from field 18 to 24. No input is allowed unless a value is indicated in field 17, FIXED/VARIABLE. In a variable limit if no collateral code is attached then an override is generated.

        Parameters:
        Returns:
        TField - The CollateralCode
