      • getSplitPositions

        public TField getSplitPositions()
        Returns the SplitPositions

        Field description: Identifies whether the POSITION records are to be created in local currency on the value-dates for the FX transactions in cross currencies. If this field is set to YES, then For the FX transactions in cross currencies, the position records are generated for the transactions done by this dealer for the two cross currencies online. On the value-date these records are split and position records for buy currency against Local currency and for sell currency against Local currency are created. If this field is set to NO or NULL then For the FX transactions in cross currencies, the position records are generated for the transactions done by this dealer for the two currencies online. On the value-date these records are not split as above Validation Rules: Acceptable values are YES , NO or NULL Default is YES

        Parameters:
        Returns:
        TField - The SplitPositions
