      • getDefRecipientRole

        public TField getDefRecipientRole()
        Returns the DefRecipientRole

        Field description: The fields DEF.RECIPIENT.ROLE and DEF.RECIPIENT.SEQ are used to define the default recipient(s) for the Advice Type. A default is required for an Advice Type which has a Status Trigger Rule as it will be automatic. If the advice is triggered "manually" then the defaults will be used to "default" in recipients but the user can choose which recipients to actually send the advice to. This field will specify the Role of one or more Default recipients. It will be linked to AA.CUSTOMER.ROLE table.

        Parameters:
        Returns:

      • getDefRecipientSeq

        public TField getDefRecipientSeq()
        Returns the DefRecipientSeq

        Field description: If the advice is for a specific recipient of a certain role then the appropriate sequence number can be specified. For instance if the advice is only for Applicant 1 (as opposed to all Applicants) then the role should be Applicant and Seq should be 1. If no sequence number is specified in this field, then advice will send to all parties in the Role (Applicant).
