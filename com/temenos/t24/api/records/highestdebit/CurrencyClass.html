        public TField getCurrency()
        Returns the Currency

        Field description: Defines the Currency for which the Highest Debit Charge details in the associated Fields 11-13 apply. For Accounts in Currencies not input to this Field, the Default Amounts specified in Fields 7-9 are used. Fields 10-13 are used to define Amounts for adjusting Highest Debit Charges on Accounts denominated in specific Currencies. For any Accounts in Currencies not specified in Field 10, the Default (local currency equivalent) Amount specified in Fields 7-9 are used. Validation Rules: 3 type SSS (SWIFT) characters Currency code. or 1-3 numeric characters (will be converted to Currency code). Multivalue Field associated with fields 11-13. (Mandatory Input if any input in associated Fields 11-13 otherwise not allowed.) It must be a valid code on the CURRENCY table (Ref: General Tables).

      • getFreeAmount

        public TField getFreeAmount()
        Returns the FreeAmount

        Field description: Specifies the amount to be deducted from the total charge calculated, subject to the Minimum and Maximum Amounts, for Accounts in the Currency specified in associated CURRENCY field. Fields CURRENCY to MAXIMUM.AMOUNT are used to define Amounts for adjusting Highest Debit charges in specific Currencies. The Default Amounts specified in fields DEFAULT.FREE.AMT to DEFAULT.MAX.AMT are used if there is no entry in CURRENCY field for the Currency of the Account. The Free Amount is subtracted from the calculated charge and further adjustments are made as specified in fields MINIMUM.AMOUNT and MAXIMUM.AMOUNT. The COMPANY record specifies when the charges are to be applied. If, however, the associated GENERAL.CHARGE record specifies that charges should be calculated on a monthly basis (Calcul Step Period = M), the Free Amount is subtracted from the charge calculated for each month. Note : (i) The result may be zero, but not negative, i.e. the Customer will not receive a Credit. (ii) Even if the Free Amount is greater than or equal to the calculated charge, any Minimum specified in MINIMUM.AMOUNT, is still applicable. Validation Rules: 1-14 numeric characters plus a decimal point. Multivalue field associated with fields CURRENCY to MAXIMUM.AMOUNT. (Optional input, only allowed if CURRENCY is input.) The number of decimals may not exceed the number for Currency in CURRENCY.

        Parameters:
        Returns:

      • getMaximumAmount

        public TField getMaximumAmount()
        Returns the MaximumAmount

        Field description: Specifies the Maximum Amount to be charged to the Account, for Accounts in the Currency specified in the associated CURRENCY field. Fields CURRENCY to MAXIMUM.AMOUNT are used to define Amounts for calculating Highest Debit Charges in specific Currencies. The Default Amounts specified in fields DEFAULT.FREE.AMT to DEFAULT.MAX.AMT are used if there is no entry in CURRENCY field for the Currency of the Account. If after the Free Amount has been subtracted from the calculated charge the result is greater than the Maximum Amount, the Maximum Amount is used. The COMPANY record specifies when the charges are to be applied. If, however, the associated GENERAL.CHARGE record specifies that charges should be calculated on a monthly basis (Calcul Step Period = M), the charge for each month is compared with the Maximum Amount and adjusted if necessary. Validation Rules: 1-14 numeric characters plus a decimal point. Multivalue field associated with fields CURRENCY to MAXIMUM.AMOUNT. (Optional input only allowed if CURRENCY is input.) The number of decimals may not exceed the number for Currency in CURRENCY.
