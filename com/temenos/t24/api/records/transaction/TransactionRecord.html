      • addChargeKey

        public void addChargeKey(java.lang.CharSequence param)
        Appends the specified ChargeKey to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Indicates which, if any, Charges are to be applied for this Transaction code. This field identifies the record in the TRANSACTION.CHARGE table (ref: ACCOUNT) which contains details of the bank charges to be applied for this particular Transaction. For example, cash dispenser, standing order, direct debit, etc. When the value of the next field is defined as NO (not immediate charge), the Interest and Charges Application will take all these charges into account for the calculation and application of the charges to the relevant accounts. Validation Rules: 1-3 numeric character Transaction Charge code. Multivalue field (Optional input. Default Value = Blank (i.e. no charge). It must be an existing TRANSACTION.CHARGE record. Date of TRANSACTION.CHARGE record must be less than or equal to the Run Date. (Run Date is the date of the data being processed which may not always be today's date. If different, the Run Date is displayed next to your User ID on the screen). Allowed to enter only 1 transaction based calculated charge. This means that no two transaction charges with the CHARGE.PERIOD field on TRANSACTION set to 'T' may be entered.

        Parameters:
        param - the ChargeKey to append
        Returns:

        public void addCombineTxn(java.lang.CharSequence param)
        Appends the specified CombineTxn to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The fields COMBINE.PROD.CAT, COMBINE.SYSTEM.ID and COMBINE.TXN are used to define the way that T24 combines accounting entries together. For example, to specify that in the funds transfer application, contracts will have the entries combined for all product categories: 20 COMBINE.PROD.CAT.. ALL All Categories 21 COMBINE.SYSTEM.ID. FT FUNDS TRANSFER This field is used to specify the transactions code that may be netted under this transaction code. Entering a list of transaction codes will mean that any entry with any of the transaction codes may be netted under this new transaction: 22. 1 COMBINE.TXN.... 1 CREDITS 22. 2 COMBINE.TXN.... 2 DEBITS Entries must have the same value date, account and currency market to be netted. Note: In COMBINE.TXN field, along with other TRANSACTION codes that are to be combined with this selected TRANSACTION, the ID of this selected TRANSACTION code also is to be mentioned. Example: If Charge defined by TRANSACTION 231 and tax defined by TRANSACTION 464 are to be combined with the charge defined by TRANSACTION 622 (ID of the selected TRANSACTION) , then select the multi value COMBINE.TXN field in TRANSACTION 622 and enter codes 231, 464 and 622 also. Validation Rules: Must be a key to a valid record on this file, or ALL. Where ALL is defined, no other transaction codes may be defined. Duplicates not allowed. Defaults to ALL if either the COMBINE.PROD.CAT field on the COMBNE.SYSTEM.ID field is populated.

      • getChequeInd

        public TField getChequeInd()
        Returns the ChequeInd

        Field description: Specifies whether the inclusion of a cheque number is to be mandatory for entries input via the DATA.CAPTURE System, and using this transaction code. The purpose of this field is twofold: 1.
