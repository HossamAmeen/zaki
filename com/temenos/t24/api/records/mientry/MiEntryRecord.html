        public TField getAccountOfficer()
        Returns the AccountOfficer

        Field description: Department or account officer to which this Profit or Loss entry should be allocated. Where the entry has been generated from P&L allocation, this will not always match the Account Officer on the originating P&L entry. Reallocation is the process of reversing a portion of P&L from the originating department, and allocating it to another. Thus, the MI.AUTO.ENTRY will show: -For Debit entries, the Department/Account Officer from whom the P&L is being taken. This will match the originating entry -For Credit entries, the Department/Account Officer to whom the P&L is being reallocated. This will normally be different to the originating entry. Validation Rules: 1-4 numeric characters Department Account Officer code. (Internal file, no user input.) Mandatory field

      • getAmountLcy

        public TField getAmountLcy()
        Returns the AmountLcy

        Field description: The amount of the Entry in local currency. (All Profit and Loss Entries are also expressed in Local Currency, even when they are the result of a foreign currency transaction, or interest and charges calculated on a foreign currency Account.) Where the Entry has been generated from P&L Allocation, this field is calculated from the Local Currency amount of the original entry being reallocated, and the percentage set for reallocation in the MI.PL.ALLOCATION application. Where the NETTED field in this record is set to 'Y', the entry is a result of netting during P&L Allocation. This happens where : - netting has been requested in MI.PL.ALLOCATION, and - the Allocation process could otherwise produce several entries with similar characteristics. One Netted entry is created, with a Local currency amount equal to the sum of the individual entries. Validation Rules: 1-14 numeric characters plus decimal point (standard amount format, type AMT), signed. (Internal file, no user input.) Mandatory field

      • getBookingDate

        public TField getBookingDate()
        Returns the BookingDate

        Field description: Contains the run date on which MI takes as the date the P&L of this entry was generated. This date determines which database period the entry will be reported in. Where the entry is generated from P&L Allocation, this date will be the same as the booking date of the original P&L entry. This is normally, but not necessarily the actual date at the time the original Entry was generated, e.g. if the previous day's business was still being processed after midnight, the actual date changes, but the run date is still the previous day. Thus the reallocated P&L will appear in MI with the same date as the original P&L, even if P&L allocation is run some time after the original P&L was earned. The date on which the record was physically generated (ie the T24 date on which the reallocation was run) is stored for audit purposes in the field CREATED.ON. Validation Rules: Standard Date format. (Internal file, no user input.) Mandatory field

      • getPlCategory

        public TField getPlCategory()
        Returns the PlCategory

        Field description: Identifies the type of Profit and Loss updated by this Entry. If the Entry is for a Product type Profit and Loss Category (e.g. Commission), then the Product Category code is also stored in this record, in the field of the same name. For Entries to base level Profit and Loss Categories (e.g. Purchase of Goods), then the Profit and Loss Category code needs no further breakdown by Product Category. Note: 1) Profit and Loss Category codes are in the range 50000 - 69999. 2) Where the entry is generated from P&L reallocation, this field is derived from the value of the 'allocate from' or 'allocate to' category code on the reallocation record. This need not be the same as the category of the originating entry. Validation Rules: 5 numeric characters Category Code. (Internal file, no user input.) Mandatory input

      • getProductCategory

        public TField getProductCategory()
        Returns the ProductCategory

        Field description: Specifies the type of product the interest or commission has been earned on. For entries generated from P&L allocation, it is derived from the product category of the originating P&L entry. Originating entries updating base level types of Profit and Loss e.g. Purchase of Goods, have no product categories, and thus will leave this field blank. For interest and charges earned on Customers' Accounts this field contains the Category code from the Account record. For Profit and Loss earned on other products, the Product Category is entered in the originating transaction or generated by the originating Application.
