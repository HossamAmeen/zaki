      • getCreditLineNo

        public TField getCreditLineNo()
        Returns the CreditLineNo

        Field description: Identifies the limit record for which the currency and/or value of CHECK.LIMIT is to be changed. The limit records for which the limit currency is to be changed are to be entered individually. The old currency is shown as part of the field enrichment. The top level limit for which the CHECK.LIMIT value is to be changed should be entered. All limits under this will be changed to the new value. Validation Rules: Input as follows: a) 1-6 numeric characters Liability Number or 3-10 Customer Mnemonic code b) 3-7 numeric character Limit Reference code c) 2 numeric character Limit Serial number d) 1-6 numeric character Customer Number or 3-10 Customer Mnemonic code If data is to be input in this field: a), b), c) and d) must be separated by '.';a) and b) are Mandatory; c) defaults to '01'; d) is Optional, only used to specify Customer sub-limit. Multivalue field associated with field 4.1. Optional Input Input must correspond to a valid ID on the limit file. Duplications will not be allowed.

        Parameters:
        Returns:
        TField - The CreditLineNo

      • getNewCheckLimit

        public TField getNewCheckLimit()
        Returns the NewCheckLimit

        Field description: New value of the field CHECK.LIMIT for the specified limit structure. This field will be used to change the current value of the CHECK.LIMIT field on each LIMIT record in the LIMIT structure which the CREDIT.LINE.NO is the top of. Validation Rules: Y or N Multi value field associated with 3.1 and 4.1. Optional input when CREDIT.LINE.NO (field 3.1) input, otherwise not allowed. The associated CREDIT.LINE.NO must be the top of the LIMIT structure, ie must be a key to the LIMIT.LINES file. The value entered must not be the same as that on the LIMIT record of the CREDIT.LINE.NO. Input only allowed when CREDIT.LINE.NO has been entered.

        Parameters:
        Returns:

        public TField getNewCurrency()
        Returns the NewCurrency

        Field description: Identifies the new currency in which all amount fields specified as part of the limit will be denominated. Validation Rules: 1-3 numeric character Currency code or 3 character SWIFT Currency code Multivalue field associated with field 3.1 Optional input when CREDIT.LINE.NO (field 3.1) input, otherwise not allowed.
