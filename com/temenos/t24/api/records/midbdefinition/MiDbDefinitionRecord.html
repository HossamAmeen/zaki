        public void addAdditional(TField param)
        Appends the specified Additional to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Additional, specific budget records may be specified here. If the BUDGET.FOR mechanism is not able to specify certain, specific budget combinations then they may be manually input here. For example if the majority of budgets are being maintained just for departments then the budget.for field would contain just DEPARTMENT. However if few specific customers within a particular department are also to have budgets set then these specific combinations may be specified in the ADDITIONAL field. For example: CONSOLIDATE.BY DEPARTMENT CONSOLIDATE.BY CUSTOMER.NO BUDGET.BY DEPARTMENT BUDGET.FOR EXIST ADDITIONAL 400_12547 ADDITIONAL 400_12554 The system would then create 'empty' budget structures for all departments that existed on the database and two additional budgets for department 400 and customers 12547 and 12554. Validation Rules: Optional Fields must be in same sequence as consolidate.by and budget.by Fields must exist in the appropriate related file (REL.FILE) Fields must be separated by an underscore character (_)

        param - the Additional to append
        Returns:
