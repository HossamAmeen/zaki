      • getDrBasicRate

        public TField getDrBasicRate()
        Returns the DrBasicRate

        Field description: Field range DR.BASIC.RATE to DR.LIMIT.AMT is used to specify rates applicable to different balance limits. Specifies a floating Basic Rate which is held on the BASIC.INTEREST table (Ref: General Tables). Rates can be fixed or linked to Basic Rates plus or minus a Margin. Validation Rules: 1-4 numeric characters. Multivalue field associated with field range DR.BASIC.RATE to DR.LIMIT.AMT. (Optional input. Not allowed if corresponding DR.INT.RATE is input or if INTEREST.DAY.BASIS contains NONE or GENERAL.) There must be a record in the BASIC.INTEREST table (Ref: General Tables) with a key consisting of this code, the Currency of the Account and an effective date which is less than or equal to the date specified in ID. Examples: The rate or rates of interest which will be applied are defined by the following multivalue group of fields: BASIC RATE INT RATE MARGIN OPER MARGIN RATE LIMIT AMOUNT Examples of their use is as follows: 1) If the interest rate is to be equal to a floating Basic Rate, only enter the following field: BASIC RATE - Code of the applicable Basic Rate. 2) If the interest rate is to be fixed, only enter the following field: INT RATE - the actual rate of interest. 3) If the interest rate is to be equal to a floating Basic Rate plus a margin, enter the following fields: BASIC RATE - Code of the applicable Base Rate MARGIN OPER - ADD MARGIN RATE - the actual Marginal Rate. 4) If the interest rate is to be equal to a floating Basic Rate plus a Percentage of the floating Basic Rate; enter the following fields: BASIC RATE - Code of the applicable Basic Rate MARGIN OPER - MULTIPLY MARGIN RATE - Percentage to be added 5) If the interest rate is to be equal to a floating Basic Rate, plus a Margin up to a limiting amount and a floating Basic Rate, plus a different Margin above that limiting amount, enter the following fields: BASIC RATE - Code of the applicable Basic Rate MARGIN OPER - ADD MARGIN RATE - first Margin rate LIMIT AMOUNT - The limit amount for the above defined rate BASIC RATE - Code of the applicable Basic Rate MARGIN OPER - ADD MARGIN RATE - 2nd Margin rate.

        Parameters:
        Returns:
        TField - The DrBasicRate

      • getDrMarginOper

        public TField getDrMarginOper()
        Returns the DrMarginOper

        Field description: Field range DR.BASIC.RATE to DR.LIMIT.AMT is used to specify rates applicable to different balance limits. Determines the method by which the Rate specified in DR.BASIC.RATE or DR.INT.RATE is to be adjusted by the Margin specified in DR.MARGIN.RATE. If the Rate specified in DR.BASIC.RATE or DR.INT.RATE is to be adjusted by a Margin, this field specifies whether the Margin Rate is to be added to, subtracted from or multiplied by the basic rate to arrive at the effective interest rate. If Multiply is specified the final rate is (100 + M) percent of the Rate from DR.BASIC.RATE or DR.INT.RATE , where M is the Margin Rate percentage from DR.MARGIN.RATE. For examples see DR.BASIC.RATE. Validation Rules: ADD, SUBTRACT or MULTIPLY, or any abbreviations thereof. Multivalue field associated with field range DR.BASIC.RATE to DR.LIMIT.AMT. (Optional input. Only allowed if corresponding DR.BASIC.RATE or DR.INT.RATE is input.)

        Parameters:
        Returns:
        TField - The DrMarginOper

        Field description: This field defines the capped rate (or maximum rate) for debit interest to be applied to the DR balance type.
