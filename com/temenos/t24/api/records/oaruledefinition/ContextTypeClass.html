      • getContext

        public TField getContext()
        Returns the Context

        Field description: This field will indicate the rule is based on which context.

        Input should be mandatory and allowed only when CONTEXT.TYPE is EB.CONTEXT/POLICY.ITEM/POLICY.DEVIATION.

        The field would be used to specify the from where the content for the Key words of the Rule would be extracted.

        1. If the CONTEXT.TYPE is EB.CONTEXT then the CONTEXT should be valid record from EB.CONTEXT. 2. If the CONTEXT.TYPE is STATUS.CODE then input not allowed. 3. If the CONTEXT.TYPE is POLICY.ITEM then input would be valid record from the OA.POLICY.ITEM. 4. If the CONTEXT.TYPE is POLICY.DEVIATION then input would be in the format of <POLICY.ITEM>-<DOMAIN.TYPE>-<ROLE>-<SEQUENCE>. Role is applicable only for Context which are belongs to Party Type. Here 1st Key word should be valid OA.POLICY.ITEM, 2nd Key word would be valid OA.DOMAIN.TYPE and 3rd key word would be OA.ROLE. If Role is not applicable, it can be left blank.

        Parameters:
        Returns:
        TField - The Context

      • getContextInstance

        public TField getContextInstance()
        Returns the ContextInstance

        Field description: This fields describes the name of the instance could be used to extract the data for rule evaluation. If the CONTEXT is of a class type which can have multiple instances, then the user must specify the instance name in this field.

        Input is allowed only when CONTEXT.TYPE is EB.CONTEXT.

        Example : Suppose ADDRESS is one formlet class, and user creates the multiple instances HOME.ADDRESS and OFFICE.ADDRESS, in this case, when defining rule definition, the user should define the INSTANCE name against to the Context to extract the data from corresponding arrangement condition.

      • getContextType

        public TField getContextType()
        Returns the ContextType

        Field description: This field will indicate the type of context to be used in this rule item. Allowed types currently are: EB.CONTEXT - Indicating the rule would refer to an EB.CONTEXT entity (for example, a table).

        STATUS.CODE - Indicating the rule would refer to a "Status code" in its rule. The actual code would be defined where these items are used.

        POLICY.ITEM - This type will accepts the CONTEXT as valid OA.POLICY.ITEM which would be part of the RULE.

        POLICY.DEVIATION - This type will accepts the CONTEXT in the format of <POLICY.ITEM>-<DOMAIN.TYPE>-<ROLE>-<SEQUENCE>. Role is applicable only for Context which are belongs to Party Type.

        1. mandatory field

        Parameters:
        Returns:
