      • getInwardOfsRtn

        public TField getInwardOfsRtn()
        Returns the InwardOfsRtn

        Field description: Identifies a routine used to process inward FUNDS.TRANSFER messages, a default routine FT.OFS.DEFAULT.MAPPING is available for 100, 200, 202 and 205 DE.MESSAGE types. The OFS based processing of inward messages used to create Funds Transfer records works as follows. An OFS.SOURCE record should be set up with FT.OFS.INWARD.MAPPING defined in the IN.DIR.RTN field. An EB.PHANTOM record with OFS.REQUEST.MANAGER defined in RUN.ROUTINE should be set up and activated. Whenever the OFS inward message directory is empty then FT.OFS.INWARD.MAPPING will be called, this routine will process inward messages stored in DE.I.MSG.FT, which is updated by the delivery system. For each inward message type processed, if a routine is defined in this field of the relevant DE.MESSAGE record, then it will be called with the following parameters

        ID.INThe @id of the DE.I.MSG.FT record
        R.INWARDThe DE.I.MSG.FT record which has been mapped from the incoming SWIFT message
        R.SWIFTThe incoming SWIFT message
        MESSAGE.TYPEe.g. 100, 200, 202, 205
        R.DE.MESSAGEThe DE.MESSAGE record, i.e. this record.
        OFS.KEYReturned from the called routine, defines the @ID of the OFS message which will be copied into the OFS input directory. This will be the FT record key when returned from the default routine.
        OFS.MESSAGEThe OFS message text, returned from the called routine.
        R.DE.MESSAGEThe DE.MESSAGE record, i.e. this record.
        A detailed description of the default mapping routine logic can be found in the API developers guide. This routine will create and return an OFS message used to create a Funds Transfer. The OFS message returned is copied into the OFS input directory as defined on the OFS source record, normal OFS processing will then take place and if all validation is successful will result in an FT record being created, either inward or outward depending on the type of incoming message. It is possible to automatically generate outward SWIFT 205 messages if zero authoriser OFS versions are used together with FWD.IN.TXN being set to "YES" on the relevant FT.TXN.TYPE.CONDITION record. Validation Rules: Optional input If input then must de defined as a type "S" routine on PGM.FILE.

        Parameters:
        Returns:
        TField - The InwardOfsRtn

      • setInwardOfsRtn

        public void setInwardOfsRtn(TField value)
        Set the specified value for InwardOfsRtn
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Identifies a routine used to process inward FUNDS.TRANSFER messages, a default routine FT.OFS.DEFAULT.MAPPING is available for 100, 200, 202 and 205 DE.MESSAGE types. The OFS based processing of inward messages used to create Funds Transfer records works as follows. An OFS.SOURCE record should be set up with FT.OFS.INWARD.MAPPING defined in the IN.DIR.RTN field. An EB.PHANTOM record with OFS.REQUEST.MANAGER defined in RUN.ROUTINE should be set up and activated. Whenever the OFS inward message directory is empty then FT.OFS.INWARD.MAPPING will be called, this routine will process inward messages stored in DE.I.MSG.FT, which is updated by the delivery system. For each inward message type processed, if a routine is defined in this field of the relevant DE.MESSAGE record, then it will be called with the following parameters

        ID.INThe @id of the DE.I.MSG.FT record
        R.INWARDThe DE.I.MSG.FT record which has been mapped from the incoming SWIFT message
        R.SWIFTThe incoming SWIFT message
        MESSAGE.TYPEe.g. 100, 200, 202, 205
        R.DE.MESSAGEThe DE.MESSAGE record, i.e. this record.
        OFS.KEYReturned from the called routine, defines the @ID of the OFS message which will be copied into the OFS input directory. This will be the FT record key when returned from the default routine.
        OFS.MESSAGEThe OFS message text, returned from the called routine.
        R.DE.MESSAGEThe DE.MESSAGE record, i.e. this record.
        A detailed description of the default mapping routine logic can be found in the API developers guide. This routine will create and return an OFS message used to create a Funds Transfer. The OFS message returned is copied into the OFS input directory as defined on the OFS source record, normal OFS processing will then take place and if all validation is successful will result in an FT record being created, either inward or outward depending on the type of incoming message. It is possible to automatically generate outward SWIFT 205 messages if zero authoriser OFS versions are used together with FWD.IN.TXN being set to "YES" on the relevant FT.TXN.TYPE.CONDITION record. Validation Rules: Optional input If input then must de defined as a type "S" routine on PGM.FILE.

        Parameters:
        value - The value to set
        Returns:
