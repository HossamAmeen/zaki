      • addDiscPortfolio

        public void addDiscPortfolio(java.lang.CharSequence param)
        Appends the specified DiscPortfolio to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specficies the list of Managed Accounts that correspond to Discretionary Portfolios. The portfolios that are linked to one of these Managed Accounts (through MANAGED.ACCOUNT field in the SEC.ACC.MASTER record that defines the portfolio) will be considered as Discretionary Portfolios and all other Portfolios will be considered as Non-Discretionary Portfolios.

        Discretionary Portfolios - Many banks offer a "Discretionary" service to investors, whereby the bank's officers have the "Power of Attorney" over decisions affecting the portfolio account (i.e. what, and when, to buy or sell, at what price etc.) They will also decide the content of that portfolio with regards, to its holdings in for example equities, bonds etc. In this case, the whole portfolio will be considered as a single asset when attached to the Collateral.

        Non-Discretionary Portfolios - Some customers insist on running their own portfolios and these would be set up as "Non-Discretionary" accounts. In this case, each asset linked to the Portfolio will be considered seperately when attached to a Collateral.

        Validation Rules: Optional Multi-Value field. Duplicates are not allowed. Must be a valid record from MANAGED.ACCOUNT table.

        Parameters:
        param - the DiscPortfolio to append
        Returns:
        nothing

      • getAllocationValue

        public TField getAllocationValue()
        Returns the AllocationValue

        Field description: Specifies if the collaterals have to be allocated against the limits based on the maximum total or outstanding amount of the limits.

        If "MAXIMUM.TOTAL" is specified in this field, then the maximum total value specified in the limits linked to the collateral right will be considered as the value to be allocated. If Null or "TOTAL.OS" is specified in this field,then the total outstanding value of the limits linked to the collateral right will be considered as the value to be allocated.

        The total outstanding will be calculated by adding the TOTAL.OS value of the limit and balance of the account linked to the limit, if account balance is in debit(less than zero) and ACCOUNT.AS.LIAB flag is set as YES. Or else the total outstanding will be the value specified in TOTAL.OS field of the limit.

        Validation Rules: Optional field. Valid values are Null, TOTAL.OS and MAXIMUM.TOTAL.

        Parameters:
        Returns:
