      • getLossFixCat

        public TField getLossFixCat()
        Returns the LossFixCat

        Field description: Specifies the profit and loss category to be used when realising the revaluation loss for a Forex deal between 2 fixed currencies. When the 2 currencies of a Forex deal are linked to a Fixed currency, i.e. the Euro, there is no longer any fluctuation between the exchange rate as the rate is fixed. Any revaluation unrealised profit and loss can therefore be realised. If the field REALISE.FIX.PL is set to Y in Forex deals between 2 fixed currencies, the system will post the revaluation loss on the deal to the category specified in this field, reversing the unrealised loss from the category defined in the field LOSS.CATEG. Realisation takes place the night before the fixed rates apply (FIXED.START.DATE in the CURRENCY record), or the next night if the fixed start date has passed. Accounting Entries: Cr LOSS.CATEG reval amt Dr LOSS.FIX.CATEG reval amt Cr LOSS.FIX.INT.CT reval amt Dr LOSS.INT.CATEG reval amt Validation Rules: Optional Input Must be different to associated LOSS.CATEG.

        Parameters:
        Returns:
        TField - The LossFixCat

        public TField getRevalType()
        Returns the RevalType

        Field description: This field signifies the revaluation method that must be used for revaluing deals arising out of the application in the APPKIC.ID field. The are only two ways in which revalution is performed, these being : 'SP' - Spot revaluation that uses the spot rate. 'RB' - Rebate revaluation that uses the forward rates table. Validation Rules: Must be either 'SP' or 'RB' If APPLIC.ID = FX.RB then must be 'RB' If APPLIC.ID = FX or FX.SP then must be 'SP' If APPLIC.ID = REST then must be 'SP' All other applications can be set to either 'SP' or 'RB'
