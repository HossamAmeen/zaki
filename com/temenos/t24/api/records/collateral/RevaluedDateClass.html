      • getRevaluedAmount

        public TField getRevaluedAmount()
        Returns the RevaluedAmount

        Field description: The execution value(s) prior to the most recent revaluation of the collateral. The most recent revaluation(s) of the EXECUTION.VALUE. Upon revaluation by the collateral end-of-day programs, a new value is placed by the system in EXECUTION.VALUE. The value which it replaces is recorded in this field together with the associated REVALUED.DATE. The most recent information is stored by the system in the first value in this field. Subsequent values contain a history of previous values attached to the collateral. The number of values stored is determined by the collateral parameter file. Multi-value field in association with REVALUED.DATE. Validation Rules: Up to 19-digit numeric inclusive decimal point (amount format). Multi-value field. Internal field. This is a NOINPUT field.

        Parameters:
        Returns:
