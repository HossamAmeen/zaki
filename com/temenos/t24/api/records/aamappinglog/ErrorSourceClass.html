      • getAccountNo

        public TField getAccountNo()
        Returns the AccountNo

        Field description: It is the actual T24 account number. When creating the mapping request for arrangement fulfilment, system will generate account number and this account number will be used when creating new arrangement through FULFILMENT service. But if the fulfilment transaction is failed, then this account number should not lost. So, this system generated account number will be stored in this field. So, when re submitting the mapping log, this account number would be used when creating new arrangement. This field will get updated only for Arrangement type of fulfilment transactions. It will holds the null value for Push mapping type of failure logs.

      • getErrorSource

        public TField getErrorSource()
        Returns the ErrorSource

        Field description: Error Source will be holding two type of values based on the type of log. 1. If the log is created for any PUSH mapping failure then it will hold the name of the target application name. For example - OA.APP.PERSONAL.INFORMATION. 2. If the log is created for any fulfilment transaction failure then it will hold the name of the FULFILMENT.
