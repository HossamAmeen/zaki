      • getFirstDetailLine

        public TField getFirstDetailLine()
        Returns the FirstDetailLine

        Field description: The physical line where the first entry, on a page, should be printed. To determine this it is best to print data, outside T24, on every physical line of the passbook. You can then check not only which line to start the entries on but also whether the data is output on the pre-printed lines. It is likely that you will have to modify the line spacing, best handled by ATTRIBUTEs, in order to marry the two exactly. The balance brought forward line is normally printed on the line preceding the FIRST.DETAIL.LINE. Validation Rules: 1-3 Numeric characters.

        public TField getLastDetailLine()
        Returns the LastDetailLine

        Field description: The last phyical line, of the page, to print an entry. When an entry is printed on this line it will cause one or all of the following: LAST Data to be output at the end of a page. FIRST A new passbook page to be started. END Data to be output at the end of the book. NEW A new passbook to be issued. Validation Rules: 1-3 Numeric characters

      • getNumberOfPages

        public TField getNumberOfPages()
        Returns the NumberOfPages

        Field description: The number of pages in the passbook - including any 'header' page where the customer's name etc is printed. When this is exceeded then any "END" items are printed, a new passbook is issued "NEW" and the balance carried forward is printed "FIRST". Note: This occurs when the last entry is printed. Which means that a new passbook is issued when the current one is full & NOT when a new entry needs to be printed. Validation Rules: 1-3 Numeric characters.

      • getSpineLines

        public TField getSpineLines()
        Returns the SpineLines

        Field description: The physical line positions of the spine of the passbook where entries cannot be printed. Identifies the middle of the passbook where entries cannot be printed. This is only required for passbooks printed with the spine running horizontally rather than the vertically. The format is XX-YY where XX is the line number where the spine area begins and YY the end. Eg 20-22 would mean that entries should not be printed on lines 20,21 or 22. Validation Rules: 5 Alphanumeric characters in the format XX-YY.
