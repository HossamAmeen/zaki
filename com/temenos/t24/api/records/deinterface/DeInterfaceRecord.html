        public TField getAckRequired()
        Returns the AckRequired

        Field description: Determines whether or not an ACK is required by Delivery. If the interface being used does not send back an acknowledgement (either positive or negative), this field should be set to "NO". When messages are sent by the generic delivery program, if this field is set to "NO", the msg.disposition on the header record will automatically be set to "ACK"; if this field is set to "Y", the msg.disposition will be set to "AWAK" (awaiting acknowledgement) and the message will be written to the AWAK queue. Validation Rules: "Y", "NO" or blank Default "Y"

        public TField getAuditSeqChecking()
        Returns the AuditSeqChecking

        Field description: Specifies whether sequence number checking should be performed by the generic delivery interface. If set to "Y", messages sent and received are checked to ensure that the sequence number assigned is the next sequence number expected. If set to "N", no sequence number checking is performed and the sequence numbers assigned are not validated (e.g. for inward messages, they do not have to be numeric). Default is blank as the many interface types do not permit a default of either Yes or No. This must be set by the user as required. Validation Rules: Y or N Default is blank - user must set correct value.

      • getLineComm

        public TField getLineComm()
        Returns the LineComm

        Field description: Can be used to define the line communications parameters, for example for ATE or TCPIP protocols. There is no validation on this field, to allow for maximum flexibility. If any validation is required, it should be done through a routine specified on VERSION. EXAMPLE 1 If ATE is specified in the PROTOCOL field, then this field could specify five pieces of information about the line communications as follows : Line , Baud , Data , Stop , Parity Where : Line Defines the &DEVICES& file devicename for the asynchronous line connected to the SWIFT device. For example '8' will look for a tty line called 'TTY08' or '0' will look for 'TTY00'. Baud Defines the baud rate of the line. Data Defines the number of data bits. Stop Defines the number of stop bits. Parity Defines parity as '0' - None '1' - Even '2' - Odd EXAMPLE 2 If TCPIP is specified in the PROTOCOL field then this field could specify three pieces of information about the communications interface as follows : Blocksize , Internet address and port , pipes location Where : Blocksize Specifies the minimum datagram size in bytes which will be exchanged with the 'c' server. This is present for performance reasons since it allows a block of fixed size to be expected by each side. It should be set at installation to a value determined appropriate for likely message sizes. The default should be 512. Internet Specifies the Internet address as connected on the network address with the port number as an extension e.g. 100.100.10.10:1234. This is the address the 'c' server will connect its sockets to. Pipes Specifies the unix pathname of the directory containing the location pipes used for communications between T24 and the 'c' server for TCPIP. Validation Rules: 1-35 type "A" (alphanumeric) characters Optional input

        Returns:
