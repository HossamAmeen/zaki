        public void addRptAttributes(java.lang.CharSequence param)
        Appends the specified RptAttributes to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies the special print attributes to be used when the report/delivery output is printed. This field should contain the names of any special print attributes that are to be set up before the report/delivery output is printed e.g. LANDSCAPE, PORTRAIT mode etc. for laser printers. The attributes should relate to specific printers and must be defined on the PRINTER.ATTRIBUTES file. The key to this field is a combination of the printer type and attribute name, e.g. QMS.LANDSCAPE. The printer type should be defined in PRINTER.TYPE of the PRINTER.ID file. Validation Rules: 1-10 type A (alphanumeric) characters. (Optional input)

      • getPrinterId

        public TField getPrinterId()
        Returns the PrinterId

        Field description: Specifies the ID of a printer that will print the Form Type defined by this record. Must be a valid entry on the PRINTER.ID file. For Delivery message output this should be a valid entry on the PRINTER.ID file, specifying the printer to which output should be directed. If this field is blank, T24 will refer to the USER record field PRINTER.FOR.RPTS For the REPORT.CONTROL system this field should be left blank unless you require all reports of this form to be sent to a single printer. This should only be used for special stationery requirements. The report control system will send the output to the printer defined on the PRINTER.ID file. Validation Rules: 1-30 type A (alphanumeric) characters.
