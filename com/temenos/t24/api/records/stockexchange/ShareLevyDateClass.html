      • getShareLevyDate

        public TField getShareLevyDate()
        Returns the ShareLevyDate

        Field description: This field and the fields BR.SHR.CNTY to CU.SHR.COMM associated with it allows the used to define any stock exchange levies that will be charged by this Stock Exchange, EBV fees, PTM levy etc. They will be used to update the CU.EBV.FEES or BR.EBV.FEES fields on a SEC.TRADE that uses this STOCK.EXCHANGE. This set of fields will be used to define the FT.COMMISSION.TYPE records to be used for these fees for shares. The fields BOND.LEVY.DATE to CU.BOND.COMM do the equivalent for bonds. A share is defined in T24 as a Security Master record where the BOND.OR.SHARE flag is set to 'S'. This set of flags works as follows; The system will check whether we are dealing with a bond or a share to work out which set of fields to use. It will then take the trade date and compare it against the set of dates in this field until it finds the one that is less than the trade date. It will then decide if we are calculating a customer levy (CU.EBV.FEES) or a broker levy (BR.EBV.FEES). If it is a broker levy it will use BR.SHR.CNTY and BR.SHR.COMM if it is a customer CU.SHR.CNTY and CU.SHR.COMM. It will then take the security domicile of the share and look for this country in the BR.SHR.CNTY (or CU.SHR.CNTY) field. If it locates it, it will take the associated FT.COMMISSION.TYPE ID from the BR.SHR.COMM (or CU.SHR.COMM) field and use it to calculate the levy. If it does not find the country code then it will look for 'ALL' in the BR.SHR.CNTY(or CU.SHR.CNTY) field and if it is there use the FT.COMMISSION.TYPE record associated with it. Validation Rules: Optional Input. Any input must be of a standard T24 date format.
