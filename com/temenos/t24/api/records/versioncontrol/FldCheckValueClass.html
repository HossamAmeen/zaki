      • getFldCheckType

        public TField getFldCheckType()
        Returns the FldCheckType

        Field description: Allows you to specify the environment variable to be tested. The system will use this variable to check against the value specified in FLD.CHECK.VALUE. If it matches then the field or fields in FLD.NAME will be included in the version, provided FLD.INC.EXC is set to &#8220;INCLUDE&#8221;. The format can be Application>fieldname (e.g USER>COMPANY.CODE) where the application can be USER, COMPANY or COMPANY.CONSOLID only. It can also be a routine. If it is routine the following conditions apply, a) When a subroutine needs to be executed then the subroutine name should be preceded by an @ symbol b) The subroutine entered should exist in PGM.FILE with PGM.TYPE = 'S' c) The APPLICATION in the record key should exist in field APPL.FOR.SUBR in the PGM.FILE record for the subroutine. d) This subroutine gets executed only once when the Version is first launched and the attributes set will thereafter remain unaltered. Validation Rules: Up to 50 type A (alphanumeric) characters. Multi-value field associated with fields FLD.CHECK.VALUE to FLD.INC.EXC. Optional input. No default value.

        Parameters:
        Returns:
        TField - The FldCheckType

      • getFldCheckValue

        public TField getFldCheckValue()
        Returns the FldCheckValue

        Field description: This field contains a value to check against the environment variable specified in FLD.CHECK.TYPE. The outcome of this check determines whether the field or fields named in FLD.NAME are included in the VERSION being used. E.g. If FLD.CHECK.TYPE has USER>COMPANY.CODE then the value in FLD.CHECK.VALUE should be the COMPANY code in which the field is to be processed. Validation Rules: Up to 50 type A (alphanumeric) characters. Multivalue field associated with fields FLD.CHECK.VALUE to FLD.INC.EXC. Optional input. No default value.
