      • addProcessError

        public void addProcessError(java.lang.CharSequence param)
        Appends the specified ProcessError to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Identifies any process errors or overrides that may have occurred on the individual payments. If any errors or overrides are found on the main payment then, if the application is being run on-line, the User may input YES or NO to the override condition. An input of NO will result in the Inward Bulk Standing Order going to the Exception file, in which case it will need to be authorised in order to execute it again. An input of YES will override the condition and execution will carry on and build the Funds Transfer records. However if any of the individual payments encounter any overrides the Funds Transfer records will be built as normal, except they will have a Record Status of 'IHLD'. The actual override condition will be held in this field. Validation Rules: (Internal multivalue field.)
