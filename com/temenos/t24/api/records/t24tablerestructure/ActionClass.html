      • addCriteria

        public void addCriteria(TField param)
        Appends the specified Criteria to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Used to define criteria which matching data items only be restructured by service based on actions defined. Validation Rules: Should be in the form of: [fieldname operand value] Where, fieldname should be a valid field name of the table for which restructure definition is being created. Allowed Operands are: EQ, GE, GT, LE, LK, LT, NE, UL, RG, NR Value is mandatory. Null value can be specified as '' or "" This field becomes mandatory for OBSOLETE.DATA.RECORDS action, if no FIELD.VALUE inputted with list of data records to be deleted. Criteria not required for MODIFY.FILE.LEVEL action as it creates required files in database and move entire data from F.[tablename] to master company file.

        Parameters:
        param - the Criteria to append
        Returns:
        nothing

      • getAction

        public TField getAction()
        Returns the Action

        Field description: The type of restructure action required for particular table can be defined here. Only one action allowed per record in T24.TABLE.RESTRUCTURE. Validation Rules: Mandatory input. Allowed values are: 1. FIELD.AMENDMENT :- To define a set of field values of an application can be modified or cleared. 2. SPLIT.RECORD :- New record and its key for same table will be formed through a hook routine attached in TARGET.RECORD.KEY field. 3. OBSOLETE.DATA.RECORDS :- To define a list of specific data records to be deleted for a table or criteria can be defined for the same. 4. MODIFY.KEY :- To define a new key for data items of a table, criteria can be defined to select ony specific list of data items. 5. MODIFY.FILE.LEVEL :- Only INT to FIN level conversion allowed. And all data items from F.[tablename] will be moved to master company file. Restructure service will action these definitions.

        Parameters:
        Returns:
        TField - The Action

      • getTargetRecordKey

        public TField getTargetRecordKey()
        Returns the TargetRecordKey

        Field description: Target record key can be derived from the value of this field. Validation Rules: Input required only for SPLIT.RECORD and MODIFY.KEY actions. Value can be constant or @routine of naming convention RSTR.[tablename].[anyname] where this routine should have valid entry in EB.API. For SPLIT.RECORD action, only @routine allowed with above said naming convention. Restructure service will receive current and target record id and its content from this routine and then write into database. For MODIFY.KEY action, restructure service will receive modified key from this routine for current data record.

        Parameters:
        Returns:
        TField - The TargetRecordKey
