      • add

        public void add(java.lang.CharSequence param)
        Appends the specified UnixBackup to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Unix system backup commands to be used for BACKUP.CYCLE.1 and BACKUP.CYCLE.2 should be entered in this field. Unix system backup commands may be entered in this field to be used with MAIN.ACCOUNT, BACKUP.CYCLE.1 and BACKUP.CYCLE.2 The pipe command may contain a backup device such as '> /dev/rmt1'. The two literals '&M&' or '&B&' if used in this field will be replaced as follows; when the Unix backup is executed: &M& will be replaced by the contents of MAIN.ACCOUNT. &B& will be replaced by the contents of BACKUP.CYCLE.1 or BACKUP.CYCLE.2 depending on whether start of day or start of batch is being run. This field is sub-valued and can be expanded to allow long commands to be entered over 1 or more lines. Use of a sub-values will result in the command being added to that of the previous sub-value. Use of one or more multi-values will result in separate commands. Note.Continuation lines may need a leading space to prevent the resulting command being incorrect Examples To split the unix command over 2 lines 24.1.1 UNIX.BACKUP find &M& -print | cpio -ovcBdum 24.1.2 UNIX.BACKUP > /dev/rmt1 Results in the command To process 2 commands 24.1.1 UNIX.BACKUP find &M&-print | cpio -ovcBdum > /dev/rmt1 24.2.1 UNIX.BACKUP mt -f /dev/rmt1 rewoffl Results in 2 separate commands The shell script 'backup.sh' is built and run at backup time in order to ensure the latest command is used from the authorised SPF record. If you experience trouble with the execution of the backup it can be checked to ensure the commands are correct. It is stored in the 'run' directory. Refer to your Systems Administrator if you wish to check where this is. Note Using a shell script in the command line effectively builds the 'backup.sh' to run that command. However, this can be an advantage if complex or lengthy commands are required locally, also centralised shell scripts can be used. This technique can be used to make disk-to-disk backups (an example 'backup.hd' can normally be found with the demo system we supply). Validation Rules: Up to 50 characters(Optional input)

        Parameters:
        param - the UnixBackup to append
        Returns:
