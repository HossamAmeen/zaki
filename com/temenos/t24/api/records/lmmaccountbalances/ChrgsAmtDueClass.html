      • getAmtRec

        public TField getAmtRec()
        Returns the AmtRec

        Field description: The amount of the charge which has been received. This is the amount of the particular charge which has been received. Usually it will be equal to the CHRGS AMT DUE (due amount) as all charges are liquidated automatically. Exceptionally, if for instance the charge liquidation account is closed, it can be different. The field is always signed and it is always positive. Associated Field Considerations. The multivalue association CHRGS AMT DUE to DATE RECEIVED contains details of charges defined on the contract. It contains of both cash (non amortised) and amortised charges. The associated fields are as follows:- CHRGS AMT DUE CHRGS DUE DATE CHRG CODE AMT REC DATE RECEIVED Validation Rules: Upto 14 AMT (Standard amount format) type characters plus decimal point. Multivalue association: CHRGS AMT DUE to DATE RECEIVED. (For LD.LOANS.AND.DEPOSITS only.)

      • getChrgCode

        public TField getChrgCode()
        Returns the ChrgCode

        Field description: The charge code of the charge. This is the code indicated on the main contract and refers to an entry on LMM CHARGE CONDITIONS. The value in field CATEGORY CODE on LMM CHARGE CONDITIONS indicates which profit and loss account the charges will be posted to. Also indicated is the period over which the contract is to be amortised plus the charge description. Associated Field Considerations. The multivalue association CHRGS AMT DUE to DATE RECEIVED contains details of charges defined on the main contract. It contains details of both cash (non amortised) and amortised charges. The associated fields are as follows:- CHRGS AMT DUE CHRGS DUE DATE CHRG CODE AMT REC DATE RECEIVED Validation Rules: Upto 3 numeric characters. Multivalue association: CHRGS AMT DUE to DATE RECEIVED. (For LD.LOANS.AND.DEPOSITS only.)

      • getChrgsAmtDue

        public TField getChrgsAmtDue()
        Returns the ChrgsAmtDue

        Field description: The amount of the fee or charge which is due. This field represents the amount of the charge which is due on the contract. That is those charges which have a value date prior to or equal to the process date. Each charge on the contract will have a set of multivalues entered into this association on the claim date of the charge. This field is signed and is always negative. Associated Field Considerations. The multivalue association CHRGS AMT DUE to DATE RECEIVED contains details of charges defined on the contract. It contains of both cash (non amortised) and amortised charges. The associated fields are as follows:- CHRGS AMT DUE CHRGS DUE DATE CHRG CODE AMT REC DATE RECEIVED Validation Rules: Upto 14 AMT (Standard amount format) type characters plus decimal point. Multivalue association: CHRGS AMT DUE to DATE RECEIVED. (For LD.LOANS.AND.DEPOSITS only.)
