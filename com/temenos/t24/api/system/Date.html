com.temenos.t24.api.system

Class Date

Packaged in : EB.DateApi.jar

Utility class that provides access to some of the date calculation routines in T24.
Emphasis is on providing access to those routines that can't be replicated with pure java, for example determining whether a date is a working day or not, which is only known by T24.

      • addFrequency

        public String addFrequency(String frequency)
        Cycles forwards to a new frequency based on a given T24 frequency.
        A T24 frequency is an initial date combined with a 'step' value. For example a value like '201711005M0630' the step value is 'M0630' which means 'every six months on the thirtieth (or end of month)' This method is a simple way to calculate the next frequency value.
        It is intended to support java developers who receive such a value and need to increment it, without needing to know about the different types of frequency objects.

        T24 Details: : Uses the CFQ routine in T24.
        Parameters:
        frequency - The T24 frequency value, like '20171105BSNSS'.
        Returns:
        String - A string representing a new frequency in T24.
        Throws:
        T24CoreException if the given frequency is invalid

      • getMonthDifference

        public TNumber getMonthDifference(TDate startDate, TDate endDate)
        Returns the calculated number of complete months between two given dates. For example, 15 Jan - 14 Feb is 0 complete months, but 15 Jan to 15 Feb is 1 complete month. 1 Jan to 28 Feb is also 1 complete month. No distinction is made between working days and holidays in this routine and it would be possible to write a pure java implementation of this routine, which is provided for convenience.
        T24 Details: : Uses the EB.NO.OF.MONTHS routine for the calculation.
        Parameters:
        startDate - The start date.
        endDate - The end date.
        Returns:
        TNumber - The number of complete months between two given dates.

      • getWorkingDayDifference

        public TNumber getWorkingDayDifference(TDate startDate, TDate endDate)
        Returns the calculated number of working days between two given dates. Note that core java libraries also provide this eg the java.time.Duration. Uses the HOLIDAY table and current logged in COMPANY in T24 to determine which days are working days. endDate is not counted in the calculation, for example getting the number of working days between a Monday and a Tuesday will return '1'. Getting the number of working days between Thursday and Tuesday with a 2 day weekend in between will return '3'.
        T24 Details: : Uses the CDD routine in T24.
        Parameters:
        startDate - The start date.
        endDate - The end date.
        Returns:
        TNumber - The calculated number of working days between the given dates.
