      • getSubCatEnd

        public TField getSubCatEnd()
        Returns the SubCatEnd

        Field description: Within each of the four main CONTRACT.TYPE there are user defined sub-types which have their own range of category codes. These must be within the overall range defined for the main CONTRACT.TYPE. The range can either be one category or large enough to cater for the different types that will be created locally. The SUB.CAT.START and SUB.CAT.END define the range. This field defines the end of the range. Validation Rules: 5 numeric characters. Mandatory Input Once the record has been authorised this field cannot be changed. Must be present on the CATEGORY table. Category code ranges cannot overlap each other. Must be within range of the CONTRACT.TYPE (defined by START.CATEGORY- END.CATEGORY)
