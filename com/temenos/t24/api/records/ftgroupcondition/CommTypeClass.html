      • getCommPercent

        public TField getCommPercent()
        Returns the CommPercent

        Field description: Specifies the rate used to calculate the Commission amount as a percentage of the Standard Commission amount. Where groups of customers are given preferential Commission rates this field is used to define what Percentage of the Standard Commission (as defined in Commission Type table) should be applied. Where no Commission is to be charged rate must be input as 0%. For any percentage rate greater than 0, the Application will compare the calculated amount with the Overall Minimum and Maximum amount defined in the Commission Type table. If the Overall Minimum or Maximum amount are greater or less respectively than the calculated amount, then these overall values will be applied regardless. This is to ensure the Bank always either cover their cost or do not charge excessive amounts. Validation Rules: Up to 10 type R characters plus a decimal point (standard rate format). Associated multivalue field. [Mandatory input if COMM.TYPE input and COMM.CCY is blank, otherwise not allowed.] Not more than 3 integers may be entered. Not more than 9 decimals may be entered.

        Parameters:
        Returns:
        TField - The CommPercent

      • getCommType

        public TField getCommType()
        Returns the CommType

        Field description: Defines the Commission Types for which the Group of Customers will be given special conditions. Where a Group of Customers is given special rates of Commission this field defines the Commission Types to which special rates apply. The Commission Type code entered into this field must first be defined in the commission Type table. The 'All' condition will indicate that the Commission percentage is applicable to all Commissions for this selection. For any Commission Type not defined in this field, the Funds Transfer Application will automatically take the normal conditions as for the other Commission Types which are included in the Transaction Type Table. Validation Rules: 2 type A (alphanumeric) character Commission type or ALL Associated multivalue field. (Optional input.)

        Returns:
