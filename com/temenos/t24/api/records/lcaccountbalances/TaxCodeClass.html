        public TField getTaxAccAmt()

        Field description: Contains the tax amount in the currency of the account that the tax is taken from. The field will be calculated by the system and no operator input will be allowed.         public TField getTaxChrgStatus()
        Returns the TaxChrgStatus

        Field description: Code to identify the current status of the tax. Used to identify if the tax has been taken, claimed, or not taken. The system will update this field whenever the tax is processed and no operator input will be allowed. The codes that are used to identify the various states of the tax are as follows: blank - The tax has not been taken yet 2 - The tax has been taken 3 - The tax has been claimed 4 - A claimed tax has been settled 7 - Tax Unpaid Reversed 8 - Tax has been taken by adding the amount to a reimbursement claim via a drawing. 9 - Tax has been taken by deducting the amount from a payment via a drawing. Notes: (i) Tax status's '8' and '9' will arise when the tax status is initially blank, ie the tax has not been taken. Then if a drawing is entered whereby payments are made or reimbursement is required, then the tax amount will be either deducted from the payment amount or added to the reimbursement and credited to the Tax suspense account. Validation Rules: 1 Digit code (see below). System generated. Associated multivalue field associated with fields TAX.CODE through to TAX.DATE.

      • getTaxRelDraw

        public TField getTaxRelDraw()
        Returns the TaxRelDraw

        Field description: Identifies the Letter of Credit or Drawing that this tax relates to. The first two digits in this field will identify either the drawing number if the value is greater than '00' or the Letter of Credit if the value is '00', that this tax relates to. If a '-' and a further two digits follows this number then this will identify the number of the drawing that has generated the tax by either a deduction from the payment amount, or addition to the reimbursement amount. If this field only contains two numeric digits, then this will indicate the drawing number that this tax relates to (i.e. this tax has been generated due to the input/change of this drawing). If the drawing number is appended with a '-' and another drawing number, then this will indicate that the tax was generated by the input/change of the first drawing number, but the entries have been produced for this tax by the second drawing number, by the deduction of the tax from its payment amount, or the addition of the tax to its reimbursement amount. For example, if a letter of credit is input and a number of charges/tax are made for the transaction, but are not taken immediately, i.e. CHARGE.STATUS is blank, then a sight payment drawing is entered under this L/C which will generate payment and reimbursement entries. When a payment has to be made, all outstanding charges/commissions, at that time, will automatically be paid by deducting the relevant amount(s) from the payment to be effected or added to the reimbursement amount and credited to the charge P/L account. So the purpose of the second number in this field is to identify which drawing has generated the entries for the outstanding tax. In order to identify how the tax was paid see the TAX.CHARGE.STATUS field below. Validation Rules: 5 type A (Alphanumeric) characters. System generated. Associated multivalue field associated with fields TAX.CODE through to TAX.DATE. The field will be calculated by the system and no operator input will be allowed.

        Parameters:
        Returns:
        TField - The TaxRelDraw

      • setTaxRelDraw

        public void setTaxRelDraw(java.lang.CharSequence value)
        Set the specified value for TaxRelDraw
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Identifies the Letter of Credit or Drawing that this tax relates to. The first two digits in this field will identify either the drawing number if the value is greater than '00' or the Letter of Credit if the value is '00', that this tax relates to. If a '-' and a further two digits follows this number then this will identify the number of the drawing that has generated the tax by either a deduction from the payment amount, or addition to the reimbursement amount. If this field only contains two numeric digits, then this will indicate the drawing number that this tax relates to (i.e. this tax has been generated due to the input/change of this drawing). If the drawing number is appended with a '-' and another drawing number, then this will indicate that the tax was generated by the input/change of the first drawing number, but the entries have been produced for this tax by the second drawing number, by the deduction of the tax from its payment amount, or the addition of the tax to its reimbursement amount. For example, if a letter of credit is input and a number of charges/tax are made for the transaction, but are not taken immediately, i.e. CHARGE.STATUS is blank, then a sight payment drawing is entered under this L/C which will generate payment and reimbursement entries. When a payment has to be made, all outstanding charges/commissions, at that time, will automatically be paid by deducting the relevant amount(s) from the payment to be effected or added to the reimbursement amount and credited to the charge P/L account. So the purpose of the second number in this field is to identify which drawing has generated the entries for the outstanding tax. In order to identify how the tax was paid see the TAX.CHARGE.STATUS field below. Validation Rules: 5 type A (Alphanumeric) characters. System generated. Associated multivalue field associated with fields TAX.CODE through to TAX.DATE. The field will be calculated by the system and no operator input will be allowed.

        value - The value to set
        Returns:
        nothing
