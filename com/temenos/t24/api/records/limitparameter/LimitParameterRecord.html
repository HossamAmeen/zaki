        public TField getNetOutstanding()
        Returns the NetOutstanding

        Field description: Defines whether, or not, Foreign Exchange contracts are netted before comparison with the respective Limit. If 'no netting' is chosen, the full value of the transaction is compared with the Limit to ensure that the transaction will not cause an excess. If 'netting' is chosen, the System will net transactions in opposite sense which involve the same two currencies, which are for the same Liability Number and which mature on the same date. For example, suppose the following two transactions are in the System:- Buy USD 100,000 Sell GBP 70,000 Value 31/12/86 Sell GBP 35,000 Buy USD 50,000 Value 31/12/86 If 'no netting' is requested, and the Limit is 150,000 US Dollars, after the first transaction, 50,000 US Dollars is available. After the second transaction, the Limit is reduced by the US Dollar equivalent (according to the latest on-line middle rate) of 35,000 GB Pounds. If this gives a figure of 50,000 US Dollars, the Limit availability is reduced to zero. If 'netting' is specified, the second transaction is subtracted from the first (100,000 US Dollars minus the middle rate equivalent of 35,000 GB Pounds), giving a net equivalent value of 50,000 US Dollars. This is then subtracted from the Limit to give an available figure of 100,000 US Dollars. It should be noted that the Limit System always uses the same side of Foreign Exchange transactions (the bought side) for all calculations. Apart from the currency, the sold side is always ignored. Note: It is important to note that the value of this field must be defined at installation. Once the value has been authorised, it can only be changed through LIMIT.CHANGE.       • setNetOutstanding

        public void setNetOutstanding(java.lang.CharSequence value)
        Set the specified value for NetOutstanding
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines whether, or not, Foreign Exchange contracts are netted before comparison with the respective Limit. If 'no netting' is chosen, the full value of the transaction is compared with the Limit to ensure that the transaction will not cause an excess. If 'netting' is chosen, the System will net transactions in opposite sense which involve the same two currencies, which are for the same Liability Number and which mature on the same date. For example, suppose the following two transactions are in the System:- Buy USD 100,000 Sell GBP 70,000 Value 31/12/86 Sell GBP 35,000 Buy USD 50,000 Value 31/12/86 If 'no netting' is requested, and the Limit is 150,000 US Dollars, after the first transaction, 50,000 US Dollars is available. After the second transaction, the Limit is reduced by the US Dollar equivalent (according to the latest on-line middle rate) of 35,000 GB Pounds. If this gives a figure of 50,000 US Dollars, the Limit availability is reduced to zero. If 'netting' is specified, the second transaction is subtracted from the first (100,000 US Dollars minus the middle rate equivalent of 35,000 GB Pounds), giving a net equivalent value of 50,000 US Dollars. This is then subtracted from the Limit to give an available figure of 100,000 US Dollars. It should be noted that the Limit System always uses the same side of Foreign Exchange transactions (the bought side) for all calculations. Apart from the currency, the sold side is always ignored. Note: It is important to note that the value of this field must be defined at installation. Once the value has been authorised, it can only be changed through LIMIT.CHANGE. Validation Rules: N for no netting or Y for netting. (Mandatory input.)

        value - The value to set
        Returns:
