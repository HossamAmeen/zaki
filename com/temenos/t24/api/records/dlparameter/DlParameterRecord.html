      • getOperation

        public TField getOperation()
        Returns the Operation

        Field description: Defines whether data is to be "S" (saved) or "R" (restored). This field is only used when the record is "V"erified. When the program is run in "V"erify mode, if the operation is set to "S" (save), the data library is saved to tape. The tape device used is specified in TAPE.DEVICE. If units are specified in SAVE.UNITS, only those units are saved to tape; otherwise the whole data library (F.DL.DATA) is saved to tape. The data library or specific units are saved to tape by the program executing the shell script, "dl_save.sh" which is created when the "V"erify mode is taken. When the program is run in "V"erify mode with the operation set to "R" (restore), units are copied from the data library specified in FROM.FILE into F.DL.DATA, creating the unit definition record on F.DL.DEFINE. The units which are copied are specified in RESTORE.UNITS. If RESTORE.UNITS is blank, all units in the from file are copied. The OVERWRITING field is used when the units to be copied already exist in F.DL.DATA. If OVERWRITING is set to "Y", the units in F.DL.DATA are overwritten; if OVERWRITING is set to "N", the units in F.DL.DATA are not overwritten; if OVERWRITING is set to "L" (latest), the units are only overwritten if they were saved in the from data library after the date they were saved in the to data library. (The default setting of OVERWRITING is "L".) OPERATION does not default: it must be entered if the "V"erify mode is taken. Validation Rules: "S" (save) or "R" (restore) Optional input Mandatory input when the "V"erify mode is taken

        Returns:

      • getOverwriting

        public TField getOverwriting()
        Returns the Overwriting

        Field description: When data units are being restored from the from data library into F.DL.DATA, OVERWRITING is used to specify whether existing units in F.DL.DATA should be overwritten. If OPERATION is "Y", units will be copied from the from data library, overwriting the units in F.DL.DATA if they already exist. If OPERATION is "N", units will only be copied from the from data library if they do not already exist in F.DL.DATA. If OPERATION is "L" (latest), units will only be copied from the from data library if they were saved in the from file after the date on which they were saved in F.DL.DATA. NOTE : Units in F.DL.DATA are only overwritten if the criteria specified in OPERATION are matched and the unit is to be copied from the from data library. Units in F.DL.DATA which are not being copied from the from data library will remain unchanged. e.g. F.DL.DATA contains two units: TMNS000-NL.REPORTING and TMNS000-CH.REPORTING. All units are to be copied from the data library specified in FROM.FILE, overwriting the records in F.DL.DATA (OPERATION = "Y"). If the from data library contains TMNS000-CH.REPORTING and TMNS000-PD.ADVICES, TMNS000-CH.REPORTING will be copied into F.DL.DATA, overwriting the unit which was already there; TMNS000-PD.ADVICES will be copied, creating a new unit; TMNS000-NL.REPORTING will remain unchanged. Validation Rules: Valid values are "Y", "N" and "L" (latest) Optional input If the operation is "R", defaults to "L" (latest)

      • getTapeDevice

        public TField getTapeDevice()
        Returns the TapeDevice

        Field description: Specifies the names of the tape device to which the data library or specific units are to be saved to. When copying units from the data library to tape, F.DL.DATA, either all units can be copied or just specific units. If all units are to be copied (i.e. SAVE.UNITS is blank), the whole of F.DL.DATA will be copied to tape; if specific units are to be copied, only the units specified in SAVE.UNITS will be copied to tape. When the "V"erify mode is taken, a shell script, "dl_save.sh" will be created to save the data library/units. This shell script will then be executed to save the file/units. Examples: Shell script to save the whole file to tape: cd .. find F.DL.DATA -print | cpio -ovcBdum > /dev/rmt1 (TAPE.DEVICE has been specified as "/dev/rmt1"; SAVE.UNITS has been left blank) Shell script to save specific units to tape: cd .. find 'F.DL.DATA/TMNS000-NL.REPORTING' 'F.DL.DATA/TMNS000-PD.ADVICES' -print |cpio -ovcBdum > /dev/rmt0.4 (TAPE.DEVICE has been specified as "/dev/rmt0.4"; two values have been entered into SAVE.UNITS: TMNS000-NL.REPORTING and TMNS000-PD.ADVICES) Validation Rules: Can only be entered if the OPERATION flag is set to "S" (save) If entered, must reference a unit in the data library (F.DL.DATA) Mandatory input if OPERATION is "S" (save) 1-50 Type "A" (alphanumeric) characters

If all units are to be copied (i.e. SAVE.UNITS is blank), the whole of F.DL.DATA will be copied to tape; if specific units are to be copied, only the units specified in SAVE.UNITS will be copied to tape. When the "V"erify mode is taken, a shell script, "dl_save.sh" will be created to save the data library/units. This shell script will then be executed to save the file/units. Examples: Shell script to save the whole file to tape: cd .. find F.DL.DATA -print | cpio -ovcBdum > /dev/rmt1 (TAPE.DEVICE has been specified as "/dev/rmt1"; SAVE.UNITS has been left blank) Shell script to save specific units to tape: cd .. find 'F.DL.DATA/TMNS000-NL.REPORTING' 'F.DL.DATA/TMNS000-PD.ADVICES' -print |cpio -ovcBdum > /dev/rmt0.4 (TAPE.DEVICE has been specified as "/dev/rmt0.4"; two values have been entered into SAVE.UNITS: TMNS000-NL.REPORTING and TMNS000-PD.ADVICES) Validation Rules: Can only be entered if the OPERATION flag is set to "S" (save) If entered, must reference a unit in the data library (F.DL.DATA) Mandatory input if OPERATION is "S" (save) 1-50 Type "A" (alphanumeric) characters
