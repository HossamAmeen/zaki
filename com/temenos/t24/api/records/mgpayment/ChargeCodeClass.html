        public TField getChargeAccount()
        Returns the ChargeAccount

        Field description: Identifies the account to which the charge amount entered in the associated CHARGE.AMOUNT field is to be debited.         public TField getChargeAmount()

      • getChargeCode

        public TField getChargeCode()
        Returns the ChargeCode

        Field description: Identifies the type of charge(s) to be levied as defined on the FT.CHARGE.TYPE table. This field identifies the charge types to be levied in respect of this payment. This code identifies a charge type defined on the FT.CHARGE.TYPE table. This table contains a record for each charge type available. Each record contains a description, a default charge amount and the P&L category which is to be credited whenever the charge is levied. The description from the FT.CHARGE.TYPE will be displayed as an enrichment to this field and the default amount from the FLAT.AMOUNT field will be used if nothing is entered in the associated CHARGE.AMOUNT field. Whenever a charge is processed the account entered in the associated CHARGE.ACCOUNT field will be debited. The P&L category defined on the FT.CHARGE.TYPE record for the charge code will be credited. The value date of charge entries is the same as that of the payment. Validation Rules: 1 - 2 alphanumeric characters. Multivalue field associated with fields CHARGE.CODE to CHARGE.ACCOUNT. (optional input) Must be a valid charge code on the FT.CHARGE.TYPE table. The same combination of charge code and charge date cannot be duplicated in this set of fields.

        Parameters:
        Returns:
        TField - The ChargeCode
