      • getCollatRight

        public TField getCollatRight()
        Returns the CollatRight

        Field description: The underlying Collateral Right(s) belonging to the limit. Indicates the Collateral Right(s) which underly the limit. This field is only maintained by the system for secured limits. It is updated whenever the relationship between the limit record and the collateral records is altered (for example by user-input of a new limit reference against the collateral, expiry of the collateral, etc). The associated value of the underlying collateral is maintained under field 24, SECURED AMT. Note: The multivalues within this field correspond to the multivalues in fields 18 & 19, COLLATERAL CODE & MAXIMUM SECURED. The Collateral Right id's are therefore grouped according to the defined set of Collateral Codes for the secured limit, if any. Other types of Collateral Right are shown seperately. Sub- values in this field correspond to sub-groupings within this arrangement. Validation Rules: 1 to 9-character Collateral Right id (1 to 6-digit Customer id; period; 1 to 2-digit sequence number). (No input, system-generated field) Multivalue & sub-value field associated with field 24.

        Parameters:
        Returns:

      • getSecuredAmt

        public TField getSecuredAmt()
        Returns the SecuredAmt

        Field description: The underlying secured amount(s) belonging to the limit. Indicates the actual secured amounts which underly the limit. This field is only maintained by the system for secured limits. It is updated whenever the value of an underlying collateral changes in such a way as to impact the limit. In the case of user-input to the Collateral record, this update is performed on-line. In the case of balance movements on an account, changes to the value of a portfolio etc., the updated collateral values are registered at end-of-day. The corresponding Collateral Right id is indicated in field 23, COLLAT RIGHT. Validation Rules: 1 to 19 type AMT (standard amount format). (No input, system-generated field) Multivalue & sub-value field associated with field 23.
