      • getOverdrawnAction

        public TField getOverdrawnAction()
        Returns the OverdrawnAction

        Field description: Indicates the action to be performed by the system for the given activity / activity class, in the event of insufficient funds while performing balance check. Has 3 values • Post – Would reduce account balance & exception record is created for funds decision by user. • Reject – Transaction rejected at source and would not be posted in T24. No charge assessed. • Settle – Transaction is paid and updates account balance. Exception is logged as system decision and charge is assessed. Validation Rules Not allowed to input if CREDIT.CHECK field value is not equal to "COMPONENT". Not allowed to input when activity class/on activity is not specified. Must be input for first multi value as default setup if Overdraft Processing flag is set. Not allowed to input if Overdraft Processing flag is not set.
