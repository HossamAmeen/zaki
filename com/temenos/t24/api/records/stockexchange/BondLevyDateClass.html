      • getBondLevyDate

        public TField getBondLevyDate()
        Returns the BondLevyDate

        Field description: This field and the fields BR.BOND.CNTY to CU.BOND.COMM associated with it allows the used to define any stock exchange levies that will be charged by this Stock Exchange, EBV fees, PTM levy etc. They will be used to update the CU.EBV.FEES or BR.EBV.FEES fields on a SEC.TRADE that uses this Stock Exchange. This set of fields will be used to define the FT.COMMISSION.TYPE records to be used for these fees for bonds. The fields SHARE.LEVY.DATE to CU.SHR.COMM do the equivalent for shares. A bond is defined in T24 as a Security Master record where the BOND.OR.SHARE flag is set to 'B'. This set of flags works as follows; The system will check whether we are dealing with a bond or a share to work out which set of fields to use. It will then take the trade date and compare it against the set of dates in this field until it finds the one that is less than the trade date. It will then decide if we are calculating a customer levy (CU.EBV.FEES) or a broker levy (BR.EBV.FEES). If it is a broker levy it will use BR.BOND.CNTY and BR.BOND.COMM if it is a customer CU.BOND.CNTY and CU.BOND.COMM. It will then take the security domicile of the bond and look for this country in the BR.BOND.CNTY (or CU.BOND.CNTY) field. If it locates it, it will take the associated FT.COMMISSION.TYPE ID from the BR.BOND.COMM (or CU.BOND.COMM) field and use it to calculate the levy. If it does not find the country code then it will look for 'ALL' in the BR.BOND.CNTY(or CU.BOND.CNTY) field and if it is there use the FT.COMMISSION.TYPE record associated with it. Validation Rules: Optional Input. Any input must be of a standard T24 date format.
