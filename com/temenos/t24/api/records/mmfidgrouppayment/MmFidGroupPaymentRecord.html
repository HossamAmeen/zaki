      • getIntSetDate

        public TField getIntSetDate()
        Returns the IntSetDate

        Field description: This field holds the interest settlement date for the current interest period and an optional frequency for the automatic renewal of this date. Input of a date in this field will trigger the calculation of interest and commission for call/notice contracts associated with this fiduciary group payment record. The calculation is performed by the end of day routine on the date entered. The interest settlement date will be changed automatically by the period indicated by the frequency code whenever the date equates to the end-of-day process date. The following are examples of input into this field: a) 310388 M0131 - represents the date from which the frequency is initiated, where: M = month 01 = at one month intervals 31 = the last day of the month. b) 310388 M0331 - represents a quarterly interest due date frequency payable at quarter end. Thus, in example a), interest will be liquidated on 31/03/88 and then on 30/04/88 and so on. In example b), interest will be liquidated on 31/03/88 and then on 30/06/88 and so on. Examples: Field Input 2.1 INTEREST RATE 1.0% 3.1 EFF DATE 10 JAN 1988 2.2 INTEREST RATE 1.5% 3.2 EFF DATE 15 JAN 1988 4 INT SET DATE 30 JAN 1988 5 LAST INT SET DATE 05 JAN 1988 6 LAST RATE APPLIED 0.75% For a contract with a principal of 1,000,000, with the interest basis E (366/365) and a value date prior to the last interest settlement date, then the calculation of interest for the period would be as follows:- a) from 05 JAN to 09 JAN 1,000,000 x 5 x 0.75% = 102.74 36500 b) from 10 JAN to 14 JAN 1,000,000 x 5 x 1.0% = 136.99 36500 c) from 15 JAN to 29 JAN 1,000,000 x 15 x 1.5% = 616.44 36500 The total interest due for the period is therefore: 102.74 + 136.99 + 616.44 = 856.17 Validation Rules: Two part input consisting of a date and a frequency. Both are optional and may be entered independently or as a combination. Numeric Date Format. The format of the frequency is type 'AA'. The date input must be greater than the previous interest settlement date in Field 5 LAST.INT.SET.DATE. Where interest is to be paid via a nostro account, the frequency part of this field must be greater than the days delivery of the payment currency. For example, if DEM payments are sent two days prior to their value date (DAYS.DELIVERY field in CURRENCY record DEM has a value of 2), then a DAILY frequency will not be allowed for a DEM Call/Notice deposit that has a nostro account as the interest liquidation account. For deposits, this is to ensure that payments are not made for future interest periods that have yet to begin. For placements, this is to ensure that 'advice to receives' are not sent for future interest periods that have yet to begin.

      • getLastIntSetDate

        public TField getLastIntSetDate()
        Returns the LastIntSetDate

        Field description: This field contains the date on which interest was last settled. When interest is settled an end of day routine creates a new fiduciary group record for the new interest period. The settlement date from the old record is stored in this field. This is used to validate any interest settlement date entered on the new record to ensure it is greater than the previous one. Validation Rules: Internal field.

      • getLastRateApplied

        public TField getLastRateApplied()
        Returns the LastRateApplied

        Field description: This field contains the last interest rate form the previous interest period. When interest is settled an end of day routine creates a new fiduciary group record for the new interest period. The last rate applied on the old record is copied into this field. This rate will then apply to the new period up to the first date entered in field 3 EFF.DATE at which point the associated rate in field 2 INTEREST.RATE will come into effect. See field 4 INT.SET.DATE for an example. Validation Rules: Internal field.

        public void setLastRateApplied(TField value)
        Set the specified value for LastRateApplied
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field contains the last interest rate form the previous interest period. When interest is settled an end of day routine creates a new fiduciary group record for the new interest period. The last rate applied on the old record is copied into this field. This rate will then apply to the new period up to the first date entered in field 3 EFF.DATE at which point the associated rate in field 2 INTEREST.RATE will come into effect. See field 4 INT.SET.DATE for an example. Validation Rules: Internal field.
