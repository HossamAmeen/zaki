      • setUpdateFields

        public void setUpdateFields(java.lang.CharSequence param, int idx)
        Set the specified UpdateFields to a specific position in collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Used to specify the valid field name(s) from the application entered in TABLE.NAME field. The event xml generated will have an OFS message with just these fields instead of the full record. Validation Rules: Value allowed only for REQUEST.TYPE=UPDATE. Allows only the standard fields of an application which are eligible for input. Fields which are not allowed, though may exist in the standard selection record: a. NOINPUT and EXTERN fields b. Key fields (ID type fields) c. Fields with type S, P , I ,J etc. which are not real fields for input in application page. When expandable (Multi-value or sub-value type ) fields are entered in UPDATE.FIELDS, an override message will be given. An override class for this can be set-up, if required, to keep control of the impact of updating these fields. Since this table is used to manage tables on tenant systems which should not be updated by the tenants themselves it is expected that the format and content of the tenant records are known. However, in the event that the tables have been modified locally then only a full record replacement will give the same value as the master record. Updating multi-value or sub-value fields, where the tenant record has those fields modified locally, may not result in an exact duplication on the tenant record. For this reason an override is given to remind (or prevent) such an update.
