      • getAccrueAmort

        public TField getAccrueAmort()
        Returns the AccrueAmort

        Field description: Optional input. Indicates whether the calculated charge is to be accrued or amortised. If a charge is to be amortised the P&L entry will be changed by the core accouting to a CRF CAP entry at the time the charge is booked. The P&L will than be amortised by the core accrual pocess up to the next charge date. If accrual is to take place the projected charge will be accrued over the next charge period and the P&L entry at the next charge date will also become a CRF CAP ENTRY. Any difference between the projected charge and the actual charge will result in an accrual adjustment at the end of the accrual period, i.e. when the customer account is debited.

      • getCalStepPeriod

        public TField getCalStepPeriod()
        Returns the CalStepPeriod

        Field description: Defines the calculation period for generic charge products . This field specifies whether the charges defined in the charge products should be calculated once to cover the full charge period, as specified in the field CHRG.FREQUENCY or in monthly steps. If ' M' (monthly) is specified the charges are calculated every month and the total charges is posted on the capitalisation date. If ' P' (periodic) is specified the charges are calculated for the whole capitalisation period. Defaults to 'P' Validation Rules: M (monthly) or P (periodic) Input not allowed when WAIVE.ALL has value.

        Parameters:
        Returns:
        TField - The CalStepPeriod

        public TField getChrgEffDate()
        Returns the ChrgEffDate

        Field description: This field defines the date from which the generic charge is effective. Defaults to Today. Needs to have value when IC.CHG.PRODUCT has value. Existing value cannot be changed when IC.CHG.PRODUCT has value. Associated with the fields - IC.CHG.PRODUCT, CAL.STEP.PERIOD and CHRG.FREQUENCY . Input not allowed when WAIVE.ALL has value. Validation Rules: Standard T24 date field.

      • getChrgFrequency

        public TField getChrgFrequency()
        Returns the ChrgFrequency

        Field description: Defines the generic charge capitalising frequency that is the charge calculation period. This is a standard T24 frquency field, so charges can be capitalised at any time, though charges are normally capitalised monthly , quarterly, semi anually or annually.Needs to have a value when the field IC.CHG.PRODUCT has value. Care should be taken that the first date in the CHRG.FREQUENCY field is not less that the value in the CHRG.EFF.DATE field. if this occurs then charges will not be capitalised until the frequency field is changed. Associated with the fields IC.CHG.PRODUCT , CAL.STEP.PERIOD,CHRG.EFF.DATE. Input not allowed when WAIVE.ALL has value. Validation Rules: Standard T24 frequency field.
