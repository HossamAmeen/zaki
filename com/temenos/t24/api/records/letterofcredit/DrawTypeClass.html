      • getPaymentPct

        public TField getPaymentPct()
        Returns the PaymentPct

        Field description: This field represents the percentage amount of LC that is available for a particular DRAW.TYPE. For LC.TYPES with P & A defined in PAY.TYPE (within LC.TYPES) this shall be defaulted to 100 by the system. For MIXED PAYMENT, the user shall input the appropriate percentage and also the DRAW.TYPE. This is mutually exclusive to the field "PAY.PORTION". As the percentage is input the PAY.PORTION amount is calculated and stored. For revolving letters of credit this field is system generated and will hold a value in proportion to the total liability. For example, if a revolving letter of credit is opened for US$100,000 with number or revolutions 3, then this field will hold a value of 25% ($100,000*4 = $400,000, which is equal to 100%, so the lc amount will be shown as 25%)

      • getPortLimRef

        public TField getPortLimRef()
        Returns the PortLimRef

        Field description: This field displays the limit reference number to be checked for limit control on LC transactions. A check will be done on the limit parameter to update the correct reference. In case of mixed payment this will refer to the limit reference to be updated for the portion concerned. Each LC transaction will be checked against the LIMIT file to verify the availability of a limit facility. This information was formerly held in LIMIT.REFERENCE. (G10.2) If no input is made in this field, the LIMIT system will provide a default according to the values defined in the LIMIT.PARAMETER file. If required, more details can be found in the LIMIT user guide. Note: The customer that limits will be checked against is the customer defined in CON.CUS.LINK. Validation Rules: 3-7 numeric characters Input to this field must be a valid LIMIT.REFERENCE number If no limit exists for the customer, a corresponding error message will be displayed by the system with an override facility.

        Returns:

      • getProOsAmt

        public TField getProOsAmt()
        Returns the ProOsAmt

        Field description: System maintained field This field displays the provision amount outstanding. This is calculated based on the provision percentage and/or the PROVISION.AMOUNT input by the user. For mixed payment LCs, PAYMENT.PCT will be taken into account in addition to provision percentage or provision amount and the respective provision outstanding amount will be calculated. EXAMPLE Let us assume a mixed payment for LC for GBP 100,000.

        DRAW.TYPE SP50%50,000
        DRAW.TYPE AC30 50% 50,000
        Let us assume a provision percentage of 50% with provision credit account equal to GBP account. Then PRO.OS.AMT will be 25,000 each under DRAW.TYPE SP and AC30. If provision amount is entered instead of %, say 40,000 then this will be divided into two parts of 50% each and 20,000 will be stored in each DRAW.TYPES namely SP and AC30.

        Returns:
        TField - The ProOsAmt

      • setProOsAmt

        public void setProOsAmt(java.lang.CharSequence value)
        Set the specified value for ProOsAmt
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: System maintained field This field displays the provision amount outstanding. This is calculated based on the provision percentage and/or the PROVISION.AMOUNT input by the user. For mixed payment LCs, PAYMENT.PCT will be taken into account in addition to provision percentage or provision amount and the respective provision outstanding amount will be calculated. EXAMPLE Let us assume a mixed payment for LC for GBP 100,000.

        DRAW.TYPE SP50%50,000
        DRAW.TYPE AC30 50% 50,000
        Let us assume a provision percentage of 50% with provision credit account equal to GBP account. Then PRO.OS.AMT will be 25,000 each under DRAW.TYPE SP and AC30. If provision amount is entered instead of %, say 40,000 then this will be divided into two parts of 50% each and 20,000 will be stored in each DRAW.TYPES namely SP and AC30.

        Parameters:
        value - The value to set
        Returns:
        nothing
