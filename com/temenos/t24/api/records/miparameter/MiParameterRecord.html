        public void addBalMvmtApps(TField param)
        Appends the specified BalMvmtApps to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies for which Applications the BALANCE.MOVEMENT related data needs to be maintained. A value of ALL would specify that the BALANCE.MOVEMENT related data would be maintained for all MI relevant Applications. Even though the field would accept any valid EB.SYSTEM.ID, only meaningful EB.SYSTEM.IDs, for which BALANCE.MOVEMENT data would be relevant, should be specified. It would be possible to generate BALANCE.MOVEMENT records either during Batch or on-line processing, only for the Applications specified here. Validation Rules: ID of a valid EB.SYSTEM.ID record. If ALL is specified, other values would not be accepted. Duplicates are not allowed.

        Parameters:
        param - the BalMvmtApps to append
        Returns:
        nothing

      • getBuildBdBalances

        public TField getBuildBdBalances()
        Returns the BuildBdBalances

        Field description: Specifies whether Book-Date related fields in BALANCE.MOVMENT (Fields: OPEN.BD.BALANCE to BK.AVG.BAL.CAL) records of Accounts have to be populated or not. When the field value is set to YES, Book-Date related fields of BALANCE.MOVEMENT records of Accounts would be populated based on the BOOKING.DATE in STMT.ENTRY records. If the STMT.ENTRY records specify an ACCOUNTING.DATE (when the Post Closing Module is installed), then these fields would be updated based on the earliest PC.PERIOD in STMT.ENTRY records, provided the entries have been applied to Post Closing Database. When the field value is set to NO, Book-Date related fields of BALANCE.MOVEMENT records would not be populated. Validation Rules: Allowed values are YES and NO. Default value is Null equivalent to NO. Once the value is set to YES, it cannot be changed.

        Parameters:
        Returns:
        TField - The BuildBdBalances

      • getBuildPlEntries

        public TField getBuildPlEntries()
        Returns the BuildPlEntries

        Field description: Specifies whether the CATEG.ENTRY records are to be consolidated in MI Database by BOOKING.DATE or ACCOUNTING.DATE. ACCOUNTING.DATE for CATEG.ENTRY records could be specified only for records generated by entries input through DATA.CAPTURE application, provided the POST CLOSING module is installed. If an ACCOUNTING.DATE is specified in CATEG.ENTRY records, consolidation would depend on the date of the earliest Post Closing (PC) Database updated (i.e. earliest PC.PERIOD.END in the entries, provided the field PC.APPLIED=Y). Validation Rules: Allowed values are BOOKING and ACCOUNTING. Default value is Null when the CATEG.ENTRY records would be consolidated in MI Database based on BOOKING.DATE. Change of value not allowed from ACCOUNTING.

        Returns:
        TField - The BuildPlEntries
