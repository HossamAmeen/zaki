      • getScClassType

        public TField getScClassType()
        Returns the ScClassType

        Field description: SC.CLASS.TYPE and EB.CLASS.NO are used to formulate the class mapping between SC internal class andEB.MESSAGE.CLASS. The main driving concept of ACTIVITY and CLASS has lend itself to facilitate the user byautomatically trigger only the message related to the operation, which user are currently do. In a sense, thesystem will try to default a sensible CLASS tiding to corresponding activity, and populate the information on theSC transaction. Consequently, the user can take control of the class to choose whether he/she wants to produce asper suggestion by the system. The following types are valid types: + BASIC.MESSAGE + NEW.MESSAGE + MODIFICATION + CANCELLATION + USER.DEFINE1 + USER.DEFINE2 + USER.DEFINE9 NEW.MESSAGE will be triggered for new SEC.OPEN.ORDER where CONFIRMATION is set to YES MODIFICATION will be triggered If there is a change in AMT.TO.BROKER field between R.NEW and R.OLD ofSEC.OPEN.ORDER CANCELLATION will be triggered when SEC.OPEN.ORDER is reversed or BROKER is deleted from SEC.OPEN.ORDER If message class needs to be triggered irrespective of any condition then, message BASIC.MESSAGE andUSER.DEFINE1 to USER.DEFINE9 will be used. Validation Rules: Only the following values are allowed BASIC.MESSAGE NEW.MESSAGE MODIFICATION CANCELLATION USER.DEFINE1 USER.DEFINE2 USER.DEFINE3 USER.DEFINE4 USER.DEFINE5 USER.DEFINE6 USER.DEFINE7 USER.DEFINE8 USER.DEFINE9
