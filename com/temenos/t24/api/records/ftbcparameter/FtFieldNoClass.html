        public TField getFtFieldNo()
        Returns the FtFieldNo

        Field description: Defines which field in the Funds Transfer application the specification in fields FT.FIELD.NO to COMP.CHECKFILE apply. Entry of a field determines whether there are any special validation rules for the field. The absence of details for a particular field means that the usual field definitions are to be used in online and cross validation. This field is associated with field range FT.FIELD.NO to COMP.CHECKFILE. Validation Rules: 1-2 numeric characters. (Mandatory input) No duplicates allowed. Input may be one of the following fields : DEBIT AMOUNT DEBIT VALUE DATE DEBIT THEIR REF CREDIT THEIR REF CREDIT VALUE DATE TREASURY RATE PROCESSING DATE ORDERING CUST ORDERING BANK ACCT WITH BANK BEN ACCT NO BEN CUSTOMER BEN BANK CHEQUE NUMBER PAYMENT DETAILS BC BANK SORT CODE CHARGES ACCT NO COMMISSION CODE COMMISSION TYPE COMMISSION AMT CHARGE CODE CHARGE TYPE CHARGE AMT PROFIT CENTRE CUST PROFIT CENTRE DEPT

        public TField getMaxLength()
        Returns the MaxLength

        Field description: Determines the maximum length of input in the associated FT.FIELD.NO. This field should contain the maximium length allowed on the corresponding local clearing message field. Validation Rules: Numeric in the range 1-99. Input is not allowed in this field when the associated NO.LINES is NONE. The value entered may not exceed the maximum length of the Funds Transfer field. The default value if NO.LINES is not NONE will be the maximum field length in Funds Transfer.

      • getNoLines

        public TField getNoLines()
        Returns the NoLines

        Field description: Defines the maximum number of lines which the associated FT.FIELD.NO may be expanded to. The value entered in this field should correspond to the maximum number of lines for the corresponding field in the local clearing message. The FT.FIELD.NO can be made a NOINPUT field by entering NONE in this field. Validation Rules: Numeric in the range 1-9 or NONE. Multivalue field. (Mandatory input when FT.FIELD.NO is input.) If the FT.FIELD.NO is a single valued field in Funds Transfer either 1 or NONE can be entered.
