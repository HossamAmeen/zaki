        public TField getDocumentType()
        Returns the DocumentType

        Field description: Specifies the Document types which need to be tracked for the group. Validation Rules: Input optional. Multi value field. Must be the ID of a valid record in the file DOCUMENT.TYPE. For customer level documents (First part of the ID refers to CUSTOMER application), value of the field CONTRACT.SPECIFIC in DOCUMENT.TYPE should be NO. (i.e. cannot specify a Contract specific document as a required document for Customer input). For application level documents (First part of ID refers to any application other than CUSTOMER), value of the field NEW.ALL.CUST in DOCUMENT.TYPE should be NO or null.

        public TField getStopTxn()

        Field description: Specifies whether a transaction needs to be stopped, if a required Mandatory document exists with an invalid status or confirmed by the User as "Not Received" for T24 confirmation messages whether a Document has been received.       • getStopTxnDays

        public TField getStopTxnDays()
        Returns the StopTxnDays

        Field description: Specifies the Number of days (calculated from a document's Status Date in the file CUST.DOCUMENT or TRANS.DOCUMENT) after which a required Mandatory document with an invalid status, would stop a transaction input. Example DOCUMENT.REQUIRED ID-SETR*101 (For document group 101 of SEC.TRADE) DOCUMENT.TYPE- AGREEMENT MANDATORY-YES STOP.TXN-YES STOP.TXN.DAYS-10 The Document AGREEMENT is recorded in the file CUST.DOCUMENT for Customer ID 1000 with an invalid status and a Status Date of 4.6.2002. Any SEC.TRADE input, for which Document AGREEMENT is required from Customer ID 1000 would be allowed only upto 13.6.2002 (10 days from the Status Date). Any SEC.TRADE input, for which Document AGREEMENT is required from Customer ID 1000 would not be allowed after 13.6.2002. Validation Rules: Optional. Multi value field associated with field DOCUMENT.TYPE. Input allowed only if the value of the field STOP.TXN is YES. 1-3 Numeric characters followed by D in the format "xxxD". If the value of the field STOP.TXN is YES and that of STOP.TXN.DAYS is Null, then transactions which require the Document would be immeidately stopped.

        Parameters:
        Returns:
        TField - The StopTxnDays

      • getTxnProcessing

        public TField getTxnProcessing()
        Returns the TxnProcessing

        Field description: Specifies whether the document type needs to be tracked for transactions input from a related application (if value is YES) or not (if value is NO). Example DOCUMENT.REQUIRED ID: AC*101 (Document Group for the ACCOUNT application) DOCUMENT.TYPE: ELECSTMT (Electronic Statement Agreement) MANDATORY: YES TXN.PROCESSING: NO The document type ELECSTMT would be tracked whenever an Account input belongs to the Document Group 101. For a Loan input (in LD Application), ACCOUNT is defined as a related application and the appropriate Account belongs to Document Group 101. Since TXN.PROCESSING is set to NO, the document ELECSTMT would not be tracked when a loan is input in LD application. Validation Rules: Mandatory if the value of the field MANDATORY is YES. Allowed values YES/NO and defaults to YES. Multi value field associated with field DOCUMENT.TYPE.

        Parameters:
        Returns:
        TField - The TxnProcessing
