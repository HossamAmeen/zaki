      • getReferenceNo

        public TField getReferenceNo()
        Returns the ReferenceNo

        Field description: Multivalue field used to store the Security Transaction Reference number created when the dealer executes thewhole or a partial amount of an Order. Fields 22 to 28 contain details of a partially completed Open Order. They are all Multivalue and are used tostore the Reference number,the amount filled, the price paid or received and the trade date of each transaction. Note: Just as the Security Trade reference number is stored at this level as a full audit check and for easewhen following a transaction through so to the Open Order reference number is passed onto the Security Trade itselfand stored within field 45.1.1 CU.ORDER.NOS. Validation Rules Standard Security Buy /Sell Transaction Reference Number which consists of the Module Mnemonic SECTSC suffixseparated from 10 numeric characters yydddnnnnn Where yy = year (e.g 85 = 1985) ddd = julian date ( i.e. 01/jan =001, 02/jan = 002, 01/feb = 032,etc (Automatic Default)
