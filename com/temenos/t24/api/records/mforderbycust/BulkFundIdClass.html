      • addBulkCrTxn

        public void addBulkCrTxn(java.lang.CharSequence param)
        Appends the specified BulkCrTxn to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Field to determine the purchase transaction towards which the redemption would be allocated. Input allowed only for Debit TRANS.TYPE (redemptions). In case of redemptions, the same would be allocated towards the credit transaction specified. If the number of units in the Credit Transaction is more than the redemption amount, then units would be redeemed only to the extent of the redemption amount and balance units would remain. If the credit transaction units are not sufficient to cover the redemption amount, then the redemptions would be allocated towards the remaining Credit transactions on FIFO basis. If there are no other credit transactions, the redemption would only be executed to the extent of units available. If the order in which units are to be redeemed is not provided here, the redemptions would be allocated on a FIFO basis. This field could be sub-valued to specify multiple credit transactions. Validation Rules: Valid MF.TRADE ID that pertains to a purchase for the fund and by the customer in the order. Fields from BULK.FUND.ID to BULK.CR.TXN would be associated multi value fields. Input would be allowed only if ORDER.TYPE is 'BULK'

        Parameters:
        param - the BulkCrTxn to append
        Returns:
        nothing
