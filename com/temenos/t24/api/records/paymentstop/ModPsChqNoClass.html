      • getModPsChqNo

        public TField getModPsChqNo()
        Returns the ModPsChqNo

        Field description: In this field the cheque number is entered - for which the payment stop instructions are to be REVOKED. Individual Cheque Number alone can be given here to revoke single payment stop instruction or to revoke for a range of cheques - specify the start and end range of cheque Numbers (ie to revoke stop payment for cheque no 1001 to 1010 - specify here 1001-1010) Validation Rules: The cheque number already instructed for stop payment can only be input here.

        Returns:

      • getModPsDate

        public TField getModPsDate()
        Returns the ModPsDate

        Field description: The date entered here is the date from which the payment stop instructions stands withdrawn. When a future date is given here, the stop payment instruction is revoked only on that date. For this purpose PS.STOP-live file is updated and same is referred during SOD batch processing for revocation of payment stop. The cheque number and the type of cheque is stated in the MOD. PS. CHQ. NO and MOD. CHQ. TYPE Validation rules Valid date format Should be a date equal or greater than the System date.
