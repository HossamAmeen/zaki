      • getAccruedInt

        public TField getAccruedInt()
        Returns the AccruedInt

        Field description: This field contains the Consolidated amount of interest (income) that has been accrued for the Group but not yet been capitalised. If the position is for an account (AC) then this field will contain the sum of the ACCR.CR.AMOUNT, ACCR.CR2.AMOUNT, ACCR.DR.AMOUNT and ACCR.DR2.AMOUNT fields of all Account records for the group. If the position is for a Money Market (MM) contract then if the contract is a deposit then this field will contain the consolidated amount from the OUTS.CUR.ACC.I.PAY from the LMM.ACCOUNT.BALANCES records that represent this group. If the contract is a loan then this field will contain the OUTS.ACCRUED.INT field from the LMM.ACCOUNT.BALANCES records for the Group. If the position represents a Fiduciary contract (FD) then this field will be updated from the OTS.INTEREST field on the FD.BALANCES record for all the contracts that constitute this Group. If the position represents a FOREX (FX) contract or a Forward Rate Agreement then this field will contain 0. If this position is for a Security then this field represents the consolidated amount of accrued interest for the Group. Validation Rules: No input field Automatically updated by the system whenever the Group Consolidation was performed

        public TField getEstimation()

        Field description: This field contains the value of the asset in the currency contained in the associated GRP.VAL.CURRENCY field.         public TField getNoNominal()
        Returns the NoNominal

        Field description: The contents of this field varies depending on the module that the position is for. If the position is for an Account (AC) then this field will contain the consolidated ONLINE.ACTUAL.BAL balances from the Account records plus the value of any forward entries for those accounts generated by the Securities module for transactions that have updated the portfolio. If the position is for a security position (SC) then this field will contain the consolidated number of securities held by the Group. If the security position is short then the figure will be preceded by a '-' sign. Validation Rules: No input field Automatically updated by the system whenever the Group Consolidation is performed

        Set the specified value for VDateNominal
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The contents of this field varies depending on the module that the position is for. If the position if for an Account (AC) then this field will contain the consolidated ONLINE.ACTUAL.BAL balance from the Account record. If the position is for a Fiduciary Contract (FD) then this field will contain the consolidated principal amount from the AMOUNT field on the underlying FD.FID.ORDER records for the group. If the position is for a FOREX contract the this field will Contain either the consolidated amount bought (AMOUNT.BOUGHT) from the original FOREX record representing the bought side of the contract. If the position is for a Money Market (MM) contract then this field will contain the consolidated principal amount from the PRINCIPAL field on the MM.MONEY.MARKET records that belong to this Group. If the position is for a loan then the amount will have a negative '-' symbol before it. If the position is for a security position (SC) then this field will contain the consolidated number of securities settled by the various positions belonging to this group at the current date. If the security position is short then the figure will be preceded by a '-' symbol. Validation Rules: No input field Automatically updated by the system whenever the Group consolidation was performed
