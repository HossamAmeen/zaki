      • setRolloverAmt

        public void setRolloverAmt(java.lang.CharSequence value)
        Set the specified value for RolloverAmt
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The amount that has to be deducted from or added to the final maturity value of the deposit for further rollover. The amount specified in the field may be LESS, EQUAL or MORE than the deposit. In case the amount of roll over is more than the deposit amount- the system takes the difference between the deposit amount and the maturity amount from the REPAY account. In case the balance in the repay account is not sufficient to supplement the addition to the deposit - then the account is moved to AZOD category and the exception log is updated i.e. the instruction to rollover the deposit is IGNORED and the AZ is matured. In case of loans, the difference between principal amount and balance in the loan account is credited to the nominated account. If the rollover amount is for a value less than the deposit maturity value, then the difference is credited to the Nominated account. In case of loans, the difference between principal amount and balance in the loan account is debited from repay account. In case balance in the repay account is not sufficient to supplement the addition to the principal amount, then based on PD.LINK.TO.AZ field in APP the oustanding balance amount will be debited. If PD.LINK.TO.AZ is 'YES' then PD will be debited. If PD.LINK.TO.AZ is 'NO' then AZ.OVERDUES will be updated. If PD.LINK.TO.AZ is 'NULL' then FT on hold will be raised. For reducing the rollover amount, minus sign is to be prefixed with the amount specified under this field. For increasing the rollover amount, the amount by which it is to be increased must be specified under this field. Example: Maturity value of a deposit = 100200 If the user wishes to rollover for an amount of 60000, then -40200 is to be entered in this field. If the user wishes to renew for 150000, then 49800 is to be entered here. Validation Rules: Rule 1: ""-"" sign with amounts to be used in case of decrease in rollover amount. For increase in rollover amount simply the amount need to be mentioned without ""+"" sign.

        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field is used to define the tax code, to be applied on the tax component like Interest. There are two ways of specifying the tax code here. One way is to enter the code from the TAX file. If any valid code already existing in TAX file is entered here, the relevant information will be picked up from the TAX file, for calculation purposes. Another way is to specify a valid code from TAX.TYPE.CONDITION file. In this case, the system first reads the CUSTOMER.CHARGE table and then based on the charge group of the customer, picks up the relevant tax record from TAX.TYPE.CONDITION. To distinguish the code of TAX.TYPE.CONDITION from the TAX file, the code is to be entered by prefixing ' * '. For example, if ' *VAT' is entered here, it will pick up the relevant code from the TAX.TYPE.CONDITION file. A code entered like '10' (without *), will refer TAX file and pick up the record relating to code 10. Validation Rules: Should have been already defined as a valid record in TAX / TAX.TYPE.CONDITION file.
