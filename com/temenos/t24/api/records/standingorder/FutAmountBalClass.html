      • getFutAmountBal

        public TField getFutAmountBal()
        Returns the FutAmountBal

        Field description: Defines any Future Amount to be effected after the current payment instructions have been completed. This field, when input, will always be associated with the next two fields and for any Future Standing Order instructions (excluding DIARY), an amount will always be mandatory. It will allow the User to load in advance any number of future Standing Order instructions provided they are loaded in chronological date order. This field, together with its associated fields, should be input by the User as soon as the instructions have been received from the Customer. There is no need to wait until the first execution date of the new instructions to enter the details in the Application. Validation Rules: Up to 14 type AMT (Standard Amount Format) characters plus a decimal point. Multivalue field. (Mandatory when Future Standing Order instructions are loaded in advance otherwise Optional. Default Value: None.) Input will only be allowed when the Current end date and Current Amount Balance fields have been entered. Input not allowed for Standing Order types DIARY, PP and DD.

      • getFutEndDate

        public TField getFutEndDate()
        Returns the FutEndDate

        Field description: Specifies a Future End date of the Standing Order for the Amount and Frequency specified in the previous associated fields Future Amount and Future Frequency. This field together with the previous associated fields (Future Amount and Future Frequency) will allow the User to load in advance any number of future Standing Order instructions provided they are loaded in chronological date order. Example: The following example will illustrate the use and the relation existing between the first nine fields. Let us suppose that you want to pay the Rates on your house on a monthly basis and that the amount of the Rates which is equal to GBP 1,000.50 needs to be paid in 10 payments. The following Standing Order instructions will then be loaded: Field No. Field description Field Input 1 Type FI 2 Pay Method BC (for example) 3 Currency GBP 4 Current Amt-Bal 100.50 5 Current Frequency 02 MAY 85 M0102 6 Current End Date 02 MAY 85 7.1 Future Amt-Bal 100.00 8.1 Future Frequency 02 JUN 85 M0102 9.1 Future End Date 02 FEB 86 Validation Rules: Up to 9 type D Date characters (date format in range 1950-2049). (Optional input. Not allowed for types PP, PPI and DD.) Date input must be equal or greater than the date specified in the associated field Future Frequency. Input not allowed if a Future frequency has not been defined. Input mandatory when further future instructions exist.

      • getFutFrequency

        public TField getFutFrequency()
        Returns the FutFrequency

        Field description: Defines any Future Frequency of the Standing Order for the amount specified in the previous associated field (Future amount - balance) or for Diary Instructions. The User must be careful when entering the Future Frequency and must ALWAYS specify the Start Date before inputting the required frequency. If the start date of each new frequency is not entered, the System will calculate the first date according to the frequency input and the Processing date will not generate the date requested by the Customer. This field together with the previous associated field (Future Amount) and the next associated field (Future End Date) will allow the User to load in advance any number of future Standing Order instructions provided they are loaded in chronological date order. The date input or generated will correspond to the first execution of the Standing Order amount specified in the associated field. The first date calculated by the System will then be the date resulting from the frequency applied to the Processing Date. If a date was input first, the frequency will be applied on this date and not on the Processing date. Validation Rules: This field is sub-divided into two elements: 1. Date 1-9 type D (date format in range 1950-2049) characters. 2. Frequency 1-5 type SS (uppercase alpha or numeric, first character alpha) characters. (Optional input for Standing Order type BI, BO, FI, and DIARY. Not allowed for types PP, BP, PPI and DD.) The start date must be greater than the date specified in the Current End Date field for the First Future instruction and must be greater than the date specified in the previous Future End Date field. Input mandatory when the associated Future amount - balance field has been input except when it concerns a Diary Standing Order type. 'DAILY' frequency is not allowed for Standing Order types BI and BO, ('BSNSS', every working day is allowed.) Input not allowed when the associated Future Amount - balance field has been left blank except when it concerns a Diary Standing Order type. An override will be required if the date input is a non-working day and the frequency is 'BSNSS'.

        Returns:
        TField - The FutFrequency
