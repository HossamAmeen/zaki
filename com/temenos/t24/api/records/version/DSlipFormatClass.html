      • getDSlipFormat

        public TField getDSlipFormat()
        Returns the DSlipFormat

        Field description: The value(s) defined here are used to produce Deal Slips according to the function and whether the user wants to have them produced automatically or when a 'hot key' is used. Deal Slips can be printed to a connected local printer or directed to a Network device. Refer to the REPORT CONTROL section of the Systems Administration User Guide for details of setting up printers. Deal slips can be linked to a function such as Input, Reversal or Authorisation and specific Deal Slips can be made to relate to those functions. Either an ENQUIRY record or a DEAL.SLIP.FORMAT record can be entered. If a valid DEAL.SLIP.FORMAT then the system will provide an enrichment from the record so the user will know thay have the correct record. Alternatively, an ENQUIRY record can be used to provide a Deal Slip, in this case the enrrichment will inform the user that an ENQUIRY record has been selected. The user can drilldown to the related table DEAL.SLIP.FORMAT to view or select the record to use. The combination of the associated fields is used to define when the Deal Slip can be produced. To trigger them automatically you need to define:

        Field Content Required
        D.SLIP.FORMAT TELLER.SLIP.INPUT
        D.SLIP.FUNCTION I
        D.SLIP.TRIGGER OL
        This will only print the Deal Slip TELLER.SLIP.INPUT when the deal is validated at I(nput) time. To trigger them via a "hot key" or the Desktop Deal Slip icon you need
        Field Content Required
        D.SLIP.FORMAT TELLER.SLIP.INPUT
        D.SLIP.FUNCTION I
        D.SLIP.TRIGGER RQ
        This will only print the Deal Slip TELLER.SLIP.INPUT at I(nput) time when the 'hot key' is used. Note: ENQUIRY linked Deal Slips are valid for Desktop (GUI) use only. Since the ENQUIRY will be based on either the live or unauthorised file then only data that is available on those files can be accessed, whereas DEAL.SLIP.FORMAT can utilise the new data being entered on a deal. ENQUIRY deal slips therefore need to be based on the live or unauthorised record depending on when the deal slip is being produced. For example a new deal would probably be based on the unauthorised record whereas a reversal would be based on the live record. Validation Rules: Input to this field must be a valid record in either ENQUIRY or DEAL.SLIP.FORMAT files. Must have values in fields D.SLIP.FUNCTION and D.SLIP.TRIGGER to produce Deal Slips

        Parameters:
        Returns:

      • getDSlipFunction

        public TField getDSlipFunction()
        Returns the DSlipFunction

        Field description: The Deal Slips can be produced at various times according to the function (or codeword FINISH) and whether they are produced automatically or in conjunction with a 'hot key' FUNCTIONS Valid functions are:

        Data Description
        IInput
        AAuthorise
        CCopy
        RReverse
        DDelete
        HHistory Restore
        FINISH Where FINISH is specified, this processing is dependent on the application and may not be triggered for all applications. Refer to the User Guide for the application for further details. Currently, only the TELLER application makes use of this facility. Validation Rules: Valid input is I, A, C, R, D, H or FINISH. Must have input in fields D.SLIP.FORMAT and D.SLIP.TRIGGER fields to be effective.

        Parameters:
        Returns:
        TField - The DSlipFunction
