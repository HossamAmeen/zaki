      • getTypeR

        public TField getTypeR()
        Returns the TypeR

        Field description: This field indicates the type of schedule defined by the user in AZ account for interest Rate related events like rate revision etc. The value shown here indicates the effective Rate of Interest applied to the account. Schedule Type R - This type is used for effecting Interest Rate revision on specified dates. The value defined under Term Priority will be taken into account for effecting this type of schedule. Both Variable interest rate and Fixed Interest rate can be fixed to a loan account, through this type, for a certain percentage of the outstanding amount and also for various periods within the life of the loan account. Example: As per the above R schedule, a rate revision will happen on 22-05-2001, with 60% of the outstanding amount by a variable interest (key1 and spread 6 on Basic Interest) and 40% of the outstanding amount by fixed interest 15% Based on the schedule types defined in the AZ account, AZ.SCHEDULE is generated by the system. By giving the ID of the AZ account in AZ.SCHEDULE, the complete schedule details of the account can be viewed. Schedule is automatically generated by the system and no user input is allowed . Further, no automatic actions like in LD/ MG are done by the system based on AZ.SCHEDULE . In All in One Module, the SCHEDULE is used only as a reference point, for updating the limit, ADI and comparing the AZ account balance on scheduled dates, for arriving the arrears or the redraw value. The other types of schedules are: A - For defining annuity type of repayment, where a constant repayment amount is generated based on the frequency. B - This type is used for effecting loan drawdowns on pre-defined dates. On the specified dates the system will transfer the draw down amount to the credit of the nominated account. I - Defines schedule for interest payments. For I type schedule, amount cannot be specified by the user. Total period of all I type schedules are validated against the Maximum Interest Only period defined for the AZ account. P- This type defines the Repayment of Principal Validation Rules No input field in AZ.SCHEDULE Part of a multi value set of fields

        Parameters:
        Returns:
        TField - The TypeR
