      • getCostPrice

        public TField getCostPrice()
        Returns the CostPrice

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field will contain the cost of the position. The actual calculation done for this field will depend on the type of position this set of values represents. If the position is for a security then this figure contains the calculated cost for a single security in the security currency. It is calculated using the COST.INVST.SEC.CCY from the SECURITY.POSITION record that gave rise to this entry on the SC.POS.ASSET file. It is calculated as follows ; COST.PRICE = (COST.INVST * PERC.FACTOR) / (NOMINAL * MULT.FACTOR) Where COST.INVST is the value contained in the COST.INVST.SEC.CCY field on the SECURITY.POSITION record. PERC.FACTOR is 100 if the Security price is expressed in percentage terms (found from the PRICE.TYPE record associated with the security) and 1 otherwise. MULT.FACTOR is the multiplication factor for the security (found from the PRICE.TYPE record associated with the security). NOMINAL is the closing nominal balance fould from the SECURITY.POSITION record field CLOSING.BAL.NO.NOM. If the position is built from more than one SECURITY.POSITION record then the COST.PRICE is pro-rated. If the position is for a FOREX contract then this field will contain the exchange rate of the deal (if the portfolio reference currency is one of the currencies involved in the deal) or the market or forward rate between the currency in the associated SECURITY.CCY value and the portfolio reference currency at the time of the original contract. It is retained as the cost price until the contact matures and is removed from the portfolio valuation. If the position is for a Forward Rate Agreement then this field will contain the value in the INTEREST.RATE field of the original contract. If the position is for an Account then this field will be blank. If the position is for a Money Market contract or for a Loans and Deposits contract then this field will be blank. If the position is for a Fiduciary contract then this field will be blank. If the position is for a Miscellaneous Deal then this field will be blank. If this position is for a Derivatives transactions then this is the price for one lot of this contract. Validation Rules: No input field. Automatically generated by the system when the information for the portfolio concerned is rebuilt.

      • getEstimation

        public TField getEstimation()
        Returns the Estimation

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field contains the value of the asset in the currency contained in the associated SECURITY.CCY field. If the position is for an Account (AC) then this field will contain the ONLINE.ACTUAL.BAL balance from the Account record plus the value of any forward entries for this account generate by the Securities module for transactions that have updated the portfolio. If the position is for a Fiduciary Contract (FD) then this field will contain the principal amount from the AMOUNT field on the underlying FD.FID.ORDER record. If the position is for a Forward Rate Agreement (FR) then this field will contain an estimated current value of the Forward Rate Agreement calculated from the current interest rates. If the position is for a Forex contract (FX) then this field will contain the estimated profit or loss on the side of the Forex transaction represented by this set of fields. If the position is for a Loans and Deposits (LD) contract then this field will contain the principal amount from the AMOUNT field on the LD.LOANS.AND.DEPOSITS record that gave rise to the position. If the position is for a loan then the amount will have a negative '-' symbol before it. If the position is for a Money Market (MM) contract then this field will contain the principal amount from the PRINCIPAL field on the MM.MONEY.MARKET record that gave rise to the position. If the position is for a Miscellaneous Deal (MD) then this field will always contain zero. If the position is for a Security positin then this field will contain an estimated valued of the security position calculated using the market price in the associated MARKET.PRICE field. If the security position is short then the figure will be preceded by a '-' sign. If the position is for a Derivatives (DX) transaction this this field hold the current contract value. If the position is for a Swap contract (SW) then this field will contain the Net Present Value of the side of the SWAP transaction represented by this set of fields. Validation Rules: No input field Automatically updated by the system whenever the portfolio valuation amount is updated.

        Returns:

      • getMarketPrice

        public TField getMarketPrice()
        Returns the MarketPrice

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. This field is not updated if the asset is a Money Market contract, a Loans and Deposits Contract, a Fiduciary contract, an Account or a Miscellaneous Deal. If the position is for a FRA Deal then this field represents the market rate against which the FRA Deal rate will be compared. If the position is a Forex then this field contains the current exchange rate between the currency on this side of the FOREX contract and the reference currency of the portfolio. The reference currency of the portfolio is in the REFERENCE.CURRENCY field of the SEC.ACC.MASTER record that defines the portfolio. A FOREX contract on the T24 portfolio system is split into its two sides for portfolio valuation purposes and the currency this side is for is contained in the SECURITY.CCY field associated with this field. If the position is a Security Position then this field contains the price of the security from the LAST.PRICE field on the SECURITY.MASTER record for the Security. For a derivatives transaction this field holds the current market price for the contract for which the transaction is held. Validation Rules: No input field. Automatically updated by the system whenever the portfolio data is rebuilt or during the Securities Price Feed run.

        Returns:
        TField - The MarketPrice

        Field description: Multivalued field associated with all the other fields in the file to give details of a portfolio position. The contents of this field varies depending on the module that the position is for. If the position is for a FOREX contract the this field will contain either the amount bought (AMOUNT.BOUGHT) or the amount sold (AMOUNT.SOLD) from the original FOREX record depending on whether this position represents the bought or sold side of the contract. If the position is for a Loans and Deposits (LD) contract then this field will contain the principal amount from the AMOUNT field on the LD.LOANS.AND.DEPOSITS record that gave rise to the position. If the position is for a Miscellaneous Deal (MD) transaction then this field will contain the principal amount from the PRINCIPAL field on the MD.DEAL record that gave rise to the position. If the position is for a loan then the amount will have a negative '-' symbol before it. If the position is for a security position (SC) then this field will contain the number of securities held by the position. If the security position is short then the figure will be preceded by a '-' sign. If the position is for a Derivatives (DX) transaction then this field will contain the number of open lots still active on that transactions. If the position is short then the amount will have a negative '-' symbol before it. If the position is for a Swap contract the this field will contain either the AS.PRINCIPAL or LB.PRICIPAL from the original SWAP record depending on whether this position represents the Asset or Liability side of the contract Validation Rules: No input field. Automatically updated by the system whenever the portfolio valuation data is rebuilt.
