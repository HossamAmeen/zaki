      • getPercentage

        public TField getPercentage()
        Returns the Percentage

        Field description: A percentage split by which any reallocated value is to be moved to different target lines. The value in this field corresponds to the percentage of cover afforded by the collateral. Per combination of source line, collateral code and (possibly) type, the entire qualifying debit balance (i.e. that amount up to the value of the collateral) will normally be moved to the specified target line during reallocation. However, if the target line requires a certain amount of excess value to be represented by the collateral, then this field may be used to specify this percentage of cover. During the reallocation process, each related set of conditions on this table is examined in turn to match against the collateral values and debit balances recorded in the system. If a percentage is defined in this field, then this percentage limits the amount of the debit balance which may be moved. This limit is defined as the given percentage of the collateral value. Multivalue field in association with field range SOURCE.LINE to TARGET.LINE. Subvalue field associated with TARGET.LINE. Note: It is possible to specify, by entering a second percentage within one multivalue set, a second target line to which the remaining (covered) amount is to be reallocated. If no second (or subsequent) percentage is entered, the remaining debit balance is not moved from its source line. In either case, the entire value of the collateral (or that amount necessary to provide the given percentage of cover) is considered to have been used in performing the reallocation and is not available for matching against any subsequent outstandings. Validation Rules: The total percentage defined within one multivalue may not exceed 100.

      • getTargetLine

        public TField getTargetLine()
        Returns the TargetLine

        Field description: The target line to which amounts are to be reallocated. The target line is the reporting line in which debit balances are to be reported (by reallocation from other lines) on the basis of the value and type of collateral supporting them. These lines are defined by use of the CRF report line definitions on F.RE.STAT.REP.LINE. Input to this field must, when appended to the REPORT ID , form the key to a record on the report lines file. This record must be defined as either 'DETAIL' or 'LINK' (ie. total lines may not be used). Target lines may be defined in a set of multivalues, each one of which corresponds to a defined SOURCE LINE. The only exception to the singly-defined target line is if there is a further split by percentage (eg. for a given combination of source line & collateral code, the value is reallocated to one line if the percentage of cover represented by the collateral is 75% or less and to another if it is greater than this). In this case, the field's subvalued association with PERCENTAGE may be used to indicate the split. Multivalue field in association with fields SOURCE.LINE. COLLAT.CODE. COLLAT.TYPE. PERCENTAGE.. subvalue field associated with PERCENTAGE. Validation Rules: The value of the target line must not be the same as the corresponding SOURCE LINE.
