com.temenos.t24.api.hook.atm

Class AtmMessageLifecycle

Packaged in : ATMFRM.MessageHook.jar

The MessageHook component allows access to routines for customising the ISO messages and validates the responses. Customizing the values are through the exit points of T24 files like INTRF.MESSAGE, ATM.TERMINAL.ACCT, ATM.CHG.TABLE and INTRF.MAPPING.

        public void calculateCharge(AtmTransactionContext atmTransactionContext, IsoRequest isoRequest, IsoRequestHeader isoRequestHeader, TStructure currentRecord, AtmTransactionRecord atmTransactionRecord)
        This interface enables the implementer to calculate the charges.
        This interface is invoked by a routine V.ATM.GET.CHG.TYPE.
        The EB.API hook used by this interface is ATM.CHG.FLD.POS.HOOK.
        The T24 field specifying this hook is the FLD.POS.RTN field in ATM.CHG.TABLE.
        If an exception is thrown in the implementing class it will be treated as an error.
        Parameters:
        atmTransactionContext - The context set in the ATM related transactions.
        isoRequest - Incoming ISO request message
        isoRequestHeader - Incoming ISO header request message
        currentRecord - The record being processed - the implementation must specify the record type.
        atmTransactionRecord - The record of the t24 file ATM.TRANSACTION.
        Returns:
        nothing

        public TNumber getBalance(TNumber accountNumber, AtmTransactionContext atmTransactionContext, IsoRequest isoRequest, IsoRequestHeader isoRequestHeader, String intrfaceMessageId)
        This interface enables the implementer to calculate and return the working balance of the account.
        The system then updates the account's working balance with the returned value. The EB.API hook used by this interface is INTRF.MAP.PRE.RTN.HOOK.
        The T24 field specifying this hook is the PRE.RTN field in INTRF.MAPPING.
        If an exception is thrown in the implementing class it will be treated as an error.
        Parameters:
        accountNumber - The account number associated with the transaction.
        atmTransactionContext - The context set in the ATM related transactions.
        isoRequest - Incoming ISO request message
        isoRequestHeader - Incoming ISO header request message
        intrfaceMessageId - T24 file INTRF.MESSAGE.
        Returns:
        TNumber - the working balance of the account.

      • getFieldValue

        public String getFieldValue(String isoRequestFieldValue, AtmTransactionContext atmTransactionContext, IsoRequest isoRequest, String intrfaceMessageId)
        This interface enables the implementer to return a value for the associated field name.
        This interface is invoked during post ISO transaction hit into the system. It triggers from AT.GEN.ISO.REQ.MSG, AT.ISO.POST.PROCESS.MSG, AT.ISO.PRE.PROCESS.MSG, ATMFRM.FORM.CLEARING.OFS and ATM.ISO.MESSAGE.PROCESS The EB.API hook used by this interface is INTRF.MAP.FIELD.SRC.VALUE.HOOK.
        The T24 field specifying this hook is the FIELD.SRC.VALUE field in INTRF.MAPPING.
        If an exception is thrown in the implementing class it will be treated as an error.
        Parameters:
        isoRequestFieldValue - The value extracted from the incoming iso request based on the associated Interface field position value.
        atmTransactionContext - The context set in the ATM related transactions.
        isoRequest - Incoming ISO request message
        intrfaceMessageId - T24 file INTRF.MESSAGE.
        Returns:
        String - the value of the field.

      • getMappingId

        public String getMappingId(String fieldValue, AtmTransactionContext atmTransactionContext, IsoRequest isoRequest)
        This interface enables the implementer to create an id for the INTRF.MAPPING table.
        This interface is invoked by a routine ATM.ISO.MESSAGE.PROCESS.
        The EB.API hook used by this interface is INTRF.MSG.MAPPING.ID.HOOK.
        The value entered in this field should prefix with '@' and the value to the parameter should be delimited with ':' The T24 field specifying this hook is the MAPPING.ID field in INTRF.MESSAGE.
        If an exception is thrown in the implementing class it will be treated as an error.
        Parameters:
        fieldValue - The value passed through the record of INTRF.MESSAGE>MAPPING.ID delimited with ':' .
        atmTransactionContext - The context set in the ATM related transactions.
        isoRequest - Incoming ISO request message
        Returns:
        String - the generated id .
