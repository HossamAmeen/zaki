      • getBuildLcyPosn

        public TField getBuildLcyPosn()
        Returns the BuildLcyPosn

        Field description: Introduced in Release G14.1 Specifies whether a FXP Position needs to be generated in PM.TRAN.ACTIVITY for the Local Currency, when a transaction involving a Foreign Currency and a Local Currency is input in the Applications FT, TT and DC. If this parameter is set to YES, data related to FXP Position class FTFFT or TTFXP (depending on the Parameterization in PM.PC.PARAM) would be respectively updated in PM.TRAN.ACTIVITY, for Local Currency movement also, when a Transaction involving a Foreign and Local Currency is input in the Applications FT and TT. If this parameter is set to YES, data related to FXP Position class DCFDC (depending on the Parameterization in PM.PC.PARAM) would be updated in PM.TRAN.ACTIVITY for Local Currency movement also, when a Foreign Currency Transaction is input in the Application DC. In this parameter is set to NO or Null, then in the above specified cases the FXP type Position class data would be generated only for Foreign Currency movement. Validation Rules: Valid values are YES, NO or Null. NO and Null are equivalent.

        Parameters:
        Returns:

        public void setPmFxSynthRtn(java.lang.CharSequence value)
        Set the specified value for PmFxSynthRtn
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field holds the name of the user exit routine that can be triggered to amend the values calculated by the CORE position management system for transactions from the Forex application. This subroutine needs to have the following arguments Incoming parameters - PRINCIPAL.CCY - used to denote the principal currency. Outgoing Parameters - PM.FX.USR.POSN.CLASS RESERVED.1 - to be passed as Null RESERVED.2 - to be passed as Null PM.FX.USR.POSN.CLASS needs to be a dynamic array holding the following structure PM.FX.USR.POSN.CLASS<x, PM.MA.ASST.LIAB.CD> - Asset / Liability PM.FX.USR.POSN.CLASS<x, PM.MA.POSN.CLASS> - Position Class PM.FX.USR.POSN.CLASS<x, PM.MA.CCY.AMT> - Notional Amount PM.FX.USR.POSN.CLASS<x, PM.MA.RATE> - Interest Rate PM.FX.USR.POSN.CLASS<x, PM.MA.VALUE.DATE> - Value date PM.FX.USR.POSN.CLASS<x, PM.MA.POSN.TYPE> - Position Type PM.FX.USR.POSN.CLASS<x, PM.MA.CURRENCY> - Currency. X is the iterator for the number of position classes. PM.MA.ASST.LIAB.CD, PM.MA.POSN.CLASS, PM.MA.CCY.AMT, PM.MA.RATE, PM.MA.VALUE.DATE, PM.MA.POSN.TYPE, PM.MA.CURRENCY are the positions as defined in I_PM.MAT.ACTIVITY. The details such as rate, amount that is updated in the position classes FX:RevalType:S or FX:RevalType:M in PM.TRAN.ACTIVITY / PM.DLY.POSN.CLASS can be modified as per the requirement of the bank. For e.g., The bank identifies one of the currencies in the transaction as the PRINCIPAL.CCY. The definition of PRINCIPAL.CCY is, if the deal involves local currency then local currency is the PRINCIPAL.CCY and for a cross currency deal, buy currency is the PRINCIPAL.CCY . The core functionality is to default the interest rate from the Periodic Interest table for the base currency and to derive the interest rate for the Non Base currency. This can be modified in the user exit routine, as to default the Interest rate from the Periodic Interest table for the Secondary currency and based on the interest rate for the secondary currency, the interest rate for the Principal Currency can be derived. So, user exit routine builds position classes with the above details. The formed Position Classes and its details are returned to the calling routine. The calling routine modifies the built position classes with the values returned from the user exit routine or appends the newly built position classes and the position management files PM.TRAN.ACTIVITY , PM.DLY.POSN.CLASS are built accordingly. Validation Rules: The field holds the name of the user exit routine. The routine name should be available in EB.API application.
