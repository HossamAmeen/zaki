      • getConsolidateBy

        public TField getConsolidateBy()
        Returns the ConsolidateBy

        Field description: Consolidation key fields e.g. CUSTOMER.NO or DEPARTMENT. The database will be held at this level. Thus if DEPARTMENT is specified then the database will be held at department level. It will thus be possible to produce reports by department. If more than one CONSOLIDATE.BY field is specified then it will be possible to produce reports at one or both levels. This also enables a report to be produced at one level (e.g. department) and then further detailed analysis performed (such as at customer level) when required. Note that this can be an I-descriptor field enabling associated information such as an account officers department code to be included. Validation Rules: Optional input
