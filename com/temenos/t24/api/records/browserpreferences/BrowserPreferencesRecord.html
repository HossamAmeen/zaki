      • addAttributes

        public void addAttributes(TField param)
        Appends the specified Attributes to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field defines a number of Attributes for the user's Browser session.. This includes whether the user wants to see any Hot Fields (as defined by VERSION) or Auto Launch Enquiries (as defined by CONTEXT.ENQUIRY) on their contract screens. A user may be accessing T24 via a slow network link, therefore the time taken to return to the T24 server to validate hot fields or run context enquiries may be unacceptable. Therefore, these facilities can be disabled here on a per user basis. Validation Rules: Accepted values: ENABLE.TAB.INDEX = This attribute enables the ability to navigate between focusable elements with tab key on contract screens. NO.AUTO.LAUNCHES = Attribute to disable the auto launch enquiries on contract screens. NO.CHECKFILE.ENRICHMENT = Attribute to disable file enrichment on contract screens. NO.HOT.FIELDS = This attribute allows the user to disable hot field's functionality on contract screens. NO.RIGHT.CLICK = Attribute to disable right click option. NO.WEB.VALIDATION.FIELDS = Attribute to deactivate the web validation on contract screens.

        Parameters:
        param - the Attributes to append
        Returns:
        nothing

        Field description: This field determines the 'skin' to be used for a user's Browser screens. A 'skin' determines the 'look and feel' of the Browser screens in terms of colours, fonts, graphics, etc. Each Browser installation will be delivered with a number of predefined skins that can be used.
