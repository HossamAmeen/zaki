        public TField getBuildCofEntries()
        Returns the BuildCofEntries

        Field description: This field specifies whethere MI.COF.ENTRIES for refinancing costs, need to be built. If this field is specified as YES, then one of the fields PROCESS.ACCOUNTS , PROCESS.CONTRACTS must have a value of YES. If this field is YES, then the value of the fields PROCESS.ACCOUNTS and PROCESS.CONTRACTS would determine whether MI.COF.ENTRIES would be built for Accounts/Contracts. If this field is NO or Null, then MI.COF.ENTRIES would not be generated for refinancing costs irrespective of the value of PROCESS.ACCOUNTS and PROCESS.CONTRACTS. Validation Rules: Input must be Yes or No. A value of Null would be interpreted as No.

      • getRebuildDate

        public TField getRebuildDate()
        Returns the RebuildDate

        Field description: Where present, this field determines the year and month from which records on the BALANCE.MOVEMENT file will be processed to produce MI.COF.ENTRIES. The build period will run from this date until the current system date. When this date is present, any previous MI.COF.ENTRIES (appropriate to the build options), which may have been created for the build period will be removed, and replaced by new records. If this date is not present, build process if specified for Cost of Funds (BUILD.COF.ENTRIES is set to YES), will run on from the dates set on the MI.DATES record in the fields ACCOUNT.COF and CONTRACT.COF, depending on which type of file is being processed, up unitl the current system date. If this date is not present, build process if specified for Additional Cost of Funds (ADD.COF.TYPE), will run on from the dates set on the appropriate MI.COF.TYPE record in the fields LAST.RUN.DATE.ACC and LAST.RUN.DATE.CON, depending on which type of file being processed, up unitl the current system date. Validation Rules: Must be a valid year and month.
