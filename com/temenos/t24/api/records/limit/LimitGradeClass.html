        public TField getGradePercent()
        Returns the GradePercent

        Field description: This field is associated with the LIMIT.GRADE field and shows the percentage attributed to each grade specified. Validation Rules: 1 to 3 numeric characters in the range 1-100. Multivalue field associated with LIMIT.GRADE. (Optional input. Default 100.) If only one occurrence of the Multivalue fields LIMIT.GRADE and GRADE PERCENT is defined, input must be 100. If more than one set of the Multivalue fields LIMIT GRADE and GRADE PERCENT are defined, input to each occurrence of the field GRADE PERCENT is mandatory. If more than one occurrence is input, the sum total of all occurrences must be 100. Only for Customer level Group Limits (For e.g. M0000000020.2000.01.10020) this field will be considered.

      • getLimitGrade

        public TField getLimitGrade()
        Returns the LimitGrade

        Field description: Specifies the grade of the risk. The grade entered must appear on the CUSTOMER STATUS table. This field is associated with the grade percentage field so that different portions of the risk may have different grades (eg 50% good, 50% bad). Where a Limit is available to a Liability Group, there may be a different risk rating associated with each Customer within the group. In this case, it is possible to enter several values of this field, each being a Risk Rating (or Customer Status) of a different member of the group. The associated field 'GRADE PERCENT' can then be used to record a percentage of the risk at a particular Grade. For instance, assume that there are the following Limit Grades (Customer Status):- CODE STATUS 1 EXCELLENT 2 GOOD 3 POOR If a Liability group is made up of two Customers, one Rated 'Excellent' and the other Rated 'Good', it is possible to record Limit Grades of 'Excellent' and 'Good' and to associate with them percentages of 50 and 50. However, if one Customer makes much greater use of the Limit than the other, the percentages entered could be, say, 80 and 20. Validation Rules: 1 to 4 numeric character Customer Status code. Multivalue field associated with Grade percent. (Optional input. A single occurrence will default to the Customer Status from the Customer Record of the Liability number.) Only for Customer level Group Limits (For e.g. M0000000020.2000.01.10020) this field will be considered.

        Parameters:
        Returns:
        TField - The LimitGrade
