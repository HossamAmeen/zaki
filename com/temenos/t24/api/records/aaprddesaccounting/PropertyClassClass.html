      • getPcAccrAmort

        public TField getPcAccrAmort()
        Returns the PcAccrAmort

        Field description: Flag to indicate if the associated property needs to be accrued or amortised. If flagged as "AMORT" when the associated Property class is made-due (either scheduled or ad-hoc) the due amount is amortised using central EB.ACCRUAL processing outside of AA. The associated transaction codes and PL category are required for either Accrual or Amortisation. Currently only applicable for "Charge" property class.

        Allowed values: 1. AMORT - Charges will be amortised from the make-due date until the period specified in ACCRUE.PERIOD. 2. ACCRUE - This Option is allowed only for scheduled CHARGE property.However,If charge property is of type REBATE.UNAMORTIZED then this option is not allowed.If ACCRUE is set for charge property other than scheduled charge then charge amount would be posted to PL directly instead of handover to EB.ACCRUAL for accrual processing.

        Parameters:
        Returns:
        TField - The PcAccrAmort

      • getPcAccrPeriod

        public TField getPcAccrPeriod()
        Returns the PcAccrPeriod

        Field description: Indicates the period of Amortisation or Accrual for the associated property class. Applicable only for "Charge" property class.

        Allowed values: 1. MATURITY - Applicable when the property class is flagged to "AMORTISE". Amortisation end date would be the maturity date of the Arrangement. 2. PERIOD - Any valid period in the format, Pnnn - where P can be "M" - Month, "D" - Days, "Y" - Year or "W" - Week, and nnn - can be any number between 1 to 999. If a period is defined, the end date would be the date calculated as the amortisation start date(usually the make-due date) plus the period specified in calendar days. 3. SCHEDULE - To perform accrual until next schedule date. 4. CONTEXT TYPE - User-defined Context Type prefixed with "U-" can be defined for CHARGE property class and AGENT product line. The source calculation for this context will be defined in the AA.COMMISSION.TYPE record. On evaluation of the charge commission, the value to this context will be fetched.

        Returns:
        TField - The PcAccrPeriod
