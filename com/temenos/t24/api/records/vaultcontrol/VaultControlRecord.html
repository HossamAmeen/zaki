      • getRemove

        public TField getRemove()
        Returns the Remove

        Field description: This field is used to move manually created VAULT.CONTROL records to history. If a VAULT.CONTROL record was created by user input and the user now wants to indicate that the item in the bank vault that the VAULT.CONTROL record represents has been removed from the Vault then the user should input 'YES' in this field. On authorisation this will result in the VAULT.CONTROL record being written to history. With regard to VAULT.CONTROL records generated through the input of a transaction, such as SEC.TRADE, these cannot normally be removed. However, if the need arises, you can effect the removal of an automatically created VAULT.CONTROL by setting the ALLOW.REMOVAL field in the VAULT.PARAMETER file to "Y". Validation Rules: No input field if VAULT.CONTROL record has been created from a VAULT.EXPECT record - i.e. linked to a trade unless the ALLOW.REMOVE flag has been set to "Y" in VAULT.PARAMETER Only input of 'YES' or 'NO' allowed.

        Parameters:
        Returns:
        TField - The Remove

        public void setRemove(java.lang.CharSequence value)
        Set the specified value for Remove
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field is used to move manually created VAULT.CONTROL records to history. If a VAULT.CONTROL record was created by user input and the user now wants to indicate that the item in the bank vault that the VAULT.CONTROL record represents has been removed from the Vault then the user should input 'YES' in this field. On authorisation this will result in the VAULT.CONTROL record being written to history. With regard to VAULT.CONTROL records generated through the input of a transaction, such as SEC.TRADE, these cannot normally be removed. However, if the need arises, you can effect the removal of an automatically created VAULT.CONTROL by setting the ALLOW.REMOVAL field in the VAULT.PARAMETER file to "Y". Validation Rules: No input field if VAULT.CONTROL record has been created from a VAULT.EXPECT record - i.e. linked to a trade unless the ALLOW.REMOVE flag has been set to "Y" in VAULT.PARAMETER Only input of 'YES' or 'NO' allowed.
