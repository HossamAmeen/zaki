        public void addSelection(TField param)
        Appends the specified Selection to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: The data to be used in the selection process. Note: This field is subvalued to enable 'lengthy' selection data to entered. Examples: Operand Selection EQ 1000 NE 2000 GT 1000 LT 1000 GE 2000 LE 2000 RG 1000 4000 NR 5000 5999 LK 10... UL ...99... Data can be entered, depending on the selection field used, as account mnemonics, dates etc. They will be converted to their correct format when processing the selection ie when using the Verify function.       • getSelectionField

        public TField getSelectionField()
        Returns the SelectionField

        Field description: Any field or i-descriptor on the standard selection record for the file AC.STMT.HANDOFF. Used to extract individual or groups of account statements for printing. Use SELECTION.FIELD, OPERAND and SELECTION to specify which statements should be printed by this request. E.g: 2.1 SELECTION.FIELD.....: ACCT.CATEGORY 3.1 OPERAND.............: RG 4.1.1SELECTION...........: 1000 5999 This will request all statements for accounts with a category in the range 1000 to 5999. The SELECTION.FIELD can be any field, or i-descriptor, on the AC.STMT.HANDOFF file. Avoid using i-descriptors to link back to the ACCOUNT file. If the account has been closed then the account record will have been moved to the history file. Validation Rules: 18 Any characters. (Optional input) Must be a defined field on the standard selection record for AC.STMT.HANDOFF.

        Parameters:
        Returns:
        TField - The SelectionField
