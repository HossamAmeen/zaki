        public TField getChqCollId()

        Field description: Enter the Cheque Collection Id of the Cheque Collection record to be included on the Cheque Batch record. After the Cheque Batch record has been authorised the Cheque Collection Id can not be removed from the batch. Validation Rules: Mandatory. Mulitvalued. Must be a valid Cheque Collection Id. No duplicate Cheque Collection Ids allowed. Cheque Collection Ids can only be used on one Cheque Batch record.

      • getValueDate

        public TField getValueDate()
        Returns the ValueDate

        Field description: The forward or backward date on which the collection credit will be taken into customer account when the cheque/ collection instrument is Cleared. In the case of return of the cheque/ collection instrument, the debit to the specified internal account will be taken into account, on this date. Whenever the user wishes to change the original value date, for reasons like postponement of clearing etc., this field can be used. This field is available in CHEQUE.COLLECTION and in CHEQUE.BATCH also, under each cheque collection record included in the batch. User has the option to use this field or the OVERALL.VALUE.DATE available in the batch. Validation Rules: Optional field Standard Date format The date entered here will be validated against the range of minimum/ maximum backward and forward values specified in DATES file.

        Parameters:
        Returns:
        TField - The ValueDate
