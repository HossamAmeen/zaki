        public TField getPhysicalPosition()
        Returns the PhysicalPosition

        Field description: Holds actual field number of a new field (called as neighbour field) added to an existing dynamic template without requiring a layout change. It decides an array conversion order gets updated in STANDARD.SELECTION PHYSICAL.ORDER and LOGICAL.ORDER fields during authorisation stage of EB.TABLE.DEFINITION or through AA.CLASS.DEFINITION Validation Rules: 1. System updated field 2. Raises override if an existing field physical position has been changed. 3. For now, Input allowed only for the EB.TABLE.DEFINITION record created through AA.CLASS.DEFINITION.
