      • getPortfolioId

        public TField getPortfolioId()
        Returns the PortfolioId

        Field description: Defines the suffix of a portfolio. A default value of "ALL" will be updated in this field upon creation of the record. Portfolio ID with value "ALL" is considered as the default portfolio group. This can be further expanded to add several portfolios that doesn’t belong to default group. The record id along with the portfolio suffix defined in this field must be a valid record in SEC.ACC.MASTER. Validation Rules: The first multi value with "ALL" is updated by the system and is mandatory. Duplicate portfolio suffix are not allowed. Associated field with TR.DEF.GROUP and TR.ACT.GROUP.

        Parameters:
        Returns:
        TField - The PortfolioId

      • getTrActGroup

        public TField getTrActGroup()
        Returns the TrActGroup

        Field description: Defines the Actual group for the PORTFOLIO.ID. The actual group and the default group will be updated with the same value upon creation of the record. The actual group can be modified to define a specific group condition. When a customer specific group condition record exists, the following rules are applicable. a. Group condition having one or more portfolios: the actual group will be updated with the customer specific group condition id only for the portfolio defined in group condition

        b. Group condition having "NONE" as the portfolio value: the actual group will updated with the customer specific group condition for "ALL" representing that it is applicable for all the portfolios.

        Validation Rules Input field. Valid record of FX.GROUP.CONDITION. Associated field with PORTFOLIO.ID and TR.DEF.GROUP.

        Parameters:
        Returns:
        TField - The TrActGroup
