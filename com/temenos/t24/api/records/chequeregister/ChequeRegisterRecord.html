      • addAutoChequeNo

        public void addAutoChequeNo(TField param)
        Appends the specified AutoChequeNo to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Range of auto cheque no is populated here when Auto.Reorder.Type is "Cheque.No" in CHEQUE.TYPE. During the Cheque issue, based on the minimum holding for the cheque.type, auto cheque no is arrived and starting cheque no stored in Cheque.Issue and full sequence stored in the Cheque.Register. On presenting any of these cheque no Auto reorder process will be done. However, if a record is available in Cheque.Issue for that account with Request Status Less than 90 (Cheque issue), then reorder will not be done. For example: In Cheque Type CURR minimum holding specified as 5 and auto cheque type is "Cheque.No". If a cheque book is issued with Cheque type as "CURR" and start no as 1 with no of leaves as 10. In Cheque Issue, auto cheque no is populated as 6 and in Cheque register, as 6,7,8,9,10. On presenting any of the cheque no, auto reorder will be done with appropriate Request status and in cheque register, auto cheque no will be populated as "Requested"

        Parameters:
        param - the AutoChequeNo to append
        Returns:
        nothing
