      • getDrValueDate

        public TField getDrValueDate()
        Returns the DrValueDate

        Field description: This field determines the Debit value date calculation rules. It accepts relative or absolute dates. You can combine 1 or 2 date operations, always based on the frequency date, each operation being "+" or "-", followed by a number of days and followed by "C" or "W", meaning calendar or working days. "+" and "C" are taken by default. If number alone entered then it will be converted into valid date. If nothing is entered here, default calculation will be done through the Funds Transfer calculation rules, depending on the payment method. Examples : "+01C-01W" which means "Last working day before the processing date" "-02C" means processing date less 2 calendar days, "01" entered when the system month/ year is Jan/2001, then it will convert to 01012001 and so on ... Validation Rules: Up to 8 type ANY (alphanumeric) characters. "+" and "C" are optional, and zeros will be added if needed. Example : "1-1W" will be taken as "+01C-01W". Optional input, blank means default FT calculation. Defaults the system date during BULK.STO Input.

        Parameters:
        Returns:
        TField - The DrValueDate

      • getOrderingAcct

        public TField getOrderingAcct()
        Returns the OrderingAcct

        Field description: Identifies the account number of the Benefactor. The contents of this field will be placed in the DEBIT.ACCT.NO field in Funds Transfer if the pay method is 'AC'. PL accounts must be entered in the format PL61000. Validation Rules: Standard numeric Account number or 3-10 character Account Mnemonic (uppercase alphanumeric or '.') type MNE. Associated multivalue field. (Mandatory or optional input as defined in Validation Rules: . No default.) Input mandatory for Pay Method 'AC'. Input optional for Pay Method 'IM'. For the Pay Method 'AC', the Account must exist on the Account file in the same company and the field will be enriched with the first line of the Account title. The account must also be in local currency. The account input may not be the same as the Inward Bulk Standing Order account. If the Inward Bulk Standing Order is a 'PL' account, then the account input must not also be a 'PL' account.

      • setDrValueDate

        public void setDrValueDate(java.lang.CharSequence value)
        Set the specified value for DrValueDate
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field determines the Debit value date calculation rules. It accepts relative or absolute dates. You can combine 1 or 2 date operations, always based on the frequency date, each operation being "+" or "-", followed by a number of days and followed by "C" or "W", meaning calendar or working days. "+" and "C" are taken by default. If number alone entered then it will be converted into valid date. If nothing is entered here, default calculation will be done through the Funds Transfer calculation rules, depending on the payment method. Examples : "+01C-01W" which means "Last working day before the processing date" "-02C" means processing date less 2 calendar days, "01" entered when the system month/ year is Jan/2001, then it will convert to 01012001 and so on ... Validation Rules: Up to 8 type ANY (alphanumeric) characters. "+" and "C" are optional, and zeros will be added if needed. Example : "1-1W" will be taken as "+01C-01W". Optional input, blank means default FT calculation. Defaults the system date during BULK.STO Input.

        Parameters:
        value - The value to set
        Returns:
        nothing
