        public void addValidationRtn(TField param)
        Appends the specified ValidationRtn to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies additional validation routines to be applied to entry created from the interface file. User written routines may be specified to valdiate the contents of the entry. The routine, if written must contain them following arguments: Entry Record (Stmt / Categ entry to be raised) (IN) Reporting Record (AC.INWARD.ENTRY record) (IN) may be modified Error Message (OUT) any error messages preventing posting of the entry Suspense Message (OUT) any error message causing the entry to be suspensed Generally available routine which may be used include: FT.VALIDATE.DD.MANDATE This will validate an entry to see if a direct debit mandate exists for it. It relies on the following information being mapped into local reference of AC.INWARD.ENTRY, and the local reference items having the names specified below: "THEIR.ACCOUNT" - The account number to which the DD is payable. "THEIR.SORT.CODE" - The sort code of the destination account. "PROCESS.DATE" - The Processing date of the mandate. The routine will also use ACCOUNT.NUMBER and THEIR.REFERENCE to check against the Standing Order file. FT.CHECK.AUTO.LIMIT Validates the local currency equivalent of an entry against the AUTO.PROCESS.LIMIT in FT.APPL.DEFAULT. If exceeded the entry is suspensed. Validation Rules: Optional Input.

        Parameters:
        param - the ValidationRtn to append
