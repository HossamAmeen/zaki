        public void addPhantCheck(java.lang.CharSequence param)
        Appends the specified PhantCheck to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field holds the command which will be executed by UniVerse to check the status of the various delivery phantoms before they are able to be initiated. Multi-values are used where the commands are too long to fit on one line. If this field is blank then it will default to the following command, which shells out to Unix and runs a standard ps : SH -c 'ps auxw | grep DE.PHANTOM.CALL' The output from the command contained here is captured and then scanned for the phantom name to be started. N.B. The UniVerse account id is appended to the name of the phantom to uniquely identify each phantom where there may be phantoms running in different accounts on a machine.

        Example ======= If the phantom DE.O.MAP.MESSAGES is running and the user tries to start the phantom again then the captured command will contain the following: phantom DE.PHANTOM.CALL DE.O.MAP.MESSAGES*bnk.run where bnk.run is the UniVerse account name of the account. The account name is then appended to the phantom name and searched for in the captured output. If found, the user will be unable to restart the phantom.

        Parameters:
        param - the PhantCheck to append
        Returns:
        nothing

        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Defines the line communications parameters for the connection to the SWIFT device. Applicable only for ATE and TCPIP protocols otherwise ignored. If ATE protocol is specified in field SWIFT.PROTOCOL then this field will specify five pieces of information about the line communications as follows: Line , Baud , Data , Stop , Parity Where: Line Defines the &DEVICES& file devicename for the asynchronous line connected to the SWIFT device. Parity Defines parity as '0' - None '1' - Even '2' - Odd If TCPIP protocol is specified in the SWIFT.PROTOCOL field then this field will specify three pieces of information about the communications interface as follows : Blocksize, Internet address and port, pipes location Where: Blocksize Specifies the minimum datagram size in bytes which will be exchanged with the 'c' server. This is present for performance reasons since it allows a block of fixed size to be expected by each side. It should be set at installation to a value determined appropriate for likely message sizes. The default should be 512. Internet Specifies the Internet address of the SWIFT device as connected address on the network with a port number as an extension e.g. 100.100.10.10:1234. This is the address the 'c' server will connect its sockets to. Pipes Specifies the unix pathname of the directory containing the location pipes used for communications between T24 and the 'c' server for TCPIP. There are four pipes two for inward and two for outward SWIFT. This path may be absolute or relative.

        1 -50 alphanumeric characters. (Optional input) Mandatory input if SWIFT is a valid carrier and the ATE or TCPIP protocol is used. The '.' are literal, as is the ':'. The port number is the last four numerics. All elements are mandatory. - A valid Unix absolute or relative pathname
