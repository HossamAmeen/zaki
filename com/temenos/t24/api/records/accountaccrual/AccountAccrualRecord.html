      • getCompAccRtn

        public TField getCompAccRtn()
        Returns the CompAccRtn

        Field description: Identifies a local routine that can be called to adjust the compensation account data. This routine will be called with 1 multi valued argument delimited by field marks. The argument data is as follows:- Field 1 - the lead acount number i.e. the account being compensated to, this is for information only Field 2 - the start date of the calculation period, this can be adjusted if required Field 3 - The end date of the calculation period, this is for information only Field 4 - Indicates the type of processing taking palce, and is information only, "S" = capitalisation, "A" = accrual, "C" = back value correction, "I" = information only, no accounting entries are posted nb: this can be used via the INFO.ACCT.XX routines for testing purposes where XX indicates the type of interest calculation DR,DR2, CR, CR2, CH Field 5 - the current list of compensation accounts delimited by value marks marks, this can be adjusted by the local routine Field6 - associated with field5 this can be returned from the local routine and can contain the start date when the balances for the relevant compensation account in field5 are included in the compensation group balances Field7 - associated with field5 this can be returned from the local routine and can contain the end date when the balances for the relevant compensation account in field5 are excluded from the compensation group balances An example follows of how this local routien could be used to adjust the calculation of the compensation group balances. Acount 100002 was compensated to account 10001 from the second of the month to the tenth of the month. The calcuation period is the 1st to the 31st of the month. Account 100002 has a balance of 1000 for the whole period. Account 100001 has a balance of 2000 for the whole period. The calculation takes place on the 2nd of the month and is projected to the end of the month. Normally the compensation group balance will be 3000 for the whole month. However if the local routine is called the compensation balances can be calculated as follows 1st 2000 2nd 3000 10th 2000 nb: the above example will require a local development to store the start and end dates of the compensation group memberships.

      • getLastDayInclusive

        public TField getLastDayInclusive()
        Returns the LastDayInclusive

        Field description: Specifies whether interest calculations for application are inclusive or exclusive of the capitalisation date. Assuming MTH.END.UPTO.DAY (field 1) contains 20 and an Account also has Interest Capitalisation on 20th of the month, Monthend Accrual processing and interest application will both take place on the 20th of each month (or on the previous working day if 20th is not a working day). If this field contains Y, the amount booked to the Customer's Account will include interest calculated on balances with value up to and including 20th. The entry booked to the Account will have a value date of 21st. If this field contains NO, the amount booked to the Customer's Account will only include interest calculated on balances with value up to and including the last calender day before 20th. The entry booked to the Account will have a value date of the day after the last balance included. Validation Rules: Y (inclusive), N or NO (exclusive). Mandatory input. Cannot be changed once a customer account has been opened in the company.

        Returns:

        public void setLastDayInclusive(java.lang.CharSequence value)
        Set the specified value for LastDayInclusive
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies whether interest calculations for application are inclusive or exclusive of the capitalisation date. Assuming MTH.END.UPTO.DAY (field 1) contains 20 and an Account also has Interest Capitalisation on 20th of the month, Monthend Accrual processing and interest application will both take place on the 20th of each month (or on the previous working day if 20th is not a working day). If this field contains Y, the amount booked to the Customer's Account will include interest calculated on balances with value up to and including 20th. The entry booked to the Account will have a value date of 21st. If this field contains NO, the amount booked to the Customer's Account will only include interest calculated on balances with value up to and including the last calender day before 20th. The entry booked to the Account will have a value date of the day after the last balance included. Validation Rules: Y (inclusive), N or NO (exclusive). Mandatory input. Cannot be changed once a customer account has been opened in the company.
