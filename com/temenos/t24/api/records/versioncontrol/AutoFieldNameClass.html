        public TField getAutoFieldName()
        Returns the AutoFieldName

        Field description: If the value in this field is not present in the corresponding field in VERSION record, the value in this field and the associated fields are appended to the VERSION record. Enables you to define the field name, which is to be subject to an automatic default value. It must be used in conjunction with the fields AUTO.OLD.CONT and AUTO.FIELD.RTN. The definition of the value or sub-value numbers must always be included in this field. Field names can be entered in the format: a) Field name (e.g. MNEMONIC for field 1 in the customer file followed by, b) '-' When you want to define a multivalue field, followed by, c) Value Number (e.g. RELATION-1 for RELATION in the customer file) when you want to define multivalue fields, followed by, d) '.' When the User wants to define a subvalue field, followed by, e) Sub-Number when you want to define a subvalue field. Alternatively, field names can be entered as field numbers, as follows: a) Field number (e.g. 1 for MNEMONIC in the CUSTOMER file), followed by, b) '.' When you want to define a multivalue field, followed by, c) 'Value number' (e.g. 7.1 for RELATION in the CUSTOMER file) when you want to define multivalue fields, followed by, d) '.' When you want to define a subvalue field, followed by, e) 'Sub number' when you want to define a subvalue field. Field numbers will be translated to field names when entered. For local reference fields, use the format LOCAL.REF-n. Validation Rules: Up to 40 type A (alphanumeric) characters in the format XXXXXXXXXX.NNN.NN or NNN.NNN.NN Multivalue field associated with Fields 1 to 3. (Optional input. No default value). Only field names defined in the corresponding Application are allowed. Duplication of field names is not allowed.

        public TField getAutoFieldRtn()
        Returns the AutoFieldRtn

        Field description: If the value in the associated field AUTO.FIELD.NAME is not present in the corresponding field in VERSION record, the value in this field and the associated fields are appended to the VERSION record. Must be used in conjunction with AUTO.OLD.CONT and AUTO.FIELD.NAME. When the user wants to run a subroutine or an interface defined in the EB.API record VERSION.CHECK.REC.RTN.HOOK, to default a value in a particular field, he should make the following input in these three fields: AUTO.FIELD.NAME - Indicates the field number for which you want to create a automatic default AUTO.OLD.CONT - Must remain blank AUTO.FIELD.RTN - Enter the subroutine name or an EB.API record of type METHOD which implements an interface defined in the EB.API record VERSION.CHECK.REC.RTN.HOOK which needs to be executed for the field in AUTO.FIELD.NAME. See the EB.API record VERSION.CHECK.REC.HOOK for the full list of supported interfaces, initially EB.TemplateHook.defaultFieldValues(). Validation Rules: Up to 35 type A (alphanumeric) characters. Multivalue field associated with Fields 1 to 2. (Optional input. No default value). The subroutine name or an EB.API record of type METHOD which implements an interface should be preceded with an @ symbol The subroutine entered should exist in PGM.FILE with PGM.TYPE = 'S' The VERSION ID should exist in field APPL.FOR.SUBR in the PGM.FILE record for the subroutine.
