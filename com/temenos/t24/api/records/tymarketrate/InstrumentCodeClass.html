        public TField getBidRate()

        Field description: It specifies the bid rate which is applicable for the period specified in the rate code field. For FX asset type,the field accepts premium/discount and pips to points conversion will take place for all types of feeds for ON,TN and forward period. For Spot period,the incoming rates are retained. Points conversion is not applicable for asset types other than FX.

        public TField getBidTrend()

        public TField getDaysSinceToday()
        Returns the DaysSinceToday

        Field description: For FX asset type, the field specifies the difference between rest date and spot date. It represents the difference in days with positive and negative sign to differentiate between spot and forward periods. Since spot is considered as start date , the periods falling below spot period will hold the difference with negative sign and the vice versa case for forward periods. For other asset types,it specifies the difference between rest date and today's date.

        Validation Rules: 1. Non input field. 2. System manipulated field based on the rest date.

      • getRatecode

        public TField getRatecode()
        Returns the Ratecode

        Field description: Specifies the period for which the market rates can be applied. System allows to enter the period in terms of Spot/Days/Months/Years.

        Validation Rules: 1. Values to this field can be of the format 'nP', where n represents the number, P can be D(Days), M(Months), Y(Year). 2. This field allows to define ON,TN and S period for FX asset type alone . 3. When the rate code entered is of format 'nD', the days can be only between 1 and 31. 4. When the rate code entered is of format 'nM', the months can be only between 1 and 12. 5. When the rate code entered is of format 'nY', the years can be only between 1 and 30. 6. Since this is a multivalue set, the rate code must be entered in a chronological sequence.

        Parameters:
        Returns:
        TField - The Ratecode
