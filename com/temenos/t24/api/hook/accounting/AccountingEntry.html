Class AccountingEntry

Packaged in : AC.AccountHook.jar

The AccountHook component allows access to routines for raise overrides from a local development and export acccouting entries to local development. There are hooks for ACCOUNTING.SUBRTN and ACCT.NAU.SUBRTN in ACCOUNT.PARAMETER.

        public void exportEntries(String entryType, String actionType, String statementId, StmtEntryRecord accountingEntry)
        This Interface enables the implementer for export accounting entries to local developement.
        This interface is invoked during the accounting phase of any transaction within T24.

        T24 details: Interface method for the ACCOUNTING.SUBRTN field in ACCOUNT.PARAMETER in T24. The EB.API hook used by this interface is HOOK.ACCT.PARAM.ACCTNG.SUBRTN. If an exception is thrown in the implementing class exception message will be logged in the EXCEPTION.LOG file.
        Export accounting entries during the accounting phase of any transaction within T24.
        Parameters:
        entryType - Type of entry like STMT or CATG or CONSOL or FWD.
        actionType - Action Type can be "VAL" for Input Entries,"VALAUT", "ADD.AUT", "SAO".
        statementId - Id of the statement Entry.
        accountingEntry - The actual accounting entries.
        Returns:
        nothing

      • raiseOverrides

        public List<String> raiseOverrides(String entryType, String actionType, String forward, StmtEntryRecord statementEntry)
        This Interface enables the implementer to raise any overrides from local development.
        This interface is invoked during the input phase of any transaction within T24.It will accept accounting entries and returns overrides.

        T24 details: Interface method for the ACCT.NAU.SUBRTN field in ACCOUNT.PARAMETER in T24. The EB.API hook used by this interface is HOOK.ACCT.PARAM.ACCT.NAU.SUBRTN. If an exception is thrown in the implementing class exception message will be logged in the EXCEPTION.LOG file.
        Parameters:
        entryType - Type of entry- like STMT or CATG.
        actionType - Action Type can be "VAL" for Input Entries.
        forward - Optional parameter can be "".
        statementEntry - Record of the statement entry id .
        Returns:
        List<String> - a list of override messages at the input phase of any transaction.
