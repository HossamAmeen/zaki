        public void addNewAssetOrder(java.lang.CharSequence param)
        Appends the specified NewAssetOrder to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Using this field User can define their own ASSET TYPE order in which the COLLATERAL RIGHTS of a liability will be utilized. This will overwrite the order (ASCENDING/DESCENDING) specified in the field USE.COLLATERAL. Validation Rules: Must be a valid COLLATERAL.CODE record id. Input is allowed only when the PRIORITY.RECEIVE is set as ASSET.TYPE. NULL – Order mentioned in USE.COLLATERAL field otherwise system default.

        public TField getNewPriorityReceive()
        Returns the NewPriorityReceive

        Field description: This field is used to specify the order by which collateral should be sorted. Validation Rules: 1 – ASSET TYPE - COLLATERAL.RIGHT received will be sorted by Asset first. When more than one asset of same type from different customers, then it will be further sorted by Customer key. The sorting can be set to sort largest to smallest or vice versa. 2 – CUSTOMER - COLLATERAL.RIGHT received will be sorted by Customer first. When there are multiple Assets for the same customer then they will be further sorted by Asset. The sorting can be set to sort largest to smallest or vice versa. Null – System default (ASSET TYPE).
