      • getType

        public TField getType()
        Returns the Type

        Field description: This fied will list out the HARDCODED list of values that guide the system on how to retrieve value to be shown for the field. Available options are: 1. Calculation.routine.field This type of filed name will be used as routine argument. The argument can be Input or Output argument. Generally the Input argument would be configured either as constant or it can be configured to get the value from any property Conditions or Standard type of marketing catalogue field (Condition Type, Relationship margin type etc). The output argument will be used simply to display the output on the screen. Example The routine AA.MC.GET.INTEREST.RATE will calculate the effective interest rate by using the interest condition and other necessary details.

        The list of incoming arguments of this routine. These incoming arguments can calculation routine field type or another standard type of field or any other field which will get the data from the condition.

        Interest Condition Details: This is "Condition" type of argument. It will be configured to get the whole condition of INTEREST. It will get the AA.PRD.CAT.INTEREST condition and will be used to calculate the interest rate. Currency: The Currency will be taken from the Product designer record from the marketing catalogue. It is Calculation routine field type. Amount: The Loan Amount is something will user provide as Selection value From the Marketing Catalogue. It is Calculation routine field type. LoanTerm: The Loan Term is something will user provide as Selection value From the Marketing Catalogue. It is Calculation routine field type. Relationship Pricing Details: This is "Relationship margin" type of marketing catalogue fields. It will get the relationship margin details for the current customer given in the marketing catalogue selection criteria. These margin details will be used to determine the final interest rate.

        The list of outgoing arguments of this routine. These arguments will be defined as Calculation Routine Field type of arguments. So system will not perform any validation for these type of fields while designing the marketing catalogue designer definition. And these fields will be mapped to the routine as output arguments. The routine calculated interest details will be displayed in these arguments in marketing catalogue output screen.

        RelPricingRate - Relationship pricing interest rate - Discounted interest rate RelOperand - Relationship Operand RelPricingAmount - Relationship pricing charge amount - Discounted charge amount RelPercentage - Relationship Pricing Charge percentage RetError - Processing error returned from routine

        2. Charge.type - This option will be applicable for the charge property. It will displays the charge type as either Fixed or Calculated based on charge definition. Validations When user selects the Charge.Type of field then Source Property is mandatory and it should be charge property.

        3. Condition This option will get the property condition (AA.PRD.CAT.XXX) for given property from the product. Generally this condition will be used to calculate the Interest rate where whole interest definition required. The condition details will be hidden on the marketing catalogue output screen.

        4. Defined This option will specify whether the Source property given for this field exist in the product or not. If the property exist in the product then Define will be displayed as YES, if property doesn't exist in the product then Define will displayed as "NO".

        5. Forward.property.description This option will be used to display the forward dated interest and charge details. The forward condition can either dated condition change for the current condition or ARRANGEMENT, START or ANNIVERSARY type of condition which will apply on arrangement after some period. Suppose ARRANGEMENT + 1Y, START + 1Y or ANNIVERSARY + 1Y. This option will display only the forward condition description. If product doesn't have any forward condition then there is no detail displayed for this field. The interest details will be displayed like description.

        Example

        i. If it is a period, it would return as "After 3 months - " followed by the Interest/Charge description as it does today. So, for example, the current and Forward columns would like below: Interest Description - 10% Forward Description - 3 months after Disbursement - 10%

        ii. If it is a hard forward date "From" prefix would be used.

        6. Payment.schedule.calculation.type This option will be applicable only for the payment schedule property and for the loan products. It will display the payment schedule calculation type. There are only two payment calculation types exist for the loans. a. Linear The payment amount will be calculated as equal monthly principal payments and interest calculated on the outstanding principal balance. The payment amount will be different for each month. b. Annuity In this calculation type, the payment amount will be same for all schedules for entire tenure of the loan. The payment amount will consists of both Interest and principal balances.

        7. Payment.schedule.frequency This option will be applicable only for the payment schedule property and for the loan products. It will displays the hard payment frequency which is defined in the payment schedule condition. Example - e0Y e1M e0W o30D e0F

        8. Payment.schedule.frequency.description - This option will be applicable only for the payment schedule property and for the loan products. It will displays the description of the payment frequency which is defined in the payment schedule condition. Example The description will be displayed as "Monthly on day 30" for frequency "e0Y e1M e0W o30D e0F"

        9. Property.description: A field which will keep the description of the property. It will be applicable for all properties. This field will build the description automatically by using product designer condition for interest and for charge property. For other properties the description given in the FULL.DESCRIPTION field will be displayed as property description.

        10. Relationship.margin This option will display the relationship margin details for interest and for charge properties for the customer given in the marketing catalogue selection criteria. The details are consists of Relationship Operand, Relationship Margin, Relationship Pricing Amount and Relationship Percentage.

        Returns:
        TField - The Type

The output argument will be used simply to display the output on the screen. Example The routine AA.MC.GET.INTEREST.RATE will calculate the effective interest rate by using the interest condition and other necessary details.

        The list of incoming arguments of this routine. These incoming arguments can calculation routine field type or another standard type of field or any other field which will get the data from the condition.

        Interest Condition Details: This is "Condition" type of argument. It will be configured to get the whole condition of INTEREST. It will get the AA.PRD.CAT.INTEREST condition and will be used to calculate the interest rate. Currency: The Currency will be taken from the Product designer record from the marketing catalogue. It is Calculation routine field type. Amount: The Loan Amount is something will user provide as Selection value From the Marketing Catalogue. It is Calculation routine field type. LoanTerm: The Loan Term is something will user provide as Selection value From the Marketing Catalogue. It is Calculation routine field type. Relationship Pricing Details: This is "Relationship margin" type of marketing catalogue fields. It will get the relationship margin details for the current customer given in the marketing catalogue selection criteria. These margin details will be used to determine the final interest rate.

        The list of outgoing arguments of this routine. These arguments will be defined as Calculation Routine Field type of arguments. So system will not perform any validation for these type of fields while designing the marketing catalogue designer definition. And these fields will be mapped to the routine as output arguments. The routine calculated interest details will be displayed in these arguments in marketing catalogue output screen.

        RelPricingRate - Relationship pricing interest rate - Discounted interest rate RelOperand - Relationship Operand RelPricingAmount - Relationship pricing charge amount - Discounted charge amount RelPercentage - Relationship Pricing Charge percentage RetError - Processing error returned from routine

        2. Charge.type - This option will be applicable for the charge property. It will displays the charge type as either Fixed or Calculated based on charge definition. Validations When user selects the Charge.Type of field then Source Property is mandatory and it should be charge property.

        3. Condition This option will get the property condition (AA.PRD.CAT.XXX) for given property from the product. Generally this condition will be used to calculate the Interest rate where whole interest definition required. The condition details will be hidden on the marketing catalogue output screen.

        4. Defined This option will specify whether the Source property given for this field exist in the product or not. If the property exist in the product then Define will be displayed as YES, if property doesn't exist in the product then Define will displayed as "NO".

        5. Forward.property.description This option will be used to display the forward dated interest and charge details. The forward condition can either dated condition change for the current condition or ARRANGEMENT, START or ANNIVERSARY type of condition which will apply on arrangement after some period. Suppose ARRANGEMENT + 1Y, START + 1Y or ANNIVERSARY + 1Y. This option will display only the forward condition description. If product doesn't have any forward condition then there is no detail displayed for this field. The interest details will be displayed like description.

        Example

        i. If it is a period, it would return as "After 3 months - " followed by the Interest/Charge description as it does today. So, for example, the current and Forward columns would like below: Interest Description - 10% Forward Description - 3 months after Disbursement - 10%

        ii. If it is a hard forward date "From" prefix would be used.

        6. Payment.schedule.calculation.type This option will be applicable only for the payment schedule property and for the loan products. It will display the payment schedule calculation type. There are only two payment calculation types exist for the loans. a. Linear The payment amount will be calculated as equal monthly principal payments and interest calculated on the outstanding principal balance. The payment amount will be different for each month. b. Annuity In this calculation type, the payment amount will be same for all schedules for entire tenure of the loan. The payment amount will consists of both Interest and principal balances.

        7. Payment.schedule.frequency This option will be applicable only for the payment schedule property and for the loan products. It will displays the hard payment frequency which is defined in the payment schedule condition. Example - e0Y e1M e0W o30D e0F

        8. Payment.schedule.frequency.description - This option will be applicable only for the payment schedule property and for the loan products. It will displays the description of the payment frequency which is defined in the payment schedule condition. Example The description will be displayed as "Monthly on day 30" for frequency "e0Y e1M e0W o30D e0F"

        9. Property.description: A field which will keep the description of the property. It will be applicable for all properties. This field will build the description automatically by using product designer condition for interest and for charge property. For other properties the description given in the FULL.DESCRIPTION field will be displayed as property description.

        10. Relationship.margin This option will display the relationship margin details for interest and for charge properties for the customer given in the marketing catalogue selection criteria. The details are consists of Relationship Operand, Relationship Margin, Relationship Pricing Amount and Relationship Percentage.

        Parameters:
        value - The value to set
        Returns:
        nothing
