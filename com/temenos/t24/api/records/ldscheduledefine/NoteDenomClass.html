      • getNoteDenom

        public TField getNoteDenom()
        Returns the NoteDenom

        Field description: Specifies the denomination of Notes which will be repurchased as part of a new Bond Issue. Together with the field NOTE.QUANT, this field specifies the value of Bonds repurchased at this denomination. The total value of the notes purchased, must equal the associated AMOUNT. A Bond issue may contain notes of different denominations so this field is sub-valued to allow such cases to be defined. When a scheduled repayment takes place the system will attempt to reduce the value of the notes according to the denominations specified here and in NOTE.QUANT. If it is not possible to allocate the amounts accross the denominations, an exception message will be raised. Validation Rules: Optional field. Must be a positive amount obeying the currency decimal definitions. No duplicate values allowed. The denomination must be defined in the main LD record. Only allowed with a P schedule definition. Not allowed if there is no note denomination on the main LD record. Mandatory if notes are defined on the main LD record. Not allowed if associated FREQUENCY field contains data.

        Parameters:
        Returns:
        TField - The NoteDenom

      • getNoteQuant

        public TField getNoteQuant()
        Returns the NoteQuant

        Field description: Specifies the number of notes of the associated NOTE.DENOM which will repaid when these bonds are repurchased. Together with the field NOTE.DENOM, this field specifies the value of Bonds repurchased at this denomination. The total value of the notes issued, must equal the assocaited AMOUNT of the contract. A Bond issue may contain notes of different denominations so this field is multi-valued to allow such cases to be defined. When a scheduled repayment takes place the system will attempt to reduce the value of the notes issued by the number specified here. If it is not possible to allocate the amounts accross the denominations, an exception message will be raised. Validation Rules: Optional field. Must contain a positive whole number, greater than zero. Not allowed if the associated NOTE.DENOM is not specified. Not allowed if there are no denominations specified on the main LD record. Mandatory with a P schedule when denominations are defined on the main LD record. Only allowed with a P schedule. Cannot exceed the number available on the main LD record of the associated NOTE.DENOM. Not allowed if the associated FREQUENCY field contains data.
