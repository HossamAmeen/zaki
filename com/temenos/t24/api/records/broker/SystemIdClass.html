      • getCategoryFrom

        public TField getCategoryFrom()
        Returns the CategoryFrom

        Field description: Used with field CATEGORY.TO below to define a range of category codes to be matched for the rates defined below. If no values are defined, then all category codes will successfully match. The set of fields SYSTEM.ID to TXN.CODE, are used to determine different brokerage rates applicable to specific deal types. The parameters system id, category code, term, currency, and customer type are taken from the deal at input stage, and a match is located within this record. If one is found then that rate can be used for automatic brokerage calculations. If one is not found and no default value exists (i.e. a multivalue group that has all match parameters blank), then no automatic calculation can be performed and the user will be required to enter a manual amount. This field together with field CATEGORY.TO below are used to determine the category ranges. This can be used to identify separate deal types within the product type defined in field SYSTEM.ID above. So for example MM Loans can be distinguished from MM Deposits, etc. Examples: If a record on this file were set up as : SYSTEM.ID MM CATEGORY.FROM 21000 CATEGORY.TO 21999 TIME 1M CURRENCY USD BANK.RATE 0.0001 NON.BNK.RATE 0.0002 CURRENCY GBP BANK.RATE 0.001 NON.BNK.RATE 0.005 CURRENCY BANK.RATE 0.01 NON.BNK.RATE 0.02 SYSTEM.ID CATEGORY.FROM CATEGORY.TO TIME CURRENCY BANK.RATE 0.1 NON.BNK.RATE 0.5 Example 1 : If a Money Market deal was input with Category code '21050', with days from value date to maturity date as '15', with deal currency as 'GBP', and with a bank type customer, then the rate used in order to calculate the brokerage amount would be '0.001'. Example 2 : If a Money Market deal was input with Category code '21050', with days from value date to maturity date as '15', with deal currency as 'JPY', and with a non-bank type customer, then the rate used in order to calculate the brokerage amount would be '0.02'. Example 3 : If a Money Market deal was input with Category code '22050', with days from value date to maturity date as '15', with deal currency as 'JPY', and with a non-bank type customer, then the rate used in order to calculate the brokerage amount would be '0.5'. Validation Rules: 5 digits. (Optional Input) Multivalue field associated with fields SYSTEM.ID to TXN.CODE.

Example 2 : If a Money Market deal was input with Category code '21050', with days from value date to maturity date as '15', with deal currency as 'JPY', and with a non-bank type customer, then the rate used in order to calculate the brokerage amount would be '0.02'. Example 3 : If a Money Market deal was input with Category code '22050', with days from value date to maturity date as '15', with deal currency as 'JPY', and with a non-bank type customer, then the rate used in order to calculate the brokerage amount would be '0.5'. Validation Rules: Valid values are 'TF', 'MM', 'LD', 'FR', 'SW', 'MG' and 'FD'. (Optional input) Multivalue field associated with fields SYSTEM.ID to TXN.CODE.

      • getTime

        public TField getTime()
        Returns the Time

        Field description: Used to match separate time periods for the term of deals to distinguish different brokerage rates defined below. The set of fields SYSTEM.ID to TXN.CODE are used to determine different brokerage rates applicable to specific deal types. The parameters system id, category code, term, currency and customer type are taken from the deal at input stage and a match is located within this record. If one is found then that rate can be used for automatic brokerage calculations. If one is not found and no default value exists (ie. a multivalue group that has all match parameters blank), then no automatic calculation can be performed and the user will be required to enter a manual amount. The term from the deal is defined as the number of days from the value date to the maturity date. This figure is extracted at deal input time and is used to find a match here. The values in this field should be defined incrementally, i.e. 0D for call/notice deals, 7D, 1M, 3M, 6M, 1Y, etc. If no value is entered here then all terms will match successfully. Validation Rules: In the format: [N][TYPE] where N can be 1-3 numeric digits and TYPE can be either 'D' - Days, 'M' - Months or 'Y' - Years. (Optional Input) Multivalue field associated with fields SYSTEM.ID to TXN.CODE.
