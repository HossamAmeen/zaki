      • getAmtToBroker

        public TField getAmtToBroker()
        Returns the AmtToBroker

        Field description: Specifies the particular amount of an order which has been placed with the previously defined Broker. The multi value fields 21, 22 23 can be used by the Dealers to record the fact that a particular order for aparticular amount of stock has already been placed with a particular Broker. The responsibility for monitoringthese limits will then fall on the Broker himself. The multi value facility is to allow more than one Broker should the Dealer wish to spread the Order amongstseveral Brokers. These fields however are initially an aide memoire for the Dealer involved as he need not actually deal with thespecified Broker. He is at liberty to trade with others and when he processes the execution T24 will allow him theopportunity to enter a different Broker. Care should be taken from a procedural point of view, however, that thedeal is not duplicated. Validation Rules Numeric input. (Optional Input)

        public TField getBroker()
        Returns the Broker

        Field description: Specifies the counterparty with whom the trade is to be effected. If an Investment Manager would like a particular trade passed onto a particular broker he could enter apreference at this point with perhaps a reason in the free format narrative field. Normally however the choice ofthe broker would be left to the Dealers themselves. The CUSTOMER thus entered must be defined as BROKER, CLIENT,COUNTERPARTY in CUSTOMER.TYPE field of CUSTOMER.SECURITY. If a CUSTOMER is defined as CUSTOMER in CUSTOMER.TYPEfield of CUSTOMER.SECURITY, a check will be made to see if the CUSTOMER has any DEALER BOOK accounts. If theCUSTOMER has a dealer book portfolio then the customer will be allowed as a valid input. Validation Rules 1 - 6 Numeric Customer code within the range 1 - 999999. or 3 - 10 type MNE (uppercase alpha or numeric or . )character Mnemonic Customer ID. (Optional Input). Must exist as a Customer type Broker, Counterparty, Client on the Customer Security File. If a CUSTOMER id with dealer book portfolio will be allowed.

        Returns:
        TField - The Broker
