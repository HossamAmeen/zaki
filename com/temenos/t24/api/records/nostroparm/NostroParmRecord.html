      • addDivertAddress

        public void addDivertAddress(java.lang.CharSequence param)
        Appends the specified DivertAddress to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field specifies any addresses which are to be used to divert the SWIFT messages. One or more diversion addresses must be set up in order to utilise reconciliation with the NOSTRO-PC package. This means that for each company or customer, depending upon your requirements, a SWIFT.2 address should be set up on the DE.ADDRESS file where the delivery address should contain the contents of DIVERT.ADDRESS. Also the DE.PRODUCT file for 950 messages should include SWIFT.2 as one of the carrier address numbers. Validation Rules: 8 or 11 type SSS (SWIFT) characters. Mutivalue field. (Mandatory input).

        Parameters:
        param - the DivertAddress to append
        Returns:
        nothing

        public TField getReportsToPrint()
        Returns the ReportsToPrint

        Field description: This fields specifies whether to print Nostro interface reports for audit trail purposes. If this field contains JOURNAL+STATUS then two reports will be printed. Report 1 is NOSTRO.INTERFACE.REPORT which is a report by currency broken down further by account number which shows the of the pseudo SWIFT MT 950 messages which have been extracted.
