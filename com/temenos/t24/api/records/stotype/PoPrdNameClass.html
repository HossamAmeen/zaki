      • getPoPrdName

        public TField getPoPrdName()
        Returns the PoPrdName

        Field description: This field is used to specify the PAYMENT.ORDER.PRODUCT id and it is significant only if its associated field PO.VERSION is defined. This being a multi value field each product can have specific version defined to be used for creating Payment order from Standing order application. When this field is defined, then the respective version record will be defaulted in the Standing Order record when the STO type and the Payment order product defined in this field is used in Standing Order. The value defined in this field will not be defaulted to the Payment order product field in Standing Order Validation Rules: Input allowed only when PI product is installed else this will be a NOINPUT field. Should be a valid record in the file PAYMENT.ORDER.PRODUCT Associated multi-value field with PO.VERSION Optional field

        Parameters:
        Returns:
        TField - The PoPrdName

      • getPoVersion

        public TField getPoVersion()
        Returns the PoVersion

        Field description: This field is used to specify the PAYMENT.ORDER version, to be used while generating the Payment Order from Standing Order when SUPPRESS.FT is set to PAYMENT. The version defined in this field will be defaulted to Standing Order when the STO.TYPE and PO.PRD.NAME defined in the associated field is used in the Standing order. Different versions can be attached for various Payment order products using these associated multi-value fields Validation Rules: Input allowed only when PI product is installed else this will be a NOINPUT field. Must be valid Payment order version record Associated multi-value field with PO.PRD.NAME Optional field
