      • getApplication

        public TField getApplication()
        Returns the Application

        Field description: Specifies the application (and VERSION) to be used as the drilldown for a deal with details contained in CUSTOMER.POSITION linked to this parameter record. When individual deal details are displayed in an enquiry based on the customer position file, the details of the application to view should be defined in this field. An existing application, or Version of an application should be defined. Applications can be identified by the I-descriptor EB.SYSTEM.ID which extracts element 3 of the CUSTOMER.POSITION key. Alternatively the CATEGORY.CODE could also be used to identify the application. CONVERSION @ E.CUS.POS.NEXT.LEVEL --------------------------------- Within CUSTOMER.POSITION enquiries the conversion routine @ E.CUS.POS.NEXT.LEVEL can be used to define the next level for drilldown. This performs the following processing: The element DETAIL.NEXT.LEVEL should be defined either as a SELECTION field or as an element in the ENQUIRY record. IF the value associated with this is not "NO" then: - For system id = AC the enquiry STMT.ENT.BOOK will be used. This will show the entries accross the account for the last 7 working days, unless a number of days is defined in the SELECTION AC.NO.DAYS, in which case this number of days will be shown. - For system id = SC the enquiry SC.SEC.ACC.HOLD.SUM will be used. This shows all securities held for the portfolio. - For all other applications the underlying application record will be displayed according to the defininition of this field. If no relevant record applies, the APPLICATION defined in the EB.SYSTEM.ID record will be used if present. If the DETAIL.NEXT.LEVEL value is NO: - If the Limit referenc in the CUSTOMER.POSITION key is SCPORT, the enquiry SC.PORT.HOLD.SUM will be run for the customer. - A limit reference beginning CURR will run the enquiry ACCT.BAL.LAST to display the accounts held for the customer. - A limit reference of COLL will run enquiry CO.001 for the customer to display collateral details. - A limit reference of COMMIT will run the enquiry LD.COMMITMENTS to show commitment contracts held by the customer. - A limit refernce of type FX will run FX.ENQ.SUMMARY to show FX deals held by the customer. - A global limit reference will run the enquiry LIAB showing all limits. - All other limit reference values will run LIM.TXN to show limit details. Details are returned to the enquiry in the following format: Next application / enquiry ">" Selection 1 ">" Selection 2 ">" Selection 3 Where Application / Enquiry contains either: Application Name S id (E.g FOREX S FX9501212345) for an application or STMT.ENT.BOOK Selection 1 to 3 contains the required selection criteria for the next level enquiry, and is null for application drilldown. Validation Rules: Must refer to an existing application of types H,U,L,T,W in the PGM.FILE. If a VERSION is specified (id contains ",") the record must exist in the VERSION application.

        public TField getAssetLiab()
        Returns the AssetLiab

        Field description: Specifies whether the Customer Position record should be displayed as an Asset or Liability item. When CUSTOMER.POSITION records are created, each item is designated as an ASSET or LIABILITY according to the sign of the DEAL.AMOUNT. A positive value is defined as an ASSET, a -ve value as an asset. The field ASSET.LIAB.MARKER is updated with an A or L respectively. Should this value wish to be reversed for a given record type, this field should be defined to contain the correct marker, which can then be referred to as required in the enquiry. Validation Rules: Optional input. A or L only allowed.

        public TField getNarrType()
        Returns the NarrType

        Field description: Contains the key to the STMT.NARR.FORMAT record used to define the format of narrative to be displayed with details relevant to a CUSTOMER.POSITION record. The details of the format narrative should be defined using the application STMT.NARR.FORMAT, the id of which is identified by this field. Narrative details should be based on the CUSTOMER.POSITION file. CONVERSION @ E.CUS.NARRATIVE ---------------------------- The conversion @ E.CUS.NARRATIVE can be used to build the narrative defined by the NARR.TYPE. The value passed to the conversion should be the NARR.TYPE extracted from the CPEP.RECORD I-descriptor. The conversion will populate field DISPLAY.NARRATIVE in the work record used by ENQUIRY. This can then be extracted and displayed as any other enquiry field, using another field name. The narrative built may be multi-valued. Example: 1) Build the narrative 10.20 FIELD.NAME..... NARRATIVE 11.20. 1 OPERATION... FIELD CPEP.RECORD 13.20 LENGTH.MASK.... 30L 14.20. 1 CONVERSION.. FIELD >,3,1 14.20. 2 CONVERSION.. @ E.CUS.NARRATIVE 15.20. 1 COMMENTS.... This should not be displayed 15.20. 2 COMMENTS.... directly, but should extracted 15.20. 3 COMMENTS.... from the field DISPLAY.NARR... 15.20. 4 COMMENTS.... to ensure multi vlaues are 15.20. 5 COMMENTS.... preserved 18.20 SINGLE.MULTI... M 2) Display the result 10.21 FIELD.NAME..... DISPLAY.NARRATIVE 11.21. 1 OPERATION... DISPLAY.NARRATIVE 12.21 COLUMN......... 26 16.21 TYPE........... L 18.21 SINGLE.MULTI... M Validation Rules: Optional input. Input must exist in STMT.NARR.FORMAT.

      • getSortPosition

        public TField getSortPosition()
        Returns the SortPosition

        Field description: Defines a value to be used when sorting the display order of customer position records. This allows a value to be attached to a CUSTOMER.POSITION record which can be used to sort information into a required order. I-Descriptor SORT.POSITION -------------------------- The I-Descriptor SORT.POSITION in the CUSTOMER.POSITION Standard Selection record is intended to provide a means of sorting Customer Position records into a designated Category code order. Where a SORT.POSITION is defined in this field, this value will be used, otherwise the CATEGORY.CODE is used. In this way items such as Commitments (category code 21095-99) can be allocated a Sort Position of 40000 to ensure they appear at the end of deal processing. Similarly items without a category code (e.g CO, LI, SC applications) can be assigned a value. Validation Rules: Optional input. Up to 5 alpha numeric characters allowed.

        Returns:
        TField - The SortPosition

CONVERSION @ E.CUS.POS.NEXT.LEVEL --------------------------------- Within CUSTOMER.POSITION enquiries the conversion routine @ E.CUS.POS.NEXT.LEVEL can be used to define the next level for drilldown. This performs the following processing: The element DETAIL.NEXT.LEVEL should be defined either as a SELECTION field or as an element in the ENQUIRY record. IF the value associated with this is not "NO" then: - For system id = AC the enquiry STMT.ENT.BOOK will be used. This will show the entries accross the account for the last 7 working days, unless a number of days is defined in the SELECTION AC.NO.DAYS, in which case this number of days will be shown. - For system id = SC the enquiry SC.SEC.ACC.HOLD.SUM will be used. This shows all securities held for the portfolio. - For all other applications the underlying application record will be displayed according to the defininition of this field. If no relevant record applies, the APPLICATION defined in the EB.SYSTEM.ID record will be used if present. If the DETAIL.NEXT.LEVEL value is NO: - If the Limit referenc in the CUSTOMER.POSITION key is SCPORT, the enquiry SC.PORT.HOLD.SUM will be run for the customer. - A limit reference beginning CURR will run the enquiry ACCT.BAL.LAST to display the accounts held for the customer. - A limit reference of COLL will run enquiry CO.001 for the customer to display collateral details. - A limit reference of COMMIT will run the enquiry LD.COMMITMENTS to show commitment contracts held by the customer. - A limit refernce of type FX will run FX.ENQ.SUMMARY to show FX deals held by the customer. - A global limit reference will run the enquiry LIAB showing all limits. - All other limit reference values will run LIM.TXN to show limit details. Details are returned to the enquiry in the following format: Next application / enquiry ">" Selection 1 ">" Selection 2 ">" Selection 3 Where Application / Enquiry contains either: Application Name S id (E.g FOREX S FX9501212345) for an application or STMT.ENT.BOOK Selection 1 to 3 contains the required selection criteria for the next level enquiry, and is null for application drilldown. Validation Rules: Must refer to an existing application of types H,U,L,T,W in the PGM.FILE. If a VERSION is specified (id contains ",") the record must exist in the VERSION application.
