        public TField getBondCommType()
        Returns the BondCommType

        Field description: The CUSTOMER.SECURITY table contains default information at customer and/or portfolio level concerning reports,valuations and commissions. Potentially customers may have more than one portfolio and details of each of these portfolios and thecommissions applicable are contained in the related multi-value fields SEC.ACC.SUFF, SHARE.COMM.TYPE,BOND.COMM.TYPE, DIV.COMM.TYPE, RED.COMM.TYPE and STOCK.COMM.TYPE. SEC.ACC.SUFF contains the portfolio suffix number to which the commission rates within fields SHARE.COMM.TYPEand BOND.COMM.TYPE etc., would apply. This field refers to the processing of bonds and allows for complexcalculations. Validation Rules: 11 (alphanumeric) character keys. (Mandatory only if field SEC.ACC.SUFF entered) Must be a valid entry on the FT.COMMISSION.TYPE table.
