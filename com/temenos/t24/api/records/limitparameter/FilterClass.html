      • getFilter

        public TField getFilter()
        Returns the Filter

        Field description: (Included from G14.0.00) When more than one limit is available to the customer, then this field is used to filter out the limits that can be used based on the certain conditions. There are some hard coded filter conditions such as: CHECK.EXPIRED: If this is given, then from the available limits to the customer, the expired limits will be filtered out. CHECK.AVAILABLE.MARKER: If this is given, then from the available limits to the customer, the limits where the AVAILABLE.MARKER is set to NO will be filtered out. CHECK.DRAW.DATE: If this is given, then from the available limits to the customer, the limits which had crossed LAST.DRAW.DATE will be filtered out CHECK.CUSTOMER.RESTRICT: If this is given, then the available limits to the customer will be checked for ALLOWED.CUST and RESTRICTED.CUST fields and filter out the limits accordingly CHECK.CURRENCY.RESTRICT: If this is given, then the available limits to the customer will be checked for ALLOWED.CCY and RESTRICTED.CCY fields and filter out the limits accordingly. CHECK.COMPANY.RESTRICT: If this is given, then the available limits to the customer will be checked for ALLOWED.COMP and RESTRICTED.COMP fields and filter out the limits accordingly Apart from the above hard coded filter fields, we can use any of the fields in LIMIT application in this field for filtering out the LIMIT record of the customer that we wish to use for the transaction. Validation Rules: Associated Multi-value fields from 44.1 to 47.1 Default value is Null The above Hard-coded values can be given as it is shown Apart from these, any relevant field from LIMIT record can be used Default is blank

        Parameters:
        Returns:

      • getFilterFrom

        public TField getFilterFrom()
        Returns the FilterFrom

        Field description: (Included from G14.0.00) Specifies the value, or start of a range of values, with which the contents of the field indicated in Filter (Field 44.1) are to be compared. Note: The values input in this field and the next must be in agreement with the Format rules of the element specified in Field 44.1 FILTER FIELD. If, for example, the input in FILTER field is from the hard-coded values as below, then only YES will be allowed in this field: CHECK.EXPIRED, CHECK.AVAILABLE.FUNDS, CHECK.AVAILABLE.MARKER, CHECK.DRAW.DATE, CHECK.CUST.RESTRICT, CHECK.CURRENCY.RESTRICT, CHECK.COMPANY.RESTRICT. Validation Rules: Up to 35 alphanumeric characters Multivalue field associated with Fields 44 to 47. Input should be YES when FILTER field above has been defined with any one of the hard-coded values mentioned above Otherwise input should be compatible with the fields of LIMIT record. Default value: None Input must begin and/or end with 3 dots when DECISION = 'LK' or 'UL'. Input for the other FILTER values (eg GE, EQ,LE, etc) will be checked according to the rules defined in the corresponding APPLICATION. It must be noted, however, that no check will be performed for the Local reference fields.

        Parameters:
        Returns:
        TField - The FilterFrom

      • getFilterOperands

        public TField getFilterOperands()
        Returns the FilterOperands

        Field description: (added from G14.0.00) Specifies the type of DECISION which the user wants to perform on the field defined in the previous field. Validation Rules: The following values will be accepted: EQ = Equal LK = Like GE = Greater or Equal LT = Less than LE = Less or Equal UL = Unlike GT = Greater than NE = Not Equal Multivalue field associated with fields 44.1 to 47.1 Input of EQ only is allowed if the hard-coded filter values are used The hard coded filter values are CHECK.EXPIRED, CHECK.AVAILABLE.FUNDS, CHECKAVAILABLE.MARKER, CHECK.DRAW.DATE, CHECK.CUST.RESTRICT, CHECK.CURRENCY.RESTRICT, CHECK.COMPANY.RESTRICT For any relevant field from LIMIT application is used in the field FILTER, then any operand compatible to the field can be used Default value: None.

        public TField getFilterTo()
        Returns the FilterTo

        Field description: Specifies the end of a range of values with which the contents of the field indicated in FILTER (Field 44.1) are to be compared. Note: The values input in this field and the previous one must be in agreement with the format rules of the element specified in Field 44.1 FILTER.
