        public TField getSubAssocCode()
        Returns the SubAssocCode

        Field description: Specifies whether a local reference field for an application is single value, multivalue, or subvalue, and the other local reference fields that are associated. For each application that uses this local reference the user may specify if the field is single value, multivalue, or subvalue. In addition they may also specify that other local references for the same application are associated. Example: a) Single value 1.1 1 LOCAL REF 1 2.1 left blank 1.2 2 LOCAL REF 2 2.2 left blank 1.3 3 LOCAL REF 3 2.3 left blank result on application: 157.1 LOCAL REF 1 ....... 157.2 LOCAL REF 2 ....... 157.3 LOCAL REF 3 ....... Fields cannot be expanded, only single value allowed for each local reference. b) Multivalue 1.1 1 LOCAL REF 1 2.1 XX. 1.2 2 LOCAL REF 2 2.2 XX. 1.3 3 LOCAL REF 3 2.3 XX. result on application: 157.1.1 LOCAL REF 1 ....... 157.1.2 LOCAL REF 1 ....... 157.2.1 LOCAL REF 2 ....... 157.3.1 LOCAL REF 3 ....... Fields can be expanded separately to allow multiple input in each local reference. In this example only 157.1.1 LOCAL REF 1 has been expanded. c) Associated subvalue 1.1 1 LOCAL REF 1 2.1 XX< (XX< denotes start of association) 1.2 2 LOCAL REF 2 2.2 XX- (XX- denotes part of association) 1.3 3 LOCAL REF 3 2.3 XX> (XX> denotes end of association) result on application: 157.1.1 LOCAL REF 1 ....... 157.1.2 LOCAL REF 1 ....... 157.2.1 LOCAL REF 2 ....... 157.2.2 LOCAL REF 2 ....... 157.3.1 LOCAL REF 3 ....... 157.3.2 LOCAL REF 3 ....... Fields are expanded together to allow multiple input for all local references. Validation Rules: 1 - 3 alpha characters. Multivalue field (Optional Input)
