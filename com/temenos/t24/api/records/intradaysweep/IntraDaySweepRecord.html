      • addSweepTime

        public void addSweepTime(java.lang.CharSequence param)
        Appends the specified SweepTime to the end of the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specify the time at which this group has to be executed automatically using phantom process. All the cash pool records relevant to this Group id with scheduled execution date as today, will be executed at the time specified here. To trigger above process, a record in EB.PHANTOM has to be added with Run.Routine as "AC.AUTO.SWEEP.SELECT" , which can be invoked either as Phantom or Interactive. This routine selects and triggers the cash pooling process for all the AC.CASH.POOL records which are scheduled for today with scheduled time less than system time. Incase the INTRA.DAY is mentioned as SINGLE, then after the cash pool group is executed, the next execution date in AC.CASH.POOL moved as per the frequency given. Incase the INTRA.DAY is mentioned as MULTIPLE, then after the cash pool execution, the next execution remains the same. Input is in the hh:mm format where: hh is the hour (in 24 hour format). Hours of 01 to 09 can also be entered as 1 to 9. mm is the minutes past that hour. The input of : is optional. The details given here is updated in AC.AUTO.SWEEP.SCHD application with id as the scheduled time along with the list of cash pool groups relevant for the scheduled time. Please note incase the sweep time mentioned here is Less than the system time-then system will not execute it, unless and otherwise the phantom process is re-started again. Moreover whenever the phantom process is restarted, all the cash pool with scheduled time less than the system time and with scheduled execution date as of today is executed once again. Validation Rules: Optional Multi value field Duplicate not allowed Must be in standard time format.

        Parameters:
        param - the SweepTime to append
        Returns:
        nothing

      • insertSweepTime

        public void insertSweepTime(java.lang.CharSequence param, int pos)
        Insert the specified SweepTime to the specified position in the collection.
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specify the time at which this group has to be executed automatically using phantom process. All the cash pool records relevant to this Group id with scheduled execution date as today, will be executed at the time specified here. To trigger above process, a record in EB.PHANTOM has to be added with Run.Routine as "AC.AUTO.SWEEP.SELECT" , which can be invoked either as Phantom or Interactive. This routine selects and triggers the cash pooling process for all the AC.CASH.POOL records which are scheduled for today with scheduled time less than system time. Incase the INTRA.DAY is mentioned as SINGLE, then after the cash pool group is executed, the next execution date in AC.CASH.POOL moved as per the frequency given. Incase the INTRA.DAY is mentioned as MULTIPLE, then after the cash pool execution, the next execution remains the same. Input is in the hh:mm format where: hh is the hour (in 24 hour format). Hours of 01 to 09 can also be entered as 1 to 9. mm is the minutes past that hour. The input of : is optional. The details given here is updated in AC.AUTO.SWEEP.SCHD application with id as the scheduled time along with the list of cash pool groups relevant for the scheduled time. Please note incase the sweep time mentioned here is Less than the system time-then system will not execute it, unless and otherwise the phantom process is re-started again. Moreover whenever the phantom process is restarted, all the cash pool with scheduled time less than the system time and with scheduled execution date as of today is executed once again. Validation Rules: Optional Multi value field Duplicate not allowed Must be in standard time format.

        Parameters:
        param - the SweepTime to append
        pos - The position where to insert the value.
