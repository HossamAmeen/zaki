      • allocateOrderNominals

        public void allocateOrderNominals(String applicationName, TBoolean isRunningInCob, SecOpenOrderRecord secOpenOrderRecord, ScExeSecOrdersRecord scExeSecOrdersRecord)
        This interface allows the developer to allocate the executed nominal for a single security order in a bulk order group.
        This is achieved by setting the CustNominal field in the scExeSecOrdersRecord parameter.
        This interface is invoked when inputting or executing a security order and by the services which execute parent child orders.

        T24 Details: The EB.API hook used by this interface is SC.STD.SEC.TRD.ORD.ALLOC.SUBR.HOOK. The T24 field specifying this hook is the ORDER.ALLOC.SUBR field in SC.STD.SEC.TRADE.
        This interface can be invoked while inputting a TRADED SEC.OPEN.ORDER with multiple brokers or on cross validation of a SC.EXE.SEC.ORDERS record.
        This interface is also invoked while processing the SC.PARENT.CHILD.EXE service.
        If an exception is thrown in the implementing class, the order will be cancelled; if running within a service the error will additionally be stored to the EXCEPTION.LOG.
        Parameters:
        applicationName - Name of the application from where this interface is being triggered
        isRunningInCob - Is the process run by the close of business process
        secOpenOrderRecord - This is the security order being processed
        scExeSecOrdersRecord - This is the executing part of the security order being processed
        Returns:
        nothing

      • isGroupSecurityOrder

        public TBoolean isGroupSecurityOrder(String applicationName, TBoolean isRunningInCob, SecOpenOrderRecord groupOrder, SecOpenOrderRecord candidateOrder)
        This interface enables the developer to implement logic to determine whether a candidate order is to be merged into a group order.
        This interface is only invoked if the system is configured to group orders.
        It is invoked while authorising a security order, on check field validation of security order group or by the services which execute manual grouping of security orders.

        T24 Details: The EB.API hook used by this interface is SC.GROUP.PARAM.GROUP.ROUTINE.HOOK. The T24 field specifying this hook is the GROUP.ROUTINE field in SC.GROUP.PARAM.
        This interface can be invoked while authorising a SEC.OPEN.ORDER or on check field validation of a SC.GROUP.ORDERS record when grouping is enabled for the order.
        The grouping is enabled via the company's SC.PARAMETER record and configured using SC.AUTO.GROUP.ORDERS.
        This interface is also invoked while processing the SC.MANUAL.GRP.ORD.EXE service.
        If an exception is thrown in the implementing class, the grouping will be cancelled; if running within a service the error will additionally be stored to the EXCEPTION.LOG.
        Parameters:
        applicationName - Name of the application from where this interface is being triggered
        isRunningInCob - Is the process run by the close of business process
        groupOrder - This is the order into which it is proposed to merge the candidate order
        candidateOrder - This order is a candidate for incorporation into the groupOrder
        Returns:
        TBoolean - a Boolean value indicating whether to merge the candidate order into the group order.
