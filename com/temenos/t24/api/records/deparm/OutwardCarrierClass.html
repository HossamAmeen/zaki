      • getOutwardCarrier

        public TField getOutwardCarrier()
        Returns the OutwardCarrier

        Field description: Specifies one of the outward carriers. Note that when OUTWARD CARRIER is 'TELEX' it applies to both inward and outward Telex and Telexp messages. The carrier must be defined on DE.CARRIER. If the carrier is a generic carrier (i.e. CARRIER.MODULE on DE.CARRIER contains "GENERIC"), messages are sent and received by the generic carrier control program, DE.CC.GENERIC. An OUTWARD CARRIER may only be changed or removed if the corresponding SHUT OUT CARR field in 'C' - closed.

        Must reference a record on DE.CARRIER, or null OUTWARD CARRIER and SHUT OUT CARR (Field 8.1) form a multivalue set listing the outward carriers and giving their status. Optional input

        Parameters:
        Returns:
        TField - The OutwardCarrier

      • getShutOutCarr

        public TField getShutOutCarr()
        Returns the ShutOutCarr

        Field description: Specifies the status of the Carrier Control module processing the OUTWARD CARRIER named in the previous field. Note that when the Outward Carrier is 'Telex' it applies to both inward and outward Telex messages. OUTWARD CARRIER and SHUT OUT CARR form a multivalue set listing the outward carriers and giving their status. In addition to 'N' and 'U' (described above) this field may also contain the following values that cannot be set on-line: 'C' Shutdown completed. 'E' End of Period processing is being run. 'A' The relevant Carrier Control module is active.

        'N' requests a normal shutdown after processing all the outward messages for OUTWARD CARRIER (Field 7.1). (For Telex: All inward and outward Telex Messages.) or 'U' requests an urgent shutdown after processing of the current message.

        Parameters:
        Returns:
        TField - The ShutOutCarr
