      • getIssueBandLevel

        public TField getIssueBandLevel()
        Returns the IssueBandLevel

        Field description: Specifies wether charges will be calculated on a banded or on a level basis. Charges may be calculated on either a band or level basis. The principle used is common to many other T24 modules. Band charging raises a different charge for each number of cheques falling into rising bands, level charging locates the level the number falls within and charges all cheques at that level. This is best illustrated with several examples as follows: In the following configuration a flat charge has not been defined (if it were, it would be added to the result of the band/level charges). Also the example uses ascending charges for larger numbers of cheques, descending bands and levels are equally possible and would in fact be more commonly used in practice. 5 ISSUE.BAND.LEVEL.. BAND 6.1 ISSUE.UPTO...... 75 7.1 ISSUE.CHG.AMT... 0.10 6.2 ISSUE.UPTO...... 150 7.2 ISSUE.CHG.AMT... 0.15 6.3 ISSUE.UPTO...... 7.3 ISSUE.CHG.AMT... 0.20 The issue of cheques would be charged at 0.10 GBP per cheque until the number issued during the charging period exceeds 75 whereupon the charge per cheque would become 0.15 GBP and so on up the band structure. Note the last band is always empty to imply a ceiling above which all cheques are charged at this maximum band rate. So for the issue of the first 25 cheques the charge would be: 25 x 0.10 GBP = 2.50 GBP If a further four cheque books of 25 cheques were subsequently issued they would be charged at: 50 x 0.10 GBP = 5.00 GBP 50 x 0.15 GBP = 7.50 GBP 12.50 GBP The next four book issue would be charged: 25 x 0.15 GBP = 3.75 GBP 75 x 0.20 GBP = 15.00 GBP 18.75 GBP A further 6 books would be simply: 150 x 0.20 GBP = 30.00 GBP If the same configuration were used on a LEVEL charging basis the first 25 cheques would be charged at 0.10 GBP per cheque. The next issue of 100 would be charged at the next higher level of 0.15 per cheque since 100 exceeds the first level of 75. The next 100 would also be charged at 0.15 per cheque as would the last. Only an issue of more than six books would invoke the highest defined band charge of 0.20 GBP per cheque. Validation Rules: 1-5 alphanumeric characters. Mandatory input. Must be either BAND or LEVEL only.

        Parameters:
        Returns:
        TField - The IssueBandLevel

      • getPeriodBandLevel

        public TField getPeriodBandLevel()
        Returns the PeriodBandLevel

        Field description: Specifies whether periodic charges will be calculated on a banded or on a level basis. Charges may be calculated on either a band or level basis. The principle used is common to many other T24 modules. Band charging raises a different charge for each number of cheques falling into rising bands, level charging locates the level the number falls within and charges all cheques at that level. This is best illustrated with several examples as follows: In the following configuration a flat charge has not been defined (if it were, it would be added to the result of the band/level charges). Also the example uses ascending charges for larger numbers of cheques, descending bands and levels are equally possible and would in fact be more commonly used in practice. 11 PERIOD.BAND.LEVEL.. BAND 12.1 PERIOD.UPTO...... 75 13.1 PERIOD.CHG.AMT... 0.10 12.2 PERIOD.UPTO...... 150 13.2 PERIOD.CHG.AMT... 0.15 12.3 PERIOD.UPTO...... 13.3 PERIOD.CHG.AMT... 0.20 The periodic usage of cheques would be charged at 0.10 GBP per cheque for the first 75 used during the charging period whereupon the charge per cheque would become 0.15 GBP and so on up the band structure. Note the last band is always empty to imply a ceiling above which all cheques are charged at this maximum band rate. So for the use of the first 25 cheques the charge would be: 25 x 0.10 GBP = 2.50 GBP If a further four cheque books of 25 cheques were subsequently used completely they would be charged at: 50 x 0.10 GBP = 5.00 GBP 50 x 0.15 GBP = 7.50 GBP 12.50 GBP The next 100 would be charged: 25 x 0.15 GBP = 3.75 GBP 75 x 0.20 GBP = 15.00 GBP 18.75 GBP A further 150 would be simply: 150 x 0.20 GBP = 30.00 GBP If the same configuration were used on a LEVEL charging basis the total number used would fall within a certain level of usage. The charge raised would be the amount per cheque associated with this level times the number of cheque used during the usage period. Validation Rules: 5 alphanumeric characters. Mandatory input. Must be either BAND or LEVEL only.

      • setIssueBandLevel

        public void setIssueBandLevel(TField value)
        Set the specified value for IssueBandLevel
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: Specifies wether charges will be calculated on a banded or on a level basis. Charges may be calculated on either a band or level basis. The principle used is common to many other T24 modules. Band charging raises a different charge for each number of cheques falling into rising bands, level charging locates the level the number falls within and charges all cheques at that level. This is best illustrated with several examples as follows: In the following configuration a flat charge has not been defined (if it were, it would be added to the result of the band/level charges). Also the example uses ascending charges for larger numbers of cheques, descending bands and levels are equally possible and would in fact be more commonly used in practice. 5 ISSUE.BAND.LEVEL.. BAND 6.1 ISSUE.UPTO...... 75 7.1 ISSUE.CHG.AMT... 0.10 6.2 ISSUE.UPTO...... 150 7.2 ISSUE.CHG.AMT... 0.15 6.3 ISSUE.UPTO...... 7.3 ISSUE.CHG.AMT... 0.20 The issue of cheques would be charged at 0.10 GBP per cheque until the number issued during the charging period exceeds 75 whereupon the charge per cheque would become 0.15 GBP and so on up the band structure. Note the last band is always empty to imply a ceiling above which all cheques are charged at this maximum band rate. So for the issue of the first 25 cheques the charge would be: 25 x 0.10 GBP = 2.50 GBP If a further four cheque books of 25 cheques were subsequently issued they would be charged at: 50 x 0.10 GBP = 5.00 GBP 50 x 0.15 GBP = 7.50 GBP 12.50 GBP The next four book issue would be charged: 25 x 0.15 GBP = 3.75 GBP 75 x 0.20 GBP = 15.00 GBP 18.75 GBP A further 6 books would be simply: 150 x 0.20 GBP = 30.00 GBP If the same configuration were used on a LEVEL charging basis the first 25 cheques would be charged at 0.10 GBP per cheque. The next issue of 100 would be charged at the next higher level of 0.15 per cheque since 100 exceeds the first level of 75. The next 100 would also be charged at 0.15 per cheque as would the last. Only an issue of more than six books would invoke the highest defined band charge of 0.20 GBP per cheque. Validation Rules: 1-5 alphanumeric characters. Mandatory input. Must be either BAND or LEVEL only.

        Parameters:
        value - The value to set
        Returns:
        nothing
