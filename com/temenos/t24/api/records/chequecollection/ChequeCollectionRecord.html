      • getChqStatus

        public TField getChqStatus()
        Returns the ChqStatus

        Field description: This field can be used to indicate a new status for the cheque. The field will be used to indicate the current status of the cheque. These statuses could any of the following: DEPOSITED - When the cheque is initially deposited on an application it is updated with this status. CLEARING - This status indicates that the cheque has now been batched on the CHEQUE.BATCH application record CLEARED - This status indicates that the funds on the cheque have been collected. RETURNED - This status indicates that the funds on the cheque could not be cleared and the cheque was returned It is possible to change the status of a cheque mannually here if the cheque has not been batched. Once the cheque has been batched all changes of the status field must be handled through the CHEQUE.BATCH application. If the status of the cheque is changed from 'DEPOSITED' to 'CLEARED' the system will automatically raise accounting entries to credit the original client account and debit the collection suspense account for the cheque amount. This action would also prompt the deletion of the forward entries created when the cheque was initially deposited. Any charges and/or tax amounts due will be raised. Validation Rules: No Input Allowed if the cheque collection record has been batched If change allowed the valid changes to the status include DEPOSITED to CLEARED or DEPOSITED to RETURNED Valid values are DEPOSITED, CLEARING, CLEARED or RETURNED

        Returns:
        TField - The ChqStatus

        public TField getCollateralLink()
        Returns the CollateralLink

        Field description: This field recognises three values viz., null, YES and NO. When the CC record is not attached to any account as Collateral, the field is defaulted to 'null'. When the field Application.ID in COLLATERAL application is populated with CQ.XXXXXX.XXXXX ( CQ.CUSTOMER. ACCOUNT), this field immediately defaults to 'YES' which enable it to link the CC record with the COLLATERAL record. This defaulted value 'YES' shall be changed to 'NO' for delinking an already linked CC from the COLLATERAL record. Validation Rules: The only input that shall be manually input is 'NO'. The value can be input as 'NO' only if the previous value is 'YES'.
