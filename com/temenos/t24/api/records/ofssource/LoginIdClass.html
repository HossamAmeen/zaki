      • getEbPhantId

        public TField getEbPhantId()
        Returns the EbPhantId

        Field description: Defines the name of a specific EB.PHANTOM record used to monitor the online communication when the associated LOGIN.ID is used. The record specified does not need to exist, it will be created when the communication begins. If no record is specified the system will allocate a record id automatically based on the SOURCE.NAME - connection number. The EB.PHANTOM record will show the following information when the communication process is in progress: RUN.MODE - OFS STATUS - ACTIVE set to CLOSED at end of communication RUN.STATUS - Source Id Session started Date Time RECEIVED date and time of last received msg REC MSG = extract of last received message TOTAL MSG REC = Total number of messages received SENT date and time of last message sent SENT MSG = extract of last sent message TOTAL MSG SENT = extract of last message sent Validation Rules: Alphanumeric 1 -15 characters

        Parameters:
        Returns:

      • getLoginId

        public TField getLoginId()
        Returns the LoginId

        Field description: Specifies the UNIX or NT login that will automatically initiate the communication with the external source. The external source will initiate communications by issuing a login using the specified id. The system will automatically begin the communication process when this login is detected. Automatic processes initiated at login are stored in the EB.AUTO.PROCESS application. This table is updated automatically based on the values stored in this field. The status of automatic processes is monitored using the EB.PHANTOM application and enquiry EP.MONITOR. A specific EB.PHANTOM.ID can be allocated if required using the associated EB.PHANT.ID field. Unix machines can have mixed case alphanumeric Login id's so the if the id is Validation Rules: Mandatory input when SOURCE.TYPE is TELNET. On Windows NT machines the id must be prefixed with the Servername and a '/' character.

        Returns:
