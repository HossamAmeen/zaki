      • getAutNewCont

        public TField getAutNewCont()
        Returns the AutNewCont

        Field description: Enables the User to define the new automatic default value. Must be used in conjunction with AUT.OLD.CONT (Field 32) and AUT.FLD.NO (Field 31). A routine (preceded by "@") or an EB.API record of type METHOD which implements an interface defined in the EB.API record VERSION.CHECK.REC.RTN.HOOK may be specified instead of actual values. Input in this field must be in accordance with the field definition of the selected Application. Quotes should only be used where the new data actually requires quotes e.g if a new value is required to be YES it should not be input as "YES". It is important to clearly understand the relation between this field and AUT.OLD. CONT. The following examples will clarify their use. 1. When the User wants to create an automatic default at the creation of new record ID's, he should then make the following input in these three fields : AUT.FLD.NO - Indicates the field number for which you want to create an automatic default. AUT.OLD.CONT - Must remain blank. AUT.NEW.CONT - Enter the value which you want to default automatically. 2. When you want to replace automatically an old content by a new content, you should then make the following input in these three fields : AUT.FLD. NO - Indicate the field number for which you want to change automatically the old content. AUT.OLD .CONT - Enter the old content AUT.NEW.CONT - Enter the new content which you want to replace automatically. 3. When you want to run a subroutine or to implement an interface defined in the EB.API record VERSION.CHECK.REC.RTN which will default a value in a particular field, you should make the following input in these three fields : AUT.FLD.NO - Indicates the field number for which you want to create a automatic default AUT.OLD.CONT - Must remain blank. AUT.NEW.CONT - Enter the subroutine name which needs to be executed for the the field in AUT.FLD.NO. Note: If the field AUT.OLD.CONT is specified, only records containing the specified value will be changed when using this Version. If the field AUT.OLD.CONT is not specified, the content specified in the field AUT.NEW.CONT will be generated when creating a new record with this Version. a) When a subroutine or an EB.API record of type METHOD which implements an interface defined in the EB.API record VERSION.CHECK.REC.RTN.HOOK needs to be executed then the subroutine name should be preceded with an @ symbol b) The subroutine entered should exist in PGM.FILE with PGM.TYPE = 'S' c) The VERSION ID should exist in field APPR.FOR.SUBR in the PGM.FILE record for the subroutine. d) The PGM.TYPE for the VERSION record should be one of the following :- 'U', 'H', 'W', 'M', 'S', 'B' Validation Rules: Up to 35 type A (alphanumeric) characters. (Optional input. No default value). Subvalue field associated with fields 31 to 34 (AUTOM.VALUE to AUTOM.INC.EX) Input of the old and the new content must be different.

        Parameters:
        Returns:
        TField - The AutNewCont
