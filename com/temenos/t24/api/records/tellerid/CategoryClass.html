      • getTillBalance

        public TField getTillBalance()
        Returns the TillBalance

        Field description: Specifies the current amount of cash present in the till - as entered by the teller. The current amount of cash present in the till must be entered when closing the till. This will be compared with the system maintained balance and any differences are reported as overrides. If the Yes is answered to the overrides then the differences are posted to the over/short accounts and the system balance adjusted accordingly. Validation Rules: 19 Numeric characters. (Input only when closing the till).

        Parameters:
        Returns:
        TField - The TillBalance

      • getTillClosBal

        public TField getTillClosBal()
        Returns the TillClosBal

        Field description: Specifies the current system balance - from the cash account. This field is updated ONLY when the system is being opened/closed. It is used for comparison with the input till balance when the till is being closed. The balance will be shown as negative. This is because it is a cash account and funds held in this type of account are always given a negative sign. You will be prevented from closing the till if this balance is positive - this means that the till has gone 'overdrawn' which should be impossible. This will have to be rectified either by TILL TRANSFERS or by DATA.CAPTURE (if another area of the system has removed funds) before the till can be closed. Validation Rules: 18 Numeric characters. (Noinput field)

        Parameters:
        Returns:
        TField - The TillClosBal
