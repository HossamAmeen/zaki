      • getOrigParty

        public TField getOrigParty()
        Returns the OrigParty

        Field description:

        This field is used when defining all the relationships within the group in one CUSTOMER.RELATIONSHIP record. This relationship defined here can be between T24 Customers, Persons or Entities. Allowing the definition of relationships with individuals or entities who are not customers of the bank.

        Forms part of a multi value set of fields from ORIG.PARTY to OWNING.PER

        Validation rules

        Optional input

        Valid values are:

        CUSTOMER

        PERSON

        ENTITY

        No input if the fields ORIG.RELATION or REL.RELATION are entered.

        Parameters:
        Returns:
        TField - The OrigParty

      • getOrigPartyId

        public TField getOrigPartyId()
        Returns the OrigPartyId

        Field description:

        This field is used to define the CUSTOMER or PERSON.ENTITY record of the originating party.

        Forms part of a multi value set of fields from ORIG.PARTY to OWNING.PER

        Validation rules

        Optional input

        Mandatory input if the field ORIG.PARTY is entered

        If ORIG.PARTY is "Customer" must be a valid record on T24 Customer application

        If ORIG.PARTY is "Person " must be a valid record on the PERSON.ENTITY application which has been classified as a PERSON record

        If ORIG.PARTY is "ENTITY" must be a valid record on the PERSON.ENTITY application which has been classified as an ENTITY record

        Returns:

        public TField getOrigRel()

        ADVISOR

        AUDITOR

        Forms part of a multi value set of fields from ORIG.PARTY to OWNING.PER
