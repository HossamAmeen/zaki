        public TField getOperand()
        Returns the Operand

        Field description: Defines for the associated SEL.FIELD, which operands are allowed for the enquiry. This field allows the selection operands to be limited for a particular enquiry. This may be required where the enquiry is based on one particular selection value, for example an enquiry based on Account movements may wish to restrict the BOOKING.DATE selection to RG or EQ. Validation Rules: : The following are the operands provided for selection : Null, EQ, NE, GT, GE, LT, LE, RG, NR, CT, NC, LK or UL.

      • getSelField

        public TField getSelField()
        Returns the SelField

        Field description: This fields defines the selection field which references the key to the file which is defining this record, e.g. the CUSTOMER record may define the ENQUIRY record LIAB to be one of the related context sensitive enquiries. The selection field of the enquiry which references the customer file is the LIABILITY.NUMBER selection field. This information should be defined here. For enquiries that are set to auto-launch at F5, additional functionality allows further specification, in the form. Selection field name Selection field name OPERAND data Selection field name OPERAND record field name e.g. LIABILITY.NUMBER CURRENCY EQ "USD" LIABILITY.NUMBER EQ CUSTOMER.NO The first syntax supplies the key to the record as the data, and EQ as the OPERAND. Validation Rules: Must be a SELECTION.FLD as defined on the related ENQUIRY record. or For enquiries that are to be auto launched at F5; A valid SELECTION FLD followed by A valid OPERAND followed by A valid field name, or data in quotes.

        Parameters:
        Returns:
        TField - The SelField
