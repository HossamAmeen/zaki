      • getCurrency

        public TField getCurrency()
        Returns the Currency

        Field description: Defines the Currency of any amounts input in the associated fields. Commissions must be specified in default or local currency and if special Commissions are to be applicable to different Currencies (e.g. different percentages/minimums/maximums) then Commissions must also be specified for each applicable Currency. The rules which determine which Currency (Credit or Debit) is initially used to identify the Commission Currency, are as follows:- For inward payments - Currency of the Debit. For all other payments - Currency of the Credit. If a specific Currency is not represented within the multivalue set, the System will default to the details associated to the default currency. If default currency is not defined then the local currency details will be used. The Currency equivalent amount will be derived at the currency table middle rate. If a default currency is defined then the default currency commission definition is mandatory, otherwise the local currency definition is mandatory. Validation Rules: 3 type SSS (uppercase alpha) character or 1-3 numeric character Currency code. Associated multivalue field. (Mandatory input for default currency if present, otherwise mandatory for local currency.)

        Parameters:
        Returns:
        TField - The Currency

        public TField getFlatAmt()
        Returns the FlatAmt

        Field description: Defines the amount to be charged for this Commission Type in the Currency defined in the preceding field. The User will enter an amount in this field when the Commission being defined is expressed as a Flat Amount, the Calculation Type is 'FLAT' and the charge amount does not vary according to the Transfer amount. If, however, it does vary according to the transfer amount, this field will be left blank and the User would then define various Percentages with their corresponding ranges if necessary. Validation Rules: Up to 14 AMT type characters (standard amount format). Associated multivalue field. (Mandatory input if CALCULATION TYPE states 'FLAT'.)

      • getOverallMaxAmt

        public TField getOverallMaxAmt()
        Returns the OverallMaxAmt

        Field description: Defines the Overall Maximum Amount to be charged on transfers involving the Currency defined. This amount represents the maximum to be charged irrespective of any special conditions which could exist for a Customer or group of Customers except if these have been specified as a Flat amount. This amount could represent the maximum amount the bank would ever conceivably charge a customer for a transfer regardless of the size of the Transfer Amount. Example : Let us suppose that "Wonder Bank" based in London applies the following payment commissions on incoming and outgoing funds transfers: 1) For any transfer in US Dollars (received or paid): - 2% for any amount up to USD 20,000.00 with a minimum of $50.00 and a maximum of $400.00. - 1% for any amount above USD 20,000.00 with a minimum of $400.00 and a maximum of $2,000.00. - An overall minimum of $50.00 and overall maximum of $2,000.00. 2) For transfers in any other currencies: - 1% for any amount up to GBP 10,000.00 equivalent with a minimum of 20.00 and a maximum of 100.00 - 1/4% for any amount above GBP 10,000.00 with a minimum of 100.00 and a maximum of 500.00. - An overall minimum of 20.00 and overall maximum of 500.00. The following input will then be made in this table. Field Name Field Input Common CALC TYPE Band Currency GBP Flat Amount Percentage 1% UPTO Amount 10,000.00 Minimum Amount 20.00 Maximum Amount 100.00 Percentage 1/4% Upto Amount Minimum Amount 100.00 Maximum Amount 500.00 Overall Minimum Amount 20.00 Overall Maximum Amount 500.00 Currency USD Flat Amount Percentage 2% Upto Amount 20,000.00 Minimum Amount 50.00 Maximum Amount 400.00 Percentage 1% Upto Amount Minimum Amount 400.00 Maximum Amount 2,000.00 Overall Minimum Amount 50.00 Overall Maximum Amount 2,000.00 Validation Rules: Up to 14 AMT type characters (standard amount format). Associated multivalue field. (Optional input.) It must be greater than the Overall Minimum Amount. It must be greater or equal to the last Maximum Amount specified within a range for the Currency.

        Returns:

      • getOverallMaxPct

        public TField getOverallMaxPct()
        Returns the OverallMaxPct

        Field description: Defines the Overall Maximum Percentage or rate to be charged on transfers involving the Currency defined. This represents the minimum to be charged irrespective of any special conditions which could exist for a Customer or group of Customers except if these have been specified as a Flat amount.Specifies the rate used to calculate the Commission for an entry as a Percentage of its value. This amount could represent the maximum amount the bank would ever conceivably charge a customer for a transfer regardless of the size of the Transfer Amount. Available Options for input: (1) Up to 10 type R characters plus a decimal point, (standard rate format.) but must be less than 100%. (2) Rate input must be positive and less than 100%. (3) Not more than 3 integers may be entered. (4) Not more than 9 decimals may be entered. Validation Rules: : (1) It must be greater than the Overall Minimum Percentage. (2) If Flat amount entered, then Overall Maximum Percentage will not be allowed. (3) Both Overall maximum percentage and overall maximum amount per currency cannot be entered. (4) The base value used to apply the overall maximum percentage from T24 application which triggers the charge will be the same as used by BAND & LEVEL type calculations.

        Returns:

This amount could represent the actual cost to the bank of providing the service, which should never be waived. Available Options for input: (1) Up to 10 type R characters plus a decimal point, (standard rate format.) but must be less than 100%. (2) Rate input must be positive and less than 100%.
