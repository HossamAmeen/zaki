      • getUsrFldName

        public TField getUsrFldName()
        Returns the UsrFldName

        Field description: Specify the name of a field as defined in the appropriate DE.MESSAGE record. The name of the field can be from the USR or SYSTEM fields as maintained in DE.MESSAGE for the Appropriate message Type. When a same field is mapped in both SYSTEM as well as USER fields, then details mapped to USER field will be used for Handoff. Only fields described as mandatory in the appropriate DE.MESSAGE record must be mapped , but others should be mapped if possible. Validation Rules: 1-18 type A (alphanumeric) characters. USR.INPUT.POS, USR.INPUT.NAME and USR.FLD.DESC form a multivalue set. (Optional input) Must be on the DE.MESSAGE record for the appropriate Message Type

        Parameters:
        Returns:

      • getUsrInputName

        public TField getUsrInputName()
        Returns the UsrInputName

        Field description: This field allows a field name to be defined for the corresponding Record number Validation Rules: Multi value fields can be defined in the format FIELD-mv. Input is only allowed in this field when the associated USR.INPUT.POSITION is an integer. It is necessary to have a corresponding entry in INPUT.FILE for the INPUT.REC.NO Input can be either numeric or alphabetic. If a number is entered, it will automatically get converted to a field name by checking the Input File standard Selection record.

      • getUsrInputPos

        public TField getUsrInputPos()
        Returns the UsrInputPos

        Field description: Defines the location of a field within the banking application records that is passed by the application to Delivery System. When a banking application calls APPLICATION.HANDOFF it can pass between 1-9 records, but typically it will pass 2 or 3. The Delivery System accesses the data required to build a message and its header through mapping. For more details see the help text of INPUT.POSITION.
