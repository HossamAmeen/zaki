      • getBankComm

        public TField getBankComm()
        Returns the BankComm

        Field description: Indicates the FT.CHARGE.TYPE or FT.COMMISSION.TYPE to be used for brokerage calculations if the customer of the deal is a bank type customer. The set of fields SYSTEM.ID to TXN.CODE, are used to determine different brokerage rates applicable to specific deal types. The parameters system id, category code, term, currency and customer type are taken from the deal at input stage, and a match is located within this record. If one is found then that rate can be used for automatic brokerage calculations. If one is not found and no default value exists (i.e. a multivalue group that has all match parameters blank), then no automatic calculation can be performed and the user will be required to enter a manual amount. The FT.CHARGE.TYPE or FT.COMMISSION.TYPE entered here will be applied if a match is found for any deal satisfying the above selection criteria and the counterparty of the deal is a bank type customer. Validation Rules: Must be a key to a record on either the FT.CHARGE.TYPE file or the FT.COMMISSION.TYPE file. Multivalue field associated with fields SYSTEM.ID to TXN.CODE. Subvalue field associated with fields CURRENCY to NON.BNK.COMM. Either BANK.RATE or BANK.COMM may be defined for each subvalue group, but not both. Optional input.

        Returns:

      • getCalcType

        public TField getCalcType()
        Returns the CalcType

        Field description: This field defines whether the brokerage calculation will be carried out on either a FIXED basis, a PERIOD basis or a PRESENT.VALUE basis. For the FIXED basis the brokerage calculated igonres the period of the contract and returns an amount based solely on the contract principal. Example: A loan matches a deal type defined on the BROKER record which will use the FIXED basis. The principal on the contract is 1,000 GBP. If the percentage rate for the deal type is set at 1% and the broker currency set to GBP the resulting brokerage amount will be 100 GBP. For the PERIOD basis the length of the contract is used in the calculation of the brokerage amount. The unit period for calculation (360 or 365 days) is based on the default basis of the contract currency. Example: A deposit matches a deal type defined on the BROKER record which will use the PERIOD basis. The principal on the contract is 10,000 GBP for a period of two years. If the percentage rate for the deal type is set at 1% the resulting brokerage amount will be 10 GBP for each year, for a total of 200 GBP. N.B. Unless FIXED is specified on this record for the deal type defined for the sub value group, the calculation uses the PERIOD basis, except for brokerage taken on Letter of Credit Drawings, the FIXED basis is always used regardless of contents of this field. For PRESENT.VALUE the PV formula will be used in the brokerage calculation. The PRESENT.VALUE calc.type is primarily for the brokerage calculation of the interest rate swap contracts. For Example: An IRS of GBP 10M with Bank A paying 6.72% fixed semi-annually and receiving floating from Bank B. Value date is 03 Apr 1998 and maturity date is 03 Apr 2002. With brokerage rate set to 0.01% and applying the above formula, the brokerage will be calculated as GBP 3,457.15. Validation Rules: Must either be FIXED, PERIOD or PRESENT.VALUE. Multivalue field associated with fields SYSTEM.ID to TXN.CODE. Subvalue field associated with fields CURRENCY to NON.BNK.COMM. Optional input. If PRESENT.VALUE is specified no BANK.COMM and NON.BNK.COMM will be allowed.

        Parameters:
        Returns:

        public TField getCurrency()
        Returns the Currency

        Field description: Used to define different rates depending on the currency of the deal. The set of fields SYSTEM.ID to TXN.CODE are used to determine different brokerage rates applicable to specific deal types. The parameters system id, category code, term, currency and customer type are taken from the deal at input stage and a match is located within this record. If one is found then that rate can be used for automatic brokerage calculations. If one is not found and no default value exists (i.e. a multivalue group that has all match parameters blank), then no automatic calculation can be performed and the user will be required to enter a manual amount. The currency from the deal is matched against the currency list here, if a match is found then the rate within this sub-value set is used. Validation Rules: 3 characters. (Optional Input) Multivalue field associated with fields SYSTEM.ID to TXN.CODE. Subvalue field associated with fields CURRENCY to NON.BNK.COMM. Must be a valid currency on the currency file.
