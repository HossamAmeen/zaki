      • getCarrierAddrNo

        public TField getCarrierAddrNo()
        Returns the CarrierAddrNo

        Field description: Specifies a new Carrier and Address Number to replace the usual one obtained from the delivery Product file. Validation Rules: a) The carrier, e.g. SWIFT, PRINT, TELEX (Mandatory input.) b) '.' c) 1-3 numeric character Address Version Number as in the ID of the Address file. (Optional input, default value '.1') Carrier must reference a record on the carrier definition file, DE.CARRIER. A record must exist on the DE.ADDRESS file, with a key composed of: - the customer or company code in the ID of this record - the address field from the carrier record (DE.CARRIER) - the address number field from this field e.g. If the ID of the record is US0010001.C-100001.PRINT.1 and this field contains TELEXF.1, the carrier record for TELEXF is accessed initially to find out the addresses used for TELEXF. So, if the ADDRESS field for TELEXF is TELEX, a record must exist on DE.ADDRESS with a key of US0010001.C-100001.TELEX.1. The carrier module, as defined in the field CARRIER.MODULE on the DE.CARRIER record, must be in the list of outward carriers on DE.PARM.

      • getLanguage

        public TField getLanguage()
        Returns the Language

        Field description: Defines the Language to be used for printed messages. If not entered the Language on the message Header will remain unchanged. If the alternate carrier is Print, then Language is required in order to select the appropriate record from the Format Print table. Validation Rules: 1-2 type A (alphanumeric) characters. (Optional input) If entered it must be a valid Language mnemonic. Must be blank if alternate carrier is EUCLID.
