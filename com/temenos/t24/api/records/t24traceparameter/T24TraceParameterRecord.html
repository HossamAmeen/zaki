      • getDestinationStatic

        public TField getDestinationStatic()
        Returns the DestinationStatic

        Field description: Should have destination physical queue name, connection factory, reply queue name details delimited by # (activation string in the case of CALLJEE) where this trace log can be sent by T24.TRACEABILITY.SERVICE and MDB also will listen on same queue to pick up this message and send it to Micro service. The format of value should be: queue/Traceability#ConnectionFactory#queue/TraceabilityReply where, 1. queue/Traceability is a desitnation queue which will have configured in standalone.xml in jboss deployment file. 2. ConnectionFactory - can be (optional) any valid connection factory used to identify destination. The default value is 'ConnectionFactory' (whihc is used in jboss pack). 3. queue/TraceabilityReply - is a reply queue where MDB updates its reponse for CALLJEE to reply back to T24 service. This will also have configured in standalone.xml in jboss deployment file. Without these values, CALLJEE cannot send SYNC message to MDB/micro service successfully. Validation Rules: Mandatory when ID is SYSTEM. Destination static defined in application level parameter record takes precedence during T24.TRACEABILITY.SERVICE. If no destination details defined application level, then system level destination static is being used by this service.

        Returns:
        TField - The DestinationStatic
