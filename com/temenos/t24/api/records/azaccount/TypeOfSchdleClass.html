      • getTypeOfSchdle

        public TField getTypeOfSchdle()
        Returns the TypeOfSchdle

        Field description: This field indicates the type of schedule defined by the user. Based on the schedule types defined here, AZ.SCHEDULE is generated by the system. The types of schedules are: A - For defining annuity type of repayment, where a constant repayment amount is generated based on the frequency. It is possible to define multiple 'A' type of schedules within an AZ loan account for different periods during the term of the loan. In such cases, it is necessary to input at least one 'A' type schedule that includes the maturity date and brings the loan balance to zero (or the residual amount). B - In loans this type is used for effecting loan drawdowns on pre-defined dates. On the specified dates the system will transfer the draw down amount to the credit of the nominated account. It is validated to see that the sum of the amount specified in all B schedules and the Drawdowns made so far, do not exceed the Principal. In deposits this is used to specify the installment deposit credit. If MULTI is defined as 'Y' then each deposit credit will create a sub az.account. I - Defines schedule for interest payments. For I type schedule, amount cannot be specified by the user. Total period of all I type schedules are validated against the Maximum Interest Only period defined for the AZ account. N - In loans this schedule specifies the interest repayment by the customer. In deposits it specifies the interest receivable to the customer. P- This type defines the Repayment of Principal R - This type is used for effecting Interest Rate revision on specified dates. The value defined under Term Priority will be taken into account for effecting this type of schedule. Both Variable interest rate and Fixed Interest rate can be defined for a loan account, through this type, for a certain percentage of the outstanding amount and also for various periods within the life of the loan account. Example: Type of Schedule: R Frequency: 20010522 Rate Sch Key: 1 Rate Sch Spread: 6 Rate Sch Operand: Add Rate Sch Percent: 60 Sch Fixed Rate: 15 As per the above R schedule, a rate revision will happen on 22-05-2001, with 60% of the outstanding amount by a variable interest (key1 and spread 6 on Basic Interest) and 40% of the outstanding amount by fixed interest 15% For changing the values of interest rates defaulted from the AZ Product to the AZ.ACCOUNT, R type of schedule can be used at the start of the loan account. C - This is a charge schedule. Incase of CREDIT CARD The charge on individual drawdowns are shown by way of 'C'schedules in the AZ sub a/c and the combined 'C' schedule amount is shown in the schedules of AZ Main a/c. BC - This is specific to deposits where a charge can be taken if the B schedule is successful. In this case the repayment account is mandatory. CI - This is specific to CREDIT CARD type of products. This is the committed interest schedule. The amount of interest from the date of draw down to the repayment date is populated through these schedules. Interest on each of the drawdowns is populated in the respective sub a/cs and the combined CI amount is populated in the AZ main a/c. Apart from the draw down date, this schedule is created automatically at every repayment date also covering interest upto the next repayment date. CC - This is specific to CREDIT CARD type od products. This is for repayment of Principal. Raised, based on the AZ product parameter setup in either revolving ratio/Highest range etc. Similar to CI, C schedules the individual CC schedule amounts are populated in the AZ sub a/cs and the combined CC schedule is populated in the AZ main a/c. Validation Rules: Mandatory field, if the option for SCHEDULE is given as Y Schedule definition should be for one type at a time. Combinations like PI, RI etc., are not allowed Valid schedule types are A, B, BC, I, IN, N, P, R, Z, C, CC and CI. Part of a multi value set of fields.

        Returns:

      • setTypeOfSchdle

        public void setTypeOfSchdle(java.lang.CharSequence value)
        Set the specified value for TypeOfSchdle
        Throws an UnsupportedOperationException if the object is not modifiable.

        Field description: This field indicates the type of schedule defined by the user. Based on the schedule types defined here, AZ.SCHEDULE is generated by the system. The types of schedules are: A - For defining annuity type of repayment, where a constant repayment amount is generated based on the frequency. It is possible to define multiple 'A' type of schedules within an AZ loan account for different periods during the term of the loan. In such cases, it is necessary to input at least one 'A' type schedule that includes the maturity date and brings the loan balance to zero (or the residual amount). B - In loans this type is used for effecting loan drawdowns on pre-defined dates. On the specified dates the system will transfer the draw down amount to the credit of the nominated account. It is validated to see that the sum of the amount specified in all B schedules and the Drawdowns made so far, do not exceed the Principal. In deposits this is used to specify the installment deposit credit. If MULTI is defined as 'Y' then each deposit credit will create a sub az.account. I - Defines schedule for interest payments. For I type schedule, amount cannot be specified by the user. Total period of all I type schedules are validated against the Maximum Interest Only period defined for the AZ account. N - In loans this schedule specifies the interest repayment by the customer. In deposits it specifies the interest receivable to the customer. P- This type defines the Repayment of Principal R - This type is used for effecting Interest Rate revision on specified dates. The value defined under Term Priority will be taken into account for effecting this type of schedule. Both Variable interest rate and Fixed Interest rate can be defined for a loan account, through this type, for a certain percentage of the outstanding amount and also for various periods within the life of the loan account. Example: Type of Schedule: R Frequency: 20010522 Rate Sch Key: 1 Rate Sch Spread: 6 Rate Sch Operand: Add Rate Sch Percent: 60 Sch Fixed Rate: 15 As per the above R schedule, a rate revision will happen on 22-05-2001, with 60% of the outstanding amount by a variable interest (key1 and spread 6 on Basic Interest) and 40% of the outstanding amount by fixed interest 15% For changing the values of interest rates defaulted from the AZ Product to the AZ.ACCOUNT, R type of schedule can be used at the start of the loan account. C - This is a charge schedule. Incase of CREDIT CARD The charge on individual drawdowns are shown by way of 'C'schedules in the AZ sub a/c and the combined 'C' schedule amount is shown in the schedules of AZ Main a/c. BC - This is specific to deposits where a charge can be taken if the B schedule is successful. In this case the repayment account is mandatory. CI - This is specific to CREDIT CARD type of products. This is the committed interest schedule. The amount of interest from the date of draw down to the repayment date is populated through these schedules. Interest on each of the drawdowns is populated in the respective sub a/cs and the combined CI amount is populated in the AZ main a/c. Apart from the draw down date, this schedule is created automatically at every repayment date also covering interest upto the next repayment date. CC - This is specific to CREDIT CARD type od products. This is for repayment of Principal. Raised, based on the AZ product parameter setup in either revolving ratio/Highest range etc. Similar to CI, C schedules the individual CC schedule amounts are populated in the AZ sub a/cs and the combined CC schedule is populated in the AZ main a/c. Validation Rules: Mandatory field, if the option for SCHEDULE is given as Y Schedule definition should be for one type at a time. Combinations like PI, RI etc., are not allowed Valid schedule types are A, B, BC, I, IN, N, P, R, Z, C, CC and CI. Part of a multi value set of fields.

        Parameters:
        value - The value to set
        Returns:
        nothing
