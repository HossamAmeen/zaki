      • getRule

        public TField getRule()
        Returns the Rule

        Field description: Entry in this field should be a valid record from OA.RULE.DEFINITION, wherein a Boolean rule can be specified. Which would compare the field values to literals. If the rule returns true then the specified path will be taken. In the rule definition, the first word (Field Name) should always be a valid field name from the context. Second word should be an operand and the third word should be a literal with quotes. eg1 : (FOREIGNAPPLICATION == 'YES') Here, "FOREIGNAPPLICATION" is a field name from the context. "==" is an operand "'YES'" is a literal eg2 : (HIGHESTDEGREE == 'PG' AND AGESTATUS == 'ADULT') Here, "HIGHESTDEGREE" and "AGESTATUS" are the valid field names in the context. "==" is an operand 'PG' and 'ADULT' are Literals. The list of comparison operands are here. Comparison List = "<", "<=", "==", "!=", ">=", ">" Join List = "||", "&&", "OR", "AND" Here is the explanation for Comparison List operators Equals Operator == Not Equals operator != Less than operator "<" Greater than operator ">" Less than or Equals to operator "<=" Greater than or Equals to operator ">=" Some of the Rule Examples : 1. COUNTRY != "US" 2. CURRENCY == "USD" 3. (GENDER == "MALE") AND (AGE >= 35)

        Parameters:
        Returns:
        TField - The Rule
