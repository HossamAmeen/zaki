        public TField getAdjBalAmt()
        Returns the AdjBalAmt

        Field description: The field is used to specify the adjusted balance amount for the corresponding balance type. This is a signed field, that is, sign given here indicate a debit or credit entry to be passed based on the balance type and the product line of the product.

        Example: We have captured a penalty interest, say a debit balance of 100, from legacy system for a lending product. Subsequently, when the balance is to be adjusted such that the new balance amount is say 500, then a value -400 has to be given here.

        Validation rule Standard t24 signed amount field

      • getNewBalAmt

        public TField getNewBalAmt()
        Returns the NewBalAmt

        Field description: The field is used to specify the new balance amount for the corresponding balance type when we capture new balance type or when we adjust an existing balance type. This is a signed field. Sign given here will be corresponding to the Product Line of the underlying product. For write off activities, system will default zero in this field.

        Example: When we want to capture a penalty interest, say a debit balance of 100, from legacy system for a lending product, the value given here would be -100. Subsequently, when the balance is to be adjusted such that the new balance amount is say 500, then a value -500 has to be given here.

        Validation rule Standard t24 amount field can be signed

        Parameters:
        Returns:
