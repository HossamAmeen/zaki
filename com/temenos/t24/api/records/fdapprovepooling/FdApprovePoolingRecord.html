      • getGlPooledGroup

        public TField getGlPooledGroup()
        Returns the GlPooledGroup

        Field description: This field allows to globally apply a new pool group to each selected orders (GL.POOLED.GROUP is moved to the field POOLED.GROUP of each selected orders). The POOLED.GROUP can be amended for MANUAL orders only. An existing group number can be entered, provided the currency, value date and maturity date of the order are the same as those on FD.POOL.GROUP record for the pool group number. If no suitable group exists and the order is to be removed from the allocated group, input of NEW will cause a new group number to be allocated. Once the order has been approved by the dealer, the group number cannot be amended, unless the approval is cancelled. Validation Rules: 10 digit group number comprised of: 5 digit julian date followed by a 5 digit sequence number or NEW. No input allowed when POOLING is AUTO, NONE or SPLIT. The number entered must exist on the FD.POOL.GROUP file, and the Currency, Value date and Maturity Date on the order must be the same as that on the FD.POOL.GROUP record. The GROUP.STATUS on FD.POOL.GROUP cannot be CLOSED

        Parameters:
        Returns:
        TField - The GlPooledGroup
