      • getNextEffDate

        public TField getNextEffDate()
        Returns the NextEffDate

        Field description: Specifies the mandatory future effective date of a next document (document to replace the current one), with a valid status. When a next document is obtained from a Customer before the expiry of the current one, it could be made immediately effective, by directly updating the fields for the current document i.e. REFERENCE.NO to DOC.SEQUENCE However, if the next document is with a valid status (Field: NEXT.STATUS is valid), then it would be possible to make it effective from any future date. If the status of the next document is valid (Field: NEXT.STATUS is valid), on the start-of-day of its effective date, T24 would update the details of the current document with that of the next one as follows: STATUS by NEXT.STATUS, STATUS.DATE by NEXT.STATUS.DATE, END.DATE by NEXT.END.DATE, STAT.DETAILS by NEXT.DETAILS, SIG.DATE by NEXT.SIG.DATE; also the DOC.SEQUENCE would be incremented by 1. Validation Rules: Value allowed only if the status of the next document (Field: NEXT.STATUS) is specified. Value not allowed if the status of the next document (Field: NEXT.STATUS) is invalid. Value mandatory if the status of the next document (Field: NEXT.STATUS) is valid. If specified, its value should be greater than Bank Date.

        Parameters:
        Returns:
        TField - The NextEffDate

      • getNextEndDate

        public TField getNextEndDate()
        Returns the NextEndDate

        Field description: Specifies the expiry date in case of a next document (document to replace the current one). T24 would not calculate the value of this field, if the next document is with an invalid status (Field: NEXT.STATUS). T24 would calculate the value of this field, if the next document is with a valid status, as follows: 1. As per document validity defined in the record with ID equal to attached status (Field: NEXT.STATUS), in the file DOCUMENT.STATUS. 2. As per document validity defined in the record with ID equal to value of Document Type (specified in the first part of ID), in the file DOCUMENT.TYPE. 3. The calculated NEXT.END.DATE would be the least of the dates calculated as per 1 and 2 above. User could override a T24 calculated NEXT.END.DATE with a lesser date. Validation Rules: Optional. Standard T24 date format. Value allowed only if NEXT.STATUS is specified. If NEXT.STATUS is invalid, it is optional; if specified, it must be greater than or equal to the Bank Date. If NEXT.STATUS is valid, then calculated or user input NEXT.END.DATE, if not null, should be greater than or equal to NEXT.EFF.DATE.

        Parameters:
        Returns:
        TField - The NextEndDate

      • getNextStatus

        public TField getNextStatus()
        Returns the NextStatus

        Field description: Specifies the status of the next document (document to replace the current one). This might be used to specify user defined status such as "Advice sent", etc. for the next document, when an advice is sent to the customer for a new document, before the expiry of the current one. It might also represent the status of the next document which might have been obtained from the Customer before the expiry of the current one, and which could be effective from a future date. Value of this field would be used by T24 in calculation of Expiry Date (Field: NEXT.END.DATE) of the Next Document, if it has a valid status. On the End-of-day of the current document expiry (Field: END.DATE), if the next document is with an invalid status and its end date (Field:NEXT.END.DATE) is either null or not less than next working day, then T24 would update the current document details with that of the next one. i.e. the fields STATUS by NEXT.STATUS, STATUS.DATE by NEXT.STATUS.DATE, END.DATE by NEXT.END.DATE, STAT.DETAILS by NEXT.DETAILS, SIG.DATE by NEXT.SIG.DATE; also the DOC.SEQUENCE would be incremented by 1. Validation Rules: Mandatory field. Should be the ID of a valid record in the file DOCUMENT.STATUS. Value of 3 ("Expired") not allowed.

        Parameters:
        Returns:
        TField - The NextStatus
