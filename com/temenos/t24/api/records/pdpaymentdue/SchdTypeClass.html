      • getSchdDateFreq

        public TField getSchdDateFreq()
        Returns the SchdDateFreq

        Field description: Specifies the date or frequency on which the event defined in the field SCHD.TYPE is to commence. A frequency may only be entered for chaser advice schedules and the date entered may not be less than today. For rate and spread changes the date entered cannot be greater than today though they may be backvalued as long as the date entered is greater than the last interest payment date. An override will be given if the date is not a working day. This field is part of an associated multivalue set. Validation Rules: 16 character standard date/frequency code format - type FQO. (Associated multivalue field). A frequency is not allowed for "RC" and "SC" type schedules. A backvalued date is not allowed for "CH" type schedules. A forwardvalued date is not allowed for "RC" and "SC" type schedules. If a backvalued date is entered for "RC" or "SC" type schedules then the date must be greater than the last interest payment date.

        Parameters:
        Returns:
        TField - The SchdDateFreq

      • getSchdEndDate

        public TField getSchdEndDate()
        Returns the SchdEndDate

        Field description: Specifies the date on which the event defined in the field SCHD.TYPE is to end. For rate and spread change type schedules this date must be the same as the date entered in SCHD.DATE.FREQ. For the chaser advice schedule the date may be equal to or greater than the date entered in SCHD.DATE.FREQ. An override will be given if the date is not a working day. This field is part of an associated multivalue set. Validation Rules: 11 type D characters - standard date format. (Associated multivalue field). Must be the same date as SCHD.DATE.FREQ for "RC" and "SC" type schedules. Must be the same date or a later date as SCHED.DATE.FREQ for "CH" type schedules.
