      • getFldName

        public TField getFldName()
        Returns the FldName

        Field description: Contains the name of a field to be processed in a VERSION (at run-time). The field is either hidden or displayed depending on the value of the associated fields (from field FLD.CHECK.VALUE to FLD.INC.EXC). The field name must be specified in the same format present in the version record for the feature to take effect. For example, if the field name in the version record has a value field with it (OTHER.OFFICER-1) then the FIELD.NAME in the version. control record should also be OTHER.OFFICER-1. If the field name in the version record is OTHER.OFFICER, then the FIELD.NAME in the version.control record should also be OTHER.OFFICER. If the Fields per line in the version record is set to 1, the blank space that appears due to hidden fields are removed. If fields per line is &#8216;MULTI&#8217; the blanks are not removed. Validation Rules: Up to 35 type A (Alphanumeric) characters corresponding to the field name or field number identification. If a field number is entered, it is translated to the field name. Local Reference fields can also be input. Optional input. No default value. Only field names or field numbers defined in the Application (as in the record key) are allowed. Duplication of field names is not allowed. Sub-value field associated with fields FLD.CHECK.VALUE to FLD.INC.EXC.
