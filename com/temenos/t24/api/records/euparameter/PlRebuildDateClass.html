      • getOrigPlBal

        public TField getOrigPlBal()
        Returns the OrigPlBal

        Field description: Holds the original total balance of the CRB base in CONVERTED.CCY on the associated PL.REBUILD.DATE, i.e. the dates on which the CRB Profit and Loss base was rebuilt. When the local currency conversion takes place both the local equivalent amount of each CATEG.ENTRY record, and the local equivalent of the CONSOLIDATE.PRFT.LOSS files are converted from ORIGINAL.LCY to CONVERTED.LCY. Since the CONSOLIDATE.PRFT.LOSS totals are the sum of the local equivalents of consolidated entries, an inevitable rounding difference may arise when rebuilding the content of this file from the CATEG.ENTRY records. In order to ensure that the total P&L remains the same the system will compare the original value held in this field with the new value (held in REBUILD.PL.BAL) after the rebuild has taken place. Any difference will be held in the field PL.ADJUST.AMT and will be posted to a local currency suspense account with the category defined in CONV.LCY.SUSP.CAT. Validation Rules: This is a NOINPUT, system amount field.

        Returns:

      • getPlRebuildDate

        public TField getPlRebuildDate()
        Returns the PlRebuildDate

        Field description: Contains the date(s) the profit and loss CRB base was rebuilt since the local currency has been converted. Together with the fields ORIG.PL.BAL, REBUILD.PL.BAL and PL.ADJUST.AMT, this field is used to raise adjustments to ensure that the system remains in balance after conversion of the local currency. The data is stored with the latest rebuild information in value 1. Validation Rules: This is a NOINPUT, system date field.
